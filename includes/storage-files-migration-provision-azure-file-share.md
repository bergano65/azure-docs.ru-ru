Файловый ресурс Azure хранится в облаке в учетной записи хранения Azure.
Здесь есть еще один уровень факторов производительности.

При наличии высоко активных общих ресурсов (общих ресурсов, используемых многими пользователями и (или) приложениями) два файловых ресурса Azure могут достигнуть предельной производительности учетной записи хранения.

Рекомендуется развертывать учетные записи хранения с одним общим файловым ресурсом.
Вы можете использовать несколько общих файловых ресурсов Azure в одной учетной записи хранения, если у вас есть архивные общие папки или вы хотите, чтобы в них было мало повседневных действий.

Эти рекомендации применяют больше возможностей к прямому облачному доступу (через виртуальную машину Azure), чем Синхронизация файлов Azure. Если вы планируете использовать Синхронизация файлов Azure только для этих общих папок, то вам подойдет несколько групп в одной учетной записи хранения Azure.

Если вы сделали список общих папок, то каждую общую папку следует сопоставлять с учетной записью хранения, в которой они будут размещаться.

На предыдущем этапе вы определили соответствующее количество общих папок. На этом шаге вы создали сопоставление учетных записей хранения с файловыми ресурсами. Теперь разверните соответствующее количество учетных записей хранения Azure с соответствующим количеством файловых ресурсов Azure.

Убедитесь, что регион каждой учетной записи хранения совпадает и соответствует региону уже развернутого ресурса службы синхронизации хранилища.

> [!CAUTION]
> Если вы создаете файловый ресурс Azure с ограничением в 100 Тиб, этот общий ресурс может использовать только локально избыточное хранилище или параметры избыточности хранилища, избыточные в пределах зоны. Прежде чем использовать файловые ресурсы 100-Тиб, учитывайте требования к избыточности хранилища.

Файловые ресурсы Azure по умолчанию создаются с ограничением в 5 тиб. Так как вы создаете новые учетные записи хранения, обязательно следуйте [указаниям по созданию учетных записей хранения, которые позволяют использовать файловые ресурсы Azure с ограничениями 100 Тиб](../articles/storage/files/storage-files-how-to-create-large-file-share.md).

Еще один вопрос при развертывании учетной записи хранения — избыточность службы хранилища Azure. См. раздел [параметры избыточности службы хранилища Azure](../articles/storage/common/storage-redundancy.md).

Также важны имена ресурсов. Например, если вы сгруппируйте несколько общих ресурсов для отдела кадров в учетную запись хранения Azure, следует соответствующим образом присвоить имя учетной записи хранения. Аналогичным образом при именовании файловых ресурсов Azure следует использовать имена, аналогичные тем, которые используются в локальных аналогах.
