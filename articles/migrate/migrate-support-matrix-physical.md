# <a name="support-matrix-for-physical-server-assessment-and-migration"></a>Матрица поддержки для оценки физических серверов и миграции

> [!NOTE]
> Если упомянутые здесь компоненты еще не отображаются на портале службы "миграция Azure", зависает. Они будут отображаться на следующей неделе.

## <a name="assessment-physical-server-requirements"></a>Оценка — требования к физическому серверу

| **Поддержка**                | **Дополнительные сведения**               
| :-------------------       | :------------------- |
| **Развертывание физического сервера**       | Физический сервер может быть автономным или развернутым в кластере. |
| **Разрешения**           | **Windows:** Настройте локальную учетную запись пользователя на всех серверах Windows, которые необходимо включить в обнаружение. Учетная запись пользователя должна быть добавлена в эти группы — удаленный рабочий стол пользователи, монитор производительности и пользователи журнала производительности. <br/> **Linux:** Вам потребуется учетная запись root на серверах Linux, которые вы хотите обнаружить. |
| **Операционная система** | Все операционные системы [Windows](https://support.microsoft.com/help/2721672/microsoft-server-software-support-for-microsoft-azure-virtual-machines) и [Linux](https://docs.microsoft.com/azure/virtual-machines/linux/endorsed-distros) , поддерживаемые Azure. |


## <a name="assessment-appliance-requirements"></a>Оценка — требования к устройству

Для оценки служба "миграция Azure" запускает упрощенное устройство для обнаружения физических серверов, а затем отправляет метаданные сервера и данные о производительности в службу "миграция Azure". Устройство может быть запущено на физическом сервере или виртуальной машине и настроено с помощью сценария PowerShell, скачанного из портал Azure. В следующей таблице перечислены требования к устройству.

| **Поддержка**                | **Дополнительные сведения**               
| :-------------------       | :------------------- |
| **Развертывание устройства**   |  Устройство развертывается как на физическом сервере, так и на виртуальной машине.<br/>  На размещающем компьютере должен быть установлен Windows Server 2012 R2 или более поздней версии.<br/> Ведущему приложению требуется достаточно места для выделения 16 ГБ ОЗУ, 8 виртуальных ЦП, около 80 ГБ дискового пространства и внешнего коммутатора для виртуальной машины устройства.<br/> Устройству требуется статический или динамический IP-адрес, а также доступ к Интернету.
| **Проект службы "миграция Azure"**  |  Устройство может быть связано с одним проектом.<br/> С одним проектом можно связать любое количество устройств.<br/> В проекте можно оценить до 35 000 компьютеров.
| **Маршрутизатор**              | Одно устройство может обнаружить до 200 серверов.
| **Группа оценки**       | В одну группу можно добавить до 35 000 компьютеров.
| **Оценка**             | Вы можете оценить до 35 000 компьютеров за одну оценку.


## <a name="assessment-appliance-url-access"></a>Оценка — доступ по URL-адресу устройства

Чтобы оценить виртуальные машины, устройству миграции Azure требуется подключение к Интернету.

- При развертывании устройства служба "миграция Azure" выполняет проверку подключения к URL-адресам, приведенным в таблице ниже.
- Если вы используете прокси-сервер на основе URL-адресов, разрешите доступ к URL-адресам в таблице, убедившись, что прокси-сервер разрешает все записи CNAME, полученные при поиске URL-адресов.
- Если у вас есть перехваченный прокси-сервер, может потребоваться импортировать сертификат сервера с прокси-сервера на устройство.


**URL-адрес** | **Дополнительные сведения**  
--- | ---
*.portal.azure.com | Переход к портал Azure
*.windows.net <br/> *.msftauth.net <br/> *.msauth.net <br/> *. microsoft.com <br/> *. live.com  | Войдите в свою подписку Azure.
*.microsoftonline.com <br/> *.microsoftonline-p.com | Создание Azure Active Directory приложений для взаимодействия устройств между службами.
management.azure.com | Создание Azure Active Directory приложений для взаимодействия устройств между службами.
dc.services.visualstudio.com | Ведение журналов и мониторинг
*.vault.azure.net | Управление секретами в Azure Key Vault при обмене данными между устройством и службой.
aka.ms/* | Разрешение доступа к связям.
https://download.microsoft.com/download/* | Позволяет загружать файлы с веб-сайта центра загрузки Майкрософт.



## <a name="assessment-port-requirements"></a>Оценка — требования к портам

В следующей таблице перечислены требования к портам для оценки.

**Устройство** | **Connection**
--- | ---
**Appliance** | Входящие подключения через TCP-порт 3389 для разрешения подключения удаленного рабочего стола к устройству.<br/> Входящие подключения через порт 44368 для удаленного доступа к приложению управления устройством с помощью URL-адреса: ``` https://<appliance-ip-or-name>:44368 ```<br/> Исходящие подключения к портам 443, 5671 и 5672 для отправки метаданных обнаружения и производительности в службу "миграция Azure".
**Физические серверы** | **Windows:** Входящие подключения через порты 443, 5989 для извлечения метаданных конфигурации и производительности с серверов Windows. <br/> **Linux:**  Входящие подключения через порт 22 (UDP) для извлечения метаданных конфигурации и производительности с серверов Linux. |
