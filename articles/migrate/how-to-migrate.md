# <a name="add-migration-tools"></a>Добавление средств миграции

В этой статье описывается, как добавить средства миграции в службу " [Миграция Azure](migrate-overview.md)".

Служба "миграция Azure" предоставляет центр средств для оценки и миграции в Azure. Он включает в себя собственные средства, инструменты, предоставляемые другими службами Azure, и сторонние предложения независимых поставщиков программного обеспечения (ISV).

Если вы хотите добавить средство миграции и еще не настроили проект службы "миграция Azure", следуйте указаниям в этой [статье](how-to-add-tool-first-time.md).



## <a name="selecting-an-isv-tool"></a>Выбор инструментов независимого поставщика программного обеспечения

Если вы выбрали [средство ISV](migrate-services-overview.md#isv-integration) для миграции, то можете начать с получения лицензии или регистрации для получения бесплатной пробной версии в соответствии с политикой ISV. В каждом инструменте есть возможность подключиться к службе "Миграция Azure". Разверните средство и следуйте инструкциям и документации по средствам, чтобы подключить рабочую область средства к службе "миграция Azure". 

## <a name="select-a-migration-scenario"></a>Выбор сценария миграции

1. В проекте службы "Миграция Azure" щелкните **Обзор**.
2. Выберите сценарий миграции, который хотите использовать:

    - Чтобы перенести компьютеры и рабочие нагрузки в Azure, выберите **Оценка и миграция серверов**.
    - Чтобы перенести локальные компьютеры SQL, выберите **Оценка и миграция баз данных**.
    - Чтобы перенести локальные веб-приложения, выберите **Оценка и миграция веб-приложений**.
    - Чтобы перенести большие объемы локальных данных в Azure в автономном режиме, выберите **Упорядочить a Data Box**.

    ![Сценарий оценки](./media/how-to-migrate/assess-scenario.png)

## <a name="select-a-server-migration-tool"></a>Выбор средства миграции сервера

1. Щелкните **Оценка и миграция серверов**.
2. Если вы еще не добавили средства миграции в **Azure Migration-Servers**, в разделе **средства миграции**выберите **щелкните здесь, чтобы добавить средство миграции**. Если вы уже добавили средства миграции, в **меню добавить дополнительные средства миграции**выберите **изменить**.

    > [!NOTE]
    > Если вам нужно перейти к другому проекту, в разделе " **Миграция Azure — серверы**" рядом с полем **сведения о другом проекте миграции**щелкните **щелкните здесь**.

3. В **службе "миграция Azure**" выберите средство миграции, которое вы хотите использовать.
    - Если вы используете миграцию с миграцией Azure, вы можете настроить и запустить миграцию непосредственно в проекте службы "миграция Azure".
    - Если вы используете средство оценки стороннего производителя, перейдите по ссылке, предоставленной для независимого поставщика программного обеспечения, и выполните миграцию в соответствии с предоставленными ими инструкциями.

## <a name="select-a-database-migration-tool"></a>Выбор средства миграции базы данных

1. Щелкните **Оценка и миграция баз данных**
2. В окне **базы данных**щелкните **Добавить средства**.
3. В окне Добавление средства > **выберите средство миграции**и выберите инструмент, который вы хотите использовать для переноса базы данных.

## <a name="select-a-web-app-migration-tool"></a>Выберите средство миграции веб-приложения

1. Щелкните **Оценка и миграция веб-приложений**.
2. Перейдите по ссылке на средство миграции для службы приложений Azure. Используйте средство миграции, чтобы:

    - **Оценка приложений в Интернете**. Вы можете оценивать и переносить приложения с общедоступным URL-адресом в Интернете, используя службу приложений Azure помощник по миграции.
    - **.NET/PHP**. для внутренних приложений .NET и PHP можно скачать и запустить помощник по миграции.

## <a name="order-an-azure-data-box"></a>Упорядочивание Azure Data Box

Чтобы перенести большие объемы данных в Azure, можно заказать поле данных Azure для автономной пересылки данных.

1. Щелкните **заказать a Data Box**.
2. В списке **выберите Azure Data Box**укажите подписку. 
3. Перенос будет импортирован в Azure. Укажите источник данных и назначение региона Azure для данных.
