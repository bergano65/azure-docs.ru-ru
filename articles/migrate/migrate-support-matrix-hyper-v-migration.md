# <a name="support-matrix-for-hyper-v-migration"></a>Матрица поддержки для миграции Hyper-V

В этой статье приведены общие сведения о параметрах и ограничениях для миграции виртуальных машин Hyper-V с помощью службы миграции [Azure: миграция сервера](migrate-services-overview.md#azure-migrate-server-migration-tool) . Если вы ищете сведения о проверке виртуальных машин Hyper-V для миграции в Azure, см. [таблицу поддержки оценки](migrate-support-matrix-hyper-v.md).

## <a name="migration-limitations"></a>Ограничения миграции

Для репликации можно выбрать до 10 виртуальных машин. Если вы хотите перенести больше компьютеров, выполните репликацию в группы из 10.


## <a name="hyper-v-host-requirements"></a>Требования к узлу Hyper-V

| **Поддержка**                | **Сведения**               
| :-------------------       | :------------------- |
| **Развертывание**       | Узел Hyper-V может быть автономным или развернутым в кластере. <br/>Программа миграции Azure для репликации (поставщик репликации Hyper-V) установлена на узлах Hyper-V.|
| **Разрешения**           | Вам потребуются права администратора на узле Hyper-V. |
| **Операционная система узла** | Windows Server 2019, Windows Server 2016 или Windows Server 2012 R2 с последними обновлениями. Обратите внимание, что также поддерживается установка основных серверных компонентов этих операционных систем. |
| **Доступ к портам** |  Исходящие подключения через HTTPS порт 443 для отправки данных репликации виртуальной машины.


## <a name="hyper-v-vms"></a>Виртуальные машины Hyper-V

| **Поддержка**                  | **Сведения**               
| :----------------------------- | :------------------- |
| **Операционная система** | Все операционные системы [Windows](https://support.microsoft.com/help/2721672/microsoft-server-software-support-for-microsoft-azure-virtual-machines) и [Linux](../virtual-machines/linux/endorsed-distros.md) , поддерживаемые Azure. |
**Windows Server 2003** | Для виртуальных машин под управлением Windows Server 2003 необходимо [установить Hyper-V Integration Services](prepare-windows-server-2003-migration.md) перед миграцией. | 
**Виртуальные машины Linux в Azure** | Для некоторых виртуальных машин могут потребоваться изменения, позволяющие им работать в Azure.<br/><br/> Для Linux служба "миграция Azure" автоматически вносит изменения в эти операционные системы:<br/> -Red Hat Enterprise Linux 6.5 +, 7.0 +<br/> -CentOS 6.5 +, 7.0 +</br> -SUSE Linux Enterprise Server 12 SP1 +<br/> -Ubuntu 14.04 LTS, 16.04 LTS, 18.04 LTS<br/> -Debian 7, 8. Для других операционных систем [необходимые изменения](prepare-for-migration.md#linux-machines) вносятся вручную.
| **Необходимые изменения для Azure** | Для некоторых виртуальных машин могут потребоваться изменения, позволяющие им работать в Azure. Внесите корректировки вручную перед миграцией. В соответствующих статьях содержатся инструкции о том, как это сделать. |
| **Загрузка Linux**                 | Если/boot находится в выделенной секции, он должен находиться на диске ОС, а не распределяться по нескольким дискам.<br/> Если/boot является частью корневого раздела (/), то раздел "/" должен находиться на диске ОС и не охватывать другие диски. |
| **Загрузка UEFI**                  | Поддерживается. Убедитесь, что выбран размер виртуальной машины, поддерживаемый виртуальной машиной Azure поколения 2.  |
| **Размер диска**                  | 2 ТБ для диска ОС, 4 ТБ для дисков данных.|
| **Номер диска** | Не более 16 дисков на виртуальную машину.|
| **Зашифрованные диски и тома**    | Не поддерживается для миграции.|
| **RDM/транзитные диски**      | Не поддерживается для миграции.|
| **Общий диск** | Виртуальные машины, использующие общие диски, не поддерживаются для миграции.|
| **NFS**                        | Тома NFS, подключенные к виртуальным машинам как тома, не реплицируются.|
| **ISCSI**                      | Виртуальные машины с целями iSCSI не поддерживаются для миграции.
| **Целевой диск**                | Вы можете выполнить миграцию на виртуальные машины Azure только с управляемыми дисками. |
| **IPv6** | Не поддерживается.|
| **Объединение сетевых карт** | Не поддерживается.|
| **Azure Site Recovery** | Невозможно выполнить репликацию с помощью миграции сервера Azure Migration, если для виртуальной машины включена репликация с Azure Site Recovery.|
| **Порты** | Исходящие подключения через HTTPS порт 443 для отправки данных репликации виртуальной машины.|

**URL-адрес** | **Сведения**
--- | ---
login.microsoftonline.com | Управление доступом и удостоверениями с помощью Active Directory.
backup.windowsazure.com | Репликация и координация данных репликации.
*.hypervrecoverymanager.windowsazure.com | Используется для управления репликацией.
*.blob.core.windows.net | Отправка данных в учетные записи хранения. 
dc.services.visualstudio.com | Передача журналов приложений, используемых для внутреннего мониторинга.
time.windows.com | Проверяет синхронизацию времени между системой и глобальным временем.

## <a name="azure-vm-requirements"></a>Требования для виртуальных машин Azure

Все локальные виртуальные машины, реплицируемые в Azure, должны соответствовать требованиям к виртуальным машинам Azure, приведенным в этой таблице.

**Компонент** | **Требования** | **Сведения**
--- | --- | ---
Размер диска операционной системы | До 2048 ГБ | Если не поддерживается, проверка завершается ошибкой.
Число дисков операционной системы | 1 | Если не поддерживается, проверка завершается ошибкой.
Число дисков с данными | 16 или меньше. | Если не поддерживается, проверка завершается ошибкой.
Размер диска данных | До 4095 ГБ | Если не поддерживается, проверка завершается ошибкой.
Сетевые адаптеры | Поддерживаются несколько адаптеров. |
Общий виртуальный жесткий диск | Не поддерживается. | Если не поддерживается, проверка завершается ошибкой.
Диск FC | Не поддерживается. | Если не поддерживается, проверка завершается ошибкой.
BitLocker | Не поддерживается. | Прежде чем включать репликацию для компьютера, необходимо отключить BitLocker.
имя виртуальной машины; | От 1 до 63 символов.<br/> при этом допустимы только буквы, цифры и дефисы<br/><br/> Имя компьютера должно начинаться и заканчиваться буквой или цифрой. |  Обновите значение в свойствах компьютера в службе Site Recovery.
Подключение после миграции — Windows | Чтобы подключиться к виртуальным машинам Azure под Windows после миграции, выполните следующие действия.<br/><br/> — Перед миграцией включите RDP на локальной виртуальной машине. Убедитесь в том, что правила для протоколов TCP и UDP для **общедоступного** профиля добавлены, а протокол RDP разрешен в разделе **Брандмауэр Windows** > **Разрешенные программы** для всех профилей.<br/><br/> — Для доступа к VPN типа "сеть — сеть" Включите RDP и разрешите RDP в **брандмауэре Windows**  ->  **Разрешенные приложения и компоненты** для **доменов и частных** сетей. Кроме того, убедитесь, что для политики SAN операционной системы задано значение **OnlineAll**. [Подробнее.](prepare-for-migration.md) |
Подключение после миграции — Linux | Чтобы подключиться к виртуальным машинам Azure после миграции с помощью SSH, сделайте следующее:<br/><br/> — Перед миграцией на локальном компьютере убедитесь, что служба Secure Shell настроена на запуск, а правила брандмауэра разрешают SSH-подключение.<br/><br/> — После миграции на виртуальной машине Azure разрешите входящие подключения к порту SSH для правил группы безопасности сети на виртуальной машине, для которой выполнена отработка отказа, и для подсети Azure, к которой он подключен. Кроме того, добавьте общедоступный IP-адрес для виртуальной машины. |
