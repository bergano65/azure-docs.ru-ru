# <a name="azure-time-series-insights-gen2-explorer"></a>Обозреватель Gen2 временных рядов Azure

В этой статье описываются различные функции и параметры, доступные в [демонстрационной среде](https://insights.timeseries.azure.com/preview/demo)Azure Time Series Insights Gen2.

## <a name="prerequisites"></a>предварительные требования

Чтобы приступить к работе с Gen2 Explorer службы "аналитика временных рядов Azure", необходимо:

* Подготовить среду Gen2 "аналитика временных рядов Azure". Дополнительные сведения о подготовке экземпляра см. в руководстве по [Gen2 Azure Time Series Insights](./time-series-insights-update-create-environment.md) .
* [Предоставьте доступ к данным](./time-series-insights-data-access.md) в среду Gen2 "аналитика временных рядов Azure", созданную для учетной записи. Вы можете предоставить доступ как другим пользователям, так и себе.
* Добавьте источник событий в среду Gen2 "аналитика временных рядов Azure" для отправки данных в среду:
  * Сведения [о подключении к концентратору событий](./time-series-insights-how-to-add-an-event-source-eventhub.md)
  * Узнайте, [как подключиться к центру Интернета вещей](./time-series-insights-how-to-add-an-event-source-iothub.md)

## <a name="explore-the-azure-time-series-insights-gen2-explorer"></a>Изучите обозреватель Gen2 "аналитика временных рядов Azure"

Обозреватель Gen2 "аналитика временных рядов Azure" состоит из следующих семи элементов:

1. [Панель "среда](#1-environment-panel)": отображает все среды Gen2 "аналитика временных рядов Azure".
1. [Панель навигации](#2-navigation-bar): позволяет переключаться между страницами **анализ** и **модель** .
1. [Дерево иерархии и панель поиска](#3-hierarchy-tree-and-search-panel): позволяет выбирать и искать определенные элементы данных для построения диаграммы.
1. [Контейнеры временных рядов](#4-time-series-well): отображаются все выбранные в данный момент элементы данных.

## <a name="2-navigation-bar"></a>2. панель навигации

  [![Панель навигации](media/v2-update-explorer/tsi-preview-navigation-bar.png)](media/v2-update-explorer/tsi-preview-navigation-bar.png#lightbox)

Используйте панель навигации для выбора между двумя представлениями:

* **Модель**: используйте ее для отправки новых типов, иерархий и экземпляров службы "аналитика временных рядов Azure" Gen2 в модель временных рядов.

### <a name="model-authoring"></a>Создание моделей

Gen2 "аналитика временных рядов Azure" поддерживает операции создания, чтения, обновления и удаления (CRUD) в модели временных рядов.

[![Панель «Поиск модели»](media/v2-update-explorer/model-search-panel.png)](media/v2-update-explorer/model-search-panel.png#lightbox)

* **Тип модели временных рядов**: типы моделей временных рядов можно использовать для определения переменных или формул для выполнения вычислений. Они связаны с заданным экземпляром модели временных рядов. Один тип может содержать одну или несколько переменных.
* **Иерархия модели временных рядов**. иерархии представляют собой систематические организации данных. Иерархии описывают связи между разными экземплярами в модели временных рядов.
* **Экземпляр модели временных рядов**. экземпляры — это сами временные ряды. В большинстве случаев это **DeviceID** или **AssetID**, который является уникальным идентификатором ресурса в среде.

Дополнительные сведения о модели временных рядов [см. в этой статье.](./concepts-model-overview.md)

При выборе экземпляров они не только добавляются в текущую диаграмму, но и также добавляются в данные.

## <a name="5-chart-panel"></a>5. панель диаграммы

Диаграмма позволяет отображать экземпляры временных рядов в виде строк. Чтобы увеличить диаграмму, можно свернуть панель среды, модель данных и панель управления интервалом времени, щелкнув соответствующие веб-элементы управления.

  [![Общие сведения о диаграмме Gen2](media/v2-update-explorer/tsi-preview-chart-overview.png)](media/v2-update-explorer/tsi-preview-chart-overview.png#lightbox)

1. **Тип диаграммы**: определяет, какие элементы данных доступны для визуализации.

1. **Размер интервала**. ползунок «размер интервала» позволяет увеличить и уменьшить интервалы в течение одного и того же интервала времени. Это обеспечивает более точное управление перемещением между крупными срезами времени, которые показывают гладкие тенденции до срезов, например, в миллисекундах, что позволяет просматривать детализированные и разрешающие пути к данным. Начальная точка ползунка по умолчанию устанавливается как наиболее оптимальное представление данных из выбора. разрешение балансировки, скорость выполнения запросов и детализация.

1. **Масштаб и Панорама**: Выберите этот элемент управления для масштабирования и панорамирования диаграммы.

1. **Элемент управления осью y**: циклический перебор доступных параметров представления оси y:

    * `Stacked`: Каждая строка имеет отдельную ось Y.
    * `Overlap`: Используйте его для выделения нескольких строк на одной оси Y, при этом данные по оси Y изменяются в зависимости от выбранной линии.
    * `Shared`: Все данные по оси Y отображаются вместе.

1. **Элемент маркера**: выбранный в данный момент элемент данных и связанные с ним сведения.

Можно продолжить детализацию определенного среза данных, щелкнув точку данных на текущем графе, удерживая нажатой **кнопку** мыши, а затем перетащив выбранную область на конечную точку по своему усмотрению. **Щелкните правой кнопкой мыши** синюю выделенную область, а затем выберите **масштаб** , как показано ниже. Вы также можете просматривать и скачивать события телеметрии в выбранном интервале времени.

  [![Масштаб Gen2 диаграммы](media/v2-update-explorer/preview-chart-zoom.png)](media/v2-update-explorer/preview-chart-zoom.png#lightbox)

После выполнения действия **масштабирования** выбранный набор данных будет отображен. Выберите элемент управления "формат" для прохода по трем представлениям данных по оси y.

  [![Ось y Gen2 диаграммы](media/v2-update-explorer/tsi-preview-explorer-standard-chart.png)](media/v2-update-explorer/tsi-preview-explorer-standard-chart.png#lightbox)

Ниже приведен пример **перекрывающейся диаграммы** :

  [![Перекрывающиеся параметры диаграммы](media/v2-update-explorer/tsi-preview-explorer-overlapping-chart.png)](media/v2-update-explorer/tsi-preview-explorer-overlapping-chart.png#lightbox)

Кнопка **"другие действия** " разворачивается для отображения **скачивания в виде CSV-файла**, **подключения к Power BI**, **отображения данных диаграммы в виде таблицы**и **просмотра параметров необработанных событий** .

  [![Параметр "другие действия"](media/v2-update-explorer/more-actions-icon.png)](media/v2-update-explorer/more-actions-icon.png#lightbox)

Дополнительные сведения о параметре **Connect to Power BI** в [соединителе Power BI](concepts-power-bi.md).

## <a name="6-time-editor-panel"></a>6. Панель редактора времени

При работе со службой "аналитика временных рядов Azure" Gen2 сначала будет выбран временной интервал. Выбранный промежуток времени будет управлять набором данных, доступным для обработки в мини-приложениях "аналитика временных рядов Azure Gen2".

  [![Панель выбора времени](media/v2-update-explorer/tsi-preview-explorer-timeline-element.png)](media/v2-update-explorer/tsi-preview-explorer-timeline-element.png#lightbox)

> [!TIP]
> Часть временной шкалы выделяется желтым или оранжевым цветом для указания объема данных, доступного в горячий магазин.

Для выбора диапазона рабочего времени доступны следующие веб-элементы управления в службе "аналитика временных рядов Azure" Gen2.

  [![Качественный контроль](media/v2-update-explorer/exploration-well-control.png)](media/v2-update-explorer/exploration-well-control.png#lightbox)

1. **Ползунок внутреннего диапазона дат**: используйте два элемента управления конечной точки, перетаскивая их на нужное время. Этот внутренний диапазон дат ограничивается элементом управления "ползунок внешнего диапазона дат".

1. **Кнопки "увеличить" и "уменьшить диапазон дат**": Увеличьте или сократите интервал времени, выбрав любую нужную кнопку.

1. **Элемент управления свертывания временного диапазона**. Этот веб-элемент управления позволяет скрывать все элементы управления, за исключением ползунка «внутренний диапазон дат».

1. **Элемент управления ползунок внешнего диапазона дат**: Используйте элементы управления конечной точки, чтобы выбрать внешний диапазон дат, который будет доступен для внутреннего элемента управления "диапазон дат".

1. **Ползунок диапазона времени**: используйте его для быстрого переключения между предустановленными значениями интервала времени, например за последние **30 минут**, за **последние 12 часов**или на **Пользовательский диапазон**. С изменением этого значения изменяются также доступные диапазоны интервалов, описанные в параграфе о ползунке размера интервала.

   [![На панель выбора и из нее](media/v2-update-explorer/to-and-from-element.png)](media/v2-update-explorer/to-and-from-element.png#lightbox)

## <a name="gen-1-environment-controls"></a>Элементы управления среды Gen 1

### <a name="gen2-terms-panel"></a>Панель условий Gen2

Этот раздел относится только к существующим средам Gen 1, которые пытаются использовать обозреватель в обновленном пользовательском интерфейсе. Вы можете использовать продукт Gen 1 и продукт Gen2 в сочетании. Мы добавили некоторые функции из существующего пользовательского интерфейса в обновленный обозреватель, но вы можете получить полный интерфейс пользовательского интерфейса для среды Gen 1 в новом обозревателе Gen2 "аналитика временных рядов Azure".

Вместо иерархии отобразится панель "аналитика временных рядов Azure Gen2". Панель термины позволяет определять запросы в среде. Используйте его для фильтрации данных на основе предиката.

  [![Где находится панель запросов](media/v2-update-explorer/s1-s2-preview-query.png)](media/v2-update-explorer/s1-s2-preview-query.png#lightbox)

Панель редактора "аналитика временных рядов Azure" Gen2 принимает следующие параметры:

**WHERE**: использование предложения WHERE для быстрой фильтрации событий с помощью набора операндов, перечисленных в следующей таблице. При выполнении поиска путем выбора операнда предикат автоматически обновляется на основе поиска. Поддерживаются следующие типы операндов:

| Операция    | Поддерживаемые типы    | Примечания |
| --- | --- | --- |
| `<`, `>`, `<=`, `>=` | Double, DateTime, TimeSpan | |
| `=`, `!=`, `<>` | String, Bool, Double, DateTime, TimeSpan, NULL |
| `IN` | String, Bool, Double, DateTime, TimeSpan, NULL | Все операнды должны быть одного типа или являться константой NULL. |
| `HAS` | Строка | В правой части разрешены только константные строковые литералы. Пустые строки и значения NULL не допускаются. |

Дополнительные сведения о поддерживаемых операциях запросов и типах данных см. в статье [выражение временных рядов (целевой сервер)](https://docs.microsoft.com/rest/api/time-series-insights/reference-time-series-expression-syntax).

### <a name="examples-of-where-clauses"></a>Примеры предложений WHERE

  [![Примеры предложения WHERE](media/v2-update-explorer/tsi-preview-example-queries.png)](media/v2-update-explorer/tsi-preview-example-queries.png#lightbox)

**Мера**: раскрывающийся список, в котором отображаются все числовые столбцы (**Double**), которые можно использовать в качестве элементов для текущей диаграммы.

**Разделить по**: в этом раскрывающемся списке отображаются все доступные столбцы категорий (строки) в модели, по которым можно группировать данные. Можно добавить до пяти условий для просмотра на одной оси x. Введите необходимые параметры, а затем нажмите кнопку **Добавить** , чтобы добавить новый термин.
