# <a name="azure-cdn-from-verizon-premium-rules-engine-features"></a>Функции обработчика правил Azure CDN от Verizon Premium

В этой статье приводятся подробные описания доступных функций для [обработчика правил](cdn-verizon-premium-rules-engine.md) сети доставки содержимого (CDN) Azure.

Третья часть правила представляет собой функцию. Функция определяет тип действия, который применяется к типу запроса, определяемого набором условий соответствия.

| [Отключение кэша](https://docs.vdms.com/cdn/Content/HRE/F/Bypass-Cache.htm) | Определяет, может ли запрос использовать технологию кэширования. |

Наши пограничные серверы используют этот относительный путь при проверке кэшированного содержимого.  По умолчанию ключ кэша не будет содержать параметры строки запроса._ |
| [Перезапись ключа кэша](https://docs.vdms.com/cdn/Content/HRE/F/Cache-Key-Rewrite.htm) | Перезаписывает ключ кэша, связанный с запросом. |

| [Сжатие типов файлов](https://docs.vdms.com/cdn/Content/HRE/F/Compress-File-Types.htm) | Определяет форматы файлов, которые будут сжаты на сервере. |

| [Внешний максимальный срок](https://docs.vdms.com/cdn/Content/HRE/F/External-Max-Age.htm) | Определяет интервал максимального срока для повторной сверки браузера с пограничным сервером. |
| [Принудительное использование внутреннего максимального срока](https://docs.vdms.com/cdn/Content/HRE/F/Force-Internal-Max-Age.htm) | Определяет интервал максимального срока для повторной сверки пограничного сервера с сервером-источником. |
| [Поддержка H.264 (Прогрессивное скачивание через HTTP)](https://docs.vdms.com/cdn/Content/HRE/F/H.264-Support-HPD.htm) | Определяет типы форматов файлов H.264, которые могут использоваться для потоковой передачи содержимого. |
| [H. 264 поддерживает параметры поиска видео](https://docs.vdms.com/cdn/Content/HRE/F/H.264-Support-VSP-HPD.htm) | Переопределяет имена, назначенные параметрам, которые управляют поиском через носитель H. 264 при использовании прогрессивной загрузки HTTP. |
| [Учитывать запрос без кэша](https://docs.vdms.com/cdn/Content/HRE/F/Honor-No-Cache-Request.htm) | Определяет, будут ли запросы без кэша HTTP-клиента перенаправляться на сервер-источник. |
| [Не учитывать источник без кэша](https://docs.vdms.com/cdn/Content/HRE/F/Ignore-Origin-No-Cache.htm) | Определяет, будет ли сеть CDN игнорировать некоторые директивы, полученные с сервера-источника. |
| [Игнорировать неподходящие диапазоны](https://docs.vdms.com/cdn/Content/HRE/F/Ignore-Unsatisfiable-Ranges.htm) | Определяет запрос, возвращаемый клиенту, если запрос формирует код состояния 416 (Запрошенный диапазон не удовлетворяет условиям). |
| [Внутреннее максимальное значение устаревания](https://docs.vdms.com/cdn/Content/HRE/F/Internal-Max-Stale.htm) | Контролирует, как долго после обычного истечения срока действия кэшированный ресурс может обслуживаться с пограничного сервера, если пограничному серверу не удалось выполнить повторную сверку кэшированного ресурса с сервером-источником. |
| [Совместное использование частичного кэширования](https://docs.vdms.com/cdn/Content/HRE/F/Partial-Cache-Sharing.htm) | Определяет, может ли запрос создавать частично кэшированное содержимое. |
| [Предварительная проверка кэшированного содержимого](https://docs.vdms.com/cdn/Content/HRE/F/Prevalidate-Cached-Content.htm) | Определяет, подойдет ли кэшированное содержимое для ранней повторной сверки до истечения его срока действия. |
| [Обновление файлов кэша нулевой длины](https://docs.vdms.com/cdn/Content/HRE/F/Refresh-Zero-Byte-Cache-Files.htm) | Определяет способ обработки запроса HTTP-клиента к ресурсу кэша нулевой длины пограничными серверами. |
| [Набор кодов состояния кэшированного содержимого](https://docs.vdms.com/cdn/Content/HRE/F/Set-Cacheable-Status-Codes.htm) | Определяет набор кодов состояния, которые могут привести к кэшированному содержимому. |
| [Доставка устаревшего содержимого при ошибке](https://docs.vdms.com/cdn/Content/HRE/F/Stale-Content-Delivery-on-Error.htm) | Определяет, будет ли доставлено содержимое с истекшим сроком при возникновении ошибки во время повторной сверки кэша или при получении запрошенного содержимого с сервера-источника клиента. | 
| [Устаревший клиент при повторной сверке](https://docs.vdms.com/cdn/Content/HRE/F/Stale-While-Revalidate.htm) | Повышает производительность, позволяя пограничным серверам обслуживать устаревший клиент для запрашивающей стороны во время повторной сверки. |
