# <a name="app-service-environment-management-addresses"></a>Адреса управления среды службы приложений

Среда службы приложений (ASE) — это одно развертывание клиента службы приложений Azure, которое выполняется в виртуальной сети Azure.  Хотя ASE выполняется в виртуальной сети, она по-прежнему должна быть доступна из нескольких выделенных IP-адресов, используемых службой приложений Azure для управления службой.  В случае с ASE трафик управления проходит по сети, управляемой пользователем. Если этот трафик блокируется или неправильно маршрутизируется, работа ASE приостанавливается. Дополнительные сведения о зависимостях сети ASE см. в статье рекомендации по работе с [сетью и среда службы приложений][networking]. Общие сведения о ASE можно начать с [введения в среда службы приложений][intro].

Каждая среда ASE имеет общедоступный виртуальный IP-адрес, на который поступает трафик управления. Входящий трафик управления с этих адресов на поступает на порты 454 и 455 для общедоступного виртуального IP-адреса ASE. В этом документе перечислены адреса источников службы приложений, передающих трафик управления в среду ASE. Эти адреса также находятся в теге службы IP с именем Аппсервицеманажемент.

Указанные ниже адреса можно настроить в таблице маршрутов, чтобы избежать возникновения проблем с маршрутизацией трафика управления. Маршруты действуют по трафику на уровне IP-адреса и не имеют сведений о направлении трафика или о том, что трафик является частью ответного сообщения TCP. Если адрес ответа для запроса TCP отличается от адреса, на который он был отправлен, возникает проблема с асимметричной маршрутизацией. Чтобы избежать проблем с асимметричной маршрутизацией с трафиком управления ASE, необходимо убедиться, что ответы отправляются обратно с того же адреса, в который они были отправлены. Дополнительные сведения о настройке ASE для работы в среде, в которой исходящий трафик отправляется локально, см. в статье [Настройка ASE с принудительным туннелированием][forcedtunnel] .

## <a name="configuring-a-network-security-group"></a>Настройка группы безопасности сети

С помощью групп безопасности сети не нужно беспокоиться об отдельных адресах или поддержке собственной конфигурации. Существует тег службы IP с именем Аппсервицеманажемент, актуальный для всех адресов. Чтобы использовать этот тег IP-службы в NSG, перейдите на портал, откройте пользовательский интерфейс групп безопасности сети и выберите правила безопасности для входящего трафика. Если у вас есть правило, существовавшее для входящего трафика управления, измените его. Если эта группа безопасности сети не была создана с использованием ASE или если она совсем новая, выберите **Добавить**. В раскрывающемся списке источника выберите **Тег службы**.  В теге исходной службы выберите **аппсервицеманажемент**. Укажите для диапазонов портов источника значение \*, для назначения — **Любые**, для диапазонов портов назначения — **454–455**, для протокола — **TCP**, а для действия — **Разрешить**. При выполнении правила необходимо задать приоритет.

Эти маршруты необходимы при настройке принудительного туннелирования. Создать таблицу маршрутов можно с помощью портала, PowerShell или Azure CLI. ## <a name="get-your-management-addresses-from-api"></a>Получение адресов управления из API ##

С помощью вызова API системы можно получить список адресов управления, которые соответствуют ASE.

    get /subscriptions/<subscription ID>/resourceGroups/<resource group>/providers/Microsoft.Web/hostingEnvironments/<ASE Name>/inboundnetworkdependenciesendpoints?api-version=2016-09-01

API возвращает документ JSON, который включает все адреса входящего трафика для среды ASE. Список адресов включает адреса управления, виртуальный IP-адрес, используемый ASE и диапазон адресов подсети ASE.  

Для вызова API с помощью [armclient](https://github.com/projectkudu/ARMClient) используйте приведенные ниже команды, но укажите свои идентификатор подписки, группу ресурсов и имя среды ASE.  

    armclient login
    armclient get /subscriptions/<subscription ID>/resourceGroups/<resource group>/providers/Microsoft.Web/hostingEnvironments/<ASE Name>/inboundnetworkdependenciesendpoints?api-version=2016-09-01
