# <a name="understand-app-and-deployment-in-azure-spring-cloud"></a>Общие сведения о приложении и развертывании в Azure Веснного облака

**Эта статья применима к:** ✔️ Java ✔️ C#

**Приложение** и **развертывание** — это два ключевых понятия модели ресурсов в облаке Azure весны. В Azure Веснного облака *приложение* — это абстракция одного бизнес-приложения или одной микрослужбы.  Одна версия кода или двоичного файла, развернутая как *приложение* , выполняется в *развертывании*.  Приложения выполняются в *экземпляре облачной службы Azure весны*или просто в *экземпляре службы*, как показано далее.

 ![Приложения и развертывания](./media/spring-cloud-app-and-deployment/app-deployment-rev.png)

В одной подписке Azure может быть несколько экземпляров служб, но облачная служба Azure весны проще всего использовать, когда все приложения, составляющие бизнес-приложение или микрослужбы, находятся в пределах одного экземпляра службы.

Уровень "Стандартный" облачного облака Azure позволяет одному приложению иметь одно рабочее развертывание и одно промежуточное развертывание, чтобы можно было легко выполнить развертывание с помощью синего или зеленого цвета.

## <a name="app"></a>Приложение
Следующие функции и свойства определяются на уровне приложения.

| Компоненты | Описание |
|:--|:----------------|
| Общие</br>Конечная точка | URL-адрес для доступа к приложению |
| Особые настройки</br>Домен | Запись CNAME, которая защищает личный домен |
| Служба</br>Привязка | Не установленное подключение к другим службам Azure |
| Управляемые</br>Идентификация | Управляемое удостоверение с помощью Azure Active Directory позволяет приложению легко получать доступ к другим ресурсам, защищенным Azure AD, таким как Azure Key Vault |
| Постоянный</br>Память | Параметр, позволяющий сохранять данные после перезапуска приложения |

## <a name="deployment"></a>Развертывание

Следующие функции и свойства определяются на уровне развертывания. они будут передаваться при обмене рабочими и промежуточными развертываниями.

| Компоненты | Описание |
|:--|:----------------|
| ЦП | Число виртуальных ядер на экземпляр приложения |
| Память | ГБ памяти на экземпляр приложения|
| Экземпляр</br>Счетчик | Число экземпляров приложения, устанавливаемое вручную или автоматически |
| Автоматическое масштабирование | Масштабирование количества экземпляров автоматически на основе предопределенных правил и расписаний |
| ВИРТУАЛЬНОЙ машины Java</br>Варианты | Настройка параметров ВИРТУАЛЬНОЙ машины Java  |
| Среда</br>Переменные | Настройка переменных среды |
| Среда выполнения</br>Версия | Java 8 или Java 11|

## <a name="restrictions"></a>Ограничения

* **Приложение должно иметь одно рабочее развертывание**: Удаление рабочего развертывания блокируется API-интерфейсом. Перед удалением он должен быть переключен на промежуточное хранение.
* **Приложение может иметь не более двух развертываний**: создание более двух развертываний блокируется API. Разверните новый двоичный файл в существующем производственном или промежуточном развертывании.
* **Управление развертыванием недоступно на уровне "базовый**": используйте уровень "Стандартный" для функции развертывания "синий-зеленый".
