# <a name="use-persistent-storage-in-azure-spring-cloud"></a>Использование постоянного хранилища в Azure Spring Cloud

Azure Веснного облака предоставляет два типа хранилища для вашего приложения: постоянные и временные.

По умолчанию Azure Веснное облако предоставляет временное хранилище для каждого экземпляра приложения. Временное хранилище ограничено 5 ГБ на экземпляр с помощью пути подключения по умолчанию/ТМП.

> [!WARNING]
> При перезапуске экземпляра приложения связанное временное хранилище будет окончательно удалено.

Постоянное хранилище — это контейнер файлового ресурса, управляемый Azure и выделяемый для каждого приложения. Данные, хранящиеся в постоянном хранилище, совместно используются всеми экземплярами приложения. В облачном экземпляре Azure весны может быть не более 10 приложений с включенным постоянным хранилищем. Для каждого приложения выделяется 50 ГБ постоянного хранилища. Путь подключения по умолчанию для постоянного хранилища —/персистент.

> [!WARNING]
> Если отключить постоянное хранилище приложений, все это хранилище будет освобождено, а все хранящиеся данные будут утеряны.

## <a name="use-the-azure-portal-to-enable-persistent-storage"></a>Использование портал Azure для включения постоянного хранилища

1. На **домашней** странице портал Azure выберите **все ресурсы**.

    >![Перейти к значку "все ресурсы"](media/portal-all-resources.jpg)

1. Выберите облачный ресурс Azure "Весна", требующий постоянного хранения. В этом примере выбранное приложение называется « **Весна**».

    > ![Выберите приложение.](media/select-service.jpg)

1. Под заголовком **Параметры** выберите **приложения**.

1. Облачные службы Azure "Весна" отображаются в таблице.  Выберите службу, к которой необходимо добавить постоянное хранилище. В этом примере выбрана служба **шлюза** .

    > ![Выбор своей службы](media/select-gateway.jpg)

1. На странице настройки службы выберите **Конфигурация** .

1. Перейдите на вкладку **постоянное хранилище** и выберите **включить**.

    > ![Включение постоянного хранилища](media/enable-persistent-storage.jpg)

После включения постоянного хранилища его размер и путь будут показаны на странице Конфигурация.

## <a name="use-the-azure-cli-to-modify-persistent-storage"></a>Использование Azure CLI для изменения постоянного хранилища

При необходимости установите расширение "Весна Cloud" для Azure CLI:

```azurecli
az extension add --name spring-cloud
```
Другие операции:

* Создание приложения с включенным постоянном хранилищем:

    ```azurecli
    az spring-cloud app create -n <app> -g <resource-group> -s <service-name> --enable-persistent-storage true
    ```

* Чтобы включить постоянное хранилище для существующего приложения, выполните следующие действия.

    ```azurecli
    az spring-cloud app update -n <app> -g <resource-group> -s <service-name> --enable-persistent-storage true
    ```

* Отключение постоянного хранилища в существующем приложении:

    ```azurecli
    az spring-cloud app update -n <app> -g <resource-group> -s <service-name> --enable-persistent-storage false
    ```

    > [!WARNING]
    > Если отключить постоянное хранилище приложений, все это хранилище будет освобождено, а все сохраненные данные будут безвозвратно утеряны.
