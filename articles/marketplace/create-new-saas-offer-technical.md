# <a name="how-to-add-technical-details-for-your-saas-offer"></a>Как добавить технические сведения о предложении SaaS

В этой статье описывается, как ввести технические сведения, которые помогут вам подключиться к вашему решению в коммерческом магазине Майкрософт. Это подключение позволяет нам подготавливать ваше предложение для клиента, если оно запрашивает и управляет им. Дополнительные сведения об этих параметрах см. в разделе [технические сведения](plan-saas-offer.md#technical-information).

> [!NOTE]
> Если вы решите обрабатывать транзакции независимо, этот параметр отображаться не будет. Вместо этого перейдите к [способу продвижения предложения SaaS](create-new-saas-offer-marketing.md).

## <a name="technical-configuration"></a>Техническая конфигурация

На вкладке **Technical Configuration (техническая конфигурация** ) вы определите технические сведения, используемые коммерческим магазином для взаимодействия с приложением SaaS или решением. 

- **URL-адрес целевой страницы** (обязательный) — определите URL-адрес сайта SaaS (например `https://contoso.com/signup` ,), который будут сопровождать клиенты после получения предложения от коммерческого рынка и запуска процесса настройки из вновь созданной подписки SaaS.

  > [!IMPORTANT]
  > На целевой странице должна быть установлена и запущена 24/7. Это единственный способ уведомления о новых покупках предложений SaaS, сделанных в коммерческом магазине, или запросы на настройку действующей подписки на предложение.

- **Веб-перехватчик подключения** (обязательно). для всех асинхронных событий, которые должны быть отправлены корпорацией Майкрософт (например, подписка SaaS была отменена), необходимо указать URL-адрес веб-перехватчика подключения. Мы будем называть этот URL-адрес для уведомления о событии.

  > [!IMPORTANT]
  > Веб-перехватчик должен работать в 24/7, так как это единственный способ уведомления об обновлениях для подписок SaaS клиентов, приобретенных через комерЦиал Marketplace.

- **Azure Active Directory идентификатор клиента** (обязательно) — чтобы найти идентификатор клиента для приложения Azure Active Directory (Azure AD), перейдите в колонку [Регистрация приложений](https://portal.azure.com/#blade/Microsoft_AAD_RegisteredApps/ApplicationsListBlade) в Azure Active Directory. В столбце **Отображаемое имя** выберите приложение. Затем найдите **идентификатор каталога (клиента)** в списке (например, `50c464d3-4930-494c-963c-1e951d15360e` ).

- **Azure Active Directory идентификатор приложения** (обязательно) — чтобы найти [идентификатор приложения](https://docs.microsoft.com/azure/active-directory/develop/howto-create-service-principal-portal#get-values-for-signing-in), перейдите в колонку [Регистрация приложений](https://portal.azure.com/#blade/Microsoft_AAD_RegisteredApps/ApplicationsListBlade) в Azure Active Directory. В столбце **Отображаемое имя** выберите приложение. Затем найдите идентификатор приложения (клиента) в списке (например, `50c464d3-4930-494c-963c-1e951d15360e` ).

Выберите **Сохранить черновик** , прежде чем перейти к следующей вкладке: Общие сведения о плане.
