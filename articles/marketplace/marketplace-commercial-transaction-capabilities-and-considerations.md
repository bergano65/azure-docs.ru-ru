# <a name="azure-marketplace-commercial-transaction-capabilities-and-considerations"></a>Возможности коммерческих транзакций в Azure Marketplace и рекомендации по их использованию

Варианты публикации в Azure Marketplace предоставляют уникальные способы связи поставщиков облачных программ и служб с клиентами. В этой статье рассматриваются следующие темы, связанные с коммерцией в Azure Marketplace:

* Варианты публикации в Marketplace
* Общие сведения о транзакциях
* Модели выставления счетов по транзакциям
* Требования к транзакциям

## <a name="marketplace-publishing-options"></a>Варианты публикации в Marketplace

Для издателей Azure Marketplace доступны перечисленные ниже варианты публикации.

### <a name="list--trial-publishing-options"></a>Публикация описаний и пробных версий

В Azure Marketplace издатели могут использовать публикацию описаний и пробных версий в рекламных целях и для привлечения пользователей. При публикации описаний и пробных версий корпорация Майкрософт не принимает прямого участия в транзакциях с лицензиями на программы издателя, и за такие транзакции не взимается плата. Издатели несут ответственность за поддержку всех аспектов транзакции с лицензией на программу, в том числе заказа, исполнения, измерения, выставления счетов, оплаты и сбора. При публикации описаний и пробных версий издатели оставляют себе 100 % платы за лицензирование программного обеспечения, которая взимается с пользователя. 

### <a name="transact-publishing-option"></a>Публикация транзакций

Помимо публикации описаний и пробных версий, издателям Azure Marketplace доступна публикация транзакций.   При этом используются доступные в глобальных масштабах коммерческие возможности корпорации Майкрософт. Этот вариант позволяет корпорации Майкрософт размещать транзакции в облачном магазине от имени издателя.

## <a name="transact-general-overview"></a>Общие сведения о транзакциях

При использовании варианта публикации transact, корпорация Microsoft предоставляет продажи стороннего программного обеспечения и развертывания некоторых типов предложение к подписке Azure клиента. Издатель должен учитывать выставление счетов за инфраструктуру Azure и лицензирование программ издателя при выборе модели выставления счетов и типа предложения в Azure Marketplace.

Сейчас публикация транзакций в Azure Marketplace поддерживается только для следующих типов предложений: виртуальные машины, приложения Azure и приложения SaaS.

### <a name="billing-infrastructure-costs"></a>Выставление счетов за использование инфраструктуры

**Для виртуальных машин и приложений Azure**

Для виртуальных машин и приложений Azure счета за использование инфраструктуры Azure выставляются подписке Azure пользователя.  Размер платы за использование инфраструктуры устанавливается и указывается отдельно от платы за лицензирование ПО издателя в выставляемом пользователю счете.

**Для приложений SaaS**

В случае приложений SaaS издателю нужно рассматривать тарифы на использование инфраструктуры Azure и лицензирование программного обеспечения как одну позицию расходов.  Она представляется как фиксированная плата для клиента. Использование инфраструктуры Azure контролируется, а счета выставляются непосредственно партнеру.  Фактическая стоимость использования инфраструктуры не видна пользователю.  Как правило, издатели добавляют плату за использование инфраструктуры Azure в счета за лицензирование программного обеспечения.  Плата за лицензирование программного обеспечения не измеряется и не взимается на основе потребления.

## <a name="transact-billing-models"></a>Модели выставления счетов по транзакциям

В зависимости от используемого варианта транзакции, плата за лицензирование программ издателя может быть представлена перечисленными ниже способами.  

* Бесплатно. Плата за программные лицензии не взимается. 

* Использование собственных лицензий (BYOL). Управление всеми применимыми выплатами за лицензии на программное обеспечение осуществляется непосредственно между издателем и пользователем. Корпорация Майкрософт обрабатывает только плату за использование инфраструктуры Azure. (Только для виртуальных машин и приложений Azure.)

* Оплата по мере использования. Плата за лицензии на программное обеспечение взимается согласно почасовому тарифу с учетом количества ядер (виртуальных ЦП) и использования инфраструктуры Azure. Это применимо только к виртуальным машинам и приложениям Azure.

* Плата за подписки (на основе сайтов). Плата за лицензирование программного обеспечения взимается на основе ежемесячного регулярного тарифа.  Это относится только к приложениям SaaS и Azure — управляемым приложениям.

* Бесплатная пробная версия. Плата за 30- или 90-дневные лицензии на программное обеспечение не взимается.

### <a name="free-and-bring-your-own-license-byol-pricing"></a>Предоставление решений бесплатно или с использованием собственной лицензии (BYOL)

При публикации предложений бесплатно или с использованием собственной лицензии корпорация Майкрософт не принимает участия в совершении транзакций по оплате лицензий на программное обеспечение. Как и при публикации описаний и пробных версий, издатель оставляет себе 100 % стоимости лицензий. 

### <a name="pay-as-you-go-and-subscription-site-based-pricing"></a>Оплата по мере использования и тарифы на подписки (на основе сайтов)

При публикации предложений с оплатой по мере использования или по количеству подписок корпорация Майкрософт предоставляет технологии и услуги для обработки покупок и возвратов лицензий на программное обеспечение, а также возвратных платежей. В этом случае издатель разрешает корпорации Майкрософт действовать в качестве агента для этих целей. Издатель разрешает корпорации Майкрософт использовать транзакции по лицензированию программного обеспечения, в то же время сохранив свою роль продавца, поставщика, распространителя и лицензиара.

Корпорация Майкрософт позволяет клиентам заказывать, лицензировать и использовать программное обеспечение издателя, руководствуясь как условиями использования Azure Marketplace, так и лицензионным соглашением между издателем и пользователем (см. Портал Cloud Partner). Издатели должны предоставлять свое лицензионное соглашение с пользователем в предложении Marketplace.

Счета по заказам, обрабатываемым через Marketplace, выставляются подписке Azure пользователя в виде одной накладкой, как и за использование инфраструктуры Azure. Клиенты могут выбирать предпочтительный способ выставления счетов и платежные средства, используемые ими для оплаты подписки Azure.

### <a name="free-software-trials"></a>Бесплатные пробные версии

При публикации транзакций издатель может бесплатно предоставить лицензию на программное обеспечение на 30 или 90 дней. Это предложение не распространяется на стоимость использования инфраструктуры Azure, связанную с использованием партнерского решения.

### <a name="private-offers"></a>Частные предложения

Издатели могут не только использовать типы предложений и модели выставления счетов для монетизации предложения, но и предоставлять частную версию решения с оговариваемыми в личном порядке ценами и пользовательскими конфигурациями при помощи настраиваемого образа. Частные предложения поддерживаются во всех 3 вариантах публикации.

Сформированная таким образом цена может быть выше или ниже цены для широкой публики.  Частные предложения могут содержать как скидки, так и премиум-содержимое. Частные предложения можно предоставлять одному или нескольким клиентам, добавляя их подписки Azure на уровне предложения.

### <a name="customer-invoicing-payment-billing-and-collections"></a>Выставление счетов, оплата и сбор средств у клиента

**Выставление счетов и оплата**

Издатель может использовать предпочитаемый клиентом способ выставления счетов для предоставления накладных за подписки или лицензии на программное обеспечение PAYGO.

**Соглашение Enterprise** 

Если клиент предпочитает использовать для выставления счетов соглашение Microsoft Enterprise, то стоимость лицензии на программное обеспечение будет взиматься с использованием этого способа как постатейная сумма отдельно от платы за использование Azure.

**Кредитные карты и ежемесячные накладные** 

Клиенты также могут оплачивать решения с помощью кредитных карт и ежемесячных накладных. В этом случае стоимость лицензий на программное обеспечение взимается точно так же, как и при использовании Соглашения Enterprise, в виде постатейной суммы отдельно от платы за использование Azure.

Например, если клиент приобретает решение с помощью кредитной карты:

**Бесплатные кредиты и денежные обязательства** 

Некоторые клиенты предварительно оплачивают Azure с помощью денежного обязательства в Соглашении Enterprise или получают бесплатные кредиты для использования с Azure. Эти кредиты можно использовать для оплаты использования Azure, но они не подходят для оплаты лицензий на программное обеспечение издателя.

**Выставление счетов и сборы** 

Счета за лицензии на программное обеспечение издателя предоставляются при помощи выбранного клиентом способа в соответствии с расписанием выставления счетов. Клиенты без Соглашения Enterprise ежемесячно получают счета за лицензии на программное обеспечение из Marketplace. Клиенты с Соглашением Enterprise ежемесячно оплачивают счета через накладную, представляемую раз в квартал.

Если выбрана ценовая модель на основе подписок или с оплатой по мере использования, то корпорация Майкрософт выступает в качестве агента издателя и отвечает за все аспекты выставления счетов, оплаты и сбора.

### <a name="publisher-payout-and-reporting"></a>Отчеты и выплаты издателя

* Из платы за лицензирование программного обеспечения, которую собирает корпорация Майкрософт, вычитается операционный сбор в размере 20 % (если не указано иное) во время выплаты издателю.

* Как правило, клиенты покупают решения при помощи Соглашения Enterprise или соглашения об оплате по мере использования с поддержкой кредитных карт. Тип соглашения определяет расписание выставления счетов, сбора и выплат.

>[!NOTE] 
>Все отчеты и данные аналитики при публикации транзакций доступны в разделе "Аналитика" на Портале Cloud Partner.

## <a name="transact-requirements"></a>Требования к транзакциям

В этом разделе описываются требования к транзакциям для предложений различных типов.

### <a name="requirements-for-all-offer-types"></a>Требования для предложений всех типов

**Учетная запись Центра разработки и учетная запись Майкрософт** 

* Для публикации транзакций необходимы учетная запись Центра разработки и учетная запись Майкрософт, независимо от ценовой модели предложения.
* В учетной записи Центра разработки хранятся все финансовые данные, необходимые корпорации Майкрософт для сбора средств у клиентов от имени издателя и их выплаты издателю.
* Несмотря на то, что можно использовать так же организации или Microsoft входные данные в обеих учетных записях, Центр разработчиков — это отдельная учетная запись из учетной записи облачного портала для партнеров. Для публикации транзакций издатель должен зарегистрировать учетную запись Центра разработки, а также зарегистрироваться на Портале Cloud Partner.

*Дополнительные сведения о настройке этих учетных записей см. в статье [Регистрация в качестве издателя в облаке Marketplace](https://docs.microsoft.com/azure/marketplace/become-publisher).*

### <a name="requirements-for-specific-offer-types"></a>Требования для предложений определенных типов

Публикация транзакций доступна для использования только с предложениями Marketplace следующих типов: 

**Виртуальная машина** 

Выберите ценовую модель с бесплатным использованием, с использованием собственной лицензии или с оплатой по мере использования и представьте номера SKU, определенные на уровне предложения. В выставляемом клиенту счете за Azure корпорация Майкрософт указывает стоимость лицензий на программное обеспечение издателя отдельно от базовых тарифов на использование инфраструктуры Azure. Стоимость использования инфраструктуры Azure зависит от использования программного обеспечения издателя.

**Приложения Azure. Шаблон решения или управляемое приложение** 

Необходимо подготовить одну или несколько виртуальных машин и посчитать сумму тарифов за их использование. Для управляемых приложений в одном плане можно выбрать ежемесячную подписку с фиксированным тарифом вместо оплаты по количеству виртуальных машин. В обоих случаях стоимость использования инфраструктуры Azure передается клиенту отдельно от цен лицензий на программное обеспечение, но в том же платежном документе.
