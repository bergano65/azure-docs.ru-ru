# <a name="saas-application-technical-info-tab"></a>Вкладка технической информации для приложения SaaS

На вкладке Technical Info (Техническая информация) предусмотрена форма технической конфигурации. Используйте эту форму, чтобы выбрать тип создаваемого приложения SaaS и настроить способ его предоставления клиентам.

![Форма технической конфигурации](./media/saas-techinfo-techconfig.png)

## <a name="technical-configuration-form"></a>Форма технической конфигурации

Эта форма содержит 2 поля: Product (Продукт) и Call to action (Призыв к действию).

### <a name="product-field"></a>Поле Product (Продукт)

Приложения SaaS можно предоставить для обоих из следующих онлайн-магазинов:
- для бизнес-пользователей, выбрав параметр **Listing** (Список);
- для пользователя с правами ИТ-администратора, выбрав параметр **продать через Microsoft**.
Чтобы определить, какой тип приложения SaaS вы создаете, прочтите раздел [Выбор правильного онлайн-магазина](https://docs.microsoft.com/azure/marketplace/determine-your-listing-type#understand-storefront-selection).

#### <a name="sell-through-microsoft"></a>Продажа через корпорацию Майкрософт
Для создания этой возможности необходимо выполнить следующие настройки:

- Подключите веб-сайт службы SaaS к API-интерфейсам SaaS корпорации Майкрософт. Создание такого подключения описано в статье [Продажа SaaS с помощью Azure. API-интерфейсы](https://docs.microsoft.com/azure/marketplace/cloud-partner-portal-orig/cloud-partner-portal-saas-subscription-apis).
- Включите функцию продажи через Azure на Портале Cloud Partner в форме технической конфигурации и предоставьте необходимые сведения. Дополнительные сведения об этой модели выставления счетов и ее реализации см. в статье [Продажа SaaS с помощью Azure](https://docs.microsoft.com/azure/marketplace/cloud-partner-portal-orig/cloud-partner-portal-saas-offer-subscriptions#overview).

 ![Форма продажи через корпорацию Майкрософт](./media/saas-techinfo-sellthrough-ms.png)

Ниже перечислены обязательные поля для продажи через корпорацию Майкрософт.

|  **Имя поля**   |  **Описание**  |
|  ---------------  |  ---------------  |
|    Идентификаторы подписок для предварительного просмотра               |    Идентификаторы всех подписок Azure, которые будут использоваться для тестирования предложения в предварительной версии, прежде чем оно станет общедоступным.               |
|     Инструкции по началу работы              |   Инструкции, которые вы предоставите клиентам для помощи в подключении к приложению SaaS. Допускается использование базовых HTML-тегов, например &lt;p&gt;, &lt;h1&gt;, &lt;li&gt; и т. п.                |
|    URL-адрес целевой страницы  |   URL-адрес сайта, на который вы будете направлять клиентов при переходе с портала Azure. Этот же URL-адрес выполняет роль конечной точки, где будут размещены API-интерфейсы подключения для совместного бизнеса с корпорацией Майкрософт.                |
|  Веб-перехватчик подключения    |  Для всех асинхронных событий, которые корпорация Майкрософт должна отправлять вам от имени клиента (например, о прекращении действия подписки Azure), нам нужно предоставить веб-перехватчик подключения. Если вы еще не используете систему веб-перехватчиков, проще всего настроить конечную точку HTTP для приложения логики, которое будет прослушивать все поступающие на нее события и обрабатывать их соответствующим образом. Дополнительные сведения см. в статье <a href="https://docs.microsoft.com/azure/logic-apps/logic-apps-http-endpoint">Вызовы, срабатывания триггеров и создание вложенных рабочих процессов в приложениях логики</a>.                |
|  Идентификатор клиента Azure AD и идентификатор приложения      |   На портале Azure необходимо создать приложение Active Directory, чтобы мы могли проверять аутентификацию для подключения между нашими службами. Для заполнения этих полей создайте приложение AD и получите из него необходимый идентификатор клиента и идентификатор приложения. Обратите внимание на то, что идентификатор приложения связан с publisherID (идентификатором издателя). Поэтому убедитесь, что во всех предложениях используется один и тот же идентификатор приложения.             |


Наконец при выборе параметра **Sell through Microsoft** (Продажа через корпорацию Майкрософт) появляется другая вкладка нового предложения под названием **Plans** (Планы). 

На вкладке [Plans](./cpp-plans-tab.md) (Планы) перечислены поддерживаемые приложением SaaS планы с указанием их цен. На данный момент мы поддерживаем только фиксированную ежемесячную оплату с возможностью разрешить бесплатный доступ на 1 или 3 месяца. Все планы и цены должны строго соответствовать тем планам и ценам, которые указаны на вашем сайте этого приложения SaaS.

>[!NOTE] 
>Планы необходимы только в том случае, если выбрана продажа через корпорацию Майкрософт.

### <a name="call-to-action-field"></a>Поле Call to action field (Призыв к действию)

Поле Call to action field (Призыв к действию) позволяет выбрать сообщение, которое отображается на кнопке приобретения вашего предложения. Доступны следующие варианты:

- Бесплатно. Если вы выберете этот вариант, будет предложено ввести URL-адрес пробной версии, по которому клиенты могут получить доступ к приложению SaaS. Например: https://contoso.com/trial
- Бесплатная пробная версия. Если вы выберете этот вариант, будет предложено ввести URL-адрес пробной версии, по которому клиенты могут получить доступ к приложению SaaS. Например: https://contoso.com/trial
- Свяжитесь со мной

Дополнительные сведения о вариантах выбора для поля Call to action field (Призыв к действию) см. в разделе "Выбор варианта публикации".
