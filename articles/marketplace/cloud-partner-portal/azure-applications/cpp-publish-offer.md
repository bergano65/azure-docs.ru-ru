# <a name="publish-azure-application-offer"></a>Публикация предложения приложения Azure

Опубликовать предложение можно после создания предложения с информацией на странице **Новое предложение**. Выберите **Публиковать**, чтобы начать процесс публикации.

На следующей схеме представлены основные шаги в процессе публикации для запуска предложения.

![Шаги публикации предложения](./media/offer-publishing-steps.png)


## <a name="detailed-description-of-publishing-steps"></a>Подробное описание шагов публикации

В следующей таблице перечисляется и описывается каждый шаг публикации, а также приводится оценка времени для завершения каждого шага.  Временные оценки в "днях" определяются как рабочие дни, исключая выходные и праздничные дни.

|  **Шаг публикации**           | **Время**    | **Описание**                                                            |
|  -------------------           | --------    | ---------------                                                            |
| Проверка необходимых компонентов         | < 15 мин    | Информация и параметры предложения — проверенные.                        |
| Проверка параметров, которые повлияли на доход | < 15 мин  | Проверяется определение использования ресурсов Azure для предложения.             |
| Сертификация                  | < 1 день     | Команда сертификации Azure анализирует предложение. Предложение проверяется на предмет вирусов, вредоносных программ, соответствия стандартам безопасности и проблем безопасности. Это позволяет определить, соответствует ли предложение всем критериям применимости. Дополнительные сведения см. в разделе [Предварительные требования](./cpp-prerequisites.md). Отзывы предоставляются в случае обнаружения проблемы. |
| Проверка тестового выпуска          | < 2 часа   | (Необязательно.) Если присутствует тестовый выпуск, корпорация Майкрософт проверяет возможность его развертывания и репликации.  |
| Создание пакета и регистрация для создания интересов | < 1 час  | Технические ресурсы предложения упакованы для использования клиентами, а ведущие системы настроены и развернуты. |

Вы можете следить за процессом публикации на вкладке **Состояние** вашего предложения на Портале Cloud Partner.

![Вкладка "Состояние" для предложения приложения Azure](./media/offer-status-tab.png)

После публикации предложение будет добавлено в список в [категории приложений Microsoft Azure Marketplace](https://azuremarketplace.microsoft.com/marketplace/apps/).

>[!Note]
>Поставщики решений (CSP) партнер канала необязательно в облако теперь доступна.  См. в разделе [поставщиков облачных решений](../../cloud-solution-providers.md) Дополнительные сведения о маркетинг вашего предложения с помощью Microsoft CSP partner каналов.

## <a name="errors-and-review-feedback"></a>Ошибки и отзывы в результате проверки

Помимо отображения состояния публикации вашего предложения на вкладке **Состояние** также показаны сообщения об ошибках и отзывы о любых этапах публикации, на которых возникла проблема.  Если проблема является критической, публикация отменяется.  После этого необходимо устранить проблемы, о которых было сообщено, а затем повторно опубликовать предложение.  Поскольку шаг **Проверка службой Майкрософт** является подробным анализом предложения и связанных с ним технических средств (особенно шаблона Azure Resource Manager), обычно ошибки представляются в виде ссылок запроса на вытягивание (PR).  Сведения о выполнении просмотра и реагирования см. в статье [Handling review feedback](./cpp-handling-review-feedback.md) (Обработка отзывов по результатам проверки).


## <a name="next-steps"></a>Дальнейшие действия

Если ошибки возникли на одном или нескольких этапах публикации, их необходимо исправить и выполнить повторную публикацию приложения.  Если критические ошибки возникли на этапе **Проверка службой Майкрософт**, необходимо [обработать отзыв о проверке](./cpp-handling-review-feedback.md), перейдя в репозиторий Azure DevOps команды проверки Майкрософт.

После успешной публикации приложения Azure вы можете [обновить существующее предложение](./cpp-update-existing-offer.md), чтобы отразить изменяющиеся технические или бизнес-требования.
