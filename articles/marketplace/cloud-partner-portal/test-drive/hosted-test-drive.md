# <a name="hosted-test-drive"></a>Размещенный тестовый выпуск

Размещенный тестовый выпуск устраняет сложности, связанные с настройкой, за счет размещения в корпорации Майкрософт и обслуживания службы, выполняющей подготовку и отзыв пользователей тестового выпуска. Эта статья предназначена для издателей, которые имеют предложения в AppSource или создают новое и хотят предложить размещенный тестовый выпуск, который подключается к экземпляру Dynamics 365 for Customer Engagement, Dynamics 365 for Finance and Operations или Dynamics 365 Business Central.

## <a name="how-to-publish-a-test-drive"></a>Публикация тестового выпуска

Создайте предложение или перейдите к имеющемуся.

Выберите параметр "Тестовый выпуск" в боковом меню.

Выберите Да для параметра Enable a Test Drive (Включить тестовый выпуск).

Укажите значения для следующих полей в разделе Сведения.

- **Описание**: представьте обзор вашего тестового выпуска. Этот текст будет отображаться для пользователя во время подготовки тестового выпуска. Это поле поддерживает HTML, поэтому вы можете предоставить форматированное содержимое.
- **Руководство пользователя**: отправьте подробное руководство пользователя (файл типа PDF), из которого пользователи тестового выпуска смогут узнать, как использовать ваше приложение.
- **Test Drive Demo Video** (Демонстрационное видео тестового выпуска): при необходимости отправьте видео, демонстрирующее приложение.

Предоставляйте разрешение AppSource на подготовку и отзыв пользователей тестового выпуска в клиенте с помощью инструкций, расположенных [здесь](https://github.com/Microsoft/AppSource/blob/patch-1/Microsoft%20Hosted%20Test%20Drive/Setup-your-Azure-subscription-for-Dynamics365-Microsoft-Hosted-Test-Drives.md).

На этом шаге вы создадите значения идентификатора Azure AD App и ключа Azure AD App, приведенные ниже.

Укажите значения для следующих полей в разделе Technical Configuration\' (Техническая конфигурация):

- **Type of Test Drive** (Тип тестового выпуска): выберите значение Microsoft Hosted (example Dynamics 365 for Customer Engagement) (Размещено в Майкрософт (например, Dynamics 365 for Customer Engagement)). Это означает, что в корпорации Майкрософт будет размещаться и обслуживаться служба, выполняющая подготовку и отзыв пользователя тестового выпуска.
- **Max Concurrent Test Drives** (Максимальное количество одновременных тестовых выпусков): задайте для этого поля количество одновременно работающих пользователей, у которых может быть активный тестовый выпуск в любой момент времени. Каждый пользователь будет использовать лицензию Dynamics, пока активен его тестовый выпуск, поэтому вам необходимо убедиться в наличии по крайней мере такого количества доступных лицензий Dynamics для пользователей тестового выпуска. Рекомендуемое значение: 3–5.
- **Test Drive Duration (hours)** (Продолжительность тестового выпуска (в часах)): задайте для этого поля количество часов, в течение которых тестовый выпуск будет активным. По истечении указанного количества часов пользователь будет отозван из вашего клиента. Рекомендуемое значение — от 2 до 24 часов в зависимости от сложности приложения. Пользователь всегда может запросить еще один тестовый выпуск, если ему не хватило времени и требуется снова получить доступ к тестовому выпуску.
- **URL-адрес экземпляра**: укажите URL-адрес, по которому пользователь тестового выпуска изначально перейдет при запуске тестового выпуска. Как правило, это URL-адрес экземпляра Dynamics 365, в котором установлено приложение и демонстрационные данные. Пример значения: https: \//тестдриве. CRM. Dynamics. com
- **Azure AD Tenant ID** (Идентификатор клиента Azure AD): укажите идентификатор клиента Azure для экземпляра Dynamics 365. Чтобы получить это значение, войдите на портал Azure, перейдите к Azure Active Directory, выберите "Свойства" из колонки меню и скопируйте идентификатор каталога. Пример значения: 72f988bf-86f1-41af-91ab-2d7cd0111234.
- **Azure AD App ID** (Идентификатор Azure AD App): идентификатор Azure AD App, созданного на шаге 7. Пример значения: 53852862-a2ae-4e43-9461-faa49650a096.
- **Azure AD App Key** (Ключ Azure AD App): секрет для Azure AD App, созданного на шаге 7. Пример значения: IJUgaIOfq9b9LbUjeQmzNBW4VGn6grr1l/n3aMrnfdk=.
- **Azure AD Tenant Name** (Имя клиента Azure AD): укажите имя клиента Azure для экземпляра Dynamics 365. Используйте формат \<имя_клиента.\>onmicrosoft.com. Пример значения: testdrive.onmicrosoft.com.
- **Instance Web API URL** (URL-адрес веб-API экземпляра): укажите URL-адрес веб-API для экземпляра Dynamics 365. Чтобы получить это значение, войдите в экземпляр Microsoft Dynamics 365 и выберите "Параметры -\> Настройка -\> Ресурсы разработчика -\> Веб-API экземпляра" (скопируйте этот URL-адрес). Пример значения: https: \//тестдриве. CRM. Dynamics. com/API/Data/v 9.0 
- **Имя роли**: укажите имя пользовательской роли безопасности Dynamics 365, созданной для тестового выпуска. Эта роль будет назначаться пользователям во время запуска их тестового выпуска. Пример значения: тестдривероле

## <a name="next-steps"></a>Дальнейшие действия

Когда вы будете готовы **опубликовать** свое предложение, после сертификации приложения вы получите **предварительную версию** своего предложения. Запустите тестовый выпуск в пользовательском интерфейсе и убедитесь, что он работает надлежащим образом. Когда вы будете довольны своим предложением предварительной версии, наступит время **запустить его!**
