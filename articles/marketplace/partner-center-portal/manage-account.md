# <a name="manage-your-commercial-marketplace-account-in-partner-center"></a>Управление коммерческой учетной записью Marketplace в Центре партнеров

После того как вы [создали учетную запись Центра партнеров](./create-account.md), вы можете управлять коммерческой учетной записью и предложениями с помощью [панели мониторинга Marketplace](https://partner.microsoft.com/dashboard/commercial-marketplace/overview).

В этой статье мы рассмотрим, как управлять учетной записью Центра партнеров, включая следующие задачи:

- [Получение доступа к параметрам учетной записи Центра партнеров](#access-your-account-settings).
- [Нахождение идентификаторов издателя, Symantec, продавца, пользователя, MPN, а также клиентов Azure AD](#account-details).
- [Обновление контактных данных](#contact-info).
- [Настройка GUID отслеживания для мониторинга использования ресурсов клиентами](#tracking-guids).
- [Управление пользователями](#manage-users)
- [Управление группами](#manage-groups)
- [Управление приложениями Azure AD](#manage-azure-ad-applications).
- [Определение ролей пользователей и разрешений](#define-user-roles-and-permissions).
- [Управление клиентами Azure AD (рабочие учетные записи)](#manage-tenants).
- [Управление соглашениями Центра партнеров](#agreements).

## <a name="access-your-account-settings"></a>Доступ к параметрам учетной записи

Если вы еще этого не сделали, вы (или администратор вашей организации) должны получить доступ к [параметрам учетной записи](https://partner.microsoft.com/dashboard/account/management), чтобы выполнить указанные ниже действия с помощью учетной записи Центра партнеров:
- Проконтролировать состояние проверки учетной записи компании.
- Подтвердить идентификатор Symantec, продавца, Microsoft Partner Network (MPN), издателя и контактную информацию, включая контактную информацию утверждающего лица и продавца в компании.
- Создать учетные записи пользователей для всех, кто будет использовать вашу учетную запись для бизнеса в Центре партнеров.

### <a name="open-developer-settings"></a>Открытие параметров разработчика

**Параметры учетной записи** расположены в правом верхнем углу [панели мониторинга коммерческой учетной записи Marketplace](https://partner.microsoft.com/dashboard/commercial-marketplace) в Центре партнеров. Выберите значок шестеренки (в правом верхнем углу панели мониторинга), а затем щелкните  **Параметры разработчика**.

![Меню параметров учетной записи в Центре партнеров](./media/dashboard-developer-settings.png)

В разделе **Настройки учетной записи** вы сможете просматривать:
- **Сведения об учетной записи**: тип и состояние учетной записи.
- **Идентификаторы издателей**: идентификатор продавца, пользователя, издателя, а также клиенты Azure AD и т. д.
- **Контактные данные**: отображаемое имя издателя, контактное имя продавца, электронную почту, телефон и адрес.
- **GUID отслеживания**: все GUID отслеживания, связанные с вашей учетной записью.

### <a name="account-details"></a>Сведения об учетной записи

В разделе сведений об учетной записи можно просмотреть основные сведения, например **тип учетной записи** (компании или частного лица) и **состояние проверки** вашей учетной записи. Во время проверки учетной записи эти параметры будут отображать все необходимые шаги, включая проверку электронной почты, проверку данных о приеме на работу и проверку сведений об организации. Здесь вы также можете обновить электронную почту и повторно отправить данные о проверке, если это необходимо.

### <a name="publisher-ids"></a>Идентификаторы издателей

В разделе "Идентификаторы издателей" отображаются **идентификаторы Symantec**, **продавца**, **пользователя**, **MPN**, а также **клиенты Azure AD**. Эти значения назначаются корпорацией Майкрософт для уникальной идентификации учетной записи разработчика и не могут быть изменены.

### <a name="contact-info"></a>Контактные данные

В разделе "Контактные данные" можно найти **отображаемое имя издателя**, **контактные данные продавца** (имя контактного лица, адрес электронной почты, номер телефона и адрес продавца компании), а также **утверждающего лица в компании** (имя, адрес электронной почты и номер телефона лица с полномочиями на утверждение решений для компании).

#### <a name="payout-account"></a>Счет для выплат

Счет для выплат — это банковский счет, на который мы будем отправлять вашу выручку с продаж. Этот банковский счет должен находиться в той же стране или регионе, где была зарегистрирована учетная запись Центра партнеров.

Чтобы настроить учетную запись для выплат, сделайте следующее:

1. Перейдите на [страницу обзора коммерческой платформы](https://partner.microsoft.com/dashboard/commercial-marketplace/overview) в Центре партнеров.
2. В разделе "Профиль" рядом с параметром **Профиль выплат** выберите **Обновить**.
3. **Выберите метод оплаты**: банковский счет или PayPal.
4. **Добавьте платежную информацию**. Сюда может относиться выбор типа счета (текущий или сберегательный), ввод имени владельца и номера счета, номера отделения банка, адреса для выставления счетов, номера телефона или адреса электронной почты PayPal. *Дополнительные сведения об использовании PayPal в качестве метода оплаты и о том, поддерживается ли эта возможность в вашем регионе продажи, см. в разделе со [сведениями о PayPal](https://docs.microsoft.com/windows/uwp/publish/setting-up-your-payout-account-and-tax-forms#paypal-info).

> [!IMPORTANT]
> Изменение счета для выплат может привести к задержке выплат на один цикл. Эта задержка связана с необходимостью проверки изменения счета, которая обычно выполняется при добавлении первого счета для выплат. После проверки вам будет выплачена полная сумма. Все выплаты, не отправленные в текущем периоде, переносятся на следующий.  

#### <a name="tax-profile"></a>Налоговый профиль

Проверьте текущее состояние налогового профиля, подтверждая правильность отображаемого **типа сущности** и **сведений о налоговом сертификате**. Выберите **Изменить**, чтобы обновить или заполнить все необходимые формы.

Чтобы установить состояние налогообложения, необходимо указать страну или регион проживания, гражданство и заполнить соответствующие налоговые формы, связанные с вашей страной или регионом.

Для продажи любого предложения через корпорацию Майкрософт вы обязаны заполнить налоговые формы США независимо от страны или региона вашего проживания или гражданства. Партнеры, соответствующие определенным критериям проживания в США, должны заполнить форму IRS W-9. Другие партнеры, находящиеся за пределами США, заполняют форму IRS W-8. Вы можете заполнить эти формы онлайн, так же как и налоговый профиль.

ITIN (индивидуальный идентификационный номер налогоплательщика) для получения выплат от корпорации Майкрософт или льгот по соглашению об избежании двойного налогообложения не требуется.

Налоговые формы можно заполнять и отправлять в Центре партнеров в электронном виде; в большинстве случаев распечатывать и отправлять формы по почте не требуется.

В разных странах и регионах действуют разные налоговые требования. Точный объем подлежащих уплате налогов зависит от того, в каких странах и регионах вы продаете свое предложение. В некоторых странах и регионах Корпорация Майкрософт оплачивает налог на продажи и использование от вашего имени. Эти страны и регионы будут идентифицированы в процессе добавления вашего предложения. В других странах и регионах, в зависимости от того, где вы зарегистрированы, вам нужно будет уплачивать налог на продажи и использование в местные налоговые органы самостоятельно. Кроме того, получаемые вами средства от продажи могут облагаться налогами как доходы. Настоятельно рекомендуем обратиться за консультацией о том, какие налоги относятся к вашим транзакциям по продажам через корпорацию Майкрософт, в соответствующий орган вашей страны или региона.

##### <a name="withholding-rates"></a>Налоговые ставки

От того, какие данные вы укажете в налоговых формах, будет зависеть налоговая ставка. Налоги удерживаются только с продаж покупателям, находящимся в США; с продаж покупателям из других стран налоги не удерживаются. Налоговая ставка может быть разной, но для большинства разработчиков, находящихся за пределами США, она по умолчанию составляет 30 %. Ее можно уменьшить, если ваша страна или регион подписали с США соглашение об избежании двойного налогообложения.

##### <a name="tax-treaty-benefits"></a>Налоговые льготы

Если вы находитесь за пределами США, то можете воспользоваться льготами по соглашению об избежании двойного налогообложения. Они зависят от конкретной страны или региона и могут позволить вам уменьшить объем налогов, которые удерживает корпорация Майкрософт. Чтобы запросить льготы по соглашению об избежании двойного налогообложения, заполните часть II формы W-8BEN. Вы можете узнать, доступны ли вам эти льготы, обратившись в соответствующие источники в вашей стране или регионе.

[Сведения о налогах для разработчиков приложений и игр Windows и издателей Azure Marketplace](https://docs.microsoft.com/windows/uwp/publish/tax-details-for-paid-apps).

#### <a name="payout-hold-status"></a>Состояние приостановки выплаты

По умолчанию Корпорация Майкрософт выполняет платежи на ежемесячной основе. Однако вы можете при необходимости поставить оплату на удержание, что позволит предотвратить отправку платежей в вашу учетную запись. Если вы решаете приостановить выплаты, мы будем продолжать регистрировать полученный доход и предоставлять подробные сведения в вашей **сводке о выплатах**. Но мы не будем отправлять никакие платежи на ваш счет, пока вы не отмените приостановку.

Чтобы приостановить платежи, перейдите в раздел **Настройки учетной записи**. В подразделе **Финансовые сведения** раздела **состояния приостановки выплаты** установите ползунок в положение **Вкл.** Вы можете изменить состояние приостановки выплат в любое время, но имейте в виду, что ваше решение начнет действовать со следующей ежемесячной выплаты. Например, если вы хотите приостановить выплату за апрель, необходимо **включить** состояние приостановки выплат до конца марта.

После того как вы **включите** состояние приостановки выплат, все выплаты будут приостановлены, пока вы не переключите ползунок в положение **Выкл.** После этого вы будете включены в следующий ежемесячный цикл выплат (при условии, что достигнуты необходимые минимальные суммы платежей). Например, если вы приостановили свои выплаты, но хотели бы получить выплату за июнь, необходимо **выключить** состояние приостановки выплат до конца мая.

> [!NOTE]
> Ваш выбор **состояния приостановки выплаты** применяется ко **всем** источникам доходов, которые оплачиваются через Центр партнеров Майкрософт, включая Azure Marketplace, AppSource, Microsoft Store, рекламу и т. д. Для каждого источника дохода нельзя выбрать разные состояния приостановки выплаты.

### <a name="tracking-guids"></a>GUID отслеживания

Глобальные уникальные идентификаторы (GUID) — это уникальные ссылочные номера (с 32 шестнадцатеричными цифрами), которые можно использовать для отслеживания использования Azure. 

Чтобы создать идентификаторы GUID для отслеживания, следует использовать генератор GUID. Команда разработчиков службы хранилища Azure создала [форму генератора GUID](https://aka.ms/StoragePartners), которая позволяет получить GUID в правильном формате на электронную почту и которую можно использовать в различных системах отслеживания.

Рекомендуется создавать уникальные GUID для всех предложений и каналов распределения каждого продукта. Можно использовать один идентификатор GUID для нескольких каналов распределения, если дробить отчетность нежелательно.

Если развернутый с помощью шаблона продукт доступен в Azure Marketplace и на сайте GitHub, можно создать и зарегистрировать два уникальных идентификатора GUID:

- продукт А в Azure Marketplace;
- продукт А на сайте GitHub.

Отчеты подготавливаются по идентификатору партнера Майкрософт и идентификатору GUID. Вы также можете отслеживать GUID на более детальном уровне в соответствии с каждым планом в вашем предложении.

Дополнительные сведения см. в статье [об отслеживании использования клиентов Azure с идентификаторами GUID](../azure-partner-customer-usage-attribution.md#faq).

## <a name="multi-user-account-management"></a>Управление учетными записями с несколькими пользователями

Центр партнеров использует [Azure Active Directory](../../active-directory/fundamentals/active-directory-whatis.md) (Azure AD) для доступа и управления учетными записями с несколькими пользователями. Клиент Azure AD вашей организации автоматически связывается с учетной записью Центра партнеров в процессе регистрации.

## <a name="manage-users"></a>Управление пользователями

Откройте раздел **Пользователи** в Центре партнеров (в разделе **Параметры учетной записи**), чтобы использовать Azure AD для управления пользователями, группами и приложениями Azure AD, имеющими доступ к учетной записи Центра партнеров. В вашей учетной записи должны быть разрешения [**уровня менеджера**](#define-user-roles-and-permissions) для [рабочей учетной записи (клиента Azure AD)](./company-work-accounts.md), в которой вы хотите добавить или изменить пользователей. Для управления пользователями в другой рабочей учетной записи (клиенте) необходимо выйти и снова войти в систему как пользователь с разрешениями **менеджера** для этой рабочей учетной записи (клиента).

Выполнив вход с помощью рабочей учетной записи (клиент Azure AD), вы можете:

- [добавлять или удалять пользователей](#add-existing-users);
- [изменять пароль пользователя](#change-a-user-password);
- [добавлять или удалять группы](#manage-groups);
- [добавлять или удалять приложения Azure AD](#add-new-azure-ad-applications);
- [управлять ключами для приложения Azure AD](#manage-keys-for-an-azure-ad-application);
- [определять роли пользователей и разрешения](#define-user-roles-and-permissions).

Помните, что все пользователи Центра партнеров (включая группы и приложения Azure AD) должны иметь активную рабочую учетную запись в [клиенте Azure AD](#manage-tenants), связанную с вашей учетной записью Центра партнеров.

### <a name="add-existing-users"></a>Добавление существующих пользователей

Чтобы добавить пользователей в учетную запись Центра партнеров, которая уже существует в [рабочей учетной записи вашей организации (клиент Azure AD)](./company-work-accounts.md), сделайте следующее:

1. Перейдите к параметру **Пользователи** (в разделе **Параметры учетной записи**) и выберите **Добавить пользователей**.
2. Выберите одного или нескольких пользователей в появившемся списке. Для поиска конкретных пользователей можно использовать поле поиска.
*При выборе нескольких пользователей для добавления в учетную запись Центра партнеров необходимо назначить им одну и ту же роль или набор настраиваемых разрешений. Чтобы добавить нескольких пользователей с разными ролями или разрешениями, повторите эти действия для каждой роли или набора настраиваемых разрешений.
3. Завершив выбор пользователей, нажмите кнопку **Добавить выбранные**.
4. В разделе **Роли** укажите роли или настраиваемые разрешения для выбранных пользователей.
5. Щелкните **Сохранить**.

### <a name="create-new-users"></a>Создание пользователей

Чтобы создать учетные записи пользователей, необходимо иметь учетную запись с разрешениями [**глобального администратора**](../../active-directory/users-groups-roles/directory-assign-admin-roles.md).

1. Перейдите к параметру **Пользователи** (в разделе **Параметры учетной записи**), выберите **Добавить пользователей**, а затем **Создать новых пользователей**.
1. Введите имя, фамилию и имя пользователя для каждого нового пользователя. 
1. Если вы хотите, чтобы новый пользователь имел учетную запись глобального администратора в каталоге вашей организации, установите флажок **Сделать этого пользователя глобальным администратором в Azure AD с полным доступом ко всем ресурсам каталогов**. Это обеспечит пользователю полный доступ ко всем административным функциям в Azure AD вашей компании. Они смогут добавлять пользователей в рабочую учетную запись (клиент Azure AD) вашей компании и управлять ими, но не в Центре партнеров, если только вы не предоставите для учетной записи соответствующие роли или разрешения.
1. Если вы установите флажок **Сделать этого пользователя глобальным администратором**, необходимо предоставить **адрес электронной почты для восстановления пароля**, чтобы пользователь при необходимости мог восстановить пароль.
1. В разделе **Членство в группах** выберите группы, к которым должен принадлежать новый пользователь.
1. В разделе **Роли** укажите роли или настраиваемые разрешения для пользователя.
1. Щелкните **Сохранить**.

При создании пользователя в Центре партнеров также будет создана учетная запись для этого пользователя в рабочей учетной записи (клиенте Azure AD), в которую вы вошли. Изменение имени пользователя в Центре партнеров приведет к тому, что те же изменения будут внесены в рабочую учетную запись (клиент Azure AD) вашей организации.

### <a name="invite-new-users-by-email"></a>Приглашение новых пользователей по электронной почте

Чтобы пригласить пользователей, которые в настоящее время не добавлены в рабочую учетную запись (клиент Azure AD) вашей организации, по электронной почте, вам необходимо иметь учетную запись с разрешениями [**глобального администратора**](../../active-directory/users-groups-roles/directory-assign-admin-roles.md).

1. Перейдите к параметру **Пользователи** (в разделе **Параметры учетной записи**), выберите **Добавить пользователей**, а затем **Invite users by email** (Пригласить пользователей по электронной почте).
2. Введите один или несколько адресов электронной почты (до 10), разделяя их запятыми или точками с запятой.
3. В разделе **Роли** укажите роли или настраиваемые разрешения для пользователя.
4. Щелкните **Сохранить**.

Приглашенные пользователи получат по электронной почте приглашение присоединиться к учетной записи Центра партнеров. В рабочей учетной записи (клиенте Azure AD) будет создана учетная запись гостевого пользователя. Каждый пользователь должен принять приглашение, прежде чем он сможет получить доступ к вашей учетной записи.

Если вам нужно повторно отправить приглашение, перейдите на страницу **Пользователи**, найдите приглашение в списке пользователей, выберите адрес электронной почты (или сообщение *Ожидание приглашения*). Затем в нижней части страницы выберите **Повторно отправить приглашение**.

> [!NOTE]
> Если ваша организация использует [интеграцию с каталогами](https://go.microsoft.com/fwlink/p/?LinkID=724033) для синхронизации локальной службы каталогов с Azure AD, вы не сможете создавать пользователей, группы или приложения Azure AD в Центре партнеров. Вам (или другому администратору в локальном каталоге) потребуется создать их непосредственно в локальном каталоге, чтобы можно было их увидеть и добавить в Центр партнеров.

### <a name="remove-a-user"></a>Удаление пользователя

Чтобы удалить пользователя из рабочей учетной записи (клиента Azure AD), перейдите к параметру **Пользователи** (в разделе **Параметры учетной записи**), отметьте пользователя, которого нужно удалить, с помощью флажка в крайнем правом столбце, а затем выберите **Удалить** из доступных действий. Появится всплывающее окно, в котором нужно подтвердить, что вы хотите удалить выбранных пользователей.

### <a name="change-a-user-password"></a>Изменение пароля пользователя

Если одному из пользователей необходимо изменить свой пароль, он может сделать это самостоятельно, если при создании учетной записи пользователя вы указали **адрес электронной почты для восстановления пароля**. Вы также можете обновить пароль пользователя, выполнив действия ниже. Чтобы изменить пароль пользователя в вашей рабочей учетной записи организации (клиенте Azure AD), необходимо войти в учетную запись с разрешениями [**глобального администратора**](../../active-directory/users-groups-roles/directory-assign-admin-roles.md). Это приведет к изменению пароля пользователя в клиенте Azure AD, а также пароля, используемого для доступа к центру партнеров.

1. На странице **Пользователи** (в разделе **Параметры учетной записи**) выберите имя учетной записи пользователя, которую требуется изменить.
2. Нажмите кнопку **Сброс пароля** в нижней части страницы.
3. Отобразится страница подтверждения для отображения сведений для входа пользователя, включая временный пароль. Не забудьте распечатать или скопировать эти сведения и предоставить их пользователю, так как вы не сможете получить доступ к временному паролю после выхода из этой страницы.

## <a name="manage-groups"></a>Управление группами

Группы позволяют управлять несколькими ролями пользователей и разрешениями одновременно.

### <a name="add-an-existing-group"></a>Добавление существующей группы

Чтобы добавить группу, которая уже существует в рабочей учетной записи организации (клиенте Azure AD), в учетную запись Центра партнеров, сделайте следующее:

1. На странице **Пользователи** (в разделе **Параметры учетной записи**) выберите **Добавить группы**.
2. Выберите одну или несколько групп из появившегося списка. Для поиска конкретных групп можно использовать поле поиска.
При выборе нескольких групп для добавления в учетную запись Центра партнеров необходимо назначить им одну и ту же роль или набор настраиваемых разрешений. Чтобы добавить несколько групп с разными ролями или разрешениями, повторите эти действия для каждой роли или набора настраиваемых разрешений.
3. Завершив выбор групп, нажмите кнопку **Добавить выбранные**.
4. В разделе **Роли** укажите роли или настраиваемые разрешения для выбранных групп. Все члены группы будут иметь доступ к учетной записи Центра партнеров с разрешениями, применяемыми к группе, независимо от ролей и разрешений, связанных с отдельной учетной записью.
5. Щелкните **Сохранить**.

При добавлении существующей группы каждый пользователь, являющийся членом этой группы, сможет получить доступ к учетной записи Центра партнеров с разрешениями, связанными с назначенной ролью группы.

### <a name="add-a-new-group"></a>Добавление новой группы

Чтобы добавить новую группу в учетную запись Центра партнеров, выполните следующие действия:

1. На странице **Пользователи** (в разделе **Параметры учетной записи**) выберите **Добавить группы**.
2. На следующей странице выберите **Новая группа**.
3. Введите отображаемое имя для новой группы.
4. Укажите роли или настраиваемые разрешения для группы. Все члены группы будут иметь доступ к учетной записи Центра партнеров с применяемыми разрешениями, независимо от ролей и разрешений, связанных с отдельной учетной записью.
5. Выберите пользователей для новой группы из появившегося списка. Для поиска конкретных пользователей можно использовать поле поиска.
6. Завершив выбор пользователей, нажмите кнопку **Добавить выбранные** , чтобы добавить их в новую группу.
7. Выберите **Сохранить**.

Эта новая группа будет создана в рабочей учетной записи вашей организации (клиент Azure AD), а не только в учетной записи центра партнеров.

### <a name="remove-a-group"></a>Удаление группы

Чтобы удалить группу из рабочей учетной записи (клиента Azure AD), перейдите к параметру **Пользователи** (в разделе **Параметры учетной записи**), отметьте группу, которую нужно удалить, с помощью флажка в крайнем правом столбце, а затем выберите **Удалить** из доступных действий. Появится всплывающее окно, в котором нужно подтвердить, что вы хотите удалить выбранные группы.

## <a name="manage-azure-ad-applications"></a>Управление приложениями Azure AD

Вы можете разрешить приложениям или службам, которые являются частью Azure AD вашей организации, получать доступ к учетной записи Центра партнеров.

### <a name="add-existing-azure-ad-applications"></a>Добавление существующих приложений Azure AD

Чтобы добавить приложения, которые уже существуют в Azure Active Directory вашей организации, выполните следующие действия:

1. На странице **Пользователи** (в разделе **Параметры учетной записи**) выберите **Добавить приложения Azure AD**.
2. Выберите одно или несколько приложений Azure AD из появившегося списка. Для поиска конкретных приложений Azure AD можно использовать поле поиска. Если вы выбрали несколько приложений Azure AD для добавления в учетную запись Центра партнеров, необходимо назначить им одну и ту же роль или набор настраиваемых разрешений. Чтобы добавить несколько приложений Azure AD с разными ролями или разрешениями, повторите эти действия для каждой роли или набора настраиваемых разрешений.
3. Завершив выбор приложений Azure AD, нажмите кнопку **Добавить выбранные**.
4. В разделе **Роли** укажите роли или настраиваемые разрешения для выбранных приложений Azure AD.
5. Щелкните **Сохранить**.

### <a name="add-new-azure-ad-applications"></a>Добавление новых приложений Azure AD

Если вы хотите предоставить Центру партнеров доступ к новой учетной записи приложения Azure AD, вы можете создать ее в разделе **Пользователи**. Это приведет к созданию новой учетной записи в рабочей учетной записи организации (клиент Azure AD), а не только в учетной записи центра партнеров. Если вы в основном используете это приложение Azure AD для выполнения проверки подлинности в Центре партнеров и не хотите, чтобы пользователи получали прямой доступ, можно ввести любой допустимый адрес в качестве **URL-адреса ответа** и **URI идентификатора приложения**, если эти значения не используются каким-либо другим приложением Azure AD в вашем каталоге.

1. На странице **Пользователи** (в разделе **Параметры учетной записи**) выберите **Добавить приложения Azure AD**.
2. На следующей странице выберите **Создать приложение Azure AD**.
3. Введите **URL-адрес ответа** для нового приложения Azure AD. Это URL-адрес, с помощью которого пользователи могут входить в систему и использовать приложение Azure AD (также известное как URL-адрес приложения или URL-адрес входа). Длина **URL-адреса ответа** не может превышать 256 символов и должна быть уникальной в каталоге.
4. Введите **URI идентификатора приложения** для нового приложения Azure AD. Это логический идентификатор приложения Azure AD, который предоставляется при отправке запроса на единый вход в Azure AD. **URI идентификатора приложения** должен быть уникальным для каждого приложения Azure AD в каталоге. Длина этого идентификатора не может превышать 256 символов. Дополнительные сведения об универсальном коде ресурса (URI) идентификатора приложения см. в статье [Интеграция приложений с Azure Active Directoryми](../../active-directory/develop/quickstart-modify-supported-accounts.md#change-the-application-registration-to-support-different-accounts)).
5. В разделе **Роли** укажите роли или настраиваемые разрешения для приложений Azure AD.
6. Щелкните **Сохранить**.

После добавления или создания приложения Azure AD можно вернуться в раздел **Пользователи** и выбрать имя приложения, чтобы проверить его параметры, включая идентификатор арендатора, идентификатор клиента, URL-адрес ответа и URI идентификатора приложения.

### <a name="remove-an-azure-ad-application"></a>Удаление приложения Azure AD

Чтобы удалить приложение из рабочей учетной записи (клиента Azure AD), перейдите к параметру **Пользователи** (в разделе **Параметры учетной записи**), отметьте приложение, которое нужно удалить, с помощью флажка в крайнем правом столбце, а затем выберите **Удалить** из доступных действий. Появится всплывающее окно, в котором нужно подтвердить, что вы хотите удалить выбранные приложения.

### <a name="manage-keys-for-an-azure-ad-application"></a>Управление ключами для приложения Azure AD

Если приложение Azure AD считывает и записывает данные в Microsoft Azure AD, ему потребуется ключ. Вы можете создать ключи для приложения Azure AD, изменив его сведения в Центре партнеров. Вы также можете удалить ключи, которые больше не нужны.

1. На странице **Пользователи** (в разделе **Параметры учетной записи**) выберите имя приложения Azure AD. Вы увидите все активные ключи для приложения Azure AD, включая дату создания ключа и срок ее действия 50.
2. Чтобы удалить ключ, который больше не нужен, выберите **Удалить**.
3. Чтобы добавить новый ключ, выберите **Добавить новый ключ**.
4. Отобразится экран с **идентификатором клиента** и **значениями ключа**. Не забудьте распечатать или скопировать эти сведения, так как вы не сможете снова получить доступ к ним после выхода из этой страницы.
5. Если вы хотите создать дополнительные ключи, выберите **Добавить другой ключ**.

## <a name="define-user-roles-and-permissions"></a>Определение ролей пользователей и разрешений

Пользователям вашей компании можно назначить следующие роли и разрешения для программы коммерческого рынка в центре партнеров:

- **Владелец**
  - «Это лицо, которое впервые создало учетную запись и имеет полный доступ к ней, включая возможность создания и изменения всех пользователей учетных записей и изменения всех параметров финансовых и учетных записей. У учетной записи обычно есть только один владелец».
- **Менеджер**
  - имеет доступ ко всем функциям учетной записи Майкрософт, кроме параметров налогов и выплат;
  - может управлять пользователями, ролями и рабочими учетными записями (клиентами).
- **Разработчик**
  - может управлять предложениями и публиковать их;
  - может просматривать некоторые отчеты издателя.

> [!NOTE]
> Для программы коммерческой платформы роли глобального администратора, бизнес-участника, финансового участника и маркетолога не используются. Назначение этих ролей пользователям не оказывает никакого влияния. Только роли менеджера и разработчика предоставляют разрешения пользователям.

Дополнительные сведения об управлении ролями и разрешениями в других областях Центра партнеров, таких как Azure Active Directory (AD), поставщик облачных решений (CSP), поставщик панели управления (CPV), гостевые пользователи или Microsoft Partner Network (MPN), см. в [этой статье](https://docs.microsoft.com/partner-center/permissions-overview).

## <a name="manage-tenants"></a>Управление клиентами

Клиент Azure Active Directory (AD), который также называется вашей рабочей учетной записью в этой документации, — это представление настройки вашей организации на портале Azure. Он помогает управлять конкретным экземпляром облачных служб корпорации Майкрософт для внутренних и внешних пользователей. Если ваша организация подписана на облачную службу корпорации Майкрософт, например Azure, Microsoft Intune или Office 365, для вас установлен клиент Azure AD.

Вы можете настроить несколько клиентов для использования с Центром партнеров. Любой пользователь с ролью **менеджера** в учетной записи Центра партнеров будет иметь возможность добавлять и удалять клиенты Azure AD из учетной записи.  

### <a name="add-an-existing-tenant"></a>Добавление существующего клиента

Чтобы связать другой клиент Azure AD с учетной записью Центра партнеров, выполните следующие действия:

1. На странице **Клиенты** (в разделе **Параметры учетной записи**) выберите **Associate another Azure AD tenant** (Связать другой клиент Azure AD).
2. Введите учетные данные Azure AD для клиента, который необходимо связать.
3. Проверьте имя организации и доменное имя вашего клиента Azure AD. Чтобы завершить настройку связи, выберите **Подтвердить**.

Если настройка связи выполнена успешно, тогда можно добавить пользователей учетных записей и управлять ими в разделе **Пользователи** в Центре партнеров.

### <a name="create-a-new-tenant"></a>Создание клиента

Чтобы создать клиент Azure AD с помощью учетной записи Центра партнеров, выполните следующие действия:

1. На странице **Клиенты** (в разделе **Параметры учетной записи**) выберите **Создать клиент Azure AD**.
2. Введите сведения о каталоге для вашей новой службы Azure AD:
    - **Доменное имя**: уникальное имя, которое будет использоваться для домена Azure AD, а также .onmicrosoft.com. Например, если вы ввели example, домен Azure AD будет example.onmicrosoft.com.
    - **Контактный адрес электронной почты**: адрес электронной почты, по которому мы можем связаться с вами относительно вашей учетной записи, если это необходимо.
    - **Сведения об учетной записи пользователя глобального администратора**: имя, фамилия, имя пользователя и пароль, которые вы хотите использовать для новой учетной записи глобального администратора.
3. Выберите **Создать**, чтобы подтвердить новые сведения о домене и учетной записи.
4. Войдите с помощью нового имени пользователя и пароля глобального администратора Azure AD, чтобы начать [добавлять пользователей и управлять ими](#manage-users).

Дополнительные сведения о создании клиентов на вашем портале Azure, а не на портале Центра партнеров, см. в статье [Краткое руководство. Создание клиента в Azure Active Directory](../../active-directory/fundamentals/active-directory-access-create-new-tenant.md).

### <a name="remove-a-tenant"></a>Удаление клиента

Чтобы удалить клиент из учетной записи Центра партнеров, найдите его имя на странице **Клиенты** (в разделе **Параметры учетной записи**), а затем выберите **Удалить**. Вам будет предложено подтвердить удаление указанного клиента. После удаления пользователи в этом клиенте не смогут войти в учетную запись Центра партнеров, а все разрешения, настроенные для этих пользователей, будут удалены.

При удалении клиента все пользователи, добавленные в учетную запись Центра партнеров из этого клиента, больше не смогут в нее войти.

> [!TIP]
> Вы не можете удалить клиент, если вы вошли в Центр партнеров, используя учетную запись в том же клиенте. Чтобы удалить клиент, необходимо войти в Центр партнеров в качестве **менеджера** для другого клиента, связанного с этой учетной записью. Если с учетной записью связан только один клиент, его можно удалить только после входа с помощью учетной записи Майкрософт, открывшей эту учетную запись.

## <a name="agreements"></a>Соглашения

В разделе " **соглашения** " центра партнеров (в разделе **Параметры учетной записи**) можно просмотреть список лицензионных соглашений, которым вы разработали. Эти соглашения отображаются в соответствии с именем и номером версии, включая дату их принятия и имя пользователя, который принял условия соглашения.

Сообщение **Требуются действия** может появиться в верхней части этой страницы, если существуют обновления соглашений, требующие вашего внимания. Чтобы принять обновленное соглашение, сначала ознакомьтесь со сведениями о версии соглашения по ссылке, а затем выберите **Принять соглашение**.
