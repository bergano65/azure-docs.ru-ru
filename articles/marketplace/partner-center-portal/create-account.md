# <a name="create-a-commercial-marketplace-account-in-partner-center"></a>Создание коммерческой учетной записи Marketplace в центре партнеров

Чтобы опубликовать свои предложения в [Azure Marketplace](https://azuremarketplace.microsoft.com/) или [AppSource](https://appsource.microsoft.com/), необходимо создать учетную запись в программе коммерческого рынка в центре партнеров. В этой статье рассматривается создание учетной записи центра партнеров и добавление новых издателей в программу коммерческого рынка.

>[!IMPORTANT]
>Если у вас есть учетная запись в [портал Cloud Partner (CPP)](https://cloudpartner.azure.com) , которая была перемещена в центр партнеров, то создавать новую учетную запись не нужно. Дополнительные сведения см. в разделе издатели, [перемещающиеся из cpp](#publishers-moving-from-cpp) .

## <a name="before-you-begin"></a>Перед началом работы

Чтобы создать учетную запись в центре партнеров, убедитесь, что у вас есть:

- Право на подписание юридических соглашений от имени компании.
- Юридическое имя, адрес и основное контактное лицо вашей компании (это может быть вы).

Эти сведения будут проверены в процессе создания учетной записи.

Существует два способа создания учетной записи.

- Если вы не знакомы с центром партнеров и у вас нет учетной записи сети Майкрософт, создайте учетную запись, используя [страницу Создание учетной записи на странице регистрации в центре партнеров](#create-an-account-using-the-partner-center-enrollment-page).
- Если вы уже зарегистрировались в Microsoft Partner Network, создайте учетную запись непосредственно в центре партнеров, используя средства [создания учетной записи с помощью существующих регистраций центра партнеров Майкрософт](#create-an-account-using-existing-microsoft-partner-center-enrollments).

## <a name="create-an-account-using-the-partner-center-enrollment-page"></a>Создание учетной записи с помощью страницы "регистрация в центре партнеров"

Используйте этот метод, если вы не знакомы с центром партнеров и не зарегистрированы в Microsoft Partner Network. В этом разделе описано, как создать новую учетную запись центра партнеров, в том числе:

- [Регистрация на странице регистрации в центре партнеров](#register-on-the-partner-center-enrollment-page)
- [Вход с рабочей учетной записью](#sign-in-with-a-work-account)
- [Принять условия](#agree-to-terms-and-conditions)
- [Укажите профиль издателя](#provide-your-publisher-profile)

>[!IMPORTANT]
>Если у вас есть учетная запись в [портал Cloud Partner (CPP)](https://cloudpartner.azure.com) , которая была перемещена в центр партнеров, то создавать новую учетную запись не нужно. Дополнительные сведения см. в разделе издатели, [перемещающиеся из cpp](#publishers-moving-from-cpp) .

### <a name="register-on-the-partner-center-enrollment-page"></a>Регистрация на странице регистрации в центре партнеров

Просмотрите сведения на странице регистрации в [**центре партнеров Майкрософт**](https://partner.microsoft.com/dashboard/account/v3/enrollment/introduction/azureisv) , а затем зарегистрируйтесь для получения учетной записи.

### <a name="sign-in-with-a-work-account"></a>Вход с рабочей учетной записью

Войдите с помощью рабочей учетной записи, чтобы вы могли связать домен рабочей учетной записи электронной почты вашей компании с новой учетной записью центра партнеров. Связав эти учетные записи, сотрудники компании могут войти в центр партнеров, используя имена пользователей и пароли своей рабочей учетной записи.

>[!NOTE]
>Чтобы проверить, есть ли у вашей компании Рабочая учетная запись, как создать новую рабочую учетную запись или как настроить несколько рабочих учетных записей для использования с центром партнеров, посетите [рабочую учетную запись организации и центр партнеров](./company-work-accounts.md).

### <a name="agree-to-terms-and-conditions"></a>Принять условия

Необходимо согласиться с условиями [соглашения Microsoft Marketplace издателя](https://go.microsoft.com/fwlink/?LinkID=699560).

### <a name="provide-your-publisher-profile"></a>Укажите профиль издателя

Ваш профиль издателя включает название вашей компании и идентификатор MPN. Если вы еще не сделали этого, необходимо присоединиться к [Microsoft Partner Network](https://partner.microsoft.com/commercial). После приподключения к Microsoft Partner Network вам будет предоставлен идентификатор MPN.

Создайте идентификатор издателя. Идентификатор издателя однозначно определяет вашу компанию и ваши предложения в Marketplace и AppSource.

Убедившись, что данные профиля издателя подтверждены, примите условия и создайте учетную запись центра партнеров, выбрав **принять и продолжить**.

>[!IMPORTANT]
>*Чтобы принять эти условия, необходимо быть полномочным для работы от имени компании.*

После регистрации вы перейдете на страницу обзора коммерческого рынка. В левой панели навигации отобразится **коммерческое Marketplace** . Если выбрать **Параметры** > **партнерских параметров** > **программы**, вы увидите, что коммерческий магазин указан как зарегистрированный.

Благодарим за создание коммерческой учетной записи Marketplace в центре партнеров! Справочные сведения см. в статье [Обзор коммерческого рынка](./commercial-marketplace-overview.md). Дополнительные сведения о добавлении издателей см. в статье [Добавление новых издателей в программу коммерческого рынка](#add-new-publishers-to-the-commercial-marketplace-program).

## <a name="create-an-account-using-existing-microsoft-partner-center-enrollments"></a>Создание учетной записи с помощью существующих регистраций центра партнеров Майкрософт

Используйте этот раздел для создания коммерческой учетной записи Marketplace, если у вас уже есть регистрация в центре партнеров Майкрософт.

Существует два типа существующих регистраций, которые можно использовать для настройки учетной записи коммерческого магазина. Выберите ту, которая применяется к вам:

- Если у вас уже есть Microsoft Partner Network регистрации, создайте учетную запись с помощью функции [регистрации в Microsoft Partner Network](#enrolled-in-the-microsoft-partner-network).
- Если вы зарегистрировали существующую программу для разработчиков, создайте учетную запись с помощью функции [регистрации в программе разработчика](#enrolled-in-a-developer-program).

Для обоих методов вы выполните следующие действия:

- Вход в центр партнеров с использованием существующих учетных данных
- Введите сведения о профиле публикации

Убедитесь, что данные вашей учетной записи являются удобными.

>[!IMPORTANT]
>Если у вас есть учетная запись в [портал Cloud Partner (CPP)](https://cloudpartner.azure.com/) , которая была перемещена в центр партнеров, то создавать новую учетную запись не нужно. Дополнительные сведения см. в разделе издатели, [перемещающиеся из cpp](#publishers-moving-from-cpp) .

### <a name="enrolled-in-the-microsoft-partner-network"></a>Зарегистрировано в Microsoft Partner Network

Если вы уже зарегистрировались в Microsoft Partner Network, используйте этот раздел для создания учетной записи непосредственно в центре партнеров.

Войдите в [Центр партнеров](https://partner.microsoft.com/) , используя учетную запись Microsoft Partner Network, чтобы связать домен учетной записи электронной почты вашей компании с новой учетной записью центра партнеров. Связав эти учетные записи, сотрудники компании могут войти в центр партнеров, используя свои имена пользователей и пароли рабочих учетных записей.

>[!NOTE]
>Чтобы проверить, есть ли у вашей компании Рабочая учетная запись центра партнеров, как создать новую рабочую учетную запись или как настроить несколько рабочих учетных записей для использования с центром партнеров, посетите [рабочую учетную запись организации и центр партнеров](./company-work-accounts.md).

После входа выполните следующие действия, чтобы завершить настройку учетной записи.

>[!NOTE]
> Для входа в Microsoft Partner Network необходимо быть **администратором учетной записи** или ролью **глобального администратора** .

1. В [центре партнеров](https://partner.microsoft.com/)выберите **Параметры** > **Параметры партнеров** > **программы** > **начать работу**.
2. Microsoft Partner Network обнаружит подписку и отобразит страницу профиля издателя, чтобы ввести название организации и идентификатор MPN.
3. Ознакомьтесь с условиями [соглашения Microsoft Marketplace издателя](https://go.microsoft.com/fwlink/?LinkID=699560), а затем выберите **принять и продолжить** , чтобы завершить регистрацию.

    >[!IMPORTANT]
    >*Чтобы принять эти условия, необходимо быть полномочным для работы от имени компании.*

После регистрации вы перейдете на страницу обзора коммерческого рынка. В левой панели навигации отобразится **коммерческое Marketplace** . Если выбрать **Параметры** > **партнерских параметров** > **программы**, вы увидите, что коммерческий магазин указан как зарегистрированный.

Благодарим за создание учетной записи в центре партнеров! Справочные сведения см. в статье [Обзор коммерческого рынка](./commercial-marketplace-overview.md). Дополнительные сведения о добавлении издателей см. в статье [Добавление новых издателей в программу коммерческого рынка](#add-new-publishers-to-the-commercial-marketplace-program).

### <a name="enrolled-in-a-developer-program"></a>Зарегистрировано в программе разработчика

Если вы уже зарегистрировались в программе для разработчиков центра партнеров, используйте этот раздел для создания учетной записи непосредственно в центре партнеров.

>[!NOTE]
>Чтобы зарегистрироваться в коммерческом магазине под той же учетной записью, что и вы вошли в систему, необходимо зарегистрироваться в центре партнеров, используя учетную запись компании. Если вы зарегистрировались с помощью отдельной учетной записи, ваша заявка на коммерческой рынок будет отображаться под новой учетной записью.
>
>Отдельные учетные записи предназначены для отдельных разработчиков, работающих самостоятельно. Учетные записи компании предназначены для организаций и предприятий. Учетные записи компаний предоставляют доступ к отправке приложений с дополнительными функциональными возможностями. Чтобы подтвердить, что вы имеете право на настройку учетной записи организации, требуется более большая проверка учетных записей компании. Эта проверка может занять от нескольких дней до нескольких недель и часто включает телефонный звонок в компанию. Оба типа учетных записей позволяют отправлять приложения, надстройки и службы. Дополнительные сведения см. в разделе [типы счетов, местоположения и сборы](https://docs.microsoft.com/windows/uwp/publish/account-types-locations-and-fees).

Войдите в [Центр партнеров](https://partner.microsoft.com/) с помощью учетной записи центра партнеров Майкрософт, чтобы зарегистрироваться в программе коммерческого рынка в той же учетной записи. Зарегистрировавшись в коммерческом магазине, используя ту же учетную запись, вы можете видеть как существующие регистрации программы, так и коммерческие рынки в меню навигации в центре партнеров.

После входа выполните следующие действия, чтобы завершить настройку учетной записи.

1. В центре партнеров выберите **параметры** > **Параметры разработчика** > **программы** > **начать работу**и введите рабочий адрес электронной почты.
2. Заполните сведения на странице профиля издателя.
3. Ознакомьтесь с условиями [соглашения Microsoft Marketplace издателя](http://go.microsoft.com/fwlink/?LinkID=699560), а затем выберите **принять и продолжить** , чтобы завершить регистрацию.

    >[!Important]
    >*Чтобы принять эти условия, необходимо быть полномочным для работы от имени компании.*

После регистрации вы перейдете на страницу обзора коммерческого рынка. В левой панели навигации отобразится **коммерческое Marketplace** . Если выбрать **параметры** > **Параметры разработчика**@no__t **-3, вы увидите, что**коммерческий магазин указан как зарегистрированный.

Дополнительные сведения о добавлении издателей см. в статье [Добавление новых издателей в программу коммерческого рынка](#add-new-publishers-to-the-commercial-marketplace-program).

## <a name="publishers-moving-from-cpp"></a>Издатели, перемещающиеся из CPP

Если ваша учетная запись была перенесена из [портал Cloud Partner (CPP)](https://cloudpartner.azure.com), вам не нужно создавать новую учетную запись центра партнеров. Вы должны получить настроенную ссылку на новую учетную запись центра партнеров в сообщении электронной почты и в уведомлении после входа в существующую учетную запись CPP.

После того, как вы включили новую учетную запись центра партнеров, перейдя по настроенной ссылке, вы можете вернуться к своей учетной записи, посетив [панель мониторинга коммерческого рынка](https://partner.microsoft.com/dashboard/commercial-marketplace/overview) в центре партнеров.

Сведения о соглашении о публикации и профиле компании будут перенесены в новую учетную запись центра партнеров, а также все ранее настроенные данные профиля выплат учетной записи, учетные записи пользователей и разрешения, а также активные предложения, связанные с вашей учетной записью CPP.

После перемещения сведений об учетной записи из CPP в центр партнеров вы больше не будете использовать CPP для обновления учетной записи, а также для управления пользователями, разрешениями и выставлением счетов. В течение ограниченного времени все обновления учетной записи, внесенные в центре партнеров, автоматически обновляются в учетной записи CPP только для чтения до тех пор, пока портал CPP не будет считаться устаревшим.

## <a name="add-new-publishers-to-the-commercial-marketplace-program"></a>Добавление новых издателей в программу коммерческого рынка

Организация может иметь несколько издателей, связанных с коммерческой учетной записью Marketplace. Существующий пользователь может добавить дополнительных издателей после входа в центр партнеров, > выбрав **Параметры учетной записи** **Издатели** > **Добавить издатель**.

>[!NOTE]
>Перед добавлением нового издателя просмотрите имеющиеся издатели, войдя в центр партнеров и > выбрав **Параметры учетной записи** **издателей** , чтобы просмотреть список существующих издателей.

Другой пользователь из того же клиента Azure Active Directory может добавить нового издателя, выполнив следующие действия:

1. Запустите поток регистрации в [центре партнеров Майкрософт](https://partner.microsoft.com/en-us/dashboard/account/v3/enrollment/introduction/azureisv).
2. Выберите **войти с рабочей учетной записью** и введите рабочий адрес электронной почты.
3. Нажмите кнопку **добавить издателя** .
4. Выберите идентификатор MPN, который нужно связать с издателем.
5. Обновите **сведения об издателе** в форме.

   1. **Имя издателя**: Имя, которое будет отображаться в Azure Marketplace или AppSource с предложением.
   1. **PublisherID**: Идентификатор, используемый центром партнеров для уникальной идентификации издателя. Значение по умолчанию для этого поля сопоставляется с существующим `PublisherID` и уникальным в системе, которое нельзя использовать повторно, поэтому необходимо обновить это поле.
   1. **Контактные данные**: При необходимости обновите контактные данные.

После завершения процесса вы сможете управлять только что созданным издателем, перейдя в программу **коммерческого рынка** , которая указана в левом меню навигации. Если вы не видите программу **коммерческого рынка** , обновите страницу. Новый издатель появится в списке издателей .
