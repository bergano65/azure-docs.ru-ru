# <a name="create-a-commercial-marketplace-account-in-partner-center"></a>Создание коммерческой учетной записи Marketplace в центре партнеров

Чтобы опубликовать предложения в [Microsoft AppSource](https://appsource.microsoft.com/) или [Azure Marketplace](https://azuremarketplace.microsoft.com/), необходимо создать учетную запись в программе коммерческого рынка в центре партнеров. В этой статье описывается, как создать учетную запись центра партнеров и как добавить новых издателей в программу коммерческого рынка.

>[!NOTE]
>Если у вас есть учетная запись в Портал Cloud Partner (CPP), мы перенесли ее в центр партнеров. Вам не нужно создавать новую учетную запись. Дополнительные сведения см. [в разделе издатели, которые перешли с портал Cloud Partner](#publishers-who-moved-from-the-cloud-partner-portal).

## <a name="before-you-begin"></a>Перед началом

Чтобы создать учетную запись в центре партнеров, убедитесь, что выполнены следующие предварительные требования. Мы проверим эту информацию в процессе создания учетной записи.

- Необходимо использовать рабочую учетную запись, связанную с вашей компанией или организацией. Личные учетные записи не поддерживаются. Дополнительные сведения см. в статье [рабочие учетные записи компании и центр партнеров](company-work-accounts.md).
- Сведения о юридическом бизнесе, адресе и основном контактном лице компании. Это может быть пользователь.
- Необходимо иметь права на подписание юридических соглашений от имени компании.

Существует два способа создания учетной записи.

- Если вы новичок в центре партнеров и у вас нет учетной записи Microsoft Partner Network (MPN), продолжайте [создавать учетную запись с помощью страницы регистрации в центре партнеров](#create-an-account-using-the-partner-center-enrollment-page).
- Если вы уже зарегистрировались в Microsoft Partner Network или программе разработчика, создайте учетную запись непосредственно в центре партнеров. Перейдите к разделу [Создание учетной записи с помощью существующих регистраций в центре партнеров](#create-an-account-using-existing-partner-center-enrollments).

## <a name="create-an-account-using-the-partner-center-enrollment-page"></a>Создание учетной записи с помощью страницы "регистрация в центре партнеров"

Используйте этот метод, если вы не знакомы с центром партнеров и не зарегистрированы в Microsoft Partner Network. Выполните действия, описанные в этом разделе, чтобы создать новую учетную запись центра партнеров и профиль издателя.

### <a name="register-on-the-partner-center-enrollment-page"></a>Регистрация на странице регистрации в центре партнеров

Просмотрите сведения на странице регистрации в [**центре партнеров Майкрософт**](https://partner.microsoft.com/dashboard/account/v3/enrollment/introduction/partnership) , а затем зарегистрируйтесь для получения учетной записи.

### <a name="sign-in-with-a-work-account"></a>Вход с рабочей учетной записью

Войдите с помощью рабочей учетной записи, чтобы связать домен учетной записи электронной почты организации с новой учетной записью центра партнеров. После того как вы собрались с этими двумя учетными записями, сотрудники компании могут войти в центр партнеров, используя свои имена пользователей и пароли своей рабочей учетной записи.

>[!NOTE]
>Чтобы узнать, есть ли у вашей компании Рабочая учетная запись, Узнайте, как создать новую рабочую учетную запись или настроить несколько рабочих учетных записей для использования с центром партнеров, см. статью [рабочие учетные записи компании и центр партнеров](company-work-accounts.md).

### <a name="agree-to-the-terms-and-conditions"></a>Принять условия

В рамках процесса регистрации необходимо согласиться с условиями [соглашения Microsoft Publisher](https://go.microsoft.com/fwlink/?LinkID=699560).

### <a name="provide-your-publisher-profile"></a>Укажите профиль издателя

Ваш профиль издателя включает название вашей компании и идентификатор MPN. Если вы еще не сделали этого, присоединитесь к [Microsoft Partner Network](https://partner.microsoft.com/commercial). После приподключения к Microsoft Partner Network вам будет предоставлен идентификатор MPN.

1. Создайте идентификатор издателя. Идентификатор издателя однозначно определяет вашу компанию и ваши предложения в Azure Marketplace и AppSource.

1. После подтверждения сведений о профиле издателя примите условия, а затем создайте учетную запись центра партнеров, выбрав **принять и продолжить**.

    > [!IMPORTANT]
    > Чтобы принять эти условия, необходимо иметь права на действия от имени компании.

    После регистрации вы перейдете на страницу обзора коммерческого рынка. На левой панели отобразится учетная запись коммерческого рынка.

1. Чтобы убедиться, что учетная запись коммерческого магазина указана как зарегистрированная, щелкните **Параметры** (значок шестеренки) в правом верхнем углу > **Параметры учетной записи**  >  **Programs**.

Теперь вы создали коммерческую учетную запись Marketplace в центре партнеров. Продолжайте [добавлять новых издателей на коммерческий рынок](#add-new-publishers-to-the-commercial-marketplace-program).

## <a name="create-an-account-using-existing-partner-center-enrollments"></a>Создание учетной записи с помощью существующих регистраций в центре партнеров

Используйте этот раздел для создания коммерческой учетной записи Marketplace, если у вас уже есть регистрация в центре партнеров Майкрософт. Существует два типа существующих регистраций, которые можно использовать для настройки учетной записи коммерческого магазина. Выберите сценарий, применимый к вам:

*Что если я уже зарегистрировался в Microsoft Partner Network?*
- [Используйте существующую регистрацию Microsoft Partner Network](#use-a-microsoft-partner-network-enrollment) , чтобы создать учетную запись.

*Что делать, если я уже зарегистрировался в программе разработчика?*
- [Используйте существующую регистрацию программы разработчика](#use-a-developer-program-enrollment) для создания учетной записи.

Для обоих типов регистрации вы входите в центр партнеров с существующими учетными данными. Убедитесь, что учетная запись и сведения о профиле издателя доступны.

### <a name="use-a-microsoft-partner-network-enrollment"></a>Использование регистрации Microsoft Partner Network

При использовании учетной записи Microsoft Partner Network она связывает домен учетной записи электронной почты вашей компании с новой учетной записью центра партнеров. После того как вы собрались с этими двумя учетными записями, сотрудники компании могут войти в центр партнеров, используя свои имена пользователей и пароли своей рабочей учетной записи. 

>[!NOTE]
> Для входа в Microsoft Partner Network необходимо быть **администратором учетной записи** или ролью **глобального администратора** .

1. Войдите в [Центр партнеров](https://partner.microsoft.com/dashboard/) , используя учетную запись Microsoft Partner Network.
1. Выберите **Параметры**  >  **Параметры учетной записи**  >  **программы**  >  **Приступая к работе**.

   Microsoft Partner Network обнаружит подписку и отобразит панель **профиль издателя** .

1. Введите название компании и идентификатор MPN.
1. Ознакомьтесь с условиями [соглашения Microsoft Publisher](https://go.microsoft.com/fwlink/?LinkID=699560), а затем выберите **принять и продолжить** , чтобы завершить регистрацию.

    > [!IMPORTANT]
    > Чтобы принять эти условия, необходимо иметь права на действия от имени компании.

    После регистрации вы перейдете на страницу обзора коммерческого рынка. На левой панели отобразится учетная запись коммерческого рынка.

1. Чтобы убедиться, что учетная запись коммерческого магазина указана как зарегистрированная, выберите **Параметры** параметры  >  **учетной записи**  >  **программы**.

Теперь вы создали коммерческую учетную запись Marketplace в центре партнеров. Продолжайте [добавлять новых издателей на коммерческий рынок](#add-new-publishers-to-the-commercial-marketplace-program).

### <a name="use-a-developer-program-enrollment"></a>Использование регистрации программы для разработчиков

>[!NOTE]
>Чтобы зарегистрироваться для коммерческого рынка в той же учетной записи, в которой вы выполнили вход, необходимо зарегистрироваться в центре партнеров с учетной записью компании. Если вы зарегистрировались в отдельной учетной записи, ваша заявка на коммерческой рынок будет отображаться под новой учетной записью.
>
>Отдельные учетные записи предназначены для разработчиков, работающих над ними. Учетные записи компании предназначены для организаций и предприятий. Учетные записи компаний предоставляют доступ к отправке приложений с дополнительными функциональными возможностями.
>
> Чтобы убедиться, что вы имеете право на настройку учетной записи компании, после регистрации вам потребуется дополнительная проверка учетных записей компании. Эта проверка может занять от нескольких дней до нескольких недель, и она часто включает телефонный звонок в компанию. Оба типа учетных записей позволяют отправлять приложения, надстройки и службы. Дополнительные сведения см. в разделе [типы счетов, местоположения и сборы](/windows/uwp/publish/account-types-locations-and-fees).

При регистрации в программе коммерческого рынка с той же учетной записью, что и программа разработчика, вы можете просмотреть все имеющиеся регистрации программы в левой области центра партнеров.

1. Войдите в [Центр партнеров](https://partner.microsoft.com/dashboard/) , используя имеющуюся учетную запись.
1. Выберите **Параметры**  >  **Параметры учетной записи**  >  **программы** начало  >  **работы** и введите рабочий адрес электронной почты.
1. Заполните сведения на странице **профиля издателя** .
2. Ознакомьтесь с условиями [соглашения Microsoft Publisher](https://go.microsoft.com/fwlink/?LinkID=699560), а затем выберите **принять и продолжить** , чтобы завершить регистрацию.

    >[!Important]
    > Чтобы принять эти условия, необходимо иметь права на действия от имени компании.

    После регистрации вы перейдете на страницу обзора коммерческого рынка. На левой панели отобразится учетная запись коммерческого рынка.

1. Чтобы убедиться, что учетная запись коммерческого магазина указана как зарегистрированная, выберите **Параметры** параметры  >  **учетной записи**  >  **программы**.

Теперь вы создали коммерческую учетную запись Marketplace в центре партнеров. Чтобы получить справку по добавлению издателей, продолжайте [добавлять новых издателей на коммерческий рынок](#add-new-publishers-to-the-commercial-marketplace-program).

## <a name="publishers-who-moved-from-the-cloud-partner-portal"></a>Издатели, которые перешли с Портал Cloud Partner

Если у вас есть учетная запись Портал Cloud Partner, мы перенесли ее в центр партнеров. Вам не нужно создавать новую учетную запись центра партнеров. Вы должны получить настроенную ссылку на новую учетную запись центра партнеров как в сообщении электронной почты, так и в уведомлении в виде баннера после входа в существующую учетную запись Портал Cloud Partner.

После выбора настроенной ссылки и включения новой учетной записи центра партнеров вы можете вернуться к своей учетной записи, перейдя на [панель мониторинга коммерческого рынка](https://partner.microsoft.com/dashboard/commercial-marketplace/overview) в центре партнеров.

Сведения о соглашении о публикации и профиле компании переходят на новую учетную запись центра партнеров, а также на все ранее настроенные данные профиля выплат учетной записи, учетные записи пользователей и разрешения, а также активные предложения, связанные с вашей учетной записью Портал Cloud Partner.

## <a name="add-new-publishers-to-the-commercial-marketplace-program"></a>Добавление новых издателей в программу коммерческого рынка

Организация может иметь несколько издателей, связанных с коммерческой учетной записью Marketplace. Существующий пользователь может добавить дополнительных издателей после входа в центр партнеров, выбрав **Параметры**  >  **Параметры учетной записи**  >  **Organization Profile**  >  **идентификаторы** профиля организации  >  **Добавить издатель**.

>[!NOTE]
>Перед добавлением нового издателя проверьте список существующих издателей, войдя в центр партнеров и выбрав **Параметры учетной записи**  >  **Издатели**.

Дополнительные пользователи из того же клиента Azure Active Directory могут использовать следующие шаги для добавления нового издателя.

1. Запустите поток регистрации в [центре партнеров Майкрософт](https://partner.microsoft.com/dashboard/account/v3/enrollment/introduction/partnership).
2. Выберите **войти с рабочей учетной записью** и введите рабочий адрес электронной почты.
3. Нажмите кнопку **добавить издателя** .
4. Выберите идентификатор MPN, который нужно связать с издателем.
5. Обновите **сведения об издателе** в форме.

   * **Имя издателя**: имя, которое отображается в коммерческом магазине с предложением.  
   * **PublisherID**— идентификатор, используемый центром партнеров для уникальной идентификации издателя. Значение по умолчанию для этого поля соответствует существующему и уникальному ИДЕНТИФИКАТОРу издателя в системе. Поскольку идентификатор издателя нельзя использовать повторно, это поле необходимо обновить.  
   * **Контактные данные**. при необходимости обновите контактные данные.

После завершения этого процесса перейдите к учетной записи коммерческого рынка, указанной на левой панели, чтобы управлять только что созданным издателем. Если вы не видите учетную запись коммерческого магазина, обновите страницу. Новый издатель появится в списке **издателей** .
