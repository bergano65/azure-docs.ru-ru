<a name="defining-and-publishing-your-consulting-services-offer"></a>Определение и публикация предложения консультационных услуг
======================================================

Это руководство предназначено помочь в определении и публикации предложения консультационных служб на портале Cloud Partner.

<a name="define-your-consulting-services-offer"></a>Определение предложения консультационных услуг
-------------------------------------

Определите свое пакетное предложение консультационных услуг. Сосредоточьтесь на фиксированном объеме, фиксированной продолжительности, оценке, фиксированной цене (или бесплатном предложении) и в первую очередь на предложениях, ориентированных на предпродажи, для отдельного клиента. Выберите проверенные пакеты услуг, которые оказались наиболее популярными и эффективными для вовлечения клиентов для вашего бизнеса.

<a name="publish-a-consulting-service-offer"></a>Публикация предложения консультационных услуг
----------------------------------

В следующих разделах описывается, как опубликовать предложение консультационных услуг:

1.  Создание предложения.
2.  Определение параметров предложения.
3.  Ввод сведений об онлайн-магазине и выбор платформы публикации — Azure Marketplace или AppSource.
4.  Публикация предложения.

### <a name="create-a-new-offer"></a>Создание предложения

Чтобы создать предложение, сделайте следующее:

1.  В главном меню на портале Cloud Partner выберите **New offer** (Создать предложение).
2.  В меню New Offer (Новое предложение) выберите **Consulting Service** (Консультационная услуга).

    ![Создание предложения консультационных услуг](media/consulting-services-publishing-offer/cppselectnewconsultingoffer.png)

### <a name="define-offer-settings"></a>Определение параметров предложения

На экране New Offer (Новое предложение) сначала создается идентификатор предложения.
Идентификатор предложения состоит из трех частей:

1.  Offer ID (Идентификатор предложения)
2.  Publisher ID (Идентификатор издателя)
3.  ИМЯ

Все они описываются в следующих разделах.

#### <a name="offer-id"></a>Offer ID (Идентификатор предложения)

Это уникальное имя, которые создается при первой отправке предложения. Оно должно содержать только строчные буквы, цифры, дефисы и символы подчеркивания. Идентификатор будет отображаться в URL-адресе и влиять на результаты поисковых систем. Пример: *yourcompanyname\_exampleservice*.

Как показано в примере, идентификатор предложения добавляется к идентификатору издателя, образуя уникальный идентификатор. Он предоставляется в виде постоянной ссылки, которую можно добавить в закладки, и индексируется поисковыми системами.

**Как только предложение станет активным, его невозможно будет обновить**.

#### <a name="publisher-id"></a>Publisher ID (Идентификатор издателя)

Связан с вашей учетной записью. Когда вы войдете в учетную запись организации, ваш идентификатор издателя отобразится в раскрывающемся меню.

#### <a name="name"></a>ИМЯ

Отображается в качестве названия предложения в AppSource или Azure Marketplace.

**Важно.** Здесь необходимо ввести только имя фактической услуги. Не следует добавлять ее длительность и тип.

В следующем примере с Edgewater Fullscope показано, как формируется имя предложения. Имя предложения выглядит следующим образом:

![Создание предложения консультационных услуг](media/consulting-services-publishing-offer/cppsampleconsultingoffer.png)

Имя предложения состоит из четырех частей:

-   **Duration** (Длительность). Это значение следует указать на вкладке Storefront Details (Сведения об онлайн-магазине) редактора. Длительность может быть выражена в часах, днях или неделях.
-   **Type of service** (Тип услуги). Это значение следует указать на вкладке Storefront Details (Сведения об онлайн-магазине) редактора. Имеются следующие типы услуг: Assessment (Оценка), Briefing (Брифинг), Implementation (Реализация), Proof of concept (Подтверждение концепции) и Workshop (Семинар).
-   **Preposition** (Кем опубликовано). Указывается рецензентом.
-   **Name** (Имя). Определяется на странице Offer Settings (Параметры предложения).

В следующем списке приведены несколько понятных имен предложений:

-   Необходимое для профессиональных услуг: 1-часовой брифинг.
-   Платформа для миграции в облако: 1-часовой брифинг.
-   PowerApps и Microsoft Flow: 1-дневный семинар.
-   Службы машинного обучения Azure: подтверждение концепции за 3 недели.
-   Решение для розничной торговли Brick and Click: 1-часовой брифинг.
-   Принесите данные с собой: 1-недельный семинар.
-   Облачная аналитика: 3-дневный семинар.
-   Обучение Power BI: 3-дневный семинар.
-   Решение для управления продажами: реализация за 1 неделю.
-   Быстрый запуск CRM: 1-дневный семинар.
-   Dynamics 365 for Sales: оценка за 2 дня.

После заполнения вкладки **Offer Settings** (Параметры предложения) можно сохранить заявку. Теперь имя предложения будет отображаться над областью редактора, и его можно будет найти в разделе All Offers (Все предложения).

### <a name="enter-storefront-details"></a>Ввод сведений об онлайн-магазине

Далее нужно ввести сведения о вашем онлайн-магазине. Вкладка сведений об онлайн-магазине состоит из следующих разделов:

-   Offer Details (Сведения о предложении);
-   Publisher Information (Сведения об издателе);
-   Listing Details (Сведения списка);
-   Marketing Artifacts (Маркетинговые артефакты);

#### <a name="offer-details"></a>Сведения о предложении

В разделе сведений о предложении содержатся следующие поля:

-   Offer Summary (Сводка предложения);
-   Offer description (Описание предложения);

##### <a name="offer-summary"></a>Offer Summary (Сводка предложения);

В области кратких сведений о предложении приводится краткое описание вашего предложения, которое отображается сразу под его именем. Это описание предложения следует вводить в формате обычного текста. В нем не должно быть разрывов строк. Ниже приведены удачные примеры краткого описания предложений с соответствующими им названием.

*Пример 1*

-   **Имя предложения.** Облачная аналитика: 3-дневный семинар
-   **Краткие сведения о предложении.** Обзор Microsoft Azure и Power BI, оценка текущей среды и небольшое подтверждение концепции.

*Пример 2*

-   **Имя предложения.** Отраслевые решения для Интернета вещей Azure: подтверждение концепции за 30 дней
-   **Краткие сведения о предложении.** Создание пилотного отраслевого проекта подключенного продукта для безопасного подключения оборудования на месте установки к решению Центра Интернета вещей Azure с панелями мониторинга, отчетами и уведомлениями.

*Пример 3*

-   **Имя предложения.** Профессиональные услуги: 1-часовой брифинг
-   **Краткие сведения о предложении.** Обзор и демонстрация предварительно настроенного, расширенного решения Dynamics 365 for Operations, обеспечивающего усовершенствованное управление проектами, выставлением счетов и ресурсами для профессиональных услуг.

*Пример 4*

-   **Имя предложения.** Power BI в вашем мире: 4-часовой семинар
-   **Краткие сведения о предложении.** Запуск первой панели мониторинга и изучение практических рекомендаций. Для группы из 12 студентов проводится на месте.

*Пример 5*

-   **Имя предложения.** Dynamics и проекты: оценка за 3 дня
-   **Краткие сведения о предложении.** Сбор требований и оценка для ERP-решения, предназначенного для компаний-поставщиков профессиональных услуг и предприятий, управляемых на проектной основе.

##### <a name="offer-description"></a>Offer description (Описание предложения);

Это описание предложения консультационной услуги. Хорошее описание предложения должно содержать точные сведения о том, как будет выглядеть взаимодействие и каким будет конечный результат для клиента. Очевидно, что оно должно помочь клиенту понять, что он получит.

Не включайте в описание предложения ссылки на электронную почту или номера телефонов для связи с вами. В предложении предусмотрена кнопка Contact Me (Связаться со мной), передающая сведения о потенциальных клиентах в место назначения для управления ими, которое вы определите в предложении.

Описание предложения следует ввести в формате Markdown. Если вы не знакомы с форматом Markdown или HTML, можно просмотреть ресурсы на [сайте документации Майкрософт](https://docs.microsoft.com/contribute/how-to-write-use-markdown).

Использование этих форматов обеспечит максимальное удобство чтения для клиентов.

Стремитесь сделать описание предложения кратким и придерживаться ограничения на количество символов, так как пользователям не нравится читать большие объемы текста. У вас по-прежнему будет возможность отправлять маркетинговые брошюры, справочные материалы и другие документы с более подробным описанием вашего предложения.

В следующем примере показано хорошо составленное описание предложения, а также связанные с ним имя и краткие сведения.

**Имя предложения.** Облачная аналитика: 3-дневный семинар

**Краткие сведения о предложении.** Обзор Microsoft Azure и Power BI, оценка текущей среды и небольшое подтверждение концепции.

**Описание предложения.** Этот трехдневный семинар предназначен для руководителей организаций и технических отделов и проводится на месте на предприятии клиента.

***Программа***

День 1

-   Основное внимание уделяется тому, как защитить, масштабировать и упорядочить данных в облаке Майкрософт с помощью Azure Data Lake, HDInsight или Хранилища данных SQL Azure.

День 2

-   Описывается настройка и развертывание решений расширенной аналитики с помощью Microsoft R и службы "Машинное обучение Azure".

День 3

-   Описывается, как получать ценные аналитические сведения и операционную аналитику с помощью Power BI, включая сеанс совместной работы по созданию панели мониторинга Power BI.

К концу семинара клиент сможет самостоятельно составить общий план и график реализации решений аналитики и обработки данных в облачной среде Майкрософт.

*Пример файла Markdown для предложения в этом формате:*

    This 3-day workshop is for technical and business leaders and is held on-site at the client’s facility.

      ### Agenda

      **Day 1**

      * Focuses on how to secure, scale, and organize data within the Microsoft cloud using Azure Data Lake, HDInsight, or Azure SQL Data Warehouse

      **Day 2**

      * Covers how to configure and deploy advanced analytics solutions with Microsoft R and Azure Machine Learning

      **Day 3**

      * Covers how to draw actionable insights and operationalize analytics with Power BI, including a collaborative session to co-build a Power BI dashboard.


      ### Deliverables
      By the end of the workshop, the client will be able to define a high-level plan and an implementation roadmap for data and analytics solutions in the Microsoft cloud.


#### <a name="publisher-information"></a>**Сведения об издателе**

**MPN ID (Идентификатор MPN)**

Ваш 9-значный идентификатор в сети Microsoft Partner Network (MPN). Если у вас нет идентификатора MPN, его можно получить, посетив Центр партнеров Майкрософт.

**Partner Center ID (Идентификатор Центра партнеров)**

Новый идентификатор центра партнеров, если у вас имеется таковой.

**MPN ID (Идентификатор MPN)**

Введите секретный ключ для предварительного просмотра вашего предложения в AppSource перед активацией.
Это не пароль.

#### <a name="listing-details"></a>Указание сведений

**Consulting service type (Тип консультационной услуги)**

Корпорация Майкрософт фокусируется исключительно на фиксированном объеме, фиксированной длительности, оценочной или фиксированной цене (или бесплатном предложении) и в основном на предложениях услуг, ориентированных на предпродажи, для отдельного клиента, которые (для предложений оценки, реализации, подтверждения концепции, брифинга или семинара) проводятся либо на месте, либо виртуально (брифинги должны проводиться на месте).

Предусмотрены следующие пять типов предложений:

-   **Оценка**. Анализ среды клиента для определения применимости решения и предоставления оценки затрат и времени.
-   **Брифинг**. Представление решения или консультационной услуги для привлечения клиента с использованием схем, демонстрационных материалов и историй клиентов. Брифинги должны проводиться на месте.
-   **Реализация.** Полная установка, в результате которой клиент получит полностью рабочее решение. Для этой пилотной реализации Майкрософт рекомендует ограничиваться только теми решениями, которые могут быть реализованы за одну неделю или быстрее.
-   **Подтверждение концепции.** Реализация ограниченной области для определения того, будет ли решение соответствовать требованиям клиента.
-   **Семинар.** Интерактивное взаимодействие, проводимое на территории клиента, которое может включать в себя сеансы обучения, брифинги, оценки и демонстрации, основанные на данных или среде клиента.

**Доступность по странам и (или) регионам**

Выберите страну и регион, где будет доступно это предложение консультационных услуг. Одно предложение нельзя опубликовать в нескольких странах или регионах. Для каждой страны или региона необходимо создать новое предложение.

**Отрасли**

Выберите отрасли, к которым более всего применимо ваше предложение консультационных услуг.

**Duration**

Выберите число (например, 3, 4 и т. д.) в поле Duration (Длительность), а затем выберите Hour (часы), Day (дни) или Week (недели).

**Primary products** (Основные продукты)

Для публикации в Azure Marketplace выберите **Azure** в качестве основного продукта, а затем укажите соответствующие области решений.

Для публикации в AppSource выберите **Dynamics 365**, **Power BI** или **PowerApps** в качестве основного продукта. Можно также выбрать другие соответствующие применимые продукты, и ваше предложение консультационных услуг появится в списках предложений, связанных с каждым из этих продуктов в AppSource.

**Relevant Competencies** (Соответствующие компетенции)

Выберите компетенции, соответствующие этому предложению, чтобы они отображались вместе со сведениями о предложении.

#### <a name="marketing-artifacts"></a>Marketing Artifacts (Маркетинговые артефакты)

**Логотип компании (формат PNG, 48 x 48 пикселей)**

Отправьте изображение, которое будет отображаться на плитке вашего предложения на странице просмотра коллекции предложений. Изображение должно быть файлом в формате PNG с разрешением 48 x 48 пикселей.

**Логотип компании (формат PNG, 216 x 216 пикселей)**

Отправьте изображение, которое будет отображаться на странице сведений о вашем предложении. Изображение должно быть файлом в формате PNG с разрешением 216 x 216 пикселей.

**Видеоролики (максимум 4)**

Отправьте до четырех видеороликов с практическими примерами клиентов или со ссылками на них. Если у вас их нет, отправьте видеоролик с описанием того, как связан опыт вашей компании с этим предложением. Если есть демонстрационное видео в Power BI или PowerApps, отправьте его также. Ссылки на видео должны вести на YouTube или Vimeo.

**Документы (максимум 3)**

Отправьте маркетинговую брошюру с подробным описанием вашего предложения консультационных услуг. Вместо этого или в дополнение можно также отправить обзорный документ о компании, справочные материалы и практические примеры.

**Снимки экрана (максимум 5)**

Отправьте до пяти изображений, которые содержат дополнительные сведения о предложении, о конечных результатах предложения или о вашей компании. Например, фрагмент вашей маркетинговой брошюры, соответствующий слайд из презентации или рисунок, изображающий темп развития компании или опыт ее работы.

### <a name="publish-your-offer"></a>Публикация предложения

После заполнения параметров предложения, сведений об онлайн-магазине и контактных данных выберите **Publish** (Опубликовать) и укажите адрес электронной почты. Когда корпорация Майкрософт будет готова опубликовать ваше предложение, вы получите по электронной почте сообщение для его предварительного просмотра перед запуском. В любой момент можно вернуться на портал, чтобы проверить состояние вашего предложения.

Предложения в процессе публикации могут отображаться в состоянии Publish canceled (Публикация отменена) или Publish failed (Опубликовать не удалось). Это нормальная часть процесса, которая позволяет Майкрософт внести изменения в ваше предложение. Если ваше предложение отображается в состоянии Publish canceled (Публикация отменена), оставьте его в этом состоянии.
