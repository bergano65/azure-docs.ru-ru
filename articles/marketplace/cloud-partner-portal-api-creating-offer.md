# <a name="create-or-modify-an-offer"></a>Создание или изменение предложения

> [!NOTE]
> Портал Cloud Partner API интегрированы с и будут продолжать работать в центре партнеров. Переход содержит небольшие изменения. Ознакомьтесь с изменениями, приведенными в [справочнике по портал Cloud Partner API](./cloud-partner-portal-api-overview.md) , чтобы убедиться, что код продолжит работать после перехода в центр партнеров. API-интерфейсы CPP следует использовать только для существующих продуктов, которые уже были интегрированы до перехода в центр партнеров. новые продукты должны использовать API-интерфейсы отправки центра партнеров.

Этот вызов обновляет конкретное предложение в пространстве имен издателя или создает новое предложение.

  `PUT https://cloudpartner.azure.com/api/publishers/<publisherId>/offers/<offerId>?api-version=2017-10-31`

## <a name="uri-parameters"></a>Параметры универсального кода ресурса (URI)

|  **Имя**         |  **Описание**                      |  **Data type**  |
|  --------         |  ----------------                     |  -------------  |
| publisherid       |  Идентификатор издателя, например `contoso` |   Строка |
| offerId           |  Идентификатор предложения                     |   Строка        |
| api-version       |  Последняя версия API            |   Дата           |
|  |  |  |

## <a name="header"></a>Header

|  **имя**;        |  **Значение**               |
|  ---------       |  ----------              | 
| Content-Type     | `application/json`       |
| Авторизация    | `Bearer YOUR_TOKEN`      |
|  |  |

### <a name="response-status-codes"></a>Коды состояния ответа

| **Код**  |  **Описание**                                                                            |
| --------  |  ---------------                                                                            |
|  200      | `OK`. Запрос успешно обработан и предложение успешно изменено.           |
|  201      | `Created`. Запрос успешно обработан и предложение успешно создано.   |
|  400      | `Bad/Malformed request`. Текст ответа ошибки может предоставлять дополнительные сведения.            |
|  403      | `Forbidden`. Клиент не имеет доступа к требуемым пространствам имен.                     |
|  404      | `Not found`. Сущность, на которую ссылается клиент, не существует.                           |
|  412      | Сервер не отвечает ни одному из предусловий, указанных в запросе его инициатором. Клиент должен проверить ETAG, отправленный с запросом. |
|  |  |

## <a name="uploading-artifacts"></a>Загрузка артефактов

Артефакты, такие как образы и логотипы, должны отправляться, загружаясь в доступное расположение в Интернете, а затем включать каждый из них в качестве универсального кода ресурса (URI) в запросе отправки, как описано в примере выше. Система обнаружит, что эти файлы отсутствуют в хранилище Azure Marketplace, и скачайте эти файлы в хранилище.  В результате вы обнаружите, что будущие запросы GET будут возвращать URL-адрес службы Azure Marketplace для этих файлов.

## <a name="categories-and-industries"></a>Категории и отрасли

При создании нового предложения необходимо указать категорию для вашего предложения в Marketplace. При необходимости для некоторых типов предложений также можно указать отрасли. На основе типа предложения укажите категории и отрасли, применимые к предложению, используя конкретные значения ключа из следующих таблиц.
