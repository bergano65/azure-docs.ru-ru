# <a name="azure-security-baseline-for-azure-load-balancer"></a>Базовый план безопасности Azure для Azure Load Balancer

Базовый план безопасности Azure для Microsoft Azure Load Balancer содержит рекомендации, которые помогут повысить уровень безопасности развертывания. Базовый план безопасности для этой службы взят из [Эталона безопасности Azure версии 1.0](../security/benchmarks/overview.md), содержащего рекомендации по обеспечению безопасности облачных решений в Azure с помощью наших практических рекомендаций. Дополнительные сведения см. в статье [Обзор базовой конфигурации безопасности Azure](../security/benchmarks/security-baselines-overview.md).

## <a name="network-security"></a>Безопасность сети

*Дополнительные сведения см. в статье о [производительности системы безопасности Azure: безопасность сети](../security/benchmarks/security-control-network-security.md).*

### <a name="11-protect-azure-resources-within-virtual-networks"></a>1,1: защита ресурсов Azure в виртуальных сетях

**Руководство**. используйте внутренние подсистемы балансировки нагрузки Azure, чтобы разрешить трафик только к серверным ресурсам из определенных виртуальных сетей или в равноправных виртуальных сетях, не раскрывая Интернет. Реализуйте внешний Load Balancer с преобразованием адресов исходной сети (SNAT) для маскировки IP-адресов серверных ресурсов для защиты от прямого доступа в Интернет.

Azure предлагает два типа предложений Load Balancer, Standard и Basic. Используйте Load Balancer (цен. категория "Стандартный") для всех рабочих нагрузок. Реализуйте группы безопасности сети и разрешите доступ только к доверенным портам приложения и диапазонам IP-адресов. В случаях, когда нет группы безопасности сети, назначенной внутренней подсети или сетевому адаптеру серверных виртуальных машин, трафик не будет разрешен для этих ресурсов из балансировщика нагрузки. С помощью стандартных подсистем балансировки нагрузки предоставьте правила исходящего трафика для определения исходящего NAT с помощью группы безопасности сети. Просмотрите эти правила для исходящего трафика, чтобы настроить поведение исходящих подключений. 

Использование Load Balancer (цен. категория "Стандартный") рекомендуется для рабочих нагрузок, и обычно базовый Load Balancer используется только для тестирования, так как базовый тип открыт для подключений из Интернета по умолчанию и не требует групп безопасности сети для работы. 

- [Исходящие подключения в Azure](load-balancer-outbound-connections.md)

- [Обновление общедоступных Load Balancer Azure](./upgrade-basic-standard.md)

**Мониторинг Центра безопасности Azure**: Да

**Ответственность**: Customer

### <a name="12-monitor-and-log-the-configuration-and-traffic-of-virtual-networks-subnets-and-nics"></a>1,2. Мониторинг и запись конфигурации и трафика виртуальных сетей, подсетей и сетевых карт

**Руководство**. Load Balancer — это транзитная служба, которая зависит от правил групп безопасности сети, применяемых к внутренним ресурсам, и настроенных правил для исходящего трафика для управления доступом в Интернет.

Просмотрите правила исходящего трафика, настроенные для Load Balancer (цен. категория "Стандартный"), в колонке правила для исходящих подключений Load Balancer и в колонке правила балансировки нагрузки, где могут быть включены неявные правила исходящего трафика.

Отслеживайте количество исходящих подключений, чтобы отслеживать, как часто ресурсы помещаются в Интернет. 

Используйте центр безопасности и следуйте рекомендациям по защите сети, чтобы помочь защитить сетевые ресурсы Azure.

Следуйте рекомендациям по безопасности для внутренних ресурсов и включите журналы потоков для групп безопасности сети и отправьте журналы в учетную запись хранения Azure для аудита.

Также отправьте журналы потоков в Log Analytics рабочую область, а затем используйте Аналитика трафика, чтобы получить представление о шаблонах трафика в облаке Azure. Преимущества Аналитика трафика включают возможность визуализировать сетевые активности, выявлять горячие участки и угрозы безопасности, распознавать шаблоны потоков трафика и указывать несетевые настройки.

- [Включение журналов потоков для групп безопасности сети](../network-watcher/network-watcher-nsg-flow-logging-portal.md)

- [Как включить и использовать Аналитика трафика](../network-watcher/traffic-analytics.md)

### <a name="14-deny-communications-with-known-malicious-ip-addresses"></a>1.4. Запрет взаимодействия с известными опасными IP-адресами

**Руководство**. включение стандартной защиты "отказ в обслуживании" (от атак DDoS) в виртуальной сети Azure для защиты от атак от атак DDoS. 

Разместите брандмауэр Azure на всех границах сети Организации с включенной фильтрацией на основе аналитики угроз и настройте ее на "предупреждение и запретить" для вредоносного сетевого трафика.

 

Используйте центр безопасности для защиты от угроз, чтобы обнаружить связь с известными вредоносными IP-адресами. 

Load Balancer (цен. категория "Стандартный") по умолчанию обеспечивает безопасность и часть частной и изолированной виртуальной сети. Он закрывается с входящими потоками, если только они не открыты группами безопасности сети, чтобы явно разрешить разрешенный трафик и запретить известные вредоносные IP-адреса. Если группа безопасности сети в подсети или сетевом адаптере ресурса виртуальной машины не находится за Load Balancer, трафик не может достичь этого ресурса. 

Разверните брандмауэр Azure на всех границах сети Организации с включенной фильтрацией на основе аналитики угроз и настройте ее на "предупреждение и запретить" для предотвращения атак с вредоносных IP-адресов. 

Реализуйте рекомендации по интеграции брандмауэра Azure с Load Balancer.

Используйте средства защиты от угроз центра безопасности для обнаружения взаимодействий с известными вредоносными IP-адресами. 

Центр безопасности (уровень "Стандартный") обеспечивает JIT-доступ к виртуальным машинам и настраивает разрешенный IP-адрес для разрешения доступа только из утвержденных IP-адресов и диапазонов.
 

Используйте адаптивную функцию усиления защиты сети в центре безопасности, чтобы рекомендовать конфигурации групп безопасности сети, ограничивающие порты и исходные IP-адреса на основе фактического трафика и аналитики угроз.

### <a name="23-enable-audit-logging-for-azure-resources"></a>2.3. Включение журналов аудита для ресурсов Azure

**Руководство**. Проверьте ведение журнала управления и плоскости управления, а также данные аудита, полученные с помощью журналов действий для базового Load Balancer. Эти параметры записи включены по умолчанию. 

Используйте журналы действий для мониторинга действий с ресурсами, чтобы просмотреть все действия и их состояние. 

Определите, какие операции были выполнены с ресурсами в подписке с помощью журналов действий: 

- кто запустил операцию;
- когда была выполнена операция;
- состояние операции;

- значения других свойств, которые могут помочь в изучении операции.

Получите сведения из журнала действий с помощью Azure PowerShell, интерфейса командной строки Azure (CLI), REST API Azure или портал Azure. 

Реализуйте многомерную диагностику для возможностей настройки Load Balancer (цен. категория "Стандартный") с Azure Monitor.  К ним относятся доступные метрики для обеспечения безопасности: сведения о подключениях к источникам преобразования адресов в сети (SNAT), порты. Также доступны дополнительные метрики для пакетов SYN (синхронизация) и счетчиков пакетов.

### <a name="102-create-an-incident-scoring-and-prioritization-procedure"></a>10.2. Создание процедуры оценки инцидента и определения приоритетов

**Руководство**. Центр безопасности назначает каждому оповещению серьезность, которая поможет определить, какие предупреждения следует расследовать первыми. 

Серьезность основывается на том, насколько надежным является центр безопасности в поиске или аналитике, используемой для выдаче оповещения, а также об уровне достоверности, который был вредоносным намерением для действия, вызвавшего оповещение.
