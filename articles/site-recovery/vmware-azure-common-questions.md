# <a name="common-questions-about-vmware-to-azure-replication"></a>Распространенные вопросы о репликации из VMware в Azure

В этой статье содержатся ответы на распространенные вопросы, которые могут возникнуть при развертывании аварийного восстановления локальных виртуальных машин VMware в Azure.

## <a name="general"></a>Общие сведения

### <a name="what-do-i-need-for-vmware-vm-disaster-recovery"></a>Что нужно для аварийного восстановления виртуальных машин VMware?

[Сведения о компонентах, участвующих](vmware-azure-architecture.md) в аварийном восстановлении виртуальных машин VMware.

### <a name="can-i-use-site-recovery-to-migrate-vmware-vms-to-azure"></a>Можно ли использовать Site Recovery для переноса виртуальных машин VMware в Azure?

Да. В дополнение к использованию Site Recovery для настройки полного аварийного восстановления виртуальных машин VMware можно также использовать Site Recovery для миграции локальных виртуальных машин VMware в Azure. В этом сценарии выполняется репликация локальных виртуальных машин VMware в службу хранилища Azure. Затем выполняется отработка отказа из локальной среды в Azure. После отработки отказа приложения и рабочие нагрузки доступны и выполняются на виртуальных машинах Azure. Этот процесс аналогичен настройке полного аварийного восстановления, за исключением того, что во время миграции восстановление после сбоя из Azure невозможно.

### <a name="does-my-azure-account-need-permissions-to-create-vms"></a>Требуются ли учетной записи Azure разрешения на создание виртуальных машин?

Если вы являетесь администратором подписки, у вас есть необходимые разрешения на репликацию. Если вы не являетесь администратором, вам потребуются разрешения для выполнения следующих действий:

- Создайте виртуальную машину Azure в группе ресурсов и виртуальной сети, которую вы укажете при настройке Site Recovery.
- Запись в выбранную учетную запись хранения или на управляемый диск в соответствии с конфигурацией.

Дополнительные [сведения](site-recovery-role-based-linked-access-control.md#permissions-required-to-enable-replication-for-new-virtual-machines) о необходимых разрешениях.

### <a name="what-applications-can-i-replicate"></a>Какие приложения можно реплицировать?

Вы можете реплицировать любое приложение или рабочую нагрузку, выполняющуюся на виртуальной машине VMware, которая соответствует [требованиям репликации](vmware-physical-azure-support-matrix.md#replicated-machines).

- Site Recovery поддерживает репликацию с поддержкой приложений, чтобы обеспечить отработку отказа и отработку отказа приложений в интеллектуальное состояние.
- Site Recovery интегрируется с приложениями Майкрософт, такими как SharePoint, Exchange, Dynamics, SQL Server и Active Directory. Кроме того, он тесно работает с ведущими поставщиками, включая Oracle, SAP, IBM и Red Hat.

[Подробнее](site-recovery-workload.md) о защите рабочей нагрузки.

### <a name="can-i-use-a-guest-os-server-license-on-azure"></a>Можно ли использовать лицензию сервера гостевой ОС в Azure?

Да, клиенты Microsoft Software Assurance могут использовать [преимущество гибридного использования Azure](https://azure.microsoft.com/pricing/hybrid-benefit/) , чтобы сэкономить на стоимости лицензирования для компьютеров Windows Server, перенесенных в Azure, или для использования Azure для аварийного восстановления.

## <a name="security"></a>Безопасность

### <a name="what-access-to-vmware-servers-does-site-recovery-need"></a>Какой доступ к серверам VMware Site Recovery требуется?

Site Recovery требуется доступ к серверам VMware, чтобы:

- Настройте виртуальную машину VMware, на которой работает сервер конфигурации Site Recovery.
- автоматически обнаруживать виртуальные машины для репликации.

### <a name="what-access-to-vmware-vms-does-site-recovery-need"></a>Какой доступ к виртуальным машинам VMware Site Recovery требуется?

- Для репликации на виртуальной машине VMware должна быть установлена и запущена служба Site Recovery Mobility Service. Вы можете развернуть средство вручную или указать, что Site Recovery выполнить принудительную установку службы при включении репликации для виртуальной машины.
- Во время репликации виртуальные машины обмениваются данными с Site Recovery следующим образом:
    - Виртуальные машины обмениваются данными с сервером конфигурации через HTTPS порт 443 для управления репликацией.
    - Виртуальные машины отправляют данные репликации на сервер обработки через HTTPS порт 9443. (Этот параметр можно изменить.)
    - если включить согласованность между виртуальными машинами, они будут обмениваться данными друг с другом через порт 20004.

### <a name="is-replication-data-sent-to-site-recovery"></a>Отправляются ли данные репликации в Site Recovery?

Нет, Site Recovery не перехватывает реплицированные данные и не содержит сведений о том, что работает на ваших виртуальных машинах. Данные репликации обмениваются между гипервизорами VMware и хранилищем Azure. Служба Site Recovery не обладает способностью перехватывать эти данные. В службу Site Recovery передаются только метаданные, необходимые для управления репликацией и отработкой отказа.  

Site Recovery сертифицированы для ISO 27001:2013 и 27018, HIPAA и DPA. Он находится в процессе оценки SOC2 и FedRAMP JAB.

## <a name="managed-disks"></a>Управляемые диски

### <a name="where-does-site-recovery-replicate-data-to"></a>Куда Site Recovery реплицирует данные?

Site Recovery реплицирует локальные виртуальные машины VMware и физические серверы на управляемые диски в Azure.

- Site Recovery сервер обработки записывает журналы репликации в учетную запись хранения кэша в целевом регионе.
- Эти журналы используются для создания точек восстановления на дисках, управляемых Azure, имеющих префикс **асрсиддиск**.
- Когда происходит отработка отказа, выбранная точка восстановления используется для создания нового управляемого диска. Этот управляемый диск подключен к виртуальной машине в Azure.
- Виртуальные машины, которые ранее были реплицированы в учетную запись хранения (до 2019 марта), не затрагиваются.

### <a name="can-i-replicate-new-machines-to-storage-accounts"></a>Можно ли реплицировать новые компьютеры в учетные записи хранения?

Нет. Начиная с 2019 марта в портал Azure можно выполнять репликацию только на управляемые диски Azure.

Репликация новых виртуальных машин в учетную запись хранения доступна только с помощью PowerShell или REST API (версии 2018-01-10 или 2016-08-10).

### <a name="what-are-the-benefits-of-replicating-to-managed-disks"></a>Каковы преимущества репликации на управляемые диски?

[Узнайте, как](https://azure.microsoft.com/blog/simplify-disaster-recovery-with-managed-disks-for-vmware-and-physical-servers/) Site Recovery упрощает аварийное восстановление с помощью управляемых дисков.

### <a name="can-i-change-the-managed-disk-type-after-a-machine-is-protected"></a>Можно ли изменить тип управляемого диска после защиты компьютера?

Да, можно легко [изменить тип управляемого диска](https://docs.microsoft.com/azure/virtual-machines/windows/convert-disk-storage) для текущих репликаций. Перед изменением типа убедитесь, что на управляемом диске не создано ни одного URL-адреса подписи общего доступа:

1. Перейдите к ресурсу " **управляемый диск** " на портал Azure и проверьте, есть ли у вас заголовок URL-адреса для подписи общего доступа в колонке " **Обзор** ".
1. Если баннер имеется, выберите его для отмены текущего экспорта.
1. Измените тип диска в течение следующих нескольких минут. При изменении типа управляемого диска дождитесь создания новых точек восстановления с помощью Azure Site Recovery.
1. Используйте новые точки восстановления для тестовой отработки отказа или отработки отказа в будущем.

### <a name="can-i-switch-replication-from-managed-disks-to-unmanaged-disks"></a>Можно ли переключить репликацию с управляемых дисков на неуправляемые диски?

Нет. Переключение с управляемого на неуправляемое не поддерживается.

## <a name="replication"></a>Репликация

### <a name="what-are-the-replicated-vm-requirements"></a>Каковы требования к реплицируемой виртуальной машине?

Дополнительные [сведения](vmware-physical-azure-support-matrix.md#replicated-machines) о требованиях к поддержке для виртуальных машин VMware и физических серверов.

### <a name="how-often-can-i-replicate-to-azure"></a>Как часто можно выполнять репликацию в Azure?

Этот процесс является непрерывным при репликации виртуальных машин VMware в Azure.

### <a name="can-i-extend-replication"></a>Можно ли расширить репликацию?

Расширенная репликация и цепочка репликации не поддерживаются. Запросите эту функцию на [форуме отзывов и предложений](https://feedback.azure.com/forums/256299-site-recovery/suggestions/6097959).

### <a name="can-i-do-an-offline-initial-replication"></a>Можно ли выполнить начальную репликацию в автономном режиме?

Автономная репликация не поддерживается. Запросите эту функцию на [форуме отзывов и предложений](https://feedback.azure.com/forums/256299-site-recovery/suggestions/6227386-support-for-offline-replication-data-transfer-from).

### <a name="what-is-asrseeddisk"></a>Что такое асрсиддиск?

Для каждого исходного диска данные реплицируются на управляемый диск в Azure. Этот диск имеет префикс **асрсиддиск**. Он сохраняет копию исходного диска и все моментальные снимки точек восстановления.

### <a name="can-i-exclude-disks-from-replication"></a>Можно ли исключить диски из репликации?

Да, можно исключить диски.

### <a name="can-i-replicate-vms-that-have-dynamic-disks"></a>Можно ли реплицировать виртуальные машины с динамическими дисками?

Динамические диски могут быть реплицированы. Диск операционной системы должен представлять собой базовый диск.

### <a name="if-i-use-replication-groups-for-multi-vm-consistency-can-i-add-a-new-vm-to-an-existing-replication-group"></a>Если я использую группы репликации для согласованности нескольких виртуальных машин, могу ли я добавить новую виртуальную машину в имеющуюся группу репликации?

Да, вы можете добавить новые виртуальные машины в имеющуюся группу репликации при включении для них репликации. Несмотря

- После начала репликации вы не сможете добавить виртуальную машину в существующую группу репликации.
- Невозможно создать группу репликации для существующих виртуальных машин.

### <a name="can-i-modify-vms-that-are-replicating-by-adding-or-resizing-disks"></a>Можно ли изменить виртуальные машины, которые реплицируются, путем добавления дисков или изменения их размера?

Для репликации VMware в Azure можно изменить размер диска. Если вы хотите добавить новые диски, необходимо добавить диск и повторно включить защиту для виртуальной машины.

### <a name="can-i-migrate-on-premises-machines-to-a-new-vcenter-server-without-impacting-ongoing-replication"></a>Можно ли перенести локальные компьютеры в новую vCenter Server без влияния на текущую репликацию?

Нет. Изменение VMware vCenter или миграция повлияет на текущую репликацию. Настройте Site Recovery с новым vCenter Server и снова включите репликацию для компьютеров.

### <a name="can-i-replicate-to-a-cache-or-target-storage-account-that-has-a-virtual-network-with-azure-firewalls-configured-on-it"></a>Можно ли выполнить репликацию в кэш или целевую учетную запись хранения, для которой настроена виртуальная сеть (с брандмауэрами Azure)?

Нет, Site Recovery не поддерживает репликацию в службу хранилища Azure в виртуальных сетях.

## <a name="configuration-server"></a>Сервер конфигурации

### <a name="what-does-the-configuration-server-do"></a>Что делает сервер конфигурации?

На сервере конфигурации выполняются локальные компоненты Site Recovery, включая:

- Сам сервер конфигурации. Сервер координирует обмен данными между локальными компонентами и Azure и управляет репликацией данных.
- Сервер обработки, который выступает в качестве шлюза репликации. Этот сервер:
    1. Получает данные репликации.
    2. оптимизирует данные с помощью кэширования, сжатия и шифрования;
    3. Отправляет данные в службу хранилища Azure.
  Сервер обработки также осуществляет принудительную установку службы Mobility Service на виртуальных машинах и выполняет автоматическое обнаружение локальных виртуальных машин VMware.
- Главный целевой сервер, который обрабатывает данные репликации во время восстановления размещения из Azure.

[Подробнее](vmware-azure-architecture.md) о компонентах и процессах сервера конфигурации.

### <a name="where-do-i-set-up-the-configuration-server"></a>Где настраивается сервер конфигурации?

Для сервера конфигурации требуется одна высокодоступная локальная виртуальная машина VMware. Для аварийного восстановления физического сервера установите сервер конфигурации на физическом компьютере.

### <a name="what-do-i-need-for-the-configuration-server"></a>Что необходимо для сервера конфигурации?

Ознакомьтесь с [предварительными требованиями.](vmware-azure-deploy-configuration-server.md#prerequisites)

### <a name="can-i-manually-set-up-the-configuration-server-instead-of-using-a-template"></a>Можно ли вручную настроить сервер конфигурации, не используя шаблон?

Рекомендуется [создать виртуальную машину сервера конфигурации](vmware-azure-deploy-configuration-server.md) , используя последнюю версию шаблона Open VIRTUALIZATION Format (OVF). Если вы не можете использовать шаблон (например, если у вас нет доступа к серверу VMware), [Скачайте](physical-azure-set-up-source.md) файл установки с портала и настройте сервер конфигурации.

### <a name="can-a-configuration-server-replicate-to-more-than-one-region"></a>Может ли сервер конфигурации выполнять репликацию больше, чем в один регион?

Нет. Для репликации в несколько регионов требуется сервер конфигурации в каждом регионе.

### <a name="can-i-host-a-configuration-server-in-azure"></a>Можно ли разместить сервер конфигурации в Azure?

Хотя это возможно, виртуальная машина Azure, на которой работает сервер конфигурации, должна взаимодействовать с локальной инфраструктурой VMware и виртуальными машинами. Это взаимодействие увеличивает задержку и влияет на текущую репликацию.

### <a name="how-do-i-update-the-configuration-server"></a>Как обновить сервер конфигурации?

[Узнайте](vmware-azure-manage-configuration-server.md#upgrade-the-configuration-server) , как обновить сервер конфигурации.

- Последние сведения об обновлении можно найти на [странице обновлений Azure](https://azure.microsoft.com/updates/?product=site-recovery).
- Последнюю версию можно скачать на портале. Вы также можете скачать последнюю версию сервера конфигурации непосредственно из [центра загрузки Майкрософт](https://aka.ms/asrconfigurationserver).
- Если ваша версия более четырех версий старше текущей версии, см. [инструкции](https://aka.ms/asr_support_statement) по обновлению.

### <a name="should-i-back-up-the-configuration-server"></a>Следует ли выполнять резервное копирование сервера конфигурации?

Мы рекомендуем создавать плановые резервные копии сервера конфигурации.

- Для успешного восстановления размещения виртуальная машина, для которой выполняется откат, должна существовать в базе данных сервера конфигурации.
- Сервер конфигурации должен работать и находиться в подключенном состоянии.
- Дополнительные [сведения](vmware-azure-manage-configuration-server.md) о стандартных задачах управления сервером конфигурации.

### <a name="when-im-setting-up-the-configuration-server-can-i-download-and-install-mysql-manually"></a>Можно ли скачать и установить MySQL вручную при настройке сервера конфигурации?

Да. Скачайте MySQL и поместите его в папку К:\темп\асрсетуп Затем установите его вручную. После настройки виртуальной машины сервера конфигурации и принятия условий для MySQL в поле **Скачать и установить** будет показано значение **Уже установлено**.

### <a name="can-i-avoid-downloading-mysql-but-let-site-recovery-install-it"></a>Можно ли не скачивать MySQL и разрешить Site Recovery сделать это?

Да. Скачайте установщик MySQL и поместите его в папку К:\темп\асрсетуп При настройке виртуальной машины сервера конфигурации примите условия и выберите **скачать и установить**. На портале будет использоваться установщик, добавленный для установки MySQL.

### <a name="can-i-use-the-configuration-server-vm-for-anything-else"></a>Можно ли использовать виртуальную машину сервера конфигурации для чего-либо еще?

Нет. Используйте виртуальную машину только для сервера конфигурации.

### <a name="can-i-clone-a-configuration-server-and-use-it-for-orchestration"></a>Можно ли клонировать сервер конфигурации и использовать его для оркестрации?

Нет. Настройте новый сервер конфигурации, чтобы избежать проблем с регистрацией.

### <a name="can-i-change-the-vault-in-which-the-configuration-server-is-registered"></a>Можно ли изменить хранилище, в котором зарегистрирован сервер конфигурации?

Нет. После того как хранилище связано с сервером конфигурации, его нельзя изменить. [Узнайте](vmware-azure-manage-configuration-server.md#register-a-configuration-server-with-a-different-vault) , как зарегистрировать сервер конфигурации в другом хранилище.

### <a name="can-i-use-the-same-configuration-server-for-disaster-recovery-of-both-vmware-vms-and-physical-servers"></a>Можно ли использовать один и тот же сервер конфигурации для аварийного восстановления как виртуальных машин VMware, так и физических серверов?

Да, но обратите внимание, что физический компьютер может быть восстановлен только на виртуальной машине VMware.

### <a name="where-can-i-download-the-passphrase-for-the-configuration-server"></a>Где можно скачать парольную фразу для сервера конфигурации?

[Узнайте](vmware-azure-manage-configuration-server.md#generate-configuration-server-passphrase) , как скачать парольную фразу.

### <a name="where-can-i-download-vault-registration-keys"></a>Где можно загрузить ключи регистрации хранилища?

В хранилище служб восстановления выберите **серверы конфигурации** в **Site Recovery инфраструктуре** > **Управление**. Затем в разделе **серверы**выберите **скачать ключ регистрации** , чтобы скачать файл учетных данных хранилища.

### <a name="can-a-single-configuration-server-be-used-to-protect-multiple-vcenter-instances"></a>Можно ли использовать один сервер конфигурации для защиты нескольких экземпляров vCenter?

Да, один сервер конфигурации может защищать виртуальные машины на нескольких vCenter.  Не существует ограничения на количество экземпляров vCenter, которые могут быть добавлены на сервер конфигурации, однако ограничения на количество виртуальных машин, которые может защищать отдельный сервер конфигурации, не ограничены.

### <a name="can-a-single-configuration-server-protect-multiple-clusters-within-vcenter"></a>Может ли один сервер конфигурации защищать несколько кластеров в vCenter?

Да, Azure Site Recovery может защищать виртуальные машины в разных кластерах.

## <a name="process-server"></a>Сервер обработки

### <a name="why-am-i-unable-to-select-the-process-server-when-i-enable-replication"></a>Почему я не могу выбрать сервер обработки при включении репликации?

Обновления в версиях 9,24 и более поздних версий теперь отображают [работоспособность сервера обработки при включении репликации](vmware-azure-enable-replication.md#enable-replication). Эта функция позволяет избежать регулирования процессов и снизить использование неработоспособных серверов обработки.

### <a name="how-do-i-update-the-process-server-to-version-924-or-later-for-accurate-health-information"></a>Разделы справки обновить сервер обработки до версии 9,24 или более поздней для получения точных сведений о работоспособности?

Начиная с [версии 9,24](service-updates-how-to.md#links-to-currently-supported-update-rollups)для указания работоспособности сервера обработки были добавлены дополнительные предупреждения. [Обновите компоненты Site Recovery до версии 9,24 или более поздней](service-updates-how-to.md#links-to-currently-supported-update-rollups) , чтобы создавались все предупреждения.

## <a name="failover-and-failback"></a>Отработка отказа и восстановление размещения

### <a name="can-i-use-the-on-premises-process-server-for-failback"></a>Можно ли использовать локальный сервер обработки для восстановления размещения?

Мы настоятельно рекомендуем создать сервер обработки в Azure для целей восстановления размещения, чтобы избежать задержек при переносе данных. Кроме того, если вы разделили сеть исходных виртуальных машин с сетью, к которой подключено Azure, на сервере конфигурации, важно использовать сервер обработки, созданный в Azure, для восстановления размещения.

### <a name="can-i-keep-the-ip-address-on-failover"></a>Можно ли удержать IP-адрес при отработке отказа?

Да, вы можете удержать IP-адрес при отработке отказа. Перед отработкой отказа убедитесь, что целевой IP-адрес указан в параметрах **вычислений и сети** для виртуальной машины. Кроме того, завершите работу компьютеров во время отработки отказа, чтобы избежать конфликтов IP-адресов во время восстановления размещения.

### <a name="can-i-change-the-target-vm-size-or-vm-type-before-failover"></a>Можно ли изменить размер целевой виртуальной машины или тип виртуальной машины перед отработкой отказа?

Да, можно изменить тип или размер виртуальной машины в любое время перед отработкой отказа. На портале используйте параметры **вычислений и сети** для РЕПЛИЦИРОВАННОЙ виртуальной машины.

### <a name="how-far-back-can-i-recover"></a>До какой точки во времени можно выполнить восстановление?

Для VMware в Azure самая старая точка восстановления, которую можно использовать, составляет 72 часов.

### <a name="how-do-i-access-azure-vms-after-failover"></a>Как получить доступ к виртуальным машина Azure после отработки отказа?

После отработки отказа вы можете получить доступ к виртуальным машинам Azure через безопасное подключение к Интернету, через VPN типа "сеть — сеть" или через Azure ExpressRoute. Для подключения необходимо подготовить несколько вещей. [Узнайте больше](site-recovery-test-failover-to-azure.md#prepare-to-connect-to-azure-vms-after-failover).

### <a name="is-failed-over-data-resilient"></a>Отработка отказа данных отказоустойчива?

Среда Azure разработана для обеспечения отказоустойчивости. Site Recovery разработано для отработки отказа в дополнительный центр обработки данных Azure в соответствии с требованиями соглашения об уровне обслуживания Azure (SLA). При отработке отказа убедитесь, что метаданные и хранилища остаются в одном географическом регионе, выбранном для вашего хранилища.

### <a name="is-failover-automatic"></a>Отработка отказа выполняется автоматически?

[Отработка отказа](site-recovery-failover.md) не выполняется автоматически. Чтобы запустить отработку отказа, можно выбрать один из вариантов на портале или использовать [PowerShell](/powershell/module/az.recoveryservices) для активации отработки отказа.

### <a name="can-i-fail-back-to-a-different-location"></a>Можно ли выполнить восстановление размещения в другое расположение?

Да. Если вы отработка отказа в Azure, вы можете восстановить размещение в другое расположение, если исходное хранилище недоступно. [Узнайте больше](concepts-types-of-failback.md#alternate-location-recovery-alr).

### <a name="why-do-i-need-a-vpn-or-expressroute-with-private-peering-to-fail-back"></a>Зачем нужна виртуальная частная сеть или ExpressRoute с частным пирингом для отработки отказа?

При восстановлении после сбоя из Azure данные из Azure копируются обратно на локальную виртуальную машину, и требуется частный доступ.

### <a name="can-i-resize-the-azure-vm-after-failover"></a>Можно ли изменить размер виртуальной машины Azure после отработки отказа?

Нет, вы не сможете изменить размер или тип целевой виртуальной машины после отработки отказа.
