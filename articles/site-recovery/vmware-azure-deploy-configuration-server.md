# <a name="deploy-a-configuration-server"></a>Развертывание сервера конфигурации.

При использовании [Azure Site Recovery](site-recovery-overview.md) для аварийного восстановления виртуальных машин VMware и физических серверов в Azure развертывается локальный сервер конфигурации. Сервер конфигурации используется для управления обменом данными между локальной средой VMware и Azure. Он также управляет репликацией данных. В этой статье рассматриваются действия, необходимые для развертывания сервера конфигурации при репликации виртуальных машин VMware в Azure. Если необходимо настроить сервер конфигурации для репликации физического сервера, см. раздел [Настройка сервера конфигурации для аварийного восстановления физических серверов в Azure](physical-azure-set-up-source.md).

> [!TIP]
> Дополнительные сведения о роли сервера конфигурации в рамках архитектуры Azure Site Recovery см. в статье [архитектура аварийного восстановления из VMware в Azure](vmware-azure-architecture.md).

## <a name="deploy-a-configuration-server-through-an-ova-template"></a>Развертывание сервера конфигурации с помощью шаблона OVA

Сервер конфигурации должен быть настроен в качестве высокодоступной виртуальной машины VMware с определенными минимальными требованиями к оборудованию и размерам. Для удобства и простоты развертывания Site Recovery предоставляет загружаемый шаблон приложения Open Virtualization (OVA) для настройки сервера конфигурации, который соответствует всем требованиям, указанным здесь.

## <a name="azure-active-directory-permission-requirements"></a>Требования к разрешениям Azure Active Directory

Чтобы зарегистрировать сервер конфигурации в службах Azure Site Recovery, необходимо иметь пользователя с одним из следующих разрешений, установленным в Azure Active Directory (Azure AD).

1. Для создания приложения пользователь должен иметь роль разработчика приложения.
    - Чтобы проверить, войдите в портал Azure.</br>
    - Перейдите в раздел **Azure Active Directory** > **ролей и администраторов**.</br>
    - Убедитесь, что пользователю назначена роль разработчика приложения. В противном случае используйте пользователя с этим разрешением или обратитесь к [администратору, чтобы включить разрешение](https://docs.microsoft.com/azure/active-directory/fundamentals/active-directory-users-assign-role-azure-portal#assign-roles).
    
2. Если роль разработчика приложения не может быть назначена, убедитесь, что для параметра **пользователи могут регистрировать приложения** задано **значение true** , чтобы пользователь мог создать удостоверение. Чтобы включить эти разрешения, сделайте следующее:
    - Войдите на портал Azure.
    - Перейдите в раздел **Azure Active Directory** > **Параметры пользователя**.
    - Вразделе Регистрация приложений **пользователи могут регистрировать приложения**, а затем выбрать **Да**.

      ![AD_application_permission Azure](media/vmware-azure-deploy-configuration-server/AAD_application_permission.png)

> [!NOTE]
> Службы федерации Active Directory (AD FS) *не поддерживается*. Используйте учетную запись, управляемую с помощью [Azure Active Directory](https://docs.microsoft.com/azure/active-directory/fundamentals/active-directory-whatis).

## <a name="download-the-template"></a>Скачивание шаблона

1. В хранилище выберите **Prepare Infrastructure** (Подготовка инфраструктуры)  > **Source** (Источник).
2. В колонке **Подготовка источника** выберите **+ Сервер конфигурации**.
3. В колонке **Add Server** (Добавление сервера) в поле **Server type** (Тип сервера) должно быть указано **Configuration server for VMware** (Сервер конфигурации для VMware).
4. Скачайте шаблон OVA для сервера конфигурации.

   > [!TIP]
   >Последнюю версию шаблона сервера конфигурации также можно скачать непосредственно из [центра загрузки Майкрософт](https://aka.ms/asrconfigurationserver).

> [!NOTE]
> Лицензия, предоставленная с помощью шаблона OVA, является лицензией на ознакомительную версию, которая действительна в течение 180 дней. По истечении этого периода необходимо приобрести лицензию.

## <a name="import-the-template-in-vmware"></a>Импорт шаблона в VMware

1. Подключитесь к серверу VMware vCenter или узлу vSphere ESXi с помощью клиента VMWare vSphere.
2. В меню **файл** выберите пункт **развернуть шаблон OVF** , чтобы запустить мастер **развертывания шаблонов OVF** .

     ![Развертывание шаблона OVF](./media/vmware-azure-deploy-configuration-server/vcenter-wizard.png)

3. Укажите расположение скачанного шаблона OVF в разделе **Выбрать источник**.
4. В разделе **Просмотр сведений** выберите **Далее**.
5. Примите значения параметров по умолчанию в разделах **Select name and folder** (Выбор имени и папки) и **Select configuration** (Выбор конфигурации).
6. В разделе **Select storage** (Выбор хранилища), чтобы улучшить производительность, выберите **Thick Provision Eager Zeroed** (Быстрое обнуление плотной подготовки) в области **Select virtual disk format** (Выбор формата виртуального диска). Использование параметра тонкой подготовки может повлиять на производительность сервера конфигурации.
7. На остальных страницах мастера примите значение параметров по умолчанию.
8. На странице **Ready to complete** (Все готово к выполнению) сделайте следующее:

    * Чтобы настроить виртуальную машину с параметрами по умолчанию, выберите **Power on after deployment** (Включение после развертывания)  > **Finish** (Готово).
    * Чтобы добавить дополнительный сетевой интерфейс, снимите флажок **Power on after deployment** (Включить после развертывания), а затем щелкните **Finish** (Готово). По умолчанию шаблон сервера конфигурации развертывается с одним сетевым адаптером. Вы можете добавить дополнительные сетевые адаптеры после развертывания.

> [!IMPORTANT]
> Не изменяйте конфигурации ресурсов, такие как память, ядра и ограничение на ЦП, а также изменяйте или удаляйте установленные службы или файлы на сервере конфигурации после развертывания. Эти типы изменений влияют на регистрацию сервера конфигурации в службах Azure и производительности сервера конфигурации.

## <a name="add-an-additional-adapter"></a>Добавление дополнительного адаптера

> [!NOTE]
> Если вы планируете хранить IP-адреса исходных компьютеров на отработку отказа и хотите восстановить их в локальной среде, требуются две сетевые карты. Одна сетевая карта подключена к исходным компьютерам, и другая сетевая карта используется для подключения к Azure.

Если вы хотите добавить дополнительный сетевой адаптер к серверу конфигурации, добавьте его до регистрации сервера в хранилище. Возможность добавления дополнительных адаптеров не поддерживается после регистрации.

1. В списке клиента vSphere щелкните виртуальную машину правой кнопкой мыши и выберите **Edit Settings** (Изменить параметры).
2. В разделе **Hardware** (Оборудование) выберите **Add** > **Ethernet Adapter** (Добавить > Адаптер Ethernet). Затем нажмите кнопку **Далее**.
3. Выберите тип адаптера и сеть.
4. Чтобы подключить виртуальный сетевой адаптер на включенной виртуальной машине, выберите **Connect at power on** (Подключиться, когда включено). Затем выберите **Next** > **Finish** > **OK** (Далее > Готово > ОК).

## <a name="register-the-configuration-server-with-azure-site-recovery-services"></a>Регистрация сервера конфигурации с помощью служб Azure Site Recovery

1. Из консоли клиента VMWare vSphere включите виртуальную машину.
2. Виртуальная машина загружается в среду установки Windows Server 2016. Примите лицензионное соглашение и введите пароль администратора.
3. После установки войдите в виртуальную машину от имени администратора.
4. При первом входе в систему через несколько секунд запускается средство настройки Azure Site Recovery.
5. Введите имя, которое использовалось для регистрации сервера конфигурации в службе Site Recovery. Затем нажмите кнопку **Далее**.
6. Средство проверяет, может ли виртуальная машина подключиться к Azure. После установки подключения выберите **Sign in** (Вход), чтобы войти в подписку Azure.</br>
    a. Учетные данные должны иметь доступ к хранилищу, в котором вы хотите зарегистрировать сервер конфигурации.</br>
    Б. Убедитесь, что выбранная учетная запись пользователя имеет разрешение на создание приложения в Azure. Чтобы включить необходимые разрешения, следуйте указаниям в разделе [Azure Active Directory требования к разрешениям](#azure-active-directory-permission-requirements).
7. Средство выполняет некоторые задачи конфигурации, а затем перезагружается.
8. Снова войдите в систему. Мастер управления сервером конфигурации запускается автоматически через несколько секунд.

### <a name="configure-settings"></a>Настройка параметров

1. В мастере управления сервером конфигурации выберите **Setup connectivity** (Настройка подключения). В раскрывающихся списках сначала выберите сетевую карту, которую встроенный сервер обработки использует для обнаружения и принудительной установки службы Mobility Service на исходных компьютерах. Затем выберите сетевую карту, которую сервер конфигурации использует для подключения к Azure. Щелкните **Сохранить**. Этот параметр уже нельзя изменить после настройки. Не изменяйте IP-адрес сервера конфигурации. Убедитесь, что IP-адрес, назначенный серверу конфигурации, является статическим IP-адресом, а не IP-адресом DHCP.
2. На странице **Выбор хранилища служб восстановления**войдите в Microsoft Azure с учетными данными, которые использовались на шаге 6 в статье [Регистрация сервера конфигурации в службах Azure Site Recovery](#register-the-configuration-server-with-azure-site-recovery-services).
3. После входа выберите подписку Azure и соответствующую группу ресурсов и хранилище.

    > [!NOTE]
    > После регистрации нет никакой гибкости изменять хранилище служб восстановления.
    > Для изменения хранилища служб восстановления необходимо отменять связь сервера конфигурации с текущим хранилищем, а также остановить репликацию всех защищенных виртуальных машин на сервере конфигурации. Дополнительные сведения см. в статье [Управление сервером конфигурации для аварийного восстановления виртуальной машины VMware](vmware-azure-manage-configuration-server.md#register-a-configuration-server-with-a-different-vault).

4. По **установке стороннего программного обеспечения**:

    |Сценарий   |Инструкции  |
    |---------|---------|
    |Можно ли скачать и установить MySQL вручную?     |  Да. Скачайте приложение MySQL, поместите его в папку **к:\темп\асрсетуп**, а затем установите вручную. После принятия условий и выбора **скачивания и установки**на портале будет указано, что *уже установлено*. Можно перейти к следующему шагу.       |
    |Можно ли избежать скачивания MySQL через Интернет?     |   Да. Поместите установщик MySQL в папку **C:\Temp\ASRSetup**. Примите условия, выберите **скачать и установить**, и на портале будет использоваться установщик, добавленный для установки приложения. После завершения установки перейдите к следующему шагу.    |
    |Я хочу скачать и установить MySQL с помощью Azure Site Recovery.    |  Примите условия лицензионного соглашения и выберите **скачать и установить**. После завершения установки перейдите к следующему шагу.       |

5. Прежде чем продолжить, убедитесь, что предварительные требования включены в раздел **Проверка конфигурации устройства**.
6. На странице **настройка vCenter Server/vSphere ESXi Server**введите полное доменное имя или IP-адрес сервера vCenter или узла vSphere, где находятся виртуальные машины, которые нужно реплицировать. Введите порт, от которого ожидается передача данных на сервер. Введите понятное имя для сервера VMware в хранилище.
7. Введите учетные данные, которые будут использоваться сервером конфигурации для подключения к серверу VMware. Site Recovery использует эти учетные данные для автоматического обнаружения виртуальных машин VMware, доступных для репликации. Выберите **Добавить** > **Продолжить**. Введенные здесь учетные данные сохраняются локально.
8. На странице **Настройка учетных данных виртуальной машины**введите имя пользователя и пароль виртуальных машин для автоматической установки службы Mobility Service во время репликации. Для компьютеров **Windows** учетной записи должны быть предоставлены права локального администратора на компьютерах, которые требуется реплицировать. Для **Linux** предоставьте сведения для учетной записи root.
9. Выберите **Finalize configuration** (Завершение конфигурации) для завершения регистрации.
10. После завершения регистрации откройте портал Azure и убедитесь, что сервер конфигурации и сервер VMware указаны в **хранилище служб восстановления** > **управление** **инфраструктурой > Site Recovery** >  **Серверы конфигурации**.

## <a name="faqs"></a>Часто задаваемые вопросы

* Какова срок действия лицензии на сервере конфигурации, развернутом через OVF? Что произойдет, если я не буду повторно активировать лицензию?

    Лицензия, предоставленная с помощью шаблона OVA, является лицензией на ознакомительную версию, действующую в течение 180 дней. Перед истечением срока действия необходимо активировать лицензию. В противном случае это может привести к частому завершению работы сервера конфигурации и препятствиемию действий репликации. Дополнительные сведения см. [в статье Управление сервером конфигурации для аварийного восстановления виртуальной машины VMware](vmware-azure-manage-configuration-server.md#update-windows-license).

* Можно ли использовать виртуальную машину, на которой установлен сервер конфигурации для разных целей?

    Нет Используйте виртуальную машину только в целях сервера конфигурации. Убедитесь, что выполнены все спецификации, указанные в статье [Предварительные требования](#prerequisites) для эффективного управления аварийным восстановлением.
* Можно ли сменить хранилище, уже зарегистрированное на сервере конфигурации, на новое хранилище?

    Нет После регистрации хранилища на сервере конфигурации его нельзя изменить.
* Можно ли использовать один и тот же сервер конфигурации для защиты физических и виртуальных машин?

    Да. Один и тот же сервер конфигурации можно использовать для репликации физических и виртуальных машин. Тем не менее физический компьютер может быть восстановлен только на виртуальной машине VMware.
* Какова цель сервера конфигурации и где он используется?

    Дополнительные сведения о сервере конфигурации и его функциональных возможностях см. в статье [архитектура репликации из VMware в Azure](vmware-azure-architecture.md).
* Где можно найти последнюю версию сервера конфигурации?

    Ознакомьтесь с инструкциями по [обновлению сервера конфигурации на портале](vmware-azure-manage-configuration-server.md#upgrade-the-configuration-server). Инструкции по обновлению всех компонентов Site Recovery см. в разделе [обновления службы в Site Recovery](https://aka.ms/asr_how_to_upgrade).
* Где можно загрузить парольную фразу для сервера конфигурации?

    Чтобы скачать парольную фразу, см. раздел [Управление сервером конфигурации для аварийного восстановления виртуальной машины VMware](vmware-azure-manage-configuration-server.md#generate-configuration-server-passphrase).
* Могу ли я изменить парольную фразу?

    Нет Не изменяйте парольную фразу сервера конфигурации. Изменение парольной фразы нарушает репликацию защищенных компьютеров и ведет к критическому состоянию работоспособности.
* Где можно загрузить ключи регистрации хранилища?

    В **хранилище служб восстановления**выберите **управление** > **инфраструктуры Site Recovery** > **серверы конфигурации**. В списке **Серверы** выберите **Скачать ключ регистрации**, чтобы загрузить файл учетных данных хранилища.
* Можно ли клонировать существующий сервер конфигурации и использовать его для оркестрации репликации?

    Нет Использование клонированного компонента сервера конфигурации не поддерживается. Клонирование сервера обработки масштабирования также является неподдерживаемым сценарием. Клонирование компонентов Site Recovery влияет на текущие репликации.

* Можно ли изменить IP-адрес сервера конфигурации?

    Нет Не изменяйте IP-адрес сервера конфигурации. Убедитесь, что все IP-адреса, назначенные серверу конфигурации, являются статическими и не являются IP DHCP.
* Можно ли настроить сервер конфигурации в Azure?

    Настройте сервер конфигурации в локальной среде с прямыми сведениями с помощью v-Center и для снижения задержек при переносе данных. Резервные копии сервера конфигурации можно создавать по расписанию для [восстановления размещения](vmware-azure-manage-configuration-server.md#failback-requirements).

Дополнительные вопросы о серверах конфигурации см. в разделе [Общие вопросы по конфигурации сервера](vmware-azure-common-questions.md#configuration-server).
