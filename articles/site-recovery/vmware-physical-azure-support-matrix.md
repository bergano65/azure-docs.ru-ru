# <a name="support-matrix-for-disaster-recovery--of-vmware-vms-and-physical-servers-to-azure"></a>Таблица поддержки аварийного восстановления виртуальных машин VMware и физических серверов в Azure

В этой статье перечислены поддерживаемые компоненты и параметры для аварийного восстановления виртуальных машин VMware в Azure с помощью службы [Azure Site Recovery](site-recovery-overview.md).

Чтобы начать работу с самым простым сценарием развертывания с помощью Azure Site Recovery, изучите наши [руководства](tutorial-prepare-azure.md). Дополнительные сведения об архитектуре Azure Site Recovery см. [здесь](vmware-azure-architecture.md).

## <a name="replication-scenario"></a>Сценарий репликации

**Сценарий** | **Сведения**
--- | ---
Виртуальные машины VMware | Репликация локальных виртуальных машин VMware в Azure. Этот сценарий можно развернуть с помощью портала Azure или [PowerShell](vmware-azure-disaster-recovery-powershell.md).
физические серверы; | Репликация локальных физических серверов Windows или Linux в Azure. Этот сценарий можно развернуть с помощью портала Azure.

## <a name="site-recovery-configuration-server"></a>Сервер конфигурации Site Recovery

Сервер конфигурации размещается на локальном компьютере, где выполняются компоненты Site Recovery, включая сервер конфигурации, сервер обработки и главный целевой сервер. Для репликации VMware необходимо настроить сервер конфигурации в соответствии со всеми требованиями. Используйте шаблон OVF, чтобы создать виртуальную машину VMware. Для репликации физического сервера необходимо настроить компьютер с сервером конфигурации вручную.

## <a name="replicated-machines"></a>Реплицируемые компьютеры

Site Recovery поддерживает репликацию любой рабочей нагрузки, выполняемой на поддерживаемом компьютере.

**Компонент** | **Сведения**
--- | ---
Параметры компьютера | Компьютеры, которые реплицируются в Azure, должны соответствовать [требованиям Azure](#azure-vm-requirements).
Рабочая нагрузка компьютера | Site Recovery поддерживает репликацию любой рабочей нагрузки (например, Active Directory, SQL Server и т. д.), выполняемой на поддерживаемом компьютере. [Узнайте больше](https://aka.ms/asr_workload).
Операционная система Windows | 64-разрядная версия Windows Server 2016 (Server Core, сервер с возможностями рабочего стола), Windows Server 2012 R2, Windows Server 2012, Windows Server 2008 R2 как минимум с пакетом обновления 1. </br></br>  [Windows Server 2008 (минимум с пакетом обновления 2) — 32- и 64-разрядная версия](migrate-tutorial-windows-server-2008.md) (только миграция). </br></br> Windows Nano Server 2016 не поддерживается.
Архитектура операционной системы Linux | Поддерживается только 64-разрядной системе. 32-разрядной системе не поддерживается.
Операционная система Linux | Red Hat Enterprise Linux: 5.2–5.11<b>\*\*</b>, 6.1–6.10<b>\*\*</b>, 7.0–7.6 <br/><br/>CentOS: 5.2–5.11<b>\*\*</b>, 6.1–6.10<b>\*\*</b>, 7.0–7.6 <br/><br/>Сервер Ubuntu версии 14.04 LTS [(поддерживаемые версии ядра)](#ubuntu-kernel-versions)<br/><br/>Сервер Ubuntu версии 16.04 LTS [(поддерживаемые версии ядра)](#ubuntu-kernel-versions)<br/><br/>Debian 7 и Debian 8[ (поддерживаемые версии ядра)](#debian-kernel-versions)<br/><br/>SUSE Linux Enterprise Server 12 SP1,SP2,SP3 [ (поддерживаемые версии ядра)](#suse-linux-enterprise-server-12-supported-kernel-versions)<br/><br/>SUSE Linux Enterprise Server 11 с пакетом обновления 3 (SP3)<b>\*\*</b> или SUSE Linux Enterprise Server 11 с пакетом обновления 4 (SP4)* </br></br>Oracle Linux 6.4, 6.5, 6.6, 6.7, 6.8, 6.9, 6.10, 7.0, 7.1, 7.2, 7.3, 7.4, 7.5 с ядром, совместимым с Red Hat, или с ядром Unbreakable Enterprise Kernel Release 3 (UEK3) <br/><br/></br>- Обновление реплицированных виртуальных машин с SUSE Linux Enterprise Server 11 SP3 до SP4 не поддерживается. Для обновления отключите репликацию и включите ее снова после обновления.</br></br> - [См. дополнительные сведения](https://support.microsoft.com/help/2941892/support-for-linux-and-open-source-technology-in-azure) о поддержке Linux и технологий с открытым кодом в Azure. Site Recovery координирует отработку отказа для запуска серверов Linux в Azure. Но поставщики Linux могут поддерживать только те версии дистрибутивов, срок поддержки которых еще не окончился.<br/><br/> - В дистрибутивах Linux поддерживаются только комплектные ядра, которые являются частью выпуска или обновления с дополнительным номером версии.<br/><br/> - Обновление защищенных виртуальных машин до основных версий дистрибутивов Linux не поддерживается. Для обновления отключите репликацию, обновите операционную систему и включите репликацию снова.<br/><br/> - На серверах под управлением Red Hat Enterprise Linux 5.2-5.11 или CentOS 5.2-5.11 следует установить [компоненты Linux Integration Services (LIS)](https://www.microsoft.com/download/details.aspx?id=55106), чтобы эти серверы могли загружаться в Azure.

## <a name="azure-site-recovery-churn-limits"></a>Обработка ограничения Azure Site Recovery

В таблице ниже приведены ограничения Azure Site Recovery. Эти ограничения получены на основе наших проверок, но они не охватывают все возможные сочетания операций ввода-вывода в приложении. Фактические результаты зависят от сочетания операций ввода-вывода приложения. Для получения наилучших результатов рекомендуется для [Запуск планировщика развертывания](site-recovery-deployment-planner.md) и выполнять расширенное тестирование приложения, выполнив тестовую отработку отказа, чтобы получить точные сведения о производительности приложения.

**Целевое хранилище репликации** | **Среднее исходный диск объема ввода-вывода** |**Скорость обработки данных среднего исходного диска** | **"Обработка" общего источника данных на диске в день**
---|---|---|---
Хранилище уровня "Стандартный" | 8 КБ | 2 МБ/с | 168 ГБ на диск
Диск P10 или P15 класса Premium | 8 КБ  | 2 МБ/с | 168 ГБ на диск
Диск P10 или P15 класса Premium | 16 КБ | 4 МБ/с |  336 ГБ на диск
Диск P10 или P15 класса Premium | 32 КБ или выше | 8 МБ/с | 672 ГБ на диск
Диск P20, P30, P40 или P50 класса Premium | 8 КБ    | 5 МБ/с | 421 ГБ на диск
Диск P20, P30, P40 или P50 класса Premium | 16 КБ или выше |20 МБ в секунду. | 1684 ГБ на диск

**Скорость обработки источника данных** | **Максимальное ограничение**
---|---
Средняя скорость обработки данных на виртуальную машину| 25 МБ/с
Пиковая скорость обработки данных на всех дисках виртуальной машины | 54 МБ/с
Максимальный объем обработки данных, поддерживаемый сервером обработки | 2 ТБ

В указанных средних значениях предполагается 30-процентное перекрытие операций ввода-вывода. В зависимости от коэффициента перекрытия, размера записи и фактической рабочей нагрузки ввода-вывода Site Recovery может обрабатывать более высокую пропускную способность. В приведенных выше значениях предполагается, что длина очереди невыполненной работы составляет примерно 5 минут, то есть после передачи выполняется обработка данных, а созданная точка восстановления составляет 5 минут.
