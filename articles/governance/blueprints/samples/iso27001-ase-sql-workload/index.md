# <a name="overview-of-the-iso-27001-app-service-environmentsql-database-workload-blueprint-sample"></a>Обзор примера схемы рабочей нагрузки Среды службы приложений или Базы данных SQL на основе ISO 27001

В примере схемы рабочей нагрузки Среды службы приложений или Базы данных SQL на основе ISO 27001 предоставляется дополнительная инфраструктура для примера схемы [ISO 27001: общие службы](../iso27001-shared/index.md).
С помощью этой схемы клиенты могут развертывать облачные архитектуры, которые предоставляют решения для сценариев, включающих требования к аккредитации или соответствию.

Есть два примера схемы на основе ISO 27001: этот пример и пример схемы [ISO 27001: общие службы](../iso27001-shared/index.md).

> [!IMPORTANT]
> Этот пример зависит от инфраструктуры, развертываемой с помощью примера схемы [ISO 27001: общие службы](../iso27001-shared/index.md), который нужно развернуть первым.

## <a name="architecture"></a>Architecture

Пример схемы рабочей нагрузки Среды службы приложений или Базы данных SQL на основе ISO 27001 позволяет развернуть платформу в качестве веб-среды на основе службы. В этой среде можно размещать несколько веб-приложений, веб-интерфейсов API и экземпляров Базы данных SQL в соответствии со стандартами ISO 27001. Этот пример схемы основан на примере схемы [ISO 27001: общие службы](../iso27001-shared/index.md).

:::image type="content" source="../../media/sample-iso27001-ase-sql-workload/iso27001-ase-sql-workload-blueprint-sample-design.png" alt-text="Пример проектирования схемы рабочей нагрузки ASE или SQL на основе ISO 27001" border="false":::

Эта среда содержит несколько служб Azure, которые обеспечивают защищенную, полностью отслеживаемую инфраструктуру рабочей нагрузки корпоративного уровня на основе стандартов ISO 27001. Эта среда включает следующие компоненты:

- [Роль Azure](../../../../role-based-access-control/overview.md) с названием DevOps, а также с правами на развертывание и администрирование ресурсов, развернутых с помощью примера схемы в [Средах службы приложений Azure](../../../../app-service/environment/intro.md).
- Определения [Политики Azure](../../../policy/overview.md), которые позволяют ограничить список служб, развертываемых в среде, и запретить создание общедоступных IP-адресов (PIP).
- Виртуальная сеть с одной подсетью и обратным пиринговым подключением к существующей среде [общих служб](../iso27001-shared/index.md) для принудительной передачи всего трафика через брандмауэр [общих служб](../iso27001-shared/index.md). В виртуальной сети размещаются следующие ресурсы:
  - Экземпляры [Среды службы приложений Azure](../../../../app-service/environment/intro.md), которые можно использовать для размещения веб-приложений, веб-интерфейсов API или функций.
  - Экземпляр [Azure Key Vault](../../../../key-vault/general/overview.md), использующий конечную точку службы виртуальной сети, для хранения секретов приложений, которые работают в среде рабочей нагрузки.
  - Экземпляр [Базы данных SQL Azure](../../../../azure-sql/database/sql-database-paas-overview.md), использующий конечную точку службы виртуальной сети, для размещения баз данных для приложений в среде рабочей нагрузки.
