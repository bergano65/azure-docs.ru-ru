# <a name="understand-scope-in-azure-policy"></a>Общие сведения о области в политике Azure

Существует ряд параметров, которые определяют, какие ресурсы могут быть оценены, и какие ресурсы оцениваются политикой Azure. Основной концепцией для этих элементов управления является _область_.
Общий обзор см. [в разделе Scope в Azure Resource Manager](../../../azure-resource-manager/management/overview.md#understand-scope).
В этой статье объясняется влияние _области_ в политике Azure, а также связанные с ней объекты и свойства.

## <a name="definition-location"></a>Расположение определения

Первая область экземпляра, используемая политикой Azure, — это время создания определения политики. Определение может быть сохранено либо в группе управления, либо в подписке. Расположение определяет область, в которую может быть назначена инициатива или политика. Ресурсы должны находиться в пределах иерархии ресурсов расположения определения для назначения.

Если расположение определения является

- Только для **подписки** на ресурсы в этой подписке можно назначить определение политики.
- Определение политики может быть назначено только для ресурсов **группы** управления в дочерних группах управления и дочерних подписках. Если вы планируете применить определение политики к нескольким подпискам, расположение должно быть группой управления, содержащей каждую подписку.

Расположение должно быть контейнером ресурсов, совместно используемым всеми ресурсами, для которых вы хотите использовать определение политики. Этот контейнер ресурсов обычно является группой управления, расположенной рядом с корневой группой управления.

## <a name="assignment-scopes"></a>Области назначения

Назначение имеет несколько свойств, которые задают область. Использование этих свойств определяет, какой ресурс для политики Azure будет оцениваться, и какие ресурсы подсчитывается на соответствие. Эти свойства соответствуют следующим понятиям:

- Включение — иерархия ресурсов или отдельного ресурса должна быть оценена в соответствии с определением. `properties.scope`Свойство объекта назначения определяет, какие объекты включать и оценивать для обеспечения соответствия. Дополнительные сведения см. в разделе [Определение назначения](./assignment-structure.md).

- Исключение. не следует оценивать иерархию ресурсов или отдельный ресурс на соответствие определению. Свойство `properties.notScopes` _Array_ объекта назначения определяет, что следует исключить. Ресурсы в этих областях не оцениваются и не включаются в число соответствий. Дополнительные сведения см. в разделе [определения назначения — исключенные области](./assignment-structure.md#excluded-scopes).

В дополнение к свойствам назначения политики, это объект [исключения политики](./exemption-structure.md) . Исключения улучшают историю области, предоставляя метод для определения части присваивания, которая не должна оцениваться.

- Исключение (**бесплатное в предварительной версии** ) — иерархия ресурсов или отдельный ресурс должны оцениваться в соответствии с определением, но не будет оцениваться по причине отказа или снижению с помощью другого метода. Ресурсы в этом состоянии отображаются как **Исключенные** в отчетах о соответствии, чтобы их можно было отслеживанию. Объект исключения создается в иерархии ресурсов или отдельном ресурсе как дочерний объект, который определяет область исключения. Иерархия ресурсов или отдельный ресурс можно исключить из нескольких назначений. Для исключения можно настроить срок действия по расписанию с помощью `expiresOn` Свойства. Дополнительные сведения см. в разделе [Определение исключения](./exemption-structure.md).

  > [!NOTE]
  > Из-за влияния предоставления исключения для иерархии ресурсов или отдельного ресурса исключения имеют дополнительные меры безопасности. Помимо выполнения `Microsoft.Authorization/policyExemptions/write` операции с иерархией ресурсов или отдельным ресурсом, создатель исключения должен иметь `exempt/Action` команду в целевом назначении.
