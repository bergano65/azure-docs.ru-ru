# <a name="connect-hybrid-machines-to-azure-from-windows-admin-center"></a>Подключение гибридных компьютеров к Azure из центра администрирования Windows

Вы можете включить серверы с поддержкой ARC в Azure для одного или нескольких компьютеров Windows в вашей среде, выполнив набор действий вручную. Также можно использовать [центр администрирования Windows](/windows-server/manage/windows-admin-center/understand/what-is) для развертывания агента подключенного компьютера и регистрации локальных серверов без выполнения каких-либо действий за пределами этого средства.

## <a name="prerequisites"></a>Предварительные требования

* Серверы с поддержкой ARC. Проверьте [Предварительные требования](agent-overview.md#prerequisites) и убедитесь, что ваша подписка, учетная запись Azure и ресурсы соответствуют требованиям.

* Центр администрирования Windows. Ознакомьтесь с требованиями для [подготовки среды](/windows-server/manage/windows-admin-center/deploy/prepare-environment) к развертыванию и [настройке интеграции с Azure ](/windows-server/manage/windows-admin-center/azure/azure-integration).

* Подписка Azure. Если у вас еще нет подписки Azure, создайте [бесплатную учетную запись](https://azure.microsoft.com/free/?WT.mc_id=A261C142F), прежде чем начать работу.

* Целевые серверы Windows, которыми требуется управлять, должны иметь подключение к Интернету для доступа к Azure.

### <a name="security"></a>Безопасность

Этот метод развертывания требует наличия прав администратора на целевом компьютере Windows или сервере для установки и настройки агента. Также необходимо быть членом роли [**пользователей шлюза**](/windows-server/manage/windows-admin-center/plan/user-access-options#gateway-access-roles) .

## <a name="deploy"></a>Развертывание

Чтобы настроить Windows Server с серверами с поддержкой Arc, выполните следующие действия.

1. Войдите в центр администрирования Windows.

1. В списке подключение на странице **Обзор** в списке подключенных серверов Windows выберите сервер из списка, чтобы подключиться к нему.

1. На панели слева выберите **гибридные службы Azure**.

1. На странице **гибридные службы Azure** выберите **Обнаружение служб Azure**.

1. На странице **Обнаружение служб Azure** в разделе **Использование политик и решений Azure для управления серверами с помощью дуги Azure** щелкните **настроить**.

1. На странице **Сеттингс\азуре Arc for Servers (дуга для серверов** ) при появлении запроса на проверку подлинности в Azure, а затем выберите **начать работу**.

1. На странице **Подключение сервера к Azure** укажите следующие действия.

    1. В раскрывающемся списке **Подписка Azure** выберите подписку Azure.
    1. Для параметра **Группа ресурсов** **выберите Создать, чтобы создать** новую группу ресурсов, или в раскрывающемся списке **Группа ресурсов** выберите существующую группу ресурсов для регистрации и управления компьютером.
    1. В раскрывающемся списке **Регион** выберите регион Azure для хранения метаданных серверов.
    1. Если компьютер или сервер обменивается данными через прокси-сервер для подключения к Интернету, выберите параметр **использовать прокси-сервер**. Укажите IP-адрес или имя прокси-сервера, а также номер порта, который будет использоваться компьютером для взаимодействия с прокси-сервером.

1. Выберите **настроить** , чтобы продолжить настройку Windows Server с серверами с поддержкой ARC в Azure.

Windows Server будет подключаться к Azure, скачать агент подключенного компьютера, установить его и зарегистрировать на серверах с поддержкой Arc Azure. Чтобы отслеживать ход выполнения, выберите в меню пункт **уведомления** .

Чтобы подтвердить установку агента подключенного компьютера, в центре администрирования Windows в области слева выберите [**события**](/windows-server/manage/windows-admin-center/use/manage-servers#events) , чтобы проверить события *Мсиинсталлер* в журнале событий приложений.

## <a name="verify-the-connection-with-azure-arc"></a>Проверка подключения с помощью Azure Arc

После установки агента и настройки его подключения к серверам с поддержкой Azure Arc перейдите на портал Azure, чтобы убедиться, что сервер подключен. Просмотрите свои компьютеры на [портале Azure](https://portal.azure.com).
