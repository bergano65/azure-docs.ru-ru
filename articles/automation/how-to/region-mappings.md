# <a name="supported-regions-for-linked-log-analytics-workspace"></a>Регионы, в которых поддерживается связанная рабочая область Log Analytics

В службе автоматизации Azure можно включить Управление обновлениями, Отслеживание изменений и инвентаризацию, а также функции Запуск и остановка виртуальных машин в нерабочее время для серверов и виртуальных машин. Эти функции имеют зависимость от Log Analytics рабочей области и поэтому должны связывать рабочую область с учетной записью службы автоматизации. Однако для их связывания поддерживаются только определенные регионы. Как правило, сопоставление неприменимо, если вы планируете связать учетную запись службы автоматизации с рабочей областью, в которой эти *функции не включены* .

Эта статья содержит Поддерживаемые сопоставления для успешного включения и использования этих функций в учетной записи службы автоматизации.

Дополнительные сведения см. в статье [log Analytics Рабочая область и учетная запись службы автоматизации](../../azure-monitor/insights/solutions.md#log-analytics-workspace-and-automation-account).

## <a name="supported-mappings"></a>Поддерживаемые сопоставления

> [!NOTE]
> Как показано в следующей таблице, между Log Analytics и службой автоматизации Azure может существовать только одно сопоставление.

Поддерживаемое сопоставление показано в следующей таблице.

|**Регион рабочей области Log Analytics**|**Регион учетной записи службы автоматизации**|
|---|---|
|**США**||
|EastUS <sup>1</sup>|EastUS2|
|WestUS2|WestUS2|
|WestCentralUS|WestCentralUS|
|**Канада**||
|CanadaCentral|CanadaCentral|
|**Азиатско-Тихоокеанский регион**||
|AustraliaSoutheast|AustraliaSoutheast|
|SoutheastAsia|SoutheastAsia|
|CentralIndia|CentralIndia|
|ChinaEast2<sup>2</sup>|ChinaEast2|
|JapanEast|JapanEast|
|**Европа**||
|UKSouth|UKSouth|
|WestEurope|WestEurope|
|**US Gov**||
|USGovVirginia|USGovVirginia|
|Усговаризона<sup>2</sup>|USGov (Аризона)|

<sup>1</sup> Сопоставление EastUS рабочих областей Log Analytics с учетными записями службы автоматизации не является точным сопоставлением региона с регионом, однако оно корректно.

<sup>2</sup> в этом регионе поддерживается только управление обновлениями, а другие функции, такие как отслеживание изменений и Инвентаризация, в настоящее время недоступны.

## <a name="unlink-a-workspace"></a>Отмена связи для рабочей области

Если вам больше не требуется интеграция учетной записи службы автоматизации с Log Analytics, вы можете отменить связь с учетной записью непосредственно на портале Azure. Перед продолжением работы необходимо [удалить](move-account.md#remove-features) функции управления обновлениями, отслеживания и инвентаризации изменений, а также запуска и остановки виртуальных машин в нерабочее время, если эти функции используются. Если их не удалить, завершить операцию отмены связи будет невозможно.

После удаления компонентов можно выполнить действия, описанные ниже, чтобы отменить связь с учетной записью службы автоматизации.

> [!NOTE]
> Некоторые компоненты, в том числе решения для мониторинга Azure SQL более ранних версий, могли создать ресурсы службы автоматизации. Их также потребуется удалить, прежде чем удалить связь с рабочей областью.

1. На портале Azure выберите свою учетную запись службы автоматизации. На странице учетной записи службы автоматизации выберите **Связанная рабочая область** в разделе **Связанные ресурсы**.

2. На странице "Отмена связи с рабочей областью" нажмите кнопку **Отменить связь с рабочей областью**. Появится запрос на продолжение работы.

3. Пока служба автоматизации Azure пытается отменить привязку учетной записи к рабочей области Log Analytics, вы можете отслеживать ход выполнения этой задачи в меню **Уведомления**.

4. Если вы используете решение для управления обновлениями, при необходимости можно удалить следующие элементы, которые больше не нужны.

    * Обновление расписаний. У каждого из них есть имя, соответствующее созданному развертыванию обновления.
    * Группы гибридных рабочих ролей, созданные для компонента. Каждая из групп имеет имя, похожее на `machine1.contoso.com_9ceb8108-26c9-4051-b6b3-227600d715c8`.

5. Если вы использовали функцию запуска и остановки виртуальных машин в нерабочее время, при необходимости можно удалить следующие элементы, которые больше не нужны.

    * Расписание запуска и остановки виртуальной машины.
    * Рабочие роли Runbook запуска и остановки виртуальной машины.
    * Переменные

Кроме того, можно удалить связь рабочей области с учетной записью службы автоматизации в рабочей области.

1. В разделе **Связанные ресурсы** рабочей области выберите **Учетная запись службы автоматизации**.
2. На странице учетной записи службы автоматизации выберите **Отменить связь с учетной записью**.
