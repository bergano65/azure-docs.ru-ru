# <a name="tutorial-create-user-flows-in-azure-active-directory-b2c"></a>Руководство по Создание потоков пользователей в Azure Active Directory B2C

В приложениях могут быть [потоки пользователей](active-directory-b2c-reference-policies.md), которые позволяют пользователям регистрироваться, входить в систему и управлять профилем. В своем клиенте Azure Active Directory (Azure AD) B2C вы можете создать несколько потоков пользователей различных типов и при необходимости использовать их в приложениях. Потоки пользователей могут повторно использоваться в разных приложениях.

В этой статье раскрываются следующие темы:

> [!div class="checklist"]
> * Создание потока пользователя для регистрации и входа в систему
> * Создание потока пользователя изменения профиля
> * Создание потока пользователя сброса пароля

В этом руководстве показано, как создать несколько рекомендуемых потоков пользователей с помощью портала Azure. Сведения о том, как настроить поток проверки учетных данных пароля владельца ресурса (ROPC) в приложении в Azure AD B2C, см. в [этой статье](configure-ropc.md).

Если у вас еще нет подписки Azure, [создайте бесплатную учетную запись Azure](https://azure.microsoft.com/free/?WT.mc_id=A261C142F), прежде чем начинать работу.

## <a name="prerequisites"></a>Технические условия

[Зарегистрируйте приложения](tutorial-register-applications.md), являющиеся частью потоков пользователей, которые вы хотите создать. 

## <a name="create-a-sign-up-and-sign-in-user-flow"></a>Создание потока пользователя для регистрации и входа в систему

Потоки пользователей регистрации и входа регулируют процедуры регистрации и входа в систему в рамках одной конфигурации. Пользователи вашего приложения проходят через соответствующую процедуру в зависимости от контекста.

1. Войдите на [портале Azure](https://portal.azure.com).
2. Убедитесь, что используете каталог, содержащий клиент Azure AD B2C, щелкнув **Фильтр каталога и подписки** в верхнем меню и выбрав каталог, содержащий ваш клиент.

    ![Переключение на каталог подписки](./media/tutorial-create-user-flows/switch-directories.png)

3. Выберите **Все службы** в левом верхнем углу окна портала Azure, а затем найдите и выберите **Azure AD B2C**.
4. В левом меню выберите **Потоки пользователей**, а затем — **Создать поток пользователя**.

    ![Выберите "Новый поток пользователя"](./media/tutorial-create-user-flows/signup-signin-user-flow.png)

5. Выберите поток пользователя **Sign-up and sign-in** (Регистрация и вход) на вкладке "Рекомендуется".

    ![Выбор потока пользователя регистрации и входа](./media/tutorial-create-user-flows/signup-signin-type.png)

6. Введите **имя** потока пользователя. Например, *signupsignin1*.
7. В разделе **Поставщики удостоверений** выберите **Регистрация по электронной почте**.

    ![Установка свойств потока](./media/tutorial-create-user-flows/signup-signin-properties.png)

8. В разделе **Атрибуты пользователя и утверждения** выберите утверждения и атрибуты, которые вы хотите собирать и отправлять от пользователя во время регистрации. Например, выберите **Показать больше**, а затем — **Страна или регион**, **Отображаемое имя** и **Почтовый индекс**. Последовательно выберите **ОК**.

    ![Выбор атрибутов и утверждений](./media/tutorial-create-user-flows/signup-signin-attributes.png)

9. Для добавления потока пользователя щелкните **Создать**. Префикс *B2C_1* автоматически добавляется к имени.

### <a name="test-the-user-flow"></a>Тестирование потока пользователя

1. На странице "Обзор" созданного потока пользователя выберите **Выполнить поток пользователя**.
2. В разделе **Приложение** выберите зарегистрированное ранее веб-приложение с именем *webapp1*. В поле **URL-адрес ответа** должно содержаться значение `https://jwt.ms`.
3. Щелкните **Выполнить поток пользователя**, а затем выберите **Зарегистрироваться сейчас**.

    ![Запуск потока пользователя](./media/tutorial-create-user-flows/signup-signin-run-now.png)

4. Введите допустимый адрес электронной почты, щелкните **Отправить проверочный код**, а затем введите код проверки, который вы получили.
5. Введите новый пароль и подтвердите его.
6. Введите имя, которое будет отображаться, выберите свою страну и регион, укажите почтовый индекс и щелкните **Создать**. Токен должен вернуться по адресу `https://jwt.ms` и отобразиться пользователю.
7. Теперь вы снова можете запустить поток пользователя и на этот раз войти в учетную запись, которую создали. Возвращенный токен содержит выбранные утверждения имени, страны и почтового индекса.

## <a name="create-a-profile-editing-user-flow"></a>Создание потока пользователя изменения профиля

Чтобы добавить для пользователей приложения возможность изменять профиль, используйте поток пользователя изменения профиля.

1. В левом меню выберите **Потоки пользователей**, а затем — **Создать поток пользователя**.
2. Выберите поток пользователя **Profile editing** (Изменение профиля) на вкладке "Рекомендуется".
3. Введите **имя** потока пользователя. Например, *profileediting1*.
4. Щелкните **Поставщики удостоверений** и выберите **Local Account SignIn** (Вход с локальной учетной записью).
5. В разделе **Атрибуты пользователя** выберите атрибуты, которые клиенты будут изменять в своих профилях. Например, выберите **Показать больше**, а затем — **Отображаемое имя** и **Должность**. Последовательно выберите **ОК**.
6. Для добавления потока пользователя щелкните **Создать**. Префикс *B2C_1* автоматически добавляется к имени.

### <a name="test-the-user-flow"></a>Тестирование потока пользователя

1. На странице "Обзор" созданного потока пользователя выберите **Выполнить поток пользователя**.
2. В разделе **Приложение** выберите зарегистрированное ранее веб-приложение с именем *webapp1*. В поле **URL-адрес ответа** должно содержаться значение `https://jwt.ms`.
3. Щелкните **Выполнить поток пользователя** и выполните вход в учетную запись, созданную ранее.
4. Теперь вы как пользователь можете изменять отображаемое имя и должность. Нажмите кнопку **Продолжить**. Токен должен вернуться по адресу `https://jwt.ms` и отобразиться пользователю.

## <a name="create-a-password-reset-user-flow"></a>Создание потока пользователя сброса пароля

Вы можете разрешить пользователям вашего приложения сбрасывать пароли при необходимости. Чтобы включить сброс пароля, используйте поток пользователя сброса пароля.

1. В левом меню выберите **Потоки пользователей**, а затем — **Создать поток пользователя**.
2. Выберите поток пользователя **Password reset** (Сброс пароля) на вкладке "Рекомендуется".
3. Введите **имя** потока пользователя. Например, *passwordreset1*.
4. В разделе **Поставщики удостоверений** включите **Reset password using email address** (Сброс пароля с использованием адреса электронной почты).
5. В разделе "Утверждения приложений" щелкните **Показать больше** и выберите утверждения, которые необходимо вернуть в токенах проверки подлинности, отправляемых приложению. Например, выберите **ИД объекта пользователя**.
6. Последовательно выберите **ОК**.
7. Для добавления потока пользователя щелкните **Создать**. Префикс *B2C_1* автоматически добавляется к имени.

### <a name="test-the-user-flow"></a>Тестирование потока пользователя

1. На странице "Обзор" созданного потока пользователя выберите **Выполнить поток пользователя**.
2. В разделе **Приложение** выберите зарегистрированное ранее веб-приложение с именем *webapp1*. В поле **URL-адрес ответа** должно содержаться значение `https://jwt.ms`.
3. Щелкните **Выполнить поток пользователя** и выполните вход в учетную запись, созданную ранее.
4. Теперь вы как пользователь сможете изменять пароль. Нажмите кнопку **Продолжить**. Токен должен вернуться по адресу `https://jwt.ms` и отобразиться пользователю.
