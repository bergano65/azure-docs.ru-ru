# <a name="performance-tiers-for-managed-disks"></a>Уровни производительности для управляемых дисков

Производительность управляемого диска Azure задается при создании диска в виде его уровня производительности. Уровень производительности определяет операции ввода-вывода и пропускную способность для управляемого диска. При установке подготовленного размера диска автоматически выбирается уровень производительности. Уровень производительности можно изменить при развертывании или впоследствии без изменения размера диска.

Изменение уровня производительности позволяет подготавливать и выполнять более высокие требования без использования возможности ускорения работы с диском. Изменение уровня производительности может быть более экономичным, а не полагаться на разбивку, в зависимости от того, как долго требуется дополнительная производительность. Это идеальный вариант для событий, которые временно потребовали более высокого уровня производительности, например праздничных покупок, тестирования производительности или запуска среды обучения. Для решения этих событий можно использовать более высокий уровень производительности в течение того времени, пока он необходим. После этого можно вернуться к исходному уровню, если больше не требуется дополнительная производительность.

## <a name="how-it-works"></a>Принцип работы

При первом развертывании или подготовке диска базовый уровень производительности для этого диска задается в зависимости от размера подготовленного диска. Можно использовать уровень производительности выше, чем исходный базовый план, для удовлетворения более высоких требований. Если этот уровень производительности больше не нужен, можно вернуться к исходному базовому уровню производительности.

Изменения в оплате по мере изменения уровня производительности. Например, если вы подготавливаете диск P10 (128 гиб), базовый уровень производительности задается как P10 (500 операций ввода-вывода в секунду и 100 Мбит/с). Плата взимается по тарифу P10. Уровень можно обновить в соответствии с производительностью P50 (7 500 операций ввода-вывода в секунду и 250 Мбит/с) без увеличения размера диска. Во время обновления будет взиматься плата по ставке P50. Если больше не требуется высокая производительность, можно вернуться к уровню P10. На диск будет взиматься плата по тарифу P10.

| Размер диска | Базовый уровень производительности | Можно обновить до |
|----------------|-----|-------------------------------------|
| 4 гиб | P1 | P2, P3, P4, P6, P10, P15, P20, P30, P40, P50 |
| 8 гиб | P2 | P3, P4, P6, P10, P15, P20, P30, P40, P50 |
| 16 гиб | P3 | P4, P6, P10, P15, P20, P30, P40, P50 | 
| 32 ГиБ | P4 | P6, P10, P15, P20, P30, P40, P50 |
| 64 ГиБ | P6 | P10, P15, P20, P30, P40, P50 |
| 128 ГиБ | P10 | P15, P20, P30, P40, P50 |
| 256 Гиб | P15 | P20, P30, P40, P50 |
| 512 ГиБ | P20 | P30, P40, P50 |
| 1 ТиБ | P30 | P40, P50 |
| 2 ТиБ | P40 | P50 |
| 4 ТиБ | P50 | None |
| 8 ТиБ | P60 |  P70, P80 |
| 16 ТиБ | P70 | P80 |
| 32 ТиБ | P80 | None |

Сведения о выставлении счетов см. в разделе [цены на управляемые диски](https://azure.microsoft.com/pricing/details/managed-disks/).
