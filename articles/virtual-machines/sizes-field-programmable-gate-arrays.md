# <a name="fpga-optimized-virtual-machine-sizes"></a>Размеры виртуальных машин, оптимизированных для FPGA

Размеры ВИРТУАЛЬНЫХ машин, оптимизированных для FPGA, — это специализированные виртуальные машины, доступные в одном или нескольких FPGA. Эти размеры предназначены для рабочих нагрузок с большим объемом вычислений. В этой статье содержатся сведения о количестве и типе FPGA, виртуальных ЦП, дисков данных и сетевых адаптеров. Кроме того, для каждого размера виртуальной машины этой группы учитывается пропускная способность хранилища и сети.

- Размеры [серии NP](np-series.md) оптимизированы для рабочих нагрузок, включая определение машинного обучения, кодирование видео и поиск базы данных & Analytics. Серия NP-Series работает на базе Ксилинкс U250 Accelerator.


## <a name="deployment-considerations"></a>Рекомендации по развертыванию

- Сведения о доступности виртуальных машин серии N по регионам см. на [этой странице](https://azure.microsoft.com/regions/services/).

- Виртуальные машины серии N поддерживают только модель развертывания с помощью Resource Manager.

- Если вы хотите развернуть большое количество виртуальных машин серии N, мы рекомендуем подписку с оплатой по мере использования или другие варианты покупки. Если вы используете [бесплатную учетную запись Azure](https://azure.microsoft.com/free/), вам доступно ограниченное количество вычислительных ядер Azure.

- Вам может потребоваться увеличить квоту ядер (для каждого региона) в подписке Azure и увеличить отдельную квоту для ядер NP. Чтобы увеличить квоту, [отправьте запрос в службу поддержки](../azure-portal/supportability/how-to-create-azure-support-request.md). Это бесплатная услуга. Ограничения по умолчанию отличаются в зависимости от категории подписки.
