# <a name="use-classroom-labs-for-trainings"></a>Использование лабораторий аудитории для обучения
Можно настроить лабораторию для обучения. Класс-Labs из службы лабораторий Azure позволяют создавать лаборатории для обучения, где каждый обучаемый использует идентичных изолированных сред для обучения. Путем применения политик можно настроить лабораторию таким образом, чтобы каждый обучаемый мог использовать среды обучения только при необходимости и при достаточном количестве ресурсов, например виртуальных машин, требуемых для обучения. 

![Лаборатории аудитории](../media/classroom-labs-scenarios/classroom.png)

Аудитории Labs соответствует следующим требованиям, которые необходимы для проведения обучения в любой виртуальной среде: 

- Обучаемые могут быстро подготовить среды для обучения.
- Все машины для обучения должны быть идентичными.
- Обучаемые не видят виртуальные машины, созданные другими обучаемыми.
- Обучаемые не могут получить больше виртуальных машин, чем требуется для обучения, и завершают работу виртуальных машин, которые не используются, что гарантирует контроль расходования ресурсов.
- Каждому обучаемому можно предоставить доступ к учебной лаборатории.
- При необходимости учебные лаборатории можно использовать повторно.

В этой статье вы узнаете о различных функциях службы лабораторий Azure, которые могут использоваться в соответствии с ранее описанных требованиям, а также подробные инструкции, которые нужно выполнить для настройки лаборатории для обучения.  

## <a name="create-the-lab-account-as-a-lab-account-administrator"></a>Создание учетной записи лаборатории администратор учетной записи лаборатории
С помощью службы лабораторий Azure первым делом для создания учетной записи лаборатории на портале Azure. После того как учетная запись администратора лаборатории создаст учетную запись лаборатории, администратор добавляет пользователей, которые хотят создавать лаборатории, чтобы **создатель лаборатории** роли. Инструкторы создания лабораторий с виртуальными машинами для учащихся для выполнения упражнений для курса, который они цель. Дополнительные сведения см. в разделе [Создание и управление учетной записью лаборатории](how-to-manage-lab-accounts.md).

## <a name="create-and-manage-classroom-labs"></a>Создание и управление лабораториями аудитории
Преподаватель, являющийся членом роли создатель лаборатории в учетной записи лаборатории, можно создать один или несколько лабораторий в учетной записи лаборатории. Создание и настройка шаблона виртуальной Машины с все необходимое программное обеспечение для выполнения упражнений в свой курс. Выбрать готовый образ из доступных образов для создания лаборатории аудитории и настроить его, установив программное обеспечение, необходимое для лаборатории. Дополнительные сведения см. в разделе [Создание и управление лабораториями аудитории](how-to-manage-classroom-labs.md).

## <a name="configure-usage-settings-and-policies"></a>Настройка параметров и политик использования
Создатель лаборатории можно добавить или удалить пользователей в лабораторию, получите ссылку регистрации для отправки пользователям лаборатории, настроить политики, такие как квоты отдельных параметров каждого пользователя, обновить количество виртуальных машин в лаборатории и многое другое. Дополнительные сведения см. в разделе [настройки политик и параметров использования](how-to-configure-student-usage.md).

## <a name="create-and-manage-schedules"></a>Создание расписаний и управление ими
Расписания позволяют настроить лабораторию аудитории, например, виртуальные машины в лаборатории автоматически начинают и завершают работу в указанное время. Вы можете определить разовое расписание или повторяющееся расписание. Дополнительные сведения см. в разделе [Создание расписаний для аудитории лабораторных работ и управление ими](how-to-create-schedules.md).

## <a name="set-up-and-publish-a-template-vm"></a>Настройка и публикация шаблона виртуальной Машины
На основе шаблона, в роли которого выступает базовый образ виртуальной машины, в лаборатории создаются все виртуальные машины пользователей. Настройка шаблона виртуальной Машины, таким образом, чтобы он использует именно то, что вы хотите предоставить участникам обучения. Вы можете указать имя и описание шаблона, которое будет отображаться для пользователей лаборатории. После этого опубликуйте шаблон, чтобы сделать экземпляры шаблона виртуальной машины доступными для пользователей лаборатории. Когда вы опубликуете шаблон, Службы лабораторий Azure создадут виртуальные машины в лаборатории с помощью этого шаблона. Число созданных при этом процессе виртуальных машин соответствует максимальному числу разрешенных в лаборатории пользователей, которое можно настроить в политике использования лаборатории. Все виртуальные машины имеют ту же конфигурацию, что и шаблон. Дополнительные сведения см. в разделе [задать вверх и публикация шаблонов виртуальных машин](how-to-create-manage-template.md). 

## <a name="use-vms-in-the-classroom-lab"></a>Использование виртуальных машин в лаборатории аудитории
Учащийся или обучение участников регистрирует в лаборатории и подключается к виртуальной Машине для выполнения упражнений для курса. Дополнительные сведения см. в разделе [как получить доступ к лаборатории аудитории](how-to-use-classroom-lab.md).
