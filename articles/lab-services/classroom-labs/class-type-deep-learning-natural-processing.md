# <a name="set-up-a-lab-focused-on-deep-learning-in-natural-language-processing-using-azure-lab-services"></a>Настройка лаборатории для глубокого обучения в обработке на естественном языке с помощью служб лабораторий Azure
В этой статье показано, как настроить лабораторную работу для глубокого обучения в обработке на естественном языке (NLP) с помощью служб лаборатории Azure. Обработка на естественном языке (NLP) — это разновидность искусственного интеллекта (ии), которая позволяет компьютерам с переводом, распознаванием речи и другими возможностями понимания языка.  

Студенты, принимающие класс NLP, получают виртуальную машину Linux, чтобы узнать, как применять алгоритмы нейронной сети для разработки моделей глубокого обучения, используемых для анализа письменного пользовательского языка. 

## <a name="lab-configuration"></a>Настройка лаборатории
Чтобы настроить эту лабораторию, вам потребуется подписка Azure, чтобы приступить к работе. Если у вас еще нет подписки Azure, [создайте бесплатную учетную запись Azure](https://azure.microsoft.com/free/), прежде чем начинать работу. Создав подписку Azure, вы можете создать новую учетную запись лаборатории в службах лаборатории Azure или использовать существующую учетную запись лаборатории. Чтобы создать учетную запись лаборатории, ознакомьтесь со следующим руководством: [Руководство по настройке учетной записи лаборатории](tutorial-setup-lab-account.md).
 
После создания учетной записи лаборатории включите следующие параметры в учетной записи лаборатории. 

| Параметр учетной записи лаборатории | Инструкции |
| ----------- | ------------ |  
| Образы Marketplace | Включите образ виртуальной машины для обработки и анализа данных для Linux (Ubuntu), который будет использоваться в вашей учетной записи лаборатории.  Инструкции см. в следующих статьях: [Укажите образы Marketplace, доступные авторам лабораторий](tutorial-setup-lab-account.md#specify-marketplace-images-available-to-lab-creators). | 

Следуйте указаниям [этого учебника](tutorial-setup-classroom-lab.md) , чтобы создать лабораторию и применить следующие параметры.

| Параметры лаборатории | Значение или инструкции | 
| ------------ | ------------------ |
| Размер виртуальной машины (ВМ) | Малый GPU (вычисление). Этот размер лучше всего подходит для ресурсоемких вычислительных и ресурсоемких приложений, таких как искусственный интеллект и глубокое обучение. |
| Образ виртуальной машины | [Виртуальная машина для обработки и анализа данных для Linux (Ubuntu)](https://azuremarketplace.microsoft.com/marketplace/apps/microsoft-dsvm.linux-data-science-vm-ubuntu). Этот образ содержит платформы и средства глубокого обучения для машинного обучения и обработки и анализа данных. Полный список установленных средств для этого образа см. в следующей статье: [Что входит в DSVM?](../../machine-learning/data-science-virtual-machine/overview.md#whats-included-on-the-dsvm) |
| Включить подключение к удаленному рабочему столу | Параметр. <p>Включение этого параметра позволит преподавателям и учащимся подключаться к своим виртуальным машинам (ВМ) с помощью удаленный рабочий стол (RDP).</p><p>**Важно!** Протокол RDP уже установлен и настроен на виртуальной машине для обработки и анализа данных для образа Linux. В результате преподаватели и студенты могут подключаться к виртуальным машинам через протокол RDP без каких-либо дополнительных действий. Кроме того, если необходимо подключиться к графическому рабочему столу, в этом образе уже установлен [сервер X2Go](https://wiki.x2go.org/doku.php/doc:newtox2go) на виртуальной машине. Студенты должны установить клиент X2Go на своих локальных компьютерах и должны использовать клиент для подключения. Дополнительные сведения см. в следующих руководствах: <ul><li>[Как получить доступ к виртуальной машине для обработки и анализа данных для Linux](../../machine-learning/data-science-virtual-machine/dsvm-ubuntu-intro.md#how-to-access-the-ubuntu-data-science-virtual-machine)</li><li>[Подключение к виртуальной машине шаблона для установки пакетов RDP и GUI](how-to-enable-remote-desktop-linux.md#teachers-connecting-to-the-template-vm-using-rdp)</li></ul></p>   |

Образ виртуальной машины для обработки и анализа данных для Linux содержит необходимые платформы и средства глубокого обучения, необходимые для этого типа класса. В результате после создания шаблона компьютера вам больше не нужно настраивать его. Его можно опубликовать для использования учащимся. Нажмите кнопку **опубликовать** на странице шаблона, чтобы опубликовать шаблон в лаборатории.  

## <a name="cost"></a>Стоимость
Если вы хотите оценить стоимость этой лабораторной работы, можно использовать следующий пример: 

Для класса из 25 учащихся с 20 часов запланированного времени класса и 10 часов квоты для домашних заданий или назначений Цена за лабораторию будет составлять 25 учащихся * (20 + 10) часов * 139. единицы лаборатории * 0,01 долларов США в час = 1042,5 долл. США

Дополнительные сведения о ценах см. на странице [цен на службы лаборатории Azure](https://azure.microsoft.com/pricing/details/lab-services/).
