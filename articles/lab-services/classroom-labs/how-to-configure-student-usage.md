# <a name="add-and-manage-lab-users"></a>Добавление пользователей лаборатории и управление ими
В этой статье описывается, как добавить и зарегистрировать пользователей в лаборатории, контролировать количество часов для использования виртуальной машины и многое другое. 


## <a name="add-users-to-the-lab"></a>Добавление пользователей в лабораторию

1. В меню слева выберите **Пользователи**. По умолчанию включен параметр **Ограничить доступ**. Когда этот параметр включен, даже если пользователь имеет ссылку на регистрацию, он не сможет зарегистрироваться в лаборатории, пока не будет включен в список пользователей. С помощью отправляемой ссылки для регистрации могут зарегистрироваться только пользователи из списка. Ниже описано, как добавить пользователей в список. Параметр **Ограничить доступ** можно отключить, что позволит пользователям регистрироваться в лаборатории, используя ссылку для регистрации. 
2. На панели инструментов выберите **Добавить пользователей** , а затем щелкните **Добавить по адресам электронной почты**. 

    ![Кнопка "Добавить пользователей"](../media/how-to-configure-student-usage/add-users-button.png)
1. На странице **Добавление пользователей** введите адреса электронной почты пользователей в отдельных строках или одной строке, разделяя их точкой с запятой. 

    ![Добавление адресов электронной почты пользователей](../media/how-to-configure-student-usage/add-users-email-addresses.png)
4. Щелкните **Сохранить**. В списке отобразятся адреса электронной почты и состояние пользователей (зарегистрирован или нет). 

    ![Список пользователей](../media/how-to-configure-student-usage/users-list-new.png)

    > [!NOTE]
    > Имена пользователей в списке отображаются после их регистрации в лаборатории. Имя, показанное в списке, создается с использованием первого и последнего имени пользователя в Azure Active Directory. 

### <a name="add-users-by-uploading-a-csv-file"></a>Добавление пользователей путем отправки CSV-файла
Можно также добавить пользователей, передав CSV-файл с адресами электронной почты пользователей.

1. Создайте CSV-файл с помощью адреса электронной почты пользователей в одном столбце.

    ![CSV-файл с пользователями](../media/how-to-configure-student-usage/csv-file-with-users.png)
2. На странице **Пользователи** лаборатории на панели инструментов выберите **Добавить пользователей** , а затем щелкните **отправить CSV**-файл.

    ![Кнопка "Отправить CSV"](../media/how-to-configure-student-usage/upload-csv-button.png)
3. Выберите CSV-файл с адресами электронной почты пользователя. Щелкнув **Открыть** после того, как вы выбрали CSV-файл, вы увидите следующее окно **Добавление пользователей**. Список адресов электронной почты будет заполнен из CSV-файла. 

    ![Окно "Добавление пользователей", заполненное адресами электронной почты из CSV-файла](../media/how-to-configure-student-usage/add-users-window.png)
4. В окне **Добавление пользователей** выберите **Сохранить**. 
5. Убедитесь, что пользователи отображаются в списке пользователей. 

    ![Список добавленных пользователей](../media/how-to-configure-student-usage/list-of-added-users.png)

## <a name="send-invitations-to-users"></a>Отправка приглашений пользователям
Чтобы отправить ссылку на регистрацию учащимся, используйте один из следующих методов. Первый способ показывает, как отправить сообщения учащимся с помощью ссылки регистрации и необязательного сообщения. Второй способ показывает, как получить ссылку для регистрации, которую можно предоставить другим пользователям любым нужным образом. 

Если **Ограничение доступа** включено для лаборатории, только пользователи из списка пользователей могут использовать ссылку регистрации для регистрации в лаборатории. Этот параметр включен по умолчанию. 

### <a name="invite-all-users"></a>Пригласить всех пользователей
1. Перейдите в представление **Пользователи** , если вы еще не сделали это на этой странице, и выберите **пригласить все** на панели инструментов. 

    ![Выбор учащихся](../media/tutorial-setup-classroom-lab/invite-all-button.png)

1. На странице **Отправка приглашения по электронной почте** введите необязательное сообщение и нажмите кнопку **Отправить**. Письмо автоматически включает ссылку для регистрации. Чтобы получить ссылку для регистрации, выберите **... (многоточие)** на панели инструментов и в **ссылке Регистрация**. 

    ![Отправка ссылки для регистрации по электронной почте](../media/tutorial-setup-classroom-lab/send-email.png)
4. Состояние **приглашения** отображается в списке **пользователей** . Состояние должно измениться на **Отправка** , а затем на отправку **на \<>** . 

    Дополнительные сведения о добавлении учащихся в класс и управлении их использованием в лаборатории см. в разделе [Настройка использования учащихся](how-to-configure-student-usage.md).

### <a name="invite-selected-users"></a>Пригласить выбранных пользователей

1. Выберите пользователя или нескольких пользователей в списке. 
2. Затем выберите значок **конверта** , отображаемый в выбранной строке (или) выберите **пригласить** на панели инструментов. 

    ![Пригласить выбранных пользователей](../media/how-to-configure-student-usage/invite-selected-users.png)
3. В окне **Отправка приглашения по электронной почте** введите необязательное **сообщение**и нажмите кнопку **Отправить**. 

    ![Отправить сообщение электронной почты выбранным пользователям](../media/how-to-configure-student-usage/send-invitation-to-selected-users.png)

    Состояние этой операции отображается в столбце **приглашения** списка **пользователей** . Сообщение электронной почты с приглашением содержит регистрационную ссылку, которую пользователи могут использовать для регистрации в лаборатории.

1. Перейдите в представление **Пользователи** , если вы еще не сделали это на странице. 

## <a name="get-registration-link"></a>Получить ссылку для регистрации
Вы также можете получить ссылку на регистрацию на портале и отправить ее с помощью собственного клиентского приложения электронной почты. 

1. Перейдите к представлению **Пользователи**, выбрав **Пользователи** в меню слева. 
2. Выберите **... (многоточие)** на панели инструментов, а затем щелкните **ссылку Регистрация**.

    ![Ссылка для регистрации учащихся](../media/how-to-configure-student-usage/registration-link-button.png)
1. В диалоговом окне **Регистрация пользователя** нажмите кнопку **Копировать** . Ссылка скопируется в буфер обмена. Вставьте ее в редактор электронной почты и отправьте электронное сообщение учащемуся. 

    ![Ссылка для регистрации учащихся](../media/how-to-configure-student-usage/registration-link.png)
2. В диалоговом окне **Регистрация пользователя** выберите **Готово**. 
4. Отправьте **регистрационную ссылку** на учащуюся, чтобы студент мог зарегистрироваться для класса.

Выберите в меню слева раздел **Пользователи**, чтобы просмотреть список зарегистрированных в лаборатории пользователей.

## <a name="set-quotas-for-users"></a>Настройка квот для пользователей
Можно установить квоты на пользователя, выполнив следующие действия: 

1. Щелкните **Пользователи** в меню слева, если страница еще не активна. 
2. Выбор **квоты на пользователя: \<число > часов** на панели инструментов. 
3. На странице **Квота на пользователя** укажите количество часов, которое необходимо назначить каждому пользователю (студенту) за пределами запланированного времени класса, а затем нажмите кнопку **сохранить**.

    ![Квота для пользователя](../media/how-to-configure-student-usage/quota-per-user.png)    
5. Измененные значения отобразятся на панели инструментов: **Квота на пользователя: \<количество часов >** . 

    ![Квота на пользователя — после](../media/how-to-configure-student-usage/quot-per-user-after.png)

    > [!IMPORTANT]
    > Запланированное время запуска виртуальных машин не учитывается в [квоте, выделенной пользователю](how-to-create-schedules.md). Квота рассчитывается на время, которое выходит за пределы расписания и которое учащийся тратит на виртуальные машины. 

## <a name="set-additional-quota-for-a-specific-user"></a>Задание дополнительной квоты для конкретного пользователя
Можно указать дополнительную квоту для пользователя. Эта квота является дополнением к общей квоте, установленной для всех пользователей в предыдущем разделе. Например, если вы (как преподавателем) настроили квоту для всех пользователей на 10 часов и задали дополнительную квоту в 5 часов для конкретных пользователей, то они будут иметь значение 15 (10 + 5) часов квоты. Если вы измените общую квоту позднее на, скажем, на 15, то пользователь получит 20 (15 + 5) часов квоты. Помните, что Общая квота находится за пределами запланированного времени. Время, которое студент тратит на виртуальную машину лаборатории в течение расписания, не учитывает эту квоту. 

Для этого выполните следующие действия.

1. Выберите пользователя (учащийся) в списке пользователей на странице **Пользователи** .
2. Затем выберите **Настройка квоты** на панели инструментов. 

    ![Кнопка "настроить квоту"](../media/how-to-configure-student-usage/adjust-quota-button.png)
3. Введите число **дополнительных часов** для выбранного пользователя или пользователей, а затем нажмите кнопку **Применить**. 

    ![Дополнительная квота для пользователя](../media/how-to-configure-student-usage/additional-quota.png)
4. Вы увидите обновленное использование для пользователя в столбце **Использование** . 

    ![Новое использование для пользователя](../media/how-to-configure-student-usage/new-usage-hours.png)

## <a name="student-accounts"></a>Учетные записи учащихся
Чтобы добавить учащихся в лабораторную среду, вы используете свои учетные записи электронной почты. Можно использовать следующие типы учетных записей электронной почты:

- Учетная запись учащегося, которая предоставляется в Office 365 Azure Active Directory (AAD) университета. 
- Учетная запись электронной почты Майкрософт, например `@outlook.com`, `@hotmail.com`, `@msn.com`или `@live.com`.
- Учетная запись электронной почты, не относящаяся к Майкрософт, например Yahoo или Google. Однако эти типы учетных записей должны быть связаны с учетная запись Майкрософт.
- Учетная запись GitHub. Эта учетная запись должна быть связана с учетная запись Майкрософт.

### <a name="using-a-non-microsoft-email-account"></a>Использование учетной записи электронной почты, отличной от Майкрософт
Студенты могут использовать сторонние учетные записи электронной почты для регистрации и входа в лабораторную среду.  Однако регистрация требует, чтобы студенты сначала создавали учетная запись Майкрософт, связанные с адресом электронной почты не от Майкрософт.

У многих учащихся уже может быть учетная запись Майкрософт, связанных с адресами электронной почты, не являющимися Майкрософт. Например, студенты уже имеют учетная запись Майкрософт, если они использовали свой адрес электронной почты с другими продуктами или службами корпорации Майкрософт, такими как Office, Skype, OneDrive или Windows.  

Когда учащийся щелкает URL-адрес регистрации для входа в аудиторию, ему будет предложено ввести адрес электронной почты и пароль. Если учащийся пытается выполнить вход с помощью неучетная запись Майкрософт, который не связан с учетная запись Майкрософт, студент получит следующее сообщение об ошибке: 

![Сообщение об ошибке](../media/how-to-configure-student-usage/cant-find-account.png)

Чтобы зарегистрироваться для учетная запись Майкрософт, студенты должны переключиться на [http://signup.live.com](http://signup.live.com).  

> [!IMPORTANT]
> Когда студенты входят в лабораторную среду, им не предоставляется возможность создания учетная запись Майкрософт. Поэтому рекомендуется включить эту ссылку для регистрации в электронное письмо с регистрацией лаборатории, которое вы отправляете учащимся, использующим учетные записи сторонних производителей.

### <a name="using-a-github-account"></a>Использование учетной записи GitHub
Студенты также могут использовать существующую учетную запись GitHub для регистрации и входа в лабораторную среду. Если у учащегося уже есть учетная запись Майкрософт, связанный с учетной записью GitHub, они смогут войти в систему и ввести пароль, как показано в предыдущем разделе. Если учетная запись GitHub еще не связана с учетная запись Майкрософт, они должны выбрать **Параметры входа**:

![Ссылка на параметры входа](../media/how-to-configure-student-usage/signin-options.png)

На странице **Параметры входа** выберите **Вход с помощью GitHub**.

![Вход с помощью ссылки GitHub](../media/how-to-configure-student-usage/signin-github.png)

Наконец, им предлагается создать учетная запись Майкрософт, связанный с учетной записью GitHub. Это происходит автоматически, когда учащийся выбирает **Next (далее**).  Затем учащийся немедленно вошел в среду и подключился к лабораторной работе.
