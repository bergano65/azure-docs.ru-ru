# <a name="back-up-azure-file-shares"></a>Резервное копирование файловых ресурсов Azure

В этой статье объясняется, как выполнить резервное копирование [файловых ресурсов Azure](../storage/files/storage-files-introduction.md) из портал Azure.

В этой статье вы узнаете, как выполнять следующие задачи.

* Создайте хранилище служб восстановления,
* Настройка резервного копирования из хранилища служб восстановления
* Настройка резервного копирования из панели общих папок
* Выполнение задания резервного копирования по запросу для создания точки восстановления.

## <a name="prerequisites"></a>Предварительные требования

* [Узнайте](azure-file-share-backup-overview.md) о решении резервного копирования на основе моментального снимка файлового ресурса Azure.
* Убедитесь, что общая папка имеется в одном из [поддерживаемых типов учетных записей хранения](azure-file-share-support-matrix.md).
* Выявление или создание [хранилища служб восстановления](#create-a-recovery-services-vault) в том же регионе, что и учетная запись хранения, в которой размещается общая папка.

[!INCLUDE [How to create a Recovery Services vault](../../includes/backup-create-rs-vault.md)]

## <a name="configure-backup-from-the-recovery-services-vault"></a>Настройка резервного копирования из хранилища служб восстановления

Ниже описаны действия по настройке резервного копирования для нескольких файловых ресурсов из области хранилища службы восстановления.

1. В [портал Azure](https://portal.azure.com/)откройте хранилище служб восстановления, которое будет использоваться для настройки резервного копирования для общей папки.

1. В области **хранилище служб восстановления** в меню вверху выберите пункт **+ резервное копирование** .

   ![Хранилище Служб восстановления](./media/backup-afs/recovery-services-vault.png)

    1. В области **цель резервного копирования** укажите, **где выполняется рабочая нагрузка?** в **Azure** , выбрав параметр **Azure** в раскрывающемся списке.

          ![Выберите Azure в качестве рабочей нагрузки](./media/backup-afs/backup-goal.png)

    2. В поле **что вы хотите создать резервную копию?** выберите **файловый ресурс Azure** из раскрывающегося списка.

          ![Выберите общую папку Azure](./media/backup-afs/select-azure-file-share.png)

    3. Выберите **резервное копирование** , чтобы зарегистрировать расширение общей папки Azure в хранилище.

          ![Выберите резервное копирование, чтобы связать файловый ресурс Azure с хранилищем](./media/backup-afs/register-extension.png)

1. После выбора **резервного копирования**откроется панель **резервного копирования** . Чтобы выбрать учетную запись хранения, в которой размещена общая папка, которую необходимо защитить, выберите **текст ссылки в текстовом поле** **учетная запись хранения** .

   ![Выбор ссылки SELECT](./media/backup-afs/choose-select-link.png)

1. Справа откроется **область Выбор учетной записи хранения** с перечнем обнаруженных поддерживаемых учетных записей хранения. Они либо связаны с этим хранилищем, либо находятся в том же регионе, что и хранилище, но еще не связаны ни с одним хранилищем служб восстановления.

1. В списке обнаруженных учетных записей хранения выберите учетную запись и нажмите кнопку **ОК**.

   ![Выбор из обнаруженных учетных записей хранения](./media/backup-afs/select-discovered-storage-account.png)

1. Следующим шагом является выбор общих файловых ресурсов, для которых требуется создать резервную копию. Нажмите кнопку **Добавить** в разделе **Общие папки to Backup (создать резервную копию** ).

   ![Выберите Общие папки для резервного копирования](./media/backup-afs/select-file-shares-to-back-up.png)

1. Справа откроется панель контекст **Выбор общих файловых ресурсов** . Azure ищет учетную запись хранения для файловых ресурсов, для которых можно выполнить резервное копирование. Если вы недавно добавили файловые ресурсы и не видите их в списке, подождите некоторое время, пока файловые ресурсы будут отображаться.

1. В списке **выберите Общие файловые ресурсы** выберите одну или несколько общих папок, для которых требуется создать резервную копию. Выберите **ОК**.

   ![Выберите Общие папки](./media/backup-afs/select-file-shares.png)

1. Чтобы выбрать политику резервного копирования для общей папки, вам доступны три варианта:

   * Выберите политику по умолчанию.<br>
   Этот параметр позволяет включить ежедневное резервное копирование, которое будет храниться в течение 30 дней. Если у вас нет существующей политики архивации в хранилище, откроется панель резервного копирования с параметрами политики по умолчанию. Если вы хотите выбрать параметры по умолчанию, можно непосредственно выбрать **включить резервное копирование**.

   * Создание новой политики <br>

      1. Чтобы создать новую политику архивации для общей папки, выберите ее в раскрывающемся списке в разделе **политика резервного копирования** .<br>

         ![Создать новую политику](./media/backup-afs/create-new-policy.png)

      1. Справа откроется область контекста **политики резервного копирования** . Укажите имя политики в текстовом поле и выберите срок хранения в соответствии с вашими требованиями. По умолчанию включен только параметр ежедневного хранения. Если требуется еженедельное, ежемесячное или ежегодное хранение, установите соответствующий флажок и укажите требуемое значение срока хранения.

      1. После указания значений срока хранения и допустимого имени политики нажмите кнопку **ОК**.<br>

         ![Укажите имя политики и значения срока хранения](./media/backup-afs/policy-name.png)

   * Выберите одну из существующих политик архивации <br>

      Чтобы выбрать одну из существующих политик резервного копирования для настройки защиты, выберите нужную политику в раскрывающемся списке **Политика архивации** .<br>

      ![Выбор существующей политики](./media/backup-afs/choose-existing-policy.png)

1. Выберите **включить резервное копирование** , чтобы начать защиту общей папки.

   ![Выберите включить резервное копирование](./media/backup-afs/enable-backup.png)

После настройки политики архивации моментальный снимок общих файловых ресурсов создается в запланированное время. Точка восстановления также сохраняется в течение выбранного периода.

>[!NOTE]
>Azure Backup теперь поддерживает политики с ежедневными, еженедельными, ежемесячными или ежегодными сроками хранения для резервного копирования файловых ресурсов Azure.

## <a name="configure-backup-from-the-file-share-pane"></a>Настройка резервного копирования из панели общих папок

Ниже описано, как настроить резервное копирование для отдельных файловых ресурсов из области соответствующего файлового ресурса.

1. В [портал Azure](https://portal.azure.com/)Откройте учетную запись хранения, в которой размещена общая папка, для которой нужно создать резервную копию.

1. В учетной записи хранения выберите плитку с меткой **Общие папки**. Вы также можете переходить к **общим файловым ресурсам** , используя оглавление для учетной записи хранения.

   ![Учетная запись хранения](./media/backup-afs/storage-account.png)

1. В списке общих файловых ресурсов должны отобразиться все общие папки, имеющиеся в учетной записи хранения. Выберите общую папку, для которой необходимо создать резервную копию.

   ![Список файловых ресурсов](./media/backup-afs/file-shares-list.png)

1. Выберите **резервное копирование** в разделе **операции** на панели файлового ресурса. Панель **Настройка резервного копирования** будет загружена справа.

   ![Панель "Настройка резервного копирования"](./media/backup-afs/configure-backup.png)

1. Для выбора хранилища служб восстановления выполните одно из следующих действий.

    * Если у вас уже есть хранилище, выберите переключатель **выбрать существующее** хранилище службы восстановления и выберите одно из существующих хранилищ из раскрывающегося меню **имя хранилища** .

       ![Выбрать существующее хранилище](./media/backup-afs/select-existing-vault.png)

    * Если у вас нет хранилища, выберите переключатель **создать новое** хранилище службы восстановления. Укажите имя для хранилища. Он создается в том же регионе, что и файловый ресурс. По умолчанию хранилище создается в той же группе ресурсов, что и общая папка. Если вы хотите выбрать другую группу ресурсов, щелкните **создать** ссылку под раскрывающимся списком **тип ресурса** и укажите имя группы ресурсов. Чтобы продолжить, нажмите кнопку **ОК** .

       ![Создать новое хранилище](./media/backup-afs/create-new-vault.png)

      >[!IMPORTANT]
      >Если учетная запись хранения зарегистрирована в хранилище или имеется несколько защищенных общих папок в учетной записи хранения, где размещена общая папка, которую вы пытаетесь защитить, имя хранилища служб восстановления будет предварительно заполнено, и вы не сможете изменить его. [Дополнительные сведения](backup-azure-files-faq.md#why-cant-i-change-the-vault-to-configure-backup-for-the-file-share)см. здесь.

1. Для выбора **политики архивации** выполните одно из следующих действий.

    * Оставьте политику по умолчанию. Это расписание ежедневного резервного копирования будет храниться в течение 30 дней.

    * Выберите существующую политику архивации, если она есть, из раскрывающегося меню **политика резервного копирования** .

       ![Выбор политики резервного копирования](./media/backup-afs/choose-backup-policy.png)

    * Создайте новую политику с ежедневными, еженедельными, ежемесячными или ежегодными сроками хранения в соответствии с вашими требованиями.  

         1. Выберите текст ссылки **создать новую политику** .

         2. Справа откроется область контекста **политики резервного копирования** . Укажите имя политики в текстовом поле и выберите срок хранения в соответствии с вашими требованиями. По умолчанию включен только параметр ежедневного хранения. Если требуется еженедельное, ежемесячное или ежегодное хранение, установите соответствующий флажок и укажите требуемое значение срока хранения.

         3. После указания значений срока хранения и допустимого имени политики нажмите кнопку **ОК**.

            ![Создать новую политику архивации](./media/backup-afs/create-new-backup-policy.png)

1. Выберите **включить резервное копирование** , чтобы начать защиту общей папки.

   ![Выберите включить резервное копирование](./media/backup-afs/select-enable-backup.png)

1. Ход настройки можно отслеживать в уведомлениях на портале или с помощью наблюдения за заданиями резервного копирования в хранилище, которое используется для защиты общей папки.

   ![Уведомления на портале](./media/backup-afs/portal-notifications.png)

1. После завершения операции Настройка резервного копирования выберите **резервное копирование** в разделе **операции** на панели файлового ресурса. На панели контекста отобразится окно **хранилища Essentials** , которое будет загружено справа. После этого можно активировать операции резервного копирования и восстановления по запросу.

   ![Основы хранилища](./media/backup-afs/vault-essentials.png)

## <a name="run-an-on-demand-backup-job"></a>Выполнение задания резервного копирования по запросу

Иногда может потребоваться создать моментальный снимок резервной копии или точку восстановления за пределами запланированного времени в политике резервного копирования. Распространенной причиной создания резервной копии по запросу является сразу же после настройки политики резервного копирования. В соответствии с расписанием в политике резервного копирования это может быть часы или дни до создания моментального снимка. Чтобы защитить свои данные до резервного копирования по расписанию, инициируйте резервное копирование по запросу. Создание резервной копии по запросу часто требуется перед внесением запланированных изменений в общие файловые ресурсы.

### <a name="from-the-recovery-services-vault"></a>Из хранилища служб восстановления

1. Откройте хранилище служб восстановления, которое использовалось для резервного копирования файлового ресурса. В области **Обзор** выберите пункт **архивные элементы** в разделе **защищенные элементы** .

   ![Выбор элементов архивации](./media/backup-afs/backup-items.png)

1. После выбора **элемента архивации**на панели " **Обзор** " появится новая панель со списком всех **типов управления архивацией** .

   ![Список типов управления резервным копированием](./media/backup-afs/backup-management-types.png)

1. В списке **тип управления резервным копированием** выберите Служба **хранилища Azure (службы файлов Azure)**. Вы увидите список всех общих файловых ресурсов и соответствующих учетных записей хранения, созданных с помощью этого хранилища.

   ![Архивные элементы службы хранилища Azure (службы файлов Azure)](./media/backup-afs/azure-files-backup-items.png)

1. В списке файловых ресурсов Azure выберите нужный файловый ресурс. Отобразятся сведения об **элементе резервного копирования** . В меню **элемент архивации** выберите пункт **создать резервную копию**. Так как это задание резервного копирования находится по запросу, политика хранения не связана с точкой восстановления.

   ![Выбрать архивацию сейчас](./media/backup-afs/backup-now.png)

1. Откроется панель **Архивация** . Укажите срок хранения точки восстановления (последний день). Для резервного копирования по запросу можно использовать максимальный срок хранения в 10 лет.

   ![Выберите дату хранения](./media/backup-afs/retention-date.png)

1. Нажмите кнопку **ОК** , чтобы подтвердить выполнение задания резервного копирования по запросу.

1. Отслеживайте уведомления на портале, чтобы следить за завершением выполнения задания резервного копирования. Ход выполнения задания можно отслеживать на панели мониторинга хранилища. Выберите выполняемые **задания резервного копирования**  >  **In progress**.

### <a name="from-the-file-share-pane"></a>Из панели общей папки

1. Откройте панель **Обзор** общей папки, для которой требуется выполнить резервное копирование по запросу.

1. В разделе " **Операция** " выберите **резервное копирование** . На панели контекста отобразится окно **хранилища Essentials** , которое будет загружено справа. Выберите **архивацию сейчас** , чтобы выполнить резервное копирование по запросу.

   ![Выбрать архивацию сейчас](./media/backup-afs/select-backup-now.png)

1. Откроется панель **Архивация** . Укажите период хранения для точки восстановления. Для резервного копирования по запросу можно использовать максимальный срок хранения в 10 лет.

   ![Хранить дату резервного копирования](./media/backup-afs/retain-backup-date.png)

1. Для подтверждения нажмите кнопку **ОК**.

>[!NOTE]
>Azure Backup блокирует учетную запись хранения при настройке защиты для любой общей папки в соответствующей учетной записи. Это обеспечивает защиту от случайного удаления учетной записи хранения с архивированными файловыми ресурсами.

## <a name="best-practices"></a>Рекомендации

* Не удаляйте моментальные снимки, созданные Azure Backup. Удаление моментальных снимков может привести к потере точек восстановления и (или) сбоям при выполнении восстановления.

* Не удаляйте блокировку, сделанную в учетной записи хранения, Azure Backup. Если вы удалите блокировку, ваша учетная запись хранения будет подвержена случайному удалению, и, если она удалена, ваши моментальные снимки или резервные копии будут потеряны.
