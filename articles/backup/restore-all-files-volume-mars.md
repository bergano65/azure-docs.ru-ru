# <a name="restore-all-the-files-in-a-volume-using-the-mars-agent"></a>Восстановление всех файлов в томе с помощью агента MARS

В этой статье объясняется, как восстановить все архивированные файлы во всем томе с помощью мастера восстановления данных в агенте Службы восстановления Microsoft Azure (MARS). Вы можете выполнить следующие действия:

- Восстановите все архивированные файлы тома на том же компьютере, на котором были созданы резервные копии.
- Восстановите все архивированные файлы тома на другой компьютер.

>[!TIP]
>Параметр **Том** позволяет восстановить все резервные копии на определенном томе. Этот параметр обеспечивает более быструю скорость передачи (до 40 Мбит/с) и рекомендуется для восстановления данных большого размера или целых томов.
>
>**Параметр отдельные файлы и папки** позволяет быстро получить доступ к данным точек восстановления. Он подходит для восстановления отдельных файлов и рекомендуется для общего размера менее 80 ГБ. Она обеспечивает скорость передачи или копирования до 6 Мбит/с во время восстановления.

## <a name="volume-level-restore-to-the-same-machine"></a>Восстановление уровня тома на том же компьютере

Следующие шаги помогут восстановить все архивированные файлы в томе.

1. Откройте оснастку **Microsoft Azure Backup**. Если вы не знаете, куда была установлена оснастка, найдите на компьютере или сервере службу **Microsoft Azure Backup**. Классическое приложение должны быть в результатах поиска.

1. Выберите **Восстановить данные**, чтобы запустить мастер.

    ![Меню "восстановить данные"](./media/restore-all-files-volume-mars/recover.png)

1. На странице **Начало работы** для восстановления данных на тот же сервер или компьютер выберите **этот сервер (имя сервера)**  >  **Далее**.

    ![Страница "Начало работы"](./media/restore-all-files-volume-mars/same-machine-instant-restore.png)

1. На странице **Выбор режима восстановления** выберите **том**  >  **Далее**.

    ![Выбор режима восстановления](./media/restore-all-files-volume-mars/select-recovery-mode.png)

1. На странице **Выбор тома и даты** выберите том, который требуется восстановить.

    В календаре выберите точку восстановления. Даты **полужирным** шрифтом означают доступность по крайней мере одной точки восстановления. Если в пределах одной даты доступно несколько точек восстановления, выберите конкретную точку восстановления из раскрывающегося меню **Время**.

     ![Выбор тома и даты](./media/restore-all-files-volume-mars/select-volume-and-date.png)

1. На странице **Указание параметров восстановления** настройте поведение восстановления.
    1. Выберите назначение восстановления:
        - **Исходное расположение**: восстановление данных по исходному пути.
        - **Другое расположение**: Укажите альтернативное расположение для восстановления данных.
    1. Выберите режим работы, **когда элементы в резервной копии уже находятся в назначении восстановления**:
        - **Создать копии. чтобы иметь обе версии**: Если файл с таким именем уже существует, данные в точке восстановления будут восстановлены как копия. Копия будет иметь локализованный префикс имени файла, используя время локального задания восстановления в одном из следующих форматов:
            - `YYYY-MM-DD HH-mm Copy of <original file name>`
            - `YYYY-MM-DD HH-mm Copy (n) of <original file name>`
        - **Перезаписать существующие версии с восстановленными версиями**. Если файл с таким именем уже существует, его содержимое заменяется данными в точке восстановления.
        - **Не восстанавливайте элементы, которые уже существуют в месте назначения восстановления**. Если файл с таким именем уже существует, он будет пропущен.
    1. **Включите восстановление разрешений списка управления доступом (ACL) для восстанавливаемого файла или папки** , если файл должен быть восстановлен с исходными разрешениями в точке восстановления.
        ![Задать параметры восстановления](./media/restore-all-files-volume-mars/specify-recovery-options.png)

1. Проверьте сведения о восстановлении в области **подтверждения** и нажмите кнопку **восстановить**.

    ![Сведения о подтверждении](./media/restore-all-files-volume-mars/confirmation-details.png)

1. На странице **Ход восстановления** отслеживайте ход выполнения задания восстановления. Мастер также можно безопасно закрыть, и операция восстановления продолжится в фоновом режиме. Чтобы просмотреть ход выполнения, дважды щелкните задание восстановления на панели мониторинга.

## <a name="volume-level-restore-to-an-alternate-machine"></a>Восстановление уровня тома на другой компьютер

Следующие шаги помогут восстановить все архивированные файлы тома на другой компьютер. Эти действия можно использовать для восстановления данных из Azure Backup в случае утери всего сервера.

В нем используются следующие термины:

- *Исходный компьютер* — компьютер для данных, на котором была создана резервная копия и который в настоящий момент недоступен.
- *Целевой компьютер* — компьютер, на который восстанавливаются данные.
- *Пример хранилища* — хранилище служб восстановления, в котором зарегистрированы исходный и целевой компьютеры.

> [!NOTE]
> Резервные копии не могут быть восстановлены на целевом компьютере, на котором работает более ранняя версия операционной системы. Например, резервную копию, созданную на компьютере Windows 7, можно восстановить на компьютере под управлением Windows 7 или более поздней версии, Резервную копию, созданную на компьютере с Windows 10, нельзя восстановить на компьютер с Windows 7.

1. Откройте на целевом компьютере оснастку **Microsoft Azure Backup**.

1. Зарегистрируйте целевой и исходный компьютеры в одном хранилище Служб восстановления.

1. Выберите **Восстановить данные**, чтобы открыть **мастер восстановления данных**.

    ![Снимок экрана Azure Backup с выделенными данными восстановления (восстановление на другой компьютер)](./media/backup-azure-restore-windows-server/recover.png)

1. На странице **Приступая к работе** выберите **Другой сервер**.

    ![Снимок экрана мастера восстановления данных начало работы странице (восстановление на другой компьютер)](./media/backup-azure-restore-windows-server/alternatemachine_gettingstarted_instantrestore.png)

1. Укажите файл с учетными данными хранилища, который соответствует примеру хранилища, и выберите **Далее**.

    Если файл учетных данных хранилища является недопустимым (или срок его действия истек), [Скачайте новый файл учетных данных хранилища из примера хранилища](backup-azure-file-folder-backup-faq.md#where-can-i-download-the-vault-credentials-file) на портал Azure. После ввода действительных учетных данных хранилища появится имя соответствующего хранилища архивации.

1. На странице **Выбор резервного сервера** из списка компьютеров выберите исходный компьютер и укажите парольную фразу. Выберите **Далее**.

    ![Снимок экрана мастера восстановления данных: страница "Выбор резервного сервера" (восстановление на другой компьютер)](./media/backup-azure-restore-windows-server/alternatemachine_selectmachine_instantrestore.png)

1. На странице **Выбор режима восстановления** выберите **том**  >  **Далее**.

    ![Выбор режима восстановления](./media/restore-all-files-volume-mars/select-recovery-mode.png)

1. На странице **Выбор тома и даты** выберите том, который требуется восстановить.

    В календаре выберите точку восстановления. Даты **полужирным** шрифтом означают доступность по крайней мере одной точки восстановления. Если в пределах одной даты доступно несколько точек восстановления, выберите конкретную точку восстановления из раскрывающегося меню **Время**.

     ![Выбор тома и даты](./media/restore-all-files-volume-mars/select-volume-and-date.png)

1. На странице **Указание параметров восстановления** настройте поведение восстановления.
    1. Выберите назначение восстановления:
        - **Исходное расположение**: восстановление данных по исходному пути.
        - **Другое расположение**: Укажите альтернативное расположение для восстановления данных.
    1. Выберите режим работы, **когда элементы в резервной копии уже находятся в назначении восстановления**:
        - **Создать копии. чтобы иметь обе версии**: Если файл с таким именем уже существует, данные в точке восстановления будут восстановлены как копия. Копия будет иметь локализованный префикс имени файла, используя время локального задания восстановления в одном из следующих форматов:
            - `YYYY-MM-DD HH-mm Copy of <original file name>`
            - `YYYY-MM-DD HH-mm Copy (n) of <original file name>`
        - **Перезаписать существующие версии с восстановленными версиями**. Если файл с таким именем уже существует, его содержимое заменяется данными в точке восстановления.
        - **Не восстанавливайте элементы, которые уже существуют в месте назначения восстановления**. Если файл с таким именем уже существует, он будет пропущен.
    1. **Включите восстановление разрешений списка управления доступом (ACL) для восстанавливаемого файла или папки** , если файл должен быть восстановлен с исходными разрешениями в точке восстановления.
        ![Задать параметры восстановления](./media/restore-all-files-volume-mars/specify-recovery-options.png)

1. Проверьте сведения о восстановлении в области **подтверждения** и нажмите кнопку **восстановить**.

    ![Сведения о подтверждении](./media/restore-all-files-volume-mars/confirmation-details.png)

1. На странице **Ход восстановления** отслеживайте ход выполнения задания восстановления. Мастер также можно безопасно закрыть, и операция восстановления продолжится в фоновом режиме. Чтобы просмотреть ход выполнения, дважды щелкните задание восстановления на панели мониторинга.
