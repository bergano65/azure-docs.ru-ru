# <a name="delete-a-recovery-services-vault"></a>Удаление хранилища служб восстановления

В этой статье описывается удаление [Azure Backup](backup-overview.md) хранилище служб восстановления. Он содержит инструкции для удаления зависимостей и затем удалить хранилище и удаление хранилища принудительно.


## <a name="before-you-start"></a>Перед началом работы

Перед началом работы важно знать, что нельзя удалить хранилище служб восстановления, который имеет серверов, зарегистрированные в нем, или, содержащий данные резервного копирования.

- Чтобы корректно удалить хранилище, регистрацию серверов в нем, удалить хранилище данных и затем удалите хранилище.
- Если вы попытаетесь удалить хранилище, по-прежнему имеет зависимости, выдается сообщение об ошибке. и необходимо вручную удалить зависимости хранилища, включая:
    - Архивные элементы
    - На защищенных серверах
    - Резервное копирование серверов управления (Azure Backup Server, DPM) ![выберите хранилище, чтобы открыть ее панель мониторинга](./media/backup-azure-delete-vault/backup-items-backup-infrastructure.png)
- Если вы не хотите сохранять данные в хранилище служб восстановления и удалить хранилище, можно удалить хранилище принудительно.
- Если вы пытаетесь удалить хранилище, но у вас не получается, это означает, что хранилище по-прежнему настроено на получение данных резервного копирования.


## <a name="delete-a-vault-from-the-azure-portal"></a>Удаление хранилища на портале Azure

1. Откройте панель мониторинга хранилища.  
2. На панели мониторинга щелкните **удалить**. Убедитесь, что Вы действительно хотите удалить.

    ![выберите хранилище, чтобы открыть его панель мониторинга](./media/backup-azure-delete-vault/contoso-bkpvault-settings.png)

Если произошла ошибка, удалите [архивные элементы](#remove-backup-items), [серверы инфраструктуры](#remove-backup-infrastructure-servers), и [точек восстановления](#remove-azure-backup-agent-recovery-points), а затем удалите хранилище.

![Ошибка хранилища при удалении](./media/backup-azure-delete-vault/error.png)


## <a name="delete-the-recovery-services-vault-by-force"></a>Принудительное удаление хранилища служб восстановления

Вы можете удалить хранилище принудительно с помощью PowerShell. Принудительного удаления означает, что хранилище и все связанные данные резервного копирования удаляется без возможности восстановления.

[!INCLUDE [updated-for-az](../../includes/updated-for-az.md)]


Чтобы удалить хранилище, принудительно:

1. Войдите в подписку Azure с помощью `Connect-AzAccount` команды и следуйте инструкциям на экране инструкциям.

   ```powershell
    Connect-AzAccount
   ```
2. При первом использовании службы Azure Backup зарегистрируйте поставщик служб восстановления Azure в своей подписке с помощью команды [Register-AzResourceProvider](/powershell/module/az.Resources/Register-azResourceProvider).

   ```powershell
    Register-AzResourceProvider -ProviderNamespace "Microsoft.RecoveryServices"
   ```

3. Откройте окно PowerShell с правами администратора.
4. Удалите ограничения с помощью `Set-ExecutionPolicy Unrestricted`.
5. Выполните следующую команду, чтобы загрузить пакет клиента Azure Resource Manager с сайта chocolately.org.

    `iex ((New-Object System.Net.WebClient).DownloadString('https://chocolatey.org/install.ps1'))`

6. Установите клиент API Azure Resource Manager с помощью следующей команды.

   `choco.exe install armclient`

7. На портале Azure Соберите идентификатор и ресурсов группы имя подписки хранилища, которое требуется удалить.
8. В PowerShell выполните следующую команду, используя ваш идентификатор подписки, имя группы ресурсов и имя хранилища. Эта команда удалит хранилище и все зависимости.

   ```powershell
   ARMClient.exe delete /subscriptions/<subscriptionID>/resourceGroups/<resourcegroupname>/providers/Microsoft.RecoveryServices/vaults/<recovery services vault name>?api-version=2015-03-15
   ```
9. Если хранилище не пустой, возникает ошибка «Невозможно удалить хранилище, как существующие ресурсы в этом хранилище». Чтобы удалить содержащиеся в хранилище, сделайте следующее:

   ```powershell
   ARMClient.exe delete /subscriptions/<subscriptionID>/resourceGroups/<resourcegroupname>/providers/Microsoft.RecoveryServices/vaults/<recovery services vault name>/registeredIdentities/<container name>?api-version=2016-06-01
   ```

10. На портале Azure убедитесь, что хранилище будет удалено.


## <a name="remove-vault-items-and-delete-the-vault"></a>Удаление элементов хранилища, а затем удалите хранилище

Эти процедуры предоставляют некоторые примеры резервного копирования данных и серверов инфраструктуры. Когда все будет удален из хранилища, его можно удалить.

### <a name="remove-backup-items"></a>Удалите элементы резервного копирования

Эта хранимая процедура пример, в котором показано, как удалить резервные копии данных из файлов Azure.

1. Нажмите кнопку **архивные элементы** > **Azure хранилища (файлы Azure)**

    ![выбор типа резервной копии](./media/backup-azure-delete-vault/azure-storage-selected-list.png)

2. Щелкните правой кнопкой мыши каждый элемент службы файлов Azure для удаления и нажмите кнопку **остановить архивацию**.

    ![выбор типа резервной копии](./media/backup-azure-delete-vault/stop-backup-item.png)


3. В **остановить архивацию** > **выберите параметр**выберите **удаление данных архивации**.
4. Введите имя элемента и нажмите кнопку **остановить архивацию**.
   - Это подтверждает, что Вы действительно хотите удалить элемент.
   - **Остановить архивацию** активирует кнопку после проверки.
   - Если вы сохраняете и не удалять данные, нельзя будет удалить хранилище.

     ![Удалить данные архивации](./media/backup-azure-delete-vault/stop-backup-blade-delete-backup-data.png)

5. При необходимости указать причину причину удаления данных и добавить комментарии.
6. Чтобы убедиться, что удалить задание завершено, проверьте сообщения Azure ![Удалить данные архивации](./media/backup-azure-delete-vault/messages.png).
7. После завершения задания служба отправляет сообщение: **остановке процесса архивации и удалении данных архивации**.
8. Удалив элемент в списке, в меню **Элементы резервного копирования** щелкните **Обновить**, чтобы увидеть элементы в хранилище.

      ![Удалить данные архивации](./media/backup-azure-delete-vault/empty-items-list.png)

   - Это подтверждает, что Вы действительно хотите удалить элемент.

### <a name="remove-azure-backup-agent-recovery-points"></a>Удаление точек восстановления агента службы архивации Azure

1. В меню панели мониторинга хранилища щелкните **инфраструктура резервного копирования**.
2. Нажмите кнопку **серверы управления архивацией** Просмотр серверов инфраструктуры.

    ![выберите хранилище, чтобы открыть его панель мониторинга](./media/backup-azure-delete-vault/identify-protected-servers.png)

3. В списке **Защищенные серверы** выберите агент Azure Backup.

    ![выбор типа резервной копии](./media/backup-azure-delete-vault/list-of-protected-server-types.png)

4. Выберите сервер в список серверов, защищенных с помощью агента службы архивации Azure.

    ![выбор определенного защищенного сервера](./media/backup-azure-delete-vault/azure-backup-agent-protected-servers.png)

5. На выбранном сервере мониторинга, щелкните **удалить**.

    ![удаление выбранного сервера](./media/backup-azure-delete-vault/selected-protected-server-click-delete.png)

6. В меню **Удалить** введите имя элемента и нажмите кнопку **Удалить**.
   - Это подтверждает, что Вы действительно хотите удалить элемент.

   - Если вы сохраняете и не удалять данные, нельзя будет удалить хранилище.

     ![Удалить данные архивации](./media/backup-azure-delete-vault/delete-protected-server-dialog.png)

7. При необходимости указать причину причину удаления данных и добавить комментарии.
8. Чтобы убедиться, что удалить задание завершено, проверьте сообщения Azure ![Удалить данные архивации](./media/backup-azure-delete-vault/messages.png).
7. После завершения задания служба отправляет сообщение: **остановке процесса архивации и удалении данных архивации**.

### <a name="delete-the-vault-after-removing-dependencies"></a>Удалите хранилище после удаления зависимости

1. Когда будут удалены все зависимости, найдите службу **Essentials** панели в меню хранилища.
2. Убедитесь, что нет **архивные элементы**, **серверы управления архивацией**, или **реплицированные элементы** в списке. Если в хранилище по-прежнему есть элементы, их необходимо удалите.

2. Когда на панели мониторинга хранилища не останется элементов, щелкните **Удалить**.

    ![Удалить данные архивации](./media/backup-azure-delete-vault/vault-ready-to-delete.png)

1. Чтобы подтвердить удаление хранилища, нажмите кнопку **Да**. Хранилище удаляется, а на портале открывается меню **Новая служба** .

## <a name="what-if-i-stop-the-backup-process-but-retain-the-data"></a>Что будет, если я остановлю резервное копирование, но сохраню данные?

Если вы остановите процесс резервного копирования, но случайно сохранить данные, необходимо удалить резервные копии данных, как описано в предыдущих разделах.
