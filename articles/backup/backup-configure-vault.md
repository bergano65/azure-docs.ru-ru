# <a name="back-up-windows-machines-with-the-azure-backup-mars-agent"></a>Резервное копирование компьютеров Windows с агентом Azure Backup MARS

В этой статье объясняется, как к резервному копированию машин Windows с помощью [Azure Backup](backup-overview.md) служба и агент служб восстановления Microsoft Azure (MARS), также известный как агент службы архивации Azure.

В этой статье раскрываются следующие темы:

> [!div class="checklist"]
> * Проверка предварительных требований и создайте хранилище служб восстановления.
> * Скачивание и настройка агента служб восстановления Microsoft Azure.
> * Создание политики резервного копирования и расписания.
> * Выполнение ad-hoc резервного копирования.

## <a name="about-the-mars-agent"></a>Общие сведения об агенте MARS

Агент MARS используется службой Azure Backup для резервного копирования файлов, папок и состояния системы из локальных компьютеров и виртуальных машин Azure в резервное хранилище служб восстановления в Azure. Вы можете запустить агент следующим образом:

- Запустите агент непосредственно на компьютерах Windows в локальной, таким образом, чтобы они позволяет архивировать непосредственно в резервное хранилище служб восстановления Azure.
- Запуск агента Azure виртуальных машин под управлением Windows (side-by-side с расширением резервного копирования виртуальной Машины Azure) для резервного копирования определенных файлов и папок на виртуальной Машине.
- Запустить агент на сервере Microsoft Azure Backup Server (MABS) или на сервере System Center Data Protection Manager (DPM). В этом случае машин и рабочих нагрузок резервного копирования в MABS (или) DPM и затем MABS/DPM создает резервные копии в хранилище в Azure с помощью агента MARS.
Целевой объект резервного копирования зависит от того, где установлен агент.

> [!NOTE]
> Основной метод резервного копирования виртуальных машин Azure — с помощью расширения службы архивации Azure на виртуальной Машине. В этом случае резервное копирование будет выполняться для всей виртуальной машины. Может потребоваться установить и использовать агент MARS вместе с расширением, если вы хотите архивировать отдельных файлов и папок на виртуальной Машине. [Узнайте больше](backup-architecture.md#architecture-direct-backup-of-azure-vms).

## <a name="before-you-start"></a>Перед началом работы

- [Узнайте, как](backup-architecture.md#architecture-direct-backup-of-on-premises-windows-server-machines-or-azure-vm-files-or-folders) службы архивации Azure создает резервные копии компьютеров Windows с агентом служб восстановления Microsoft AZURE.
- [Дополнительные сведения о](backup-architecture.md#architecture-back-up-to-dpmmabs) архитектура резервного копирования, агент MARS на вторичном сервере MABS или DPM.
- [Просмотрите](backup-support-matrix-mars-agent.md) поддерживаемые, и что можно архивировать с помощью агента MARS.
- Проверка доступа к Интернету на компьютерах, которые вы хотите создать резервную копию.
- Для архивации сервера или клиента в Azure вам потребуется учетная запись Azure. Если ее нет, можно создать [бесплатную учетную запись](https://azure.microsoft.com/free/) всего за несколько минут.

### <a name="verify-internet-access"></a>Проверка доступа к Интернету

Если компьютер имеет ограниченный доступ к Интернету, убедитесь, что параметры брандмауэра на компьютере или прокси разрешают эти URL-адреса:

- www\.msftncsi.com
- *.Microsoft.com
- *.WindowsAzure.com
- *.microsoftonline.com
- *.windows.net

## <a name="create-a-recovery-services-vault"></a>Создание хранилища служб восстановления

Хранилище служб восстановления хранятся резервные копии и точки восстановления, которые создаются в течение времени, а также содержит политику архивации, примененную к машинам резервных копий. Создайте хранилище следующим образом.

1. Войдите на [портал Azure](https://portal.azure.com/) , используя свою подписку Azure.
2. В поле поиска введите **служб восстановления** и нажмите кнопку **хранилища служб восстановления**.

    ![Создание хранилища служб восстановления — шаг 1](./media/backup-try-azure-backup-in-10-mins/open-rs-vault-list.png)

3. На **хранилища служб восстановления** меню, щелкните **+ добавить**.

    ![Создание хранилища служб восстановления — шаг 2](./media/backup-try-azure-backup-in-10-mins/rs-vault-menu.png)

4. В поле **Имя**введите понятное имя хранилища.

   - Имя должно быть уникальным в пределах подписки Azure.
   - Оно может содержать от 2 до 50 символов.
   - Имя должно начинаться с буквы, оно может содержать только буквы, цифры и дефисы.

5. Выберите подписку Azure, группу ресурсов и географическом регионе, в котором должен будет создан хранилище. Резервные копии данных отправляется в хранилище. Затем щелкните **Создать**.

    ![Создание хранилища служб восстановления — шаг 3](./media/backup-try-azure-backup-in-10-mins/rs-vault-step-3.png)

   - Для создания хранилища может потребоваться некоторое время.
   - Следите за уведомлениями о состоянии на портале в верхней правой области. Если через несколько минут вы не видите хранилища, нажмите кнопку **обновить**.

     ![Кнопка "Обновить"](./media/backup-try-azure-backup-in-10-mins/refresh-button.png)

### <a name="set-storage-redundancy"></a>Установка избыточности хранилища

Служба архивации Azure автоматически обрабатывает хранилища для хранилища. Необходимо указать способ репликации этого хранилища.

1. В колонке **Хранилища служб восстановления** щелкните новое хранилище. В разделе **параметры** щелкните **свойства**.
2. В **свойства**в разделе **резервная конфигурация**, нажмите кнопку **обновления**.

3. Выберите тип репликации хранилища и нажмите кнопку **Сохранить**.

      ![Настройка конфигурации для нового хранилища](./media/backup-try-azure-backup-in-10-mins/recovery-services-vault-backup-configuration.png)

- Рекомендуется, если вы используете Azure в качестве конечной точки основного хранилища службы архивации, продолжают использовать значение по умолчанию **геоизбыточное** параметр.
- а если нет, — **локально избыточное** (это позволит снизить плату за хранилище Azure).
- Дополнительные сведения о [географически](../storage/common/storage-redundancy-grs.md) и [локального](../storage/common/storage-redundancy-lrs.md) избыточности.

## <a name="download-the-mars-agent"></a>Скачайте агент служб восстановления Microsoft AZURE

Скачайте агент MARS для установки на компьютерах, которые вы хотите создать резервную копию.

- Если вы уже установили агент на одном компьютере, убедитесь, что вы используете последнюю версию.
- Последняя версия доступна на портале или с использованием [прямого скачивания](https://aka.ms/azurebackup_agent)

1. В хранилище в разделе **Приступая к работе**, нажмите кнопку **резервного копирования**.

    ![Открытая колонка цели резервного копирования](./media/backup-try-azure-backup-in-10-mins/open-backup-settings.png)

2. В **где выполняется рабочая нагрузка?** выберите **локальной**. Этот параметр следует выбирать, даже если вы хотите установить агент MARS на виртуальной Машине Azure.
3. В **действия вы хотите резервировать?** выберите **файлов и папок** и/или **состояния системы**. Существует несколько других параметров, но они поддерживаются, только если у вас резервного копирования сервера-получателя. Нажмите кнопку **Подготовка инфраструктуры**.

      ![Настройка резервного копирования файлов и папок](./media/backup-try-azure-backup-in-10-mins/set-file-folder.png)

4. На **Подготовка инфраструктуры**в разделе **агент служб восстановления установите**, скачайте агент служб восстановления Microsoft AZURE.

    ![Download Agent for Windows Server or Windows Client](./media/backup-try-azure-backup-in-10-mins/choose-agent-for-server-client.png)

5. Во всплывающем окне скачивания нажмите кнопку **Сохранить**. По умолчанию файл **MARSagentinstaller.exe** сохраняется в папке для скачивания.

6. Теперь проверьте **уже загрузки или с помощью последнюю версию агента служб восстановления**, а затем загрузить учетные данные хранилища.

    ![Скачивание учетных данных хранилища](./media/backup-try-azure-backup-in-10-mins/download-vault-credentials.png)

7. Выберите команду **Сохранить**. Файл загружается в папку загрузки. Не удается открыть файл учетных данных хранилища.

## <a name="install-and-register-the-agent"></a>Установка и регистрация агента

1. Запустите **MARSagentinstaller.exe** файл на компьютерах, которые вы хотите создать резервную копию.
2. В мастере установки агента MARS > **параметры установки**, укажите, где требуется установить агент и расположение для кэша. Нажмите кнопку **Далее**.
   - Служба архивации Azure использует кэш для хранения моментальных снимков данных перед их отправкой в Azure.
   - Расположение кэша должно иметь объем свободного места по крайней мере 5% от размера данных резервного копирования.

     ![Параметры установки мастер режима MARS](./media/backup-configure-vault/mars1.png)

2. В **конфигурацию прокси-сервера**, укажите, как агент, запущенный на компьютере Windows будут подключаться к Интернету. Нажмите кнопку **Далее**.

   - Если вы используете настраиваемый прокси-сервера укажите параметры прокси-сервера и учетные данные, при необходимости.
   - Помните, что агент должен доступ к [эти URL-адреса](#verify-internet-access).

     ![Доступ к Интернету мастер режима MARS](./media/backup-configure-vault/mars2.png)

3. В **установки** просмотрите проверки требований и нажмите кнопку **установить**.
4. После установки агента, нажмите кнопку **перейти к регистрации**.
5. В **мастера регистрации сервера** > **Идентификация хранилища**найдите и выберите загруженный файл учетных данных. Нажмите кнопку **Далее**.

    ![Register - учетные данные хранилища](./media/backup-configure-vault/register1.png)

6. В **параметр шифрования**, укажите парольную фразу, которая будет использоваться для шифрования и расшифровки архивов для машины.

    - Сохранить парольную фразу шифрования в безопасном месте.
    - Если вы потеряли или забыли парольную фразу, корпорация Майкрософт не поможет восстановить данные архивации. Сохраните файл в безопасном месте, Он необходим для восстановления резервной копии.

7. Нажмите кнопку **Готово**. Теперь, когда агент установлен и компьютер зарегистрирован в хранилище, вы можете настроить параметры резервного копирования, в том числе расписание.

## <a name="create-a-backup-policy"></a>создание политики архивации;

Политика архивации указывает, когда следует создавать моментальные снимки данных для создания точек восстановления и как долго следует хранить точки восстановления.

- Можно настроить политику резервного копирования с помощью агента служб восстановления Microsoft AZURE.
- Служба архивации Azure не учитывает автоматически летнее время (DST). Это может привести к некоторым различиям между фактическое время и время запланированного резервного копирования.

Создайте политику следующим образом:

1. На каждом компьютере откройте агент служб восстановления Microsoft AZURE. Его можно найти, выполнив поиск строки **Microsoft Azure Backup** на компьютере.
2. В **действия**, нажмите кнопку **создать расписание архивации**.

    ![Планирование архивации Windows Server](./media/backup-configure-vault/schedule-first-backup.png)

3. В мастере планирования архивации > **Приступая к работе**, нажмите кнопку **Далее**.
4. В **Выбор элементов для архивации**, нажмите кнопку **добавить элементы**.
5. В **выбрать элементы**, выберите объект для резервного копирования. Нажмите кнопку **ОК**.
6. В **Выбор элементов для архивации** щелкните **Далее**.
7. В **Определение расписания архивации** , определите, если вы хотите создавать резервные копии ежедневно или еженедельно. Нажмите кнопку **Далее**.

    - Точка восстановления создается в том случае, когда создается резервная копия.
    - Количество точек восстановления, созданных в вашей среде зависит от расписания резервного копирования.

1. Вы можете запланировать ежедневное резервное копирование, до трех раз в день. Например на снимке экрана показаны две резервные копии ежедневно в полночь и в 18: 00.

    ![Ежедневное расписание](./media/backup-configure-vault/day-schedule.png)

9. Можно запустить слишком еженедельные резервные копии. Например на снимке экрана показаны резервные копии, создаваемые каждые альтернативный среду и воскресенье в 9:30 и 01:00:00.

    ![Еженедельное расписание](./media/backup-configure-vault/week-schedule.png)

8. На **выбора политики хранения** , определите, каким образом хранить исторические копии ваших данных. Нажмите кнопку **Далее**.

   - Параметры хранения указывают, какие точки восстановления должны быть сохранены, и как долго они должны быть сохранены для.
   - К примеру устанавливая ежедневного параметр хранения, вы указывают, что в то время, указанное для ежедневного сохранения последней точки восстановления будут храниться для указанного числа дней. Или, например, можно указать ежемесячная политика хранения, чтобы указать, что точки восстановления, созданной 30 каждого месяца должны храниться в течение 12 месяцев.
   - Хранение точки восстановления ежедневных и еженедельных обычно совпадает с расписание резервного копирования. Это означает, что при запуске резервного копирования в соответствии с расписанием точки восстановления, созданной в резервной копии хранится в течение указанного в ежедневно или еженедельно политику хранения.
   - Например, на следующем снимке экрана:
     - Ежедневное резервное копирование в полночь и 18: 00, хранятся в течение семи дней.
     - Резервные копии, созданные в субботу в полночь и 18: 00, хранятся в течение 4 недель.
     - Резервные копии, созданные в субботу на последней неделе месяца в полночь и 18: 00, хранятся в течение 12 месяцев. -Резервные копии, созданные в субботу за последнюю неделю марта, хранятся в течение 10 лет.

   ![Пример хранения](./media/backup-configure-vault/retention-example.png)

11. В **выберите тип начального архива** укажите способ выполнения начального резервного копирования, по сети или в автономном режиме. Нажмите кнопку **Далее**.

10. В **Подтверждение**, просмотрите сведения и нажмите кнопку **Готово**.
11. Когда мастер завершит создание расписания архивации, нажмите кнопку **Закрыть**.

### <a name="perform-the-initial-backup-offline"></a>Выполнение начальной архивации в автономном режиме

Можно запустить начальное резервное копирование автоматически по сети или в автономном режиме. Автономное заполнение для начальной архивации полезно в том случае, если у вас есть большое количество данных, который будет использовать большой объем пропускной способности сети для передачи. Выполните операцию в автономном режиме передачи следующим образом:

1. Запись резервного копирования данных в промежуточное расположение.
2. Используйте средство AzureOfflineBackupDiskPrep копировать данные из промежуточной папки в один или несколько дисков SATA.
3. Средство создает задание импорта Azure. [Дополнительные сведения](https://docs.microsoft.com/azure/storage/common/storage-import-export-service) об Azure импорта и экспорта.
4. Вы отправляете дисков SATA в центре обработки данных Azure.
5. В центре обработки данных данные диска копируется в учетную запись хранилища Azure.
6. Служба архивации Azure копирует данные из учетной записи хранения в хранилище и добавочное резервное копирование.

[Дополнительные сведения](backup-azure-backup-import-export.md) о автономного заполнения.

### <a name="enable-network-throttling"></a>Включение регулирования сети

Можно управлять как пропускная способность сети используется агент MARS, включив регулирование сети. Регулирование полезно в том случае, если вам нужно создавать резервные копии данных во время работы, но хотите контролировать использование пропускной способности, используемый для резервного копирования и восстановления действия.

- Сеть Azure Backup, регулирование использует [качества обслуживания (QoS)](https://docs.microsoft.com/windows-server/networking/technologies/qos/qos-policy-top) в локальной операционной системе.
- Для резервного копирования регулирования сети можно найти в Windows Server 2008 R2 и более поздние версии и Windows 7 и более поздних версий. Операционные системы должны быть запущены с последними пакетами обновления.

Включите регулирование сети следующим образом:

1. В агенте служб восстановления Microsoft AZURE, щелкните **изменить свойства**.
2. На **регулирование** вкладке **включить использование пропускной способности Интернета, регулирование для операций резервного копирования**.

    ![Регулирование сети](./media/backup-configure-vault/throttling-dialog.png)
3. Укажите допустимую пропускную способность во время работы, так и вне рабочих часов. Значения пропускной способности начинаются с 512 Кбит/с и перейти до 1023 Мбит/с. Нажмите кнопку **ОК**.

## <a name="run-an-ad-hoc-backup"></a>Выполнение нерегламентированного резервного копирования

1. В агенте служб восстановления Microsoft AZURE, щелкните **архивацию сейчас**. Эта команда запускает начальную репликацию по сети.

    ![Моментальная архивация Windows Server](./media/backup-configure-vault/backup-now.png)

2. В **Подтверждение**, проверьте параметры и нажмите кнопку **резервного копирования**.
3. Нажмите кнопку **Закрыть**, чтобы закрыть мастер. После этого до завершения резервного копирования, мастер продолжит работу в фоновом режиме.

После завершения начальной архивации в консоли службы архивации отобразится состояние **Задание выполнено**.
