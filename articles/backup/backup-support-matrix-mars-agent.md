# <a name="support-matrix-for-backup-with-the-microsoft-azure-recovery-services-mars-agent"></a>Матрица поддержки для резервного копирования с помощью агента Служб восстановления Microsoft Azure (MARS)

Можно использовать [службы архивации Azure](backup-overview.md) для резервного копирования локальных компьютеров и приложений и создания резервных копий виртуальных машин (ВМ) Azure. В этой статье перечислены параметры поддержки и ограничения при использовании агента служб восстановления Microsoft Azure (MARS) для резервного копирования машин.

## <a name="the-mars-agent"></a>Агент служб восстановления Microsoft AZURE

Служба архивации Azure использует агент MARS для резервного копирования данных из локальных компьютеров и виртуальных машин Azure в резервное хранилище служб восстановления в Azure. Агент служб восстановления Microsoft AZURE можно:
- Запустите на локальные компьютеры Windows, таким образом, чтобы они позволяет архивировать непосредственно в резервное хранилище служб восстановления Azure.
- Запустите на виртуальных машинах Windows, таким образом, они могут резервное копирование непосредственно в хранилище.
- Запустите на сервере System Center Data Protection Manager (DPM) или Microsoft Azure Backup Server (MABS). В этом случае машин и рабочих нагрузок резервного копирования сервера DPM или MABS. Агент MARS затем создает резервные копии этого сервера в хранилище Azure. 

Параметры резервного копирования зависит от того, где установлен агент. Дополнительные сведения см. в разделе [архитектуры Azure Backup, с помощью агента MARS](backup-architecture.md#architecture-direct-backup-of-on-premises-windows-server-machines-or-azure-vm-files-or-folders). Сведения о MABS и DPM архитектура резервного копирования, см. в разделе [резервное копирование DPM или MABS](backup-architecture.md#architecture-back-up-to-dpmmabs). Также см. в разделе [требования](backup-support-matrix-mabs-dpm.md) для архитектура резервного копирования.

**Установка** | **Дополнительные сведения**
--- | ---
Скачайте последнюю версию агента служб восстановления Microsoft AZURE | Последнюю версию агента можно скачать из хранилища или [напрямую](https://aka.ms/azurebackup_agent).
Установка непосредственно на компьютере | Агент служб восстановления Microsoft AZURE можно установить непосредственно на сервере Windows на локальном или на виртуальной Машине Windows, под управлением любой из [поддерживаемые операционные системы](https://docs.microsoft.com/en-us/azure/backup/backup-support-matrix-mabs-dpm#supported-mabs-and-dpm-operating-systems).
Установите на резервном сервере | При настройке сервера DPM или MABS для резервного копирования в Azure нужно скачать и установить агент MARS на этом сервере. Агент можно установить на [поддерживаемые операционные системы](backup-support-matrix-mabs-dpm.md#supported-mabs-and-dpm-operating-systems) в матрице поддержки резервного копирования сервера.

> [!NOTE]
> По умолчанию виртуальные машины Azure, которые включены для резервного копирования имеют резервного копирования Azure установки расширения. Это расширение создает резервную копию всей виртуальной машины. Вы можете установить и запустить агент MARS на виртуальной машине Azure вместе с расширением, если хотите выполнить резервное копирование определенных папок или файлов, а не всей виртуальной машины.
> При запуске агента MARS на виртуальной Машине Azure, он создает резервные копии файлов или папок, которые находятся в временного хранилища на виртуальной Машине. Резервное копирование завершается сбоем, если файлы или папки удаляются из временного хранилища или временное хранилище удаляется.


## <a name="cache-folder-support"></a>Поддержка папок кэширования

При использовании агента MARS для резервного копирования данных, агент создается моментальный снимок данных и сохраняет его в папке локального кэша, перед отправкой данных в Azure. Папка кэша (нуля) предъявляет несколько требований.

**Кэш.** | **Дополнительные сведения**
--- | ---
Размер |  Свободное место на диске должно быть по крайней мере 5 – 10 процентов от общего размера данных резервных копий. 
Расположение | Папка кэша должны храниться локально на компьютере, который выполняется резервное копирование, и необходимо подключиться к Интернету. Папка кэша не должно быть в общей сетевой папке, на съемном носителе или на томе в автономном режиме. 
Папка | Папка кэша должна быть зашифрована, когда в дедуплицированном томе или в папке, сжатые, который является разреженным или с точки повторного анализа.
Изменение места | Можно изменить расположение кэша, останавливая работу модуля архивации (`net stop bengine`) и копирование папки кэша на новый диск. (Убедитесь, что новый диск имеется достаточно свободного пространства). Затем обновите две записи реестра в разделе **HKLM\SOFTWARE\Microsoft\Windows Azure Backup** (**Config/ScratchLocation** и **Config/CloudBackupProvider/ScratchLocation**) новое расположение и перезапустить подсистему.

## <a name="support-for-initial-offline-backup"></a>Поддержка начального автономного резервного копирования

Служба архивации Azure поддерживает *автономное заполнение* для передачи начальной резервной копии данных в Azure с помощью дисков. Эта поддержка полезно в том случае, если начальное резервное копирование может находиться в диапазоне размер терабайт (ТБ). Автономное резервное копирование поддерживается для:

- Прямое резервное копирование файлов и папок на локальных компьютерах, работающих под управлением агента MARS.
- резервного копирования рабочих нагрузок и файлов с сервера DPM или MABS;

Автономное резервное копирование не может использоваться для файлов состояния системы.

## <a name="support-for-data-restoration"></a>Поддержка для восстановления данных

С помощью [мгновенное восстановление](backup-instant-restore-capability.md) компонент службы архивации Azure, вы можете восстановить данные перед копированием в хранилище. Выполняется резервное копирование компьютера должен работать под управлением .NET Framework 4.5.2 или более поздней версии.

Невозможно восстановить резервные копии на целевой компьютер, на котором выполняется более ранней версии операционной системы. Например создание резервных копий с компьютера под управлением Windows 7 можно восстановить на Windows 8 или более поздней версии. Но создание резервных копий с компьютера под управлением Windows 8 невозможно восстановить на компьютере под управлением Windows 7.
