# <a name="frequently-asked-questions-about-networking-in-azure-functions"></a>Часто задаваемые вопросы о работе в сети в функциях Azure

В этой статье приведены часто задаваемые вопросы о работе в сети в функциях Azure. Более подробный обзор см. в разделе [функции сетевых параметров](functions-networking-options.md).

## <a name="how-do-i-set-a-static-ip-in-functions"></a>Как задать статический IP-адрес в функции?

Развертывание функции в среде службы приложений — в настоящее время единственным способом статический входящий и исходящий IP-адрес для своей функции. Дополнительные сведения об использовании среды службы приложений, начните со статьи [Создание и использование внутреннего балансировщика нагрузки со средой службы приложений](../app-service/environment/create-ilb-ase.md).

## <a name="how-do-i-restrict-internet-access-to-my-function"></a>Как ограничить доступ к Интернету для Моя функция?

Можно ограничить доступ к Интернету, двумя способами:

* [Ограничения IP-адресов](../app-service/app-service-ip-restrictions.md): Ограничьте входящий трафик для приложения-функции, диапазон IP-адресов.
* Удаление всех триггеров HTTP. Для некоторых приложений достаточно просто избегать триггеров HTTP и использовать любые другие источники событий для активации функции.

Имейте в виду, что редакторе портала Azure требуется прямой доступ к своей выполнение функции. Любые изменения кода, на портале Azure требуется устройство, которое используется для просмотра на портале, чтобы его список разрешенных IP-адрес. Но вы можете использовать все вкладка функций платформы с ограничениями сети на месте.

## <a name="how-do-i-restrict-my-function-app-to-a-virtual-network"></a>Как ограничить Мои приложения-функции к виртуальной сети?

— Это единственный способ полностью ограничить функции таким образом, что весь трафик идет через виртуальную сеть для использования внутренней балансировкой нагрузки среды службы приложений. Этот параметр развертывает сайт в выделенной инфраструктуре, внутри виртуальной сети и отправляет все триггеры и трафика через виртуальную сеть. 

Дополнительные сведения об использовании среды службы приложений, начните со статьи [Создание и использование внутреннего балансировщика нагрузки со средой службы приложений](../app-service/environment/create-ilb-ase.md).

## <a name="how-can-i-access-resources-in-a-virtual-network-from-a-function-app"></a>Как получить доступ к ресурсам в виртуальной сети из приложения-функции?

Доступ к ресурсам в виртуальной сети из выполнение функции можно с помощью интеграции виртуальной сети. Дополнительные сведения см. в разделе [Интеграция виртуальной сети](functions-networking-options.md#virtual-network-integration).

## <a name="how-do-i-access-resources-protected-by-service-endpoints"></a>Как получить доступ к ресурсам, защищенным конечные точки службы?

С помощью интеграции виртуальной сети (в настоящее время в предварительной версии), защищенный конечной точки службы ресурсы доступны из выполнения функции. Дополнительные сведения см. в разделе [предварительную интеграцию виртуальной сети](functions-networking-options.md#preview-version-of-virtual-network-integration).

## <a name="how-can-i-trigger-a-function-from-a-resource-in-a-virtual-network"></a>Как можно вызвать функцию из ресурса в виртуальной сети?

Вы можете активировать функцию из ресурса в виртуальной сети только при развертывании приложения-функции в среде службы приложений. Дополнительные сведения об использовании среды службы приложений см. в разделе [Создание и использование внутреннего балансировщика нагрузки со средой службы приложений](../app-service/environment/create-ilb-ase.md).


## <a name="how-can-i-deploy-my-function-app-in-a-virtual-network"></a>Как можно развернуть Мои приложения-функции в виртуальной сети?

Развертывание в среде службы приложений — единственный способ создать приложение-функцию, полностью находящийся внутри виртуальной сети. Дополнительные сведения об использовании внутреннего балансировщика нагрузки со средой службы приложений, начните со статьи [Создание и использование внутреннего балансировщика нагрузки со средой службы приложений](https://docs.microsoft.com/azure/app-service/environment/create-ilb-ase).

Для сценариев, где требуется только односторонний доступ к ресурсам виртуальной сети, меньше полный сетевой изоляции, см. в разделе [сети Обзор функций](functions-networking-options.md).
