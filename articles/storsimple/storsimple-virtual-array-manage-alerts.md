# <a name="use-storsimple-device-manager-to-manage-alerts-for-the-storsimple-virtual-array"></a>Управление оповещениями для виртуального массива StorSimple с помощью диспетчера устройств StorSimple

## <a name="overview"></a>Обзор

Функция оповещений в службе диспетчера устройств StorSimple позволяет просматривать и удалять оповещения, связанные с виртуальным массивом StorSimple, в режиме реального времени. Вы можете использовать оповещения в колонке **Service summary** (Сводка службы), чтобы централизованно отслеживать проблемы работоспособности виртуальных массивов StorSimple и всего решения Microsoft Azure StorSimple.

В этом руководстве описывается, как настроить уведомления об оповещениях, распространенные условия оповещений и их уровни серьезности, а также, как просматривать и отслеживать оповещения. Кроме того, он включает краткую справочную таблицу оповещений, которая позволяет быстро найти нужное оповещение и реагировать соответствующим образом.

![Страница оповещений](./media/storsimple-virtual-array-manage-alerts/alerts1.png)

## <a name="configure-alert-settings"></a>Настроить параметры оповещений.

Вы можете настроить получение уведомлений по электронной почте об условиях оповещений для каждого виртуального массива StorSimple. Кроме того, можно указать других получателей оповещений. Для этого следует ввести их адреса электронной почты в поле **Дополнительные получатели письма**, разделяя их точкой с запятой.

> [!NOTE]
> Можно ввести не более 20 адресов электронной почты для каждого виртуального массива.

После включения получения уведомлений по электронной почте для виртуального массива члены списка уведомлений будут получать сообщение при возникновении каждого критического оповещения. Сообщения будут отправлены из *storsimple-Alerts-reply \@ mail.WindowsAzure.com* и будут описывать условие оповещения. Получатели могут нажать **Отменить подписку**, чтобы удалить себя из списка уведомлений по электронной почте.

#### <a name="to-enable-email-notification-for-alerts"></a>Включение уведомления по электронной почте об оповещениях

1. Откройте службу диспетчера устройств StorSimple и в разделе **Управление** щелкните **Устройства**. В отображаемом списке устройств выберите нужное устройство.
   
    ![Параметры оповещений](./media/storsimple-virtual-array-manage-alerts/alerts2.png)
2. Откроется колонка **Параметры**. В разделе **Параметры устройства** выберите **Общие**. Откроется колонка **Общие параметры**.
   
    ![На снимке экрана показана панель Параметры устройства с названием вызываемая область параметров оповещений.](./media/storsimple-virtual-array-manage-alerts/alerts4.png)
3. В колонке **Общие параметры** перейдите к разделу **Параметры оповещения** и задайте следующее.
   
   1. В поле **Включить уведомления по электронной почте** выберите **Да**.
   2. В поле **Администраторы службы электронной почты** выберите **Да**, если хотите, чтобы администратор службы и все дополнительные администраторы получали уведомления об оповещениях.
   3. В поле **Дополнительные получатели письма** введите адреса электронной почты всех получателей, которые должны получать уведомления об оповещениях. Введите имена в формате *кто-то \@ Somewhere.com*. Используйте точку с запятой для разделения адресов электронной почты. Можно указать не более 20 адресов электронной почты для каждого виртуального устройства.
      
       ![На снимке экрана отображаются сведения о параметрах оповещений с параметрами, описанными на этом шаге.](./media/storsimple-virtual-array-manage-alerts/alerts6.png)
   4. Чтобы отправить тестовое уведомление по электронной почте, щелкните **Отправить тестовое электронное сообщение**. Служба диспетчера устройств StorSimple будет отображать сообщения о состоянии по мере отправки тестового уведомления.
      
       ![На снимке экрана показано информационное диалоговое окно с подтверждением тестового сообщения.](./media/storsimple-virtual-array-manage-alerts/alerts7.png)
      
      > [!NOTE]
      > Если не удается отправить тестовое уведомление, служба диспетчера устройств StorSimple отобразит соответствующее сообщение. Щелкните **ОК**, подождите несколько минут и повторите попытку отправки тестового сообщения уведомления.
      >
      >
   5. В нижней части страницы нажмите кнопку **Сохранить** , чтобы сохранить настройки. При запросе подтверждения нажмите кнопку **Да**.
      
      ![На снимке экрана показана панель параметров с выбранной кнопкой сохранить.](./media/storsimple-virtual-array-manage-alerts/alerts10.png)

## <a name="common-alert-conditions"></a>Распространенные условия оповещений

Виртуальный массив StorSimple создает оповещения при выполнении ряда условий. Ниже приведены наиболее распространенные типы условий оповещения.

* **Проблемы с подключением** — эти оповещения появляются при возникновении сложностей в процессе передачи данных. Проблемы со связью могут возникнуть во время передачи данных в учетную запись хранения Azure и из нее или из-за отсутствия подключения между виртуальными устройствами и службой диспетчера устройств StorSimple. Проблемы связи сложнее всего исправить, поскольку существует множество точек отказа. Всегда необходимо сначала проверить, есть ли сетевое подключение и доступ к Интернету, перед тем как выполнять более сложные методы устранения неполадок. Дополнительные сведения о портах и параметрах брандмауэра см. в статье [Системные требования для виртуального массива StorSimplе](storsimple-ova-system-requirements.md). Дополнительные сведения об устранении неполадок см. в статье [Устранение неполадок в развертывании устройства StorSimple](./storsimple-8000-troubleshoot-deployment.md).
* **Проблемы производительности** – эти оповещения возникают в результате неоптимальной работы системы, например, при тяжелой нагрузке.

Кроме того могут появиться оповещения, относящиеся к безопасности, обновлениям или сбоям заданий.

## <a name="alert-severity-levels"></a>Уровни серьезности оповещений

Оповещения имеют различные уровни серьезности в зависимости от того, какое влияние окажет оповещение и какая потребуется реакция. Ниже перечислены уровни серьезности.

* **Критические** – такое оповещение создается при возникновении условий, отрицательно влияющих на успешную работу системы. Убедитесь, что работа службы StorSimple не прервана.
* **Предупреждения** – состояние может стать критическим, если не будут предприняты необходимые действия. Необходимо изучить ситуацию и предпринять любые действия, необходимые для решения этой проблемы.
* **Информация** – оповещение содержит информацию, которая может быть полезна при отслеживании и управлении системы.

## <a name="view-and-track-alerts"></a>Просмотр и отслеживание оповещений

В колонке сводных данных о службе диспетчера устройств StorSimple можно просмотреть общие сведения о количестве оповещений на виртуальных устройствах, упорядоченных по степени серьезности.

![Панель мониторинга оповещений](./media/storsimple-virtual-array-manage-alerts/alerts14.png)

Выберите уровень серьезности, чтобы открыть колонку **Оповещения**. Результаты включают только оповещения соответствующего уровня серьезности.

![Отчет об оповещениях с сортировкой по типам](./media/storsimple-virtual-array-manage-alerts/alerts15.png)

Щелкните оповещение в списке, чтобы просмотреть дополнительные сведения о нем, включая время последней доставки оповещения, число повторений оповещения на устройстве и рекомендуемые действия для разрешения оповещения.

![Список оповещений и сведения](./media/storsimple-virtual-array-manage-alerts/alerts16.png)

Можно скопировать сведения об оповещении в текстовый файл, если необходимо отправить их в службу технической поддержки Майкрософт. Как только вы выполните рекомендации и устраните условия оповещения в локальной среде, удалите оповещение из списка. Для этого выберите оповещение в списке и нажмите кнопку **Очистить**. Чтобы удалить несколько оповещений, выберите каждое оповещение, которое нужно удалить, щелкните любой столбец, кроме столбца **Оповещение**, а затем щелкните **Очистить**.

При нажатии на кнопку **Сброс** можно оставить свои комментарии об оповещении, а также указать действия, которые были предприняты для разрешения проблемы.

![комментарии к оповещениям](./media/storsimple-virtual-array-manage-alerts/alerts17.png)

Некоторые события будут удалены системой, если другое событие активируется новыми данными.

## <a name="sort-and-review-alerts"></a>Сортировка и просмотр оповещений

В колонке **Оповещения** может отображаться до 250 оповещений. В случае превышения этого числа оповещений в представлении по умолчанию отображаются не все оповещения. Вы можете сочетать следующие поля, чтобы настроить типы отображаемых оповещений.

* **Состояние** – можно отображать **активные** или **сброшенные** оповещения. Активные оповещения по-прежнему инициируются в системе, а удаленные оповещения были либо вручную удалены администратором, либо очищены программно, так как система обновила условие оповещения новыми данными.
* **Важность** – можно отображать оповещения всех уровней важности (критические, предупреждения, информация) или определенного уровня важности, например, только критические оповещения.
* **Источник** — можно отобразить оповещения из всех источников или только оповещения, которые поступили из службы либо одного из или всех виртуальных устройств.
* **Временной диапазон** – указав даты **от** и **до**, а также метки времени, можно просматривать оповещения за интересующий вас период.
