# <a name="limitations-in-azure-database-for-mysql---flexible-server-preview"></a>Ограничения в службе "база данных Azure для MySQL" — гибкий сервер (Предварительная версия)

> [!IMPORTANT] 
> Сейчас предоставляется общедоступная предварительная версия Гибкого сервера Базы данных Azure для MySQL.

В этой статье описываются ограничения службы "база данных Azure для гибкого сервера MySQL". Также применяются [Общие ограничения](https://dev.mysql.com/doc/mysql-reslimits-excerpt/5.7/en/limits.html) ядра СУБД MySQL. Если вы хотите узнать о уровнях ресурсов (вычислений, памяти, хранилища), см. статью о средах [вычислений и хранения](concepts-compute-storage.md) .

## <a name="server-parameters"></a>Параметры сервера

> [!NOTE]
> Если вы ищете значения min/max для параметров сервера, таких как `max_connections` и `innodb_buffer_pool_size` , эта информация была перемещена в раздел Параметры сервера основные сведения о [параметрах сервера](./concepts-server-parameters.md) .

База данных Azure для MySQL поддерживает настройку значений параметров сервера. Минимальное и максимальное значения некоторых параметров (например, `max_connections`, `join_buffer_size` , `query_cache_size` ) определяется уровнем вычислений и размером вычислений сервера. Дополнительные сведения об этих ограничениях см. в разделе [Параметры сервера](./concepts-server-parameters.md) .

Такие подключаемые модули паролей, как "validate_password" и "caching_sha2_password", не поддерживаются службой.

## <a name="storage-engines"></a>Модули хранилища

MySQL поддерживает множество ядер хранилища. В базе данных Azure для гибкого сервера MySQL поддерживаются и не поддерживаются следующие модули хранилища:

### <a name="supported"></a>Поддерживается
- [InnoDB](https://dev.mysql.com/doc/refman/5.7/en/innodb-introduction.html);
- [MEMORY](https://dev.mysql.com/doc/refman/5.7/en/memory-storage-engine.html).

### <a name="unsupported"></a>Не поддерживается
- [MyISAM](https://dev.mysql.com/doc/refman/5.7/en/myisam-storage-engine.html);
- [BLACKHOLE](https://dev.mysql.com/doc/refman/5.7/en/blackhole-storage-engine.html);
- [ARCHIVE](https://dev.mysql.com/doc/refman/5.7/en/archive-storage-engine.html);
- [FEDERATED](https://dev.mysql.com/doc/refman/5.7/en/federated-storage-engine.html).

## <a name="privileges--data-manipulation-support"></a>Права & поддержка обработки данных

Многие параметры и параметры сервера могут случайно снизить производительность сервера или инвертировать свойства ACID сервера MySQL. Для обеспечения целостности службы и соглашения об уровне обслуживания на уровне продукта эта служба не предоставляет несколько ролей. 

Служба MySQL не разрешает прямой доступ к базовой файловой системе. Некоторые команды обработки данных не поддерживаются. 

### <a name="unsupported"></a>Не поддерживается

Следующие действия не поддерживаются.
- Роль администратора баз данных: ограничена. Кроме того, можно использовать учетную запись администратора (созданную во время создания нового сервера), которая позволяет выполнять большинство инструкций DDL и DML. 
- SUPER Privilege. Аналогично, [супер-привилегия](https://dev.mysql.com/doc/refman/5.7/en/privileges-provided.html#priv_super) ограничена.
- DEFINER. Ограничено, требуется разрешение SUPER. Если импортируются данные с помощью резервной копии, удалите команды `CREATE DEFINER` вручную или с помощью команды `--skip-definer` при выполнении mysqldump.
- Системные базы данных. [системная база данных MySQL](https://dev.mysql.com/doc/refman/5.7/en/system-schema.html) доступна только для чтения и используется для поддержки различных функций PaaS. Нельзя вносить изменения в `mysql` системную базу данных.
- `SELECT ... INTO OUTFILE`: Не поддерживается в службе.

### <a name="supported"></a>Поддерживается
- `LOAD DATA INFILE` поддерживается, но должен быть указан параметр `[LOCAL]`, указывающий на путь в формате UNC (хранилище Azure, подключенное по протоколу SMB).

## <a name="functional-limitations"></a>Ограничения функциональных возможностей

### <a name="zone-redundant-ha"></a>Уровень доступности избыточности в зоне
- Эту конфигурацию можно задать только во время создания сервера.
- Не поддерживается на уровне пакетного вычислений.

### <a name="networking"></a>Сеть
- Невозможно изменить метод подключения после создания сервера. Если сервер создан с *закрытым доступом (интеграция с виртуальной сетью)* , он не может быть изменен на *общий доступ (разрешенные IP-адреса)* после создания, и наоборот.
- Протокол TLS/SSL включен по умолчанию и не может быть отключен.
- Минимальная поддерживаемая версия TLS на сервере — TLS 1.2. Дополнительные сведения см. в статье подключение по протоколу [TLS/SSL](./how-to-connect-tls-ssl.md) .

### <a name="stopstart-operation"></a>Операция завершения или запуска
- Не поддерживается в конфигурациях высокой доступности с избыточностью зоны (первичной и резервной).
- Не поддерживается с конфигурациями реплики чтения (как исходной, так и репликой).

### <a name="scale-operations"></a>Операции масштабирования
- Уменьшение объема подготовленного хранилища сервера не поддерживается.

### <a name="read-replicas"></a>Реплики чтения
- Не поддерживается в конфигурациях высокой доступности с избыточностью зоны (первичной и резервной).

### <a name="server-version-upgrades"></a>Обновления версии сервера
- Автоматическая миграция между основными версиями ядра СУБД не поддерживается. Если вы хотите обновить основную версию, сделайте [дамп и восстановите](../concepts-migrate-dump-restore.md) его на сервере, который был создан с использованием новой версии модуля.

### <a name="restoring-a-server"></a>Восстановление сервера
- При восстановлении до точки во времени создаются новые серверы с теми же конфигурациями вычислений и хранилища, что и на исходном сервере, на котором они основаны. После создания сервера его вычисление можно масштабировать.
- Восстановление удаленного сервера не поддерживается.
