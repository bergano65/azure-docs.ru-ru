# <a name="azure-security-baseline-for-azure-stack-edge"></a>Базовый план безопасности Azure для Azure Stack пограничных устройств

Этот базовый план безопасности применяет рекомендации из [теста безопасности Azure версии 2,0](../security/benchmarks/overview.md) к Microsoft Azure Stackм пограничных устройств. Azure Security Benchmark содержит рекомендации по обеспечению безопасности облачных решений в Azure. Содержимое группируются по **элементам управления безопасностью** , определенным в ходе тестирования системы безопасности Azure, и связанным рекомендациям, применимым к Azure Stack границе. **Элементы управления** , неприменимые к Azure Stack границе, были исключены.

Сведения о том, как Azure Stack граница полностью сопоставляется с тестовым показателем безопасности Azure, см. в разделе [полный базовый файл сопоставления безопасности Azure Stack](https://github.com/MicrosoftDocs/SecurityBenchmarks/tree/master/Azure%20Offer%20Security%20Baselines).

### <a name="ns-1-implement-security-for-internal-traffic"></a>NS-1: реализация безопасности для внутреннего трафика

**Руководство**. клиенты развертывают доступное, физическое, Azure Stack пограничных устройствах в своей частной сети для внутреннего доступа и обеспечивают дополнительную защиту. Например, Azure Stack пограничное устройство доступно по внутренней сети клиента и требует настройки IP-адреса, настроенного клиентом. Кроме того, пароль доступа выбирается клиентом для доступа к пользовательскому интерфейсу устройства. 

Внутренний трафик обеспечивает дополнительную защиту:

- Протокол TLS версии 1,2 необходим для управления портал Azure и пакетом SDK Azure Stack пограничном устройстве.

- Любой клиентский доступ к устройству осуществляется через локальный веб-интерфейс, использующий стандарт TLS 1,2 в качестве защищенного протокола по умолчанию.

- Только полномочным устройствам Azure Stack с пограничным устройством Pro разрешено присоединяться к службе пограничной Azure Stack, созданной клиентом в своей подписке Azure.

### <a name="im-2-manage-application-identities-securely-and-automatically"></a>IM-2: безопасное и автоматическое управление удостоверениями приложений

**Руководство**. всем Azure Stack пограничных устройств автоматически назначается управляемое удостоверение, назначенное системой в Azure Active Directory (Azure AD). Azure Stack пограничные устройства загружаются в заблокированное состояние для локального доступа. Для учетной записи администратора локального устройства необходимо подключиться к устройству через локальный веб-интерфейс пользователя или интерфейс PowerShell и задать надежный пароль. Храните учетные данные администратора устройства и управляйте ими в безопасном месте, таком как Azure Key Vault, и поворачивайте пароль администратора в соответствии со стандартами Организации.

### <a name="dp-4-encrypt-sensitive-information-in-transit"></a>DP-4: шифрование конфиденциальной информации во время передачи

**Руководство**. Azure Stack пограничных устройств использует безопасные каналы для данных в полете. А именно:

- Стандартный TLS 1,2 используется для данных, передаваемого между устройством и облаком Azure. Откат к TLS 1,1 и более ранним версиям отсутствует. Взаимодействие с агентом будет заблокировано, если TLS 1,2 не поддерживается. TLS 1,2 также требуется для управления портал Azure и пакетом средств разработки программного обеспечения (SDK).

- Когда клиенты обращаются к устройству через локальный веб-интерфейс пользователя браузера, в качестве защищенного протокола по умолчанию используется стандартный TLS 1,2.

Рекомендуется настроить в браузере использование TLS 1,2. Используйте SMB 3,0 с шифрованием для защиты данных при копировании с серверов данных.

- [Рекомендации по защите данных в полете](azure-stack-edge-security.md#protect-data-in-flight)
