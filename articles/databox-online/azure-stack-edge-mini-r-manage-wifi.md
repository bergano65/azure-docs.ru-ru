# <a name="use-the-local-web-ui-to-manage-wireless-connectivity-on-your-azure-stack-edge-mini-r"></a>Использование локального веб-интерфейса для управления беспроводным подключением на Azure Stack пограничной панели Mini R

В этой статье описывается, как управлять беспроводным сетевым подключением на Azure Stack пограничном устройстве R. Вы можете использовать локальный веб-интерфейс на Azure Stack пограничном устройстве R с помощью для добавления, подключения и удаления профилей Wi-Fi.

## <a name="about-wi-fi"></a>О Wi-Fi

Мини-устройство с Azure Stackным пограничным устройством R может действовать как при подключении к сети, так и через беспроводную сеть. Устройство имеет Wi-Fi порт, который должен быть включен, чтобы разрешить устройству подключаться к беспроводной сети. 

Устройство имеет пять портов, порт 1 через порт 4 и пятый Wi-Fi порт. Ниже приведена схема задней плоскости Мини-устройства R при подключении к беспроводной сети.

![Кабель для Wi-Fi](./media/azure-stack-edge-mini-r-deploy-install/wireless-cabled.png)


## <a name="add-connect-to-wi-fi-profile"></a>Добавление, подключение к профилю Wi-Fi

Выполните следующие действия в локальном пользовательском интерфейсе устройства, чтобы добавить профиль Wi-Fi и подключиться к нему.

1. Откройте страницу **Начало работы** в локальном пользовательском веб-интерфейсе устройства. На плитке **Сеть** выберите **Настроить**.  
    
    На вашем физическом устройстве доступно пять сетевых интерфейсов. Порты 1 и 2 — это сетевые интерфейсы со скоростью 1 Гбит/с. ПОРТЫ 3 и 4 — это все сетевые интерфейсы с 10 Гбит/с. Пятый порт — порт Wi-Fi. 

    [![Локальный веб-интерфейс "Параметры сети", страница 1](./media/azure-stack-edge-mini-r-deploy-configure-network-compute-web-proxy/configure-wifi-1.png)](./media/azure-stack-edge-mini-r-deploy-configure-network-compute-web-proxy/configure-wifi-1.png#lightbox)  
    
    Выберите порт Wi-Fi и настройте параметры порта. 
    
    > [!IMPORTANT]
    > Настоятельно рекомендуется настроить статический IP-адрес для порта Wi-Fi.  

    ![Страница "Параметры сети" в локальном веб-интерфейсе (2)](./media/azure-stack-edge-mini-r-deploy-configure-network-compute-web-proxy/configure-wifi-2.png)

    **Сетевая** страница обновляется после применения параметров порта Wi-Fi.

    ![Локальный веб-интерфейс "Параметры сети", стр. 3](./media/azure-stack-edge-mini-r-deploy-configure-network-compute-web-proxy/configure-wifi-4.png)

   
2. Выберите **добавить Wi-Fi профиль** и отправьте профиль Wi-Fi. 

    ![Локальный веб-пользовательский интерфейс "Параметры сети WiFi порт" 1](./media/azure-stack-edge-mini-r-deploy-configure-network-compute-web-proxy/add-wifi-profile-1.png)
    
    Профиль беспроводной сети содержит SSID (сетевое имя), ключ пароля и сведения о безопасности, которые могут быть подключены к беспроводной сети. Профиль Wi-Fi для вашей среды можно получить у администратора сети.

    ![Локальный веб-интерфейс "Параметры порта WiFi"](./media/azure-stack-edge-mini-r-deploy-configure-network-compute-web-proxy/add-wifi-profile-2.png)

    После добавления профиля список Wi-Fi профилей обновляется в соответствии с новым профилем. Профиль должен отображать **состояние подключения** " **отключено**". 

    ![Локальный веб-пользовательский интерфейс "Параметры сети WiFi порт" 3](./media/azure-stack-edge-mini-r-deploy-configure-network-compute-web-proxy/add-wifi-profile-3.png)
    
3. После успешной загрузки профиля беспроводной сети подключитесь к этому профилю. Выберите **Подключение к профилю Wi-Fi**. 

    ![Локальный веб-интерфейс "порт Wi-Fi параметры сети" 4](./media/azure-stack-edge-mini-r-deploy-configure-network-compute-web-proxy/add-wifi-profile-4.png)

4. Выберите профиль Wi-Fi, добавленный на предыдущем шаге, и нажмите кнопку **Применить**. 

    ![Локальный веб-интерфейс "порт Wi-Fi параметры сети" 5](./media/azure-stack-edge-mini-r-deploy-configure-network-compute-web-proxy/add-wifi-profile-5.png)

    **Состояние подключения** должно обновляться на " **подключено**". Уровень сигнала обновляется для указания качества сигнала. 

    ![Локальный веб-интерфейс "порт Wi-Fi параметры сети" 6](./media/azure-stack-edge-mini-r-deploy-configure-network-compute-web-proxy/add-wifi-profile-6.png)

    > [!NOTE]
    > Для перемещения больших объемов данных вместо беспроводной сети рекомендуется использовать проводное подключение. 


## <a name="download-wi-fi-profile"></a>Скачать профиль Wi-Fi

Вы можете скачать Wi-Fi профиль, который используется для подключения к беспроводной сети.

1. В локальном веб-ИНТЕРФЕЙСе устройства перейдите в раздел **конфигурация > сеть**. 

2. В разделе Wi-Fi параметры профиля выберите **скачать профиль**. При этом необходимо загрузить профиль Wi-Fi, который вы используете в данный момент.


## <a name="delete-wi-fi-profile"></a>Удалить профиль Wi-Fi

Вы можете удалить Wi-Fi профиль, который используется для подключения к беспроводной сети.


1. В локальном веб-ИНТЕРФЕЙСе устройства перейдите в раздел **конфигурация > сеть**. 

2. В разделе Wi-Fi параметры профиля выберите **удалить профиль Wi-Fi**.

3. В колонке **Удаление профиля Wi-Fi** выберите профиль, который нужно удалить. Нажмите кнопку **Применить**.


## <a name="configure-cisco-wi-fi-profile"></a>Настройка профиля Wi-Fi Cisco

Ниже приведены некоторые рекомендации по управлению и настройке беспроводного контроллера Cisco и точки доступа на устройстве. 

### <a name="dhcp-bridging-mode"></a>Режим моста DHCP

Чтобы использовать беспроводный контроллер Cisco для устройства, необходимо включить режим моста протокола DHCP на контроллере беспроводной локальной сети (Влк).

Дополнительные сведения см. в разделе [режим моста DHCP](https://www.cisco.com/c/en/us/support/docs/wireless/4400-series-wireless-lan-controllers/110865-dhcp-wlc.html#anc9).

#### <a name="bridging-configuration-example"></a>Пример конфигурации "мост"

Чтобы включить функцию моста DHCP на контроллере, необходимо отключить функцию DHCP-прокси на контроллере. Чтобы включить мост DHCP с помощью командной строки:

```powershell
(Cisco Controller) > config dhcp proxy disable
(Cisco Controller) > show dhcp proxy
DHCP Proxy Behaviour: disabled
```

Если DHCP-сервер не существует в той же сети уровня 2 (L2), что и клиент, широковещательная рассылка должна перенаправляться на DHCP-сервер в клиентском шлюзе с помощью вспомогательного метода IP. Ниже приведен пример этой конфигурации.

```powershell
Switch#conf t
Switch(config)#interface vlan <client vlan #>
Switch(config-if)#ip helper-address <dhcp server IP>
```

Мост DHCP является глобальным параметром, поэтому он влияет на все транзакции DHCP в контроллере. Необходимо добавить инструкции вспомогательного приложения IP в проводную инфраструктуру для всех необходимых виртуальных локальных сетей (VLAN) на контроллере.

### <a name="enable-the-passive-client-for-wlan"></a>Включение пассивного клиента для беспроводной сети

Чтобы включить функцию пассивного клиента для беспроводной локальной сети (WLAN) на беспроводном контроллере Cisco, сделайте следующее:

* Для интерфейса, связанного с WLAN, должна быть включена маркировка VLAN.
* Для WLAN необходимо включить многоадресную виртуальную ЛС.
* На Влк должна быть включена пересылка Гарп.

Дополнительные сведения см. в разделе [многоадресная виртуальная ЛС сведения об оптимизации многоадресной рассылки](https://www.cisco.com/c/en/us/td/docs/wireless/controller/8-5/config-guide/b_cg85/wlan_interfaces.html).

### <a name="troubleshoot"></a>Диагностика

При возникновении проблем с выделением IP-адресов на виртуальных машинах, работающих на Azure Stack пограничном устройстве R, необходимо проверить указанные выше параметры конфигурации в сетевой среде.
