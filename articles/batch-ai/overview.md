# <a name="what-is-azure-batch-ai"></a>Что такое Azure Batch AI?

Azure Batch AI — это управляемая служба, с помощью которой специалисты по обработке и анализу данных, а также исследователи искусственного интеллекта (ИИ) могут обучать и тестировать модели машинного обучения и ИИ в нужном масштабе в Azure без необходимости управлять всей инфраструктурой. Просто опишите вычислительные ресурсы, выполняемые задания, место для хранения входных и выходных данных модели, а служба Batch AI сама выполнит все остальное.

Службу Batch AI можно использовать автономно или для обучения модели в составе более крупного рабочего процесса разработки:

* С помощью автономной Batch AI вы можете обучать, тестировать и оценивать модели машинного обучения и искусственного интеллекта в пакетном режиме на объединенных в кластеры [GPU](../virtual-machines/linux/sizes-gpu.md) или обычных ЦП. 

* Чтобы использовать кластер Batch AI в рабочем процессе, можно применить [Машинное обучение Azure](../machine-learning/service/overview-what-is-azure-ml.md) или другие [средства Batch AI](https://azure.microsoft.com/overview/ai-platform/). Машинное обучение Azure предоставляет широкие возможности для подготовки данных экспериментирования и изучения журнала заданий. Машинное обучение Azure также позволяет упаковать обученную модель в контейнер и развернуть модель для вывода или на устройствах Интернета вещей.  

## <a name="train-machine-learning-and-ai-models"></a>Обучение моделей машинного обучения и искусственного интеллекта

Используйте службу Batch AI для обучения моделей машинного обучения, создания глубоких нейронных сетей (глубокое обучение) и других систем применения искусственного интеллекта. Batch AI имеет встроенную поддержку популярных библиотек и платформ искусственного интеллекта с открытым кодом, включая [TensorFlow](https://github.com/tensorflow/tensorflow), [PyTorch](https://github.com/pytorch/pytorch), [Chainer](https://github.com/chainer/chainer) и [Microsoft Cognitive Toolkit (CNTK)](https://github.com/Microsoft/CNTK).

Завершив определение проблемной области и подготовку данных, вы сможете взаимодействовать с Batch AI интерактивно для тестирования идей для модели. Когда вы будете готовы к экспериментам в реальном масштабе, запустите задания на нескольких GPU с помощью MPI или других библиотек связи и выполняйте сразу несколько экспериментов в параллельном режиме.

Batch AI поможет вам обучить модели в широком масштабе несколькими способами. Например:  

|  |  |
|---------|---------|
| **Распределение обучения**<br/>![Распределенное обучение](./media/overview/distributed-training.png)  | Увеличить масштаб одного задания обучения до нескольких GPU, подключенных к сети, чтобы обучить большие сети с большими объемами данных.|
| **Эксперимент**<br/>![Экспериментирование](./media/overview/experimentation.png) | Масштабирование обучения с несколькими заданиями. Выполните перебор по параметрам, чтобы проверить новые идеи, или настройте гиперпараметры для оптимизации точности и производительности. |
| **Параллельное выполнение**![Параллельное выполнение](./media/overview/parallel-execution.png) | Обучайте или оценивайте несколько моделей сразу, выполняя задания параллельно на большой группе серверов, чтобы завершить работу быстрее.|

Завершив обучение модели, протестируйте ее с помощью Batch AI, если процесс тестирования не включен в скрипт обучения. Также можно выполнить пакетную оценку.

## <a name="how-it-works"></a>Принцип работы

Используйте пакеты SDK для Batch AI, скрипты командной строки или портал Azure для управления вычислительными ресурсами и планирования заданий для обучения и тестирования искусственного интеллекта. 

* **Подготовка и масштабирование кластеров виртуальных машин** — укажите количество узлов (виртуальных машин) и выберите их размер (с поддержкой или без поддержки GPU) или другой размер виртуальной Машины,в зависимости от конкретных потребностей в обучении. Увеличивайте и уменьшайте число узлов автоматически или вручную, чтобы использовать ровно необходимое количество ресурсов. 

* **Управление зависимостями и контейнерами** — по умолчанию кластеры Batch AI под управлением образов виртуальных машин Linux с предварительно установленными зависимостями выполняют контейнерные платформы обучения на GPU или ЦП. Если вам нужна особая конфигурация, подключите пользовательский образ или выполняйте скрипты запуска.

* **Распределение данных** — выберите одну или несколько технологий хранения для входных данных, скриптов и выходных данных задания: служба файлов Azure, хранилище BLOB-объектов Azure или управляемый NFS-сервер. Также Batch AI поддерживает пользовательские решения для хранения, в том числе высокопроизводительные параллельные файловые системы. Подключайте файловые системы хранилища к узлам кластера и контейнерам задания с помощью несложных файлов конфигурации.

* **Планирование заданий** — передавайте задания в очередь заданий с учетом приоритетов, чтобы совместно использовать ресурсы кластера и возможности виртуальных машин с низким приоритетом и зарезервированных виртуальных машин.

* **Обработка сбоев** — отслеживайте состояние заданий и перезапускайте их в случае сбоев виртуальной машины при выполнении потенциально длительных заданий.

* **Сбор результатов** — легкий доступ к журналам выходных данных, Stdout, Stderr и обученным моделям. Настройте задания Batch AI, чтобы отправлять выходные данные непосредственно в подключенное хранилище.

Как любая служба Azure, Batch AI поддерживает такие распространенные инструменты, как управление доступом на основе ролей (RBAC) и виртуальные сети Azure для обеспечения защиты.
