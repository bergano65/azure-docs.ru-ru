# <a name="tutorial-enable-virtual-networks-integration-and-firewalls-on-event-hubs-namespace"></a>Руководство. Включение интеграции виртуальных сетей и брандмауэров с пространством имен Центров событий

[Конечные точки служб виртуальной сети](../virtual-network/virtual-network-service-endpoints-overview.md) расширяют пространство частных адресов и возможности идентификации вашей виртуальной сети в службах Azure благодаря прямому соединению. Конечные точки позволяют защищать критически важные ресурсы служб Azure в пределах отдельных виртуальных сетей. Трафик, поступающий из виртуальной сети в службу Azure, всегда остается в магистральной сети Microsoft Azure.

Брандмауэры позволяют ограничить доступ к пространству имен Центров событий из определенных IP-адресов (или диапазонов IP-адресов).

В этом руководстве описано, как выполнить интеграцию конечных точек службы виртуальных сетей и настройку брандмауэров (фильтрация IP) с имеющимся пространством имен Центров событий Azure, используя портал.

Из этого руководства вы узнаете, как выполнять такие задачи:
> [!div class="checklist"]
> * Интеграция конечных точек службы виртуальных сетей с пространством имен Центров событий.
> * Настройка брандмауэра (фильтрация IP) с пространством имен Центров событий.

>[!WARNING]
> Реализация интеграции виртуальных сетей может препятствовать взаимодействию других служб Azure с Центрами событий.
>
> Собственные интеграции не поддерживаются, когда включены виртуальные сети.
> Распространенные сценарии Azure, которые не работают с виртуальными сетями:
> * система диагностики Azure и ведение журнала;
> * Azure Stream Analytics
> * интеграция Сетки событий;
> * веб-приложения и Функции должны быть в виртуальной сети;
> * маршруты Центра Интернета вещей;
> * Device Explorer Центра Интернета вещей.


> [!IMPORTANT]
> Виртуальные сети поддерживаются в **стандартном** и **выделенном** уровнях Центра событий. В "базовом" уровне не поддерживается.

Если у вас еще нет подписки Azure, создайте [бесплатную учетную запись][], прежде чем начинать работу.

## <a name="prerequisites"></a>Предварительные требования

Мы будем использовать имеющееся пространство имен Центров событий, поэтому убедитесь, что у вас есть доступное пространство имен. Если доступного пространства имен нет, ознакомьтесь с [этим руководством](./event-hubs-create.md).

## <a name="sign-in-to-the-azure-portal"></a>Вход на портал Azure

Сначала откройте [портал Azure][Azure portal] и выполните вход, используя подписку Azure.

## <a name="select-event-hubs-namespace"></a>Выбор пространства имен Центров событий

В рамках этого руководства мы создали пространство имен Центров событий и перейдем к нему.

## <a name="navigate-to-firewalls-and-virtual-networks-experience"></a>Переход к интерфейсу брандмауэра и виртуальных сетей

Используйте меню навигации в левой области портала, чтобы выбрать параметр **Брандмауэры и виртуальные сети**.

  ![Переход к меню](./media/event-hubs-tutorial-vnet-and-firewalls/vnet-firewall-landing-page.png)

  Выберите переключатель **Все сети** при первом посещении этой страницы. Это означает, что пространство имен Центров событий разрешает все входящие подключения.

## <a name="add-virtual-network-service-endpoint"></a>Добавление конечной точки службы для виртуальной сети

Чтобы ограничить доступ, необходимо интегрировать конечную точку службы для виртуальной сети для этого пространства имен Центров событий.

1. Щелкните переключатель **Выбранные сети** в верхней части страницы, чтобы включить оставшуюся часть страницы с параметрами меню.
  ![выбранные сети](./media/event-hubs-tutorial-vnet-and-firewalls/vnet-firewall-selecting-selected-networks.png)
2. В разделе "Виртуальная сеть" на этой странице выберите параметр ***+Add existing virtual network*** (+Добавить существующую виртуальную сеть). Появится область, где можно будет выбрать созданную виртуальную сеть.
  ![добавление имеющейся виртуальной сети](./media/event-hubs-tutorial-vnet-and-firewalls/vnet-firewall-adding-vnet-from-portal-slide-in-pane.png)
3. Выберите виртуальную сеть из списка, а затем выберите подсеть.
   ![выбор подсети](./media/event-hubs-tutorial-vnet-and-firewalls/vnet-firewall-adding-vnet-from-portal-slide-in-pane-with-subnet-query.png)
4. Прежде чем добавлять виртуальную сеть в список, необходимо включить конечную точку службы. Если конечная точка службы не включена, портал предложит включить ее.
  ![выбор подсети и включение конечной точки](./media/event-hubs-tutorial-vnet-and-firewalls/vnet-firewall-adding-vnet-from-portal-slide-in-pane-after-enabling.png)
    > [!NOTE]
    > Если конечную точку службы не удалось включить, вы можете проигнорировать предупреждение об отсутствии конечной точки службы для виртуальной сети с помощью шаблона ARM. Эта функция недоступна на портале.

5. После включения конечной точки службы по выбранной подсети можно добавить ее в список разрешенных виртуальных сетей.
  ![добавление подсети после включения конечной точки](./media/event-hubs-tutorial-vnet-and-firewalls/vnet-firewall-adding-vnet-from-portal-slide-in-pane-after-adding.png)

6. Нажмите кнопку **Сохранить** в верхней ленте, чтобы сохранить конфигурацию виртуальной сети в службе. Подождите несколько минут, чтобы подтверждение появилось в уведомлениях портала.

## <a name="add-firewall-for-specified-ip"></a>Добавление брандмауэра для указанного IP-адреса

Доступ к пространству имен Центров событий можно ограничить частью диапазона IP-адресов или определенным IP-адресом, используя правила брандмауэра.

1. Щелкните переключатель **Выбранные сети** в верхней части страницы, чтобы включить оставшуюся часть страницы с параметрами меню.
  ![выбранные сети](./media/event-hubs-tutorial-vnet-and-firewalls/vnet-firewall-selecting-selected-networks.png)
2. В разделе **Брандмауэр** в сетке ***Диапазон адресов*** можно добавить один или несколько определенных IP-адресов или их диапазоны.
  ![добавление IP-адресов](./media/event-hubs-tutorial-vnet-and-firewalls/vnet-firewall-adding-firewall.png)
3. После добавления нескольких IP-адресов (или диапазонов IP-адресов) нажмите кнопку **Сохранить** в верхней ленте, чтобы убедиться, что конфигурация сохранилась на стороне службы. Подождите несколько минут, чтобы подтверждение появилось в уведомлениях портала.
  ![добавление IP-адресов и нажатие кнопки "Сохранить"](./media/event-hubs-tutorial-vnet-and-firewalls/vnet-firewall-adding-firewall-hitting-save.png)

## <a name="adding-your-current-ip-address-to-the-firewall-rules"></a>Добавление текущего IP-адреса в правила брандмауэра

1. Вы также можете быстро добавить текущий IP-адрес, установив флажок ***Add your client IP address (YOUR CURRENT IP ADDRESS)*** (Добавить IP-адрес клиента (Текущий IP-адрес)) прямо над сеткой ***Диапазон адресов***.
  ![добавление текущего IP-адреса](./media/event-hubs-tutorial-vnet-and-firewalls/vnet-firewall-adding-current-ip-hitting-save.png)
2. После добавления текущего IP-адреса в правила брандмауэра нажмите кнопку **Сохранить** в верхней ленте, чтобы убедиться, что конфигурация сохранилась на стороне службы. Подождите несколько минут, чтобы подтверждение появилось в уведомлениях портала.
  ![добавление текущего IP-адреса и нажатие кнопки "Сохранить"](./media/event-hubs-tutorial-vnet-and-firewalls/vnet-firewall-adding-current-ip-hitting-save-after-saving.png)

В этом руководстве вы интегрировали конечные точки виртуальной сети и правила брандмауэра с существующим пространством имен Центров событий. Вы научились выполнять следующие задачи:
> [!div class="checklist"]
> * Интеграция конечных точек службы виртуальных сетей с пространством имен Центров событий.
> * Настройка брандмауэра (фильтрация IP) с пространством имен Центров событий.
