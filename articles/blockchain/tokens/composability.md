# <a name="azure-blockchain-tokens-composability"></a>Компонуемости маркеров Azure Блокчейн

[!INCLUDE [Preview note](./includes/preview.md)]

Token компонуемости обеспечивает гибкость создания маркеров для расширенных сценариев. У вас может быть сложный сценарий, который не может быть реализован с помощью [четырех предварительно созданных шаблонов маркеров](templates.md#base-token-types). Token компонуемости позволяет проектировать собственные шаблоны маркеров путем добавления или удаления определенных поведений для создания собственного шаблона токена. При создании нового шаблона токена Azure Блокчейн tokens проверяет все правила грамматики маркеров. Составные шаблоны сохраняются в службе маркеров Azure Блокчейн для выдачи в подключенных сетях блокчейн.

Вы можете использовать [поведение маркеров](templates.md#token-behaviors) в следующих разделах для проектирования шаблона токена.

## <a name="burnable-b"></a>Записываемый (b)

Возможность удаления токенов из источника.

Например, при активации Интернет-точек кредитных карт для подарочной карты записываются точки кредитных карт.

## <a name="delegable-g"></a>Делегируемых (g)

Возможность делегировать действия, выполненные с маркером, который вы владеете.

Делегат может выполнять действия в качестве владельца маркера. Например, можно использовать токен делегируемых, чтобы реализовать голосование. Маркер делегируемых позволяет владельцу токена голосования проголосовать за кого-нибудь другого.

## <a name="logable-l"></a>Логабле (l)

Возможность записи в журнал.

Например, можно выдать маркер логабле для распределения фильма по каждому кинотеатру, отображающему конкретный фильм. Чтобы фильм воспроизводился, на экране должна регистрироваться транзакция для каждого показа, так как выводятся сведения о выплатах за использование во время выполнения выпуска фильма. Сборка Actors может использовать маркеры фильмов для проверки выдачи на каждый из кинотеатров в дистрибутиве.

## <a name="mint-able-m"></a>Mint-возможность (m)

Возможность Mint дополнительные токены для класса Token. Роль Минтер включает в себя поведение минтабле.

Например, розничная компания, которая хочет реализовать программу лояльности, может использовать маркеры минтабле для своей программы лояльности. Они могут Mint дополнительные баллы по программе лояльности клиентов по мере роста своей базы клиентов.  

## <a name="non-subdividable-or-whole-d"></a>Не является разделителем или целым (~ d)

Ограничение, предотвращающее разделение маркера на более мелкие части.

Например, одно художественное Рисование нельзя разделить на несколько меньших частей. 

## <a name="non-transferable-t"></a>Не передается (~ t)

Ограничение для предотвращения смены владельца первоначального маркера.

Например, диплом университета — это непереносимый токен. После того, как диплом передается выпускнику, он не может быть передан другому человеку.

## <a name="roles-r"></a>Роли (r)

Возможность определять роли в классе шаблона токена для конкретных поведений.

Вы можете предоставить список имен ролей, поддерживаемых маркером, во время создания маркера. Если указаны роли, пользователь может назначить роли для этих вариантов поведения. В настоящее время поддерживается только роль Минтер.

## <a name="singleton-s"></a>Одноэлементные элементы

Ограничение, разрешающее предоставление одного маркера.

Например, музей-артефакт является одноэлементным токеном. Артефакты музей являются уникальными. Маркер, представляющий артефакт, содержит только один элемент в блоке.

## <a name="subdividable-d"></a>С разделителями (d)

Возможность разделения маркера на более мелкие части.

Например, доллар можно разделить на центы.

## <a name="transferable-t"></a>Передаваемые (t)

Возможность передавать владение маркером.

Например, заголовок свойства — это передаваемый токен, который может быть передан от одного человека другому при продаже свойства.
