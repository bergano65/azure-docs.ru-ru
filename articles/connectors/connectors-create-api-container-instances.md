# <a name="deploy-and-manage-azure-container-instances-by-using-azure-logic-apps"></a>Развертывание и управление экземплярами контейнеров Azure с помощью Azure Logic Apps

С помощью Azure Logic Apps и соединителя экземпляров контейнеров Azure можно настроить автоматизированные задачи и рабочие процессы, которые развертывают [группы контейнеров](../container-instances/container-instances-container-groups.md)и управляют ими. Соединитель экземпляров контейнера поддерживает следующие действия.

* Создание или удаление группы контейнеров
* Получение свойств группы контейнеров
* Получение списка групп контейнеров
* Получение журналов экземпляра контейнера

Используйте эти действия в приложениях логики для выполнения таких задач, как выполнение рабочей нагрузки контейнера в ответ на триггер Logic Apps. Кроме того, можно использовать и другие действия, выводимые из действий экземпляра контейнера. 

Этот соединитель предоставляет только действия, поэтому для запуска приложения логики используйте отдельный триггер, например триггер **повторения** для выполнения рабочей нагрузки контейнера по обычному расписанию. Или может потребоваться активировать развертывание группы контейнеров после такого события, как поступление сообщения электронной почты Outlook. 

Если вы не знакомы с приложениями логики, ознакомьтесь со статьей [Что такое Azure Logic Apps](../logic-apps/logic-apps-overview.md).

## <a name="prerequisites"></a>Предварительные требования

* Подписка Azure. Если у вас еще нет подписки Azure, [зарегистрируйтесь для получения бесплатной учетной записи Azure](https://azure.microsoft.com/free/). 

* Основные сведения о [создании приложений логики](../logic-apps/quickstart-create-first-logic-app-workflow.md) и [о создании экземпляров контейнеров и управлении ими](../container-instances/container-instances-quickstart.md)

* Приложение логики, в котором вы хотите получить доступ к экземплярам контейнера. Чтобы использовать действие, запустите приложение логики с помощью другого триггера, например триггера **повторения** .

## <a name="add-a-container-instance-action"></a>Добавление действия экземпляра контейнера

[!INCLUDE [Create connection general intro](../../includes/connectors-create-connection-general-intro.md)]

1. Войдите на [портал Azure](https://portal.azure.com) и откройте свое приложение логики в конструкторе приложений логики, если оно еще не открыто.

1. Выберите путь: 

   * На последнем шаге, в котором необходимо добавить действие, выберите **Новый шаг**. 

     -или-

   * Между шагами, в которых вы хотите добавить действие, переместите указатель на стрелку между шагами. 
   Выберите появившийся знак "плюс" ( **+** ), а затем щелкните **Добавить действие**.

1. В поле поиска введите "экземпляр контейнера" в качестве фильтра. В списке действия выберите нужное действие соединителя экземпляров контейнеров Azure.

1. Укажите имя подключения. 

1. Укажите необходимые сведения для выбранного действия и продолжайте создание рабочего процесса приложения логики.

  Например, выберите **создать группу контейнеров** и введите свойства для группы контейнеров и один или несколько экземпляров контейнеров в группе, как показано на следующем рисунке (частичная детализация):

  ![Создание группы контейнеров](./media/connectors-create-api-container-instances/logic-apps-aci-connector.png)
