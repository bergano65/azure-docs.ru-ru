# <a name="reactivate-a-disabled-azure-subscription"></a>Повторная активация отключенной подписки Azure

Подписка Azure может быть отключена, так как срок действия средств на счете истек, достигнута предельная сумма расходов, есть просроченный платеж, исчерпан лимит кредитной карты или администратор учетной записи отменил подписку. Выберите подходящую ситуацию и выполните действия, описанные в этой статье, чтобы активировать подписку повторно.

## <a name="your-credit-is-expired"></a>Истек срок действия денег на счете

При регистрации бесплатной учетной записи Azure вы получите бесплатную пробную версию подписки, которая предоставляет 200 долл. на счете в Azure на 30 дней и 12 месяцев бесплатных служб. Через 30 дней Azure отключит вашу подписку. Это делается, чтобы предотвратить тарификацию за случайное использование денег на счете и бесплатных служб сверх ограничений в рамках подписки. Чтобы продолжить использование служб Azure, необходимо [повысить уровень подписки](upgrade-azure-subscription.md). После обновления ваша подписка по-прежнему имеет доступ к бесплатным службам в течение 12 месяцев. Плата взимается только за использование платных служб и ресурсов.

## <a name="you-reached-your-spending-limit"></a>Достигнута предельная сумма расходов

В подписках Azure с деньгами на счете, таких как бесплатная пробная версия и Visual Studio Enterprise, есть предельные суммы расходов на них. Это означает, что использовать службы можно только в пределах предоставленных денег на счете. По достижении предельной суммы расходов Azure отключает подписку до истечения периода выставления счетов. Это делается, чтобы предотвратить тарификацию за случайное использование денег на счете сверх ограничений в рамках подписки. Дополнительные сведения см. в разделе [Снятие ограничения на предельную сумму расходов](spending-limit.md#remove).

> [!NOTE]
> Если у вас есть бесплатная пробная подписка, после отключения предельной суммы расходов произойдет переход на отдельную подписку с оплатой по мере использования. После создания подписки оставшиеся деньги на счете хранятся в течение полных 30 дней. Кроме того, вам предоставляется доступ к бесплатным службам в течение 12 месяцев.

Сведения о мониторинге и контроле операций выставления счетов для Azure см. в статье [Планирование управления затратами на Azure](../understand/plan-manage-costs.md).


## <a name="your-bill-is-past-due"></a>Счет просрочен

Сведения об устранении этой проблемы см. в статье [Решение проблемы просроченных платежей по подписке Azure после получения электронного сообщения от Azure](resolve-past-due-balance.md).

## <a name="the-bill-exceeds-your-credit-card-limit"></a>Сумма счета превышает баланс кредитной карты

Для решения этой проблемы [выберите другую кредитную карту](change-credit-card.md). Если вы являетесь представителем компании, вы можете [выбрать оплату по выставленному счету](pay-by-invoice.md).

## <a name="the-subscription-was-accidentally-canceled"></a>Подписка случайно отменена

Если вы являетесь администратором учетной записи и случайно отменили отдельную подписку с оплатой по мере использования, ее можно повторно активировать в Центре управления учетной записью.

1. Войдите в [Центр управления учетной записью](https://account.windowsazure.com/Subscriptions).
1. Выберите отмененную подписку.
1. Щелкните **Повторная активация**.

    ![Снимок экрана, на котором показаны ссылки для повторной активации в области справа.](./media/subscription-disabled/reactivate-sub.png)

Для повторной активации подписок других типов [обратитесь в службу поддержки](https://portal.azure.com/?#blade/Microsoft_Azure_Support/HelpAndSupportBlade).

## <a name="after-reactivation"></a>После повторной активации

После повторной активации подписки может возникнуть задержка при создании ресурсов или управлении ими. Если задержка превышает 30 минут, обратитесь за помощью в [службу поддержки Azure по выставлению счетов](https://go.microsoft.com/fwlink/?linkid=2083458). Большинство ресурсов Azure автоматически возобновляют работу и не требуют каких-либо действий. Но мы рекомендуем вам проверить ресурсы служб Azure и перезапустить их, если они не возобновили работу автоматически.
