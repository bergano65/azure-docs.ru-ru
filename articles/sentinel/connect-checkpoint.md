# <a name="connect-check-point-to-azure-sentinel"></a>Подключение точки проверки к Azure Sentinel



В этой статье объясняется, как подключить устройство контрольной точки к Azure Sentinel. Соединитель данных контрольной точки позволяет легко подключать журналы контрольных точек с помощью Azure Sentinel, просматривать панели мониторинга, создавать пользовательские оповещения и улучшать исследование. С помощью контрольной точки в Azure Sentinel вы получите более подробные сведения об использовании Интернета в вашей организации и улучшите возможности ее работы. 

## <a name="forward-check-point-logs-to-the-syslog-agent"></a>Пересылка журналов контрольной точки в агент системного журнала

Настройте устройство контрольной точки для пересылки сообщений Syslog в формате CEF в рабочую область Azure с помощью агента системного журнала.

1. Последовательно выберите пункты [Проверка журнал экспорта](https://aka.ms/asi-syslog-checkpoint-forwarding).
1. Прокрутите вниз до раздела **основное развертывание** и следуйте инструкциям по настройке подключения, следуя приведенным ниже рекомендациям.
   - Задайте для **порта системного журнала** значение **514** или порт, заданный в агенте.
     - Замените **имя** и **IP-адрес целевого сервера** в CLI на имя агента системного журнала и IP-адрес.
     - Задайте формат **CEF**.
1. Если вы используете версию R 77.30 или R 80.10, прокрутите **экран до пункта установки и следуйте** инструкциям по установке средства экспорта журнала для вашей версии.
1. Перейдите к процедуре [Шаг 3. Проверка подключения](connect-cef-verify.md).
