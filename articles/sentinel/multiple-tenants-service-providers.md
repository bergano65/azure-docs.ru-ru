# <a name="manage-multiple-tenants-in-azure-sentinel-as-an-mssp"></a>Управление несколькими клиентами в Azure Sentinel в качестве МССП

Если вы являетесь поставщиком управляемой службы безопасности (МССП) и используете [Azure лигхсаусе](../lighthouse/overview.md) для предоставления службам управления доступом (SoC) для клиентов, вы можете управлять ресурсами-метками клиентов Azure напрямую из собственного клиента Azure без подключения к клиенту клиента. 

## <a name="prerequisites"></a>Предварительные требования

- [Подключение Лигхсаусе Azure](../lighthouse/how-to/onboard-customer.md)

- Для правильной работы клиент (клиент МССП) должен иметь зарегистрированные поставщики ресурсов Sentinel Azure по крайней мере в одной подписке. Кроме того, каждый клиент клиентов должен иметь зарегистрированные поставщики ресурсов. Если вы зарегистрировали метку Azure в клиенте и в своих клиентах, то можете приступить к работе. Чтобы проверить регистрацию, выполните следующие действия.

    1. Выберите **подписки** из портал Azure, а затем выберите соответствующую подписку в меню.

    1. В меню навигации на экране подписки в разделе **Параметры**выберите **поставщики ресурсов**.

    1. Из ** *имени подписки* | На экране поставщики ресурсов** найдите и выберите *Microsoft. OperationalInsights* и *Microsoft. Секуритинсигхтс*и проверьте столбец **состояние** . Если поставщик имеет состояние *зарегистрировано*, выберите **зарегистрировать**.
    
        :::image type="content" source="media/multiple-tenants-service-providers/check-resource-provider.png" alt-text="Проверка поставщиков ресурсов":::

## <a name="how-to-access-azure-sentinel-in-managed-tenants"></a>Как получить доступ к Sentinel Azure в управляемых клиентах

1. В разделе **Каталог + подписка**выберите делегированные каталоги (каталог = клиент) и подписки, в которых находятся рабочие области Sentinel Azure клиента.

    :::image type="content" source="media/multiple-tenants-service-providers/directory-subscription.png" alt-text="Проверка поставщиков ресурсов":::

1. Откройте Azure Sentinel. Вы увидите все рабочие области в выбранных подписках, и вы сможете работать с ними с легкостью, например с любой рабочей областью в вашем клиенте.

> [!NOTE]
> Вы не сможете развернуть соединители в Azure Sentinel из управляемой рабочей области. Чтобы развернуть соединитель, необходимо напрямую войти в клиент, на котором требуется развернуть соединитель, и проверить подлинность с необходимыми разрешениями.
