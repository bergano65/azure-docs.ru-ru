# <a name="connect-your-external-solution-using-syslog"></a>Подключите решение внешних, с помощью системного журнала

> [!IMPORTANT]
> Сейчас Azure Sentinel предоставляется в общедоступной предварительной версии.
> Эта предварительная версия предоставляется без соглашения об уровне обслуживания и не рекомендована для использования рабочей среде. Некоторые функции могут не поддерживаться или их возможности могут быть ограничены. Дополнительные сведения см. в статье [Дополнительные условия использования предварительных выпусков Microsoft Azure](https://azure.microsoft.com/support/legal/preview-supplemental-terms/).

Вы можете подключить любой на локальном устройстве, поддерживающем системный журнал на Azure Sentinel. Это делается с помощью агента на основе машины Linux между устройством и Azure Sentinel. Если компьютер Linux в Azure, можно осуществлять потоковую передачу журналов из устройство или приложение для выделенной рабочей области можно создать в Azure и подключить его. Если компьютер Linux не входит в Azure, можно осуществлять потоковую передачу журналов от устройства к выделенному на локальном компьютере или Виртуальной машины на который вы установите агент для Linux. 

> [!NOTE]
> Если ваше устройство поддерживает CEF системного журнала, подключение становится более полным и следует выбрать этот параметр и следуйте инструкциям в [подключения данных из CEF](connect-common-event-format.md).

## <a name="how-it-works"></a>Принцип работы

Syslog подключения выполняется с помощью агента для Linux. По умолчанию агент для Linux, получает события из управляющей программы системного журнала по протоколу UDP, но в случаях, где ожидается машины Linux для сбора большого количества событий системного журнала, например, когда агент Linux получает события от других устройств, Конфигурация изменена для Использование транспорта TCP между управляющую программу Syslog и агент.

## <a name="connect-your-syslog-appliance"></a>Подключение устройства системного журнала

1. На портале Azure Sentinel выберите **соединители данных** и выберите **Syslog** плитку.
2. Если компьютер Linux не входит в Azure, загрузите и установите Azure Sentinel **агент для Linux** на устройстве. 
1. Если вы работаете в Azure, выберите или создайте виртуальную Машину, в рабочей области Azure Sentinel, предназначенный для получения сообщений системного журнала. Выберите виртуальную Машину в рабочих областях Sentinel Azure и нажмите кнопку **Connect** в верхней части левой панели.
3. Нажмите кнопку **конфигурировать журналы, должен быть подключен** обратно в настройке соединителя системного журнала. 
4. Нажмите кнопку **нажмите здесь, чтобы открыть колонку конфигурации**.
1. Выберите **данных** и затем **Syslog**.
   - Убедитесь, что каждый средством, которое вы отправляете по системного журнала находится в таблице. Для каждого устройства нужно наблюдать, задайте уровень серьезности. Нажмите кнопку **Применить**.
1. Убедитесь, что вы отправляете эти средства только на компьютере системного журнала. 

3. Для соответствующей схемы в Log Analytics можно использовать для журналов системного журнала, поиск **Syslog**.
