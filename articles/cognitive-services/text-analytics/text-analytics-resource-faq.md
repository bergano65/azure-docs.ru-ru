# <a name="frequently-asked-questions-faq-about-the-text-analytics-cognitive-service"></a>Часто задаваемые вопросы об API анализа текста (Cognitive Service)

 Найдите ответы на часто задаваемые вопросы о понятиях, коде и сценариях, связанных с API анализа текста для служб Microsoft Cognitive Services в Azure.

## <a name="can-text-analytics-identify-sarcasm"></a>Умеет ли API анализа текста распознавать сарказм?

Анализ предназначен для распознавания положительных и отрицательных тональностей, а не настроения говорящего.

В анализе тональности всегда сохраняется некоторая степень неточности, но наша модель работает гораздо лучше, когда отсутствуют скрытые значения или подтекст. Передача намерений с использованием иронии, сарказма, юмора и других нюансов зависит от регионального контекста и культурных норм. Такой тип содержимого относится к наиболее сложным для анализа. Обычно наибольшее несоответствие между результатом работы анализатора и субъективной оценки человеком отмечается именно для содержимого со смысловыми нюансами.

## <a name="can-i-add-my-own-training-data-or-models"></a>Могу ли я добавить собственные данные или модели для обучения?

Нет, все модели предоставляются предварительно обученными. Для отправленных данных выполняются только операции оценки, извлечения ключевых фраз и определения языка. Мы не размещаем пользовательские модели. Чтобы создавать и разместить пользовательские модели машинного обучения, изучите [возможности машинного обучения в Microsoft R Server](https://docs.microsoft.com/r-server/r/concept-what-is-the-microsoftml-package).

## <a name="can-i-request-additional-languages"></a>Могу ли я запросить поддержку дополнительных языков?

Анализ тональности и извлечение ключевых фраз доступны для [ограниченного числа языков](text-analytics-supported-languages.md). Обработка естественного языка — это сложная задача, которая должна пройти много проверок, прежде чем будет выпущена в качестве новой функции. По этой причине мы стараемся не объявлять о начале поддержки заранее, чтобы не создавать преждевременных надежд на появление функций, отладка которых может потребовать больше времени, чем ожидалось. 

Чтобы помочь нам в выборе языков, над которыми мы будем работать далее, проголосуйте за определенный язык участвуйте на сайте [User Voice](https://cognitive.uservoice.com/forums/555922-text-analytics). 

## <a name="why-does-key-phrase-extraction-return-some-words-but-not-others"></a>Почему извлечение ключевых фраз возвращает не все, а лишь некоторые слова?

Извлечение ключевых фраз отбрасывает незначимые слова и отдельно стоящие прилагательные. Сочетания "прилагательное+существительное", например "великолепный вид " или "туманная погода" всегда возвращаются вместе.

Обычно выходные данные содержат все существительные и объекты, существующие в предложении. Выходные данные сортируются в порядке убывания важности. Важность оценивается по количеству упоминаний определенных концепций или по связям между элементами в тексте.

## <a name="why-does-output-vary-given-identical-inputs"></a>Почему выходные данные при идентичных входных данных могут различаться?

Усовершенствования моделей и алгоритмов могут объявляться, если они существенные, или незаметно интегрироваться в службу, если изменения считаются незначительными. Вполне вероятно, что через некоторое время для одного и того же текста будут возвращаться другая оценка тональности или другой список ключевых фраз. Это вполне нормальный и ожидаемый результат для облачных ресурсов управляемого машинного обучения.
