# <a name="translator-speech-api-languages"></a>API перевода речи: Languages

[!INCLUDE [Deprecation note](../../../includes/cognitive-services-translator-speech-deprecation-note.md)]

Список языков, поддерживаемых службой "Перевод речи", постоянно пополняется. Этот API позволяет получить набор языков, которые сейчас доступны для использования в службе "Перевод речи".

Примеры кода, демонстрирующие использование API для получения списка доступных языков, см. на [сайте GitHub для Microsoft Translator](https://github.com/MicrosoftTranslator).

## <a name="implementation-notes"></a>Примечания по реализации

### <a name="get-languages"></a>GET /languages

Преобразование речи в текст, перевод распознанного текста и синтез речи из переведенного текста поддерживается для множества языков.

С помощью параметра `scope` клиент может указать, какие наборы языков его интересуют.

* **Преобразование речи в текст.** Используйте параметр запроса `scope=speech`, чтобы получить набор языков, для которых поддерживается преобразование речи в текст.
* **Перевод текста.** Используйте параметр запроса `scope=text`, чтобы получить набор языков, для которых поддерживается перевод расшифрованного текста.
* **Преобразование текста в речь.**  Используйте параметр запроса `scope=tts`, чтобы получить набор языков, для которых поддерживается синтез переведенного текста в голосовую речь.

Клиент может получить одновременно несколько наборов, предоставив нужные варианты в виде списка, разделенного запятыми. Например, `scope=speech,text,tts`.

Успешный ответ содержит объект JSON с установленным свойством для каждого запрошенного набора.

```
{
    "speech": {
        //... Set of languages supported for speech-to-text
    },
    "text": {
        //... Set of languages supported for text translation
    },
    "tts": {
        //... Set of languages supported for text-to-speech
    }
}
```

Так как клиент может использовать параметр запроса `scope`, чтобы выбрать интересующие его языки, фактический ответ может содержать не все свойства, представленные выше.

Для каждого свойства возвращаются указанные ниже значения.

### <a name="speech-to-text-speech"></a>Преобразование речи в текст (speech)

Значение свойства `speech`, связанного с преобразованием речи в текст, представляет собой словарь с парами "ключ-значение". Каждый ключ обозначает язык, поддерживаемый для преобразования речи в текст. Этот ключ служит идентификатором, который клиент передает в API-интерфейс. Значение, связанное с ключом, является объектом со следующими свойствами:

* `name`: отображаемое имя языка.
* `language`: тег языка для соответствующего письменного языка. См. в разделе «Текст транзакции» ниже.
Пример.

```
{
    "speech": {
        "en-US": { name: "English", language: "en" }
    }
}
```

### <a name="text-translation-text"></a>Перевод текста (text)

Значение, связанное со свойством `text`, также представляет собой словарь, в котором каждый ключ определяет поддерживаемый язык для перевода текста. Значение, связанное с ключом, описывает язык:

* `name`: отображаемое имя языка.
* `dir`: направление письма на этом языке, где `rtl` означает "справа налево", а `ltr` — "слева направо".

Пример.

```
{
    "text": {
        "en": { name: "English", dir: "ltr" }
    }
}
```

### <a name="text-to-speech-tts"></a>Преобразование текста в речь (tts)

Значение, связанное со свойством преобразования текста в речь (tts), также представляет собой словарь, в котором каждый ключ определяет поддерживаемый язык. Ниже перечислены атрибуты объекта voice (голос):

* `displayName`: отображаемое имя голоса;
* `gender`: пол голоса (мужской или женский);
* `locale`: тег языка голоса с подтегами для основного языка и региона;
* `language`: тег языка для соответствующего письменного языка.
* `languageName`: отображаемое имя языка.
* `regionName`: отображаемое имя региона для этого языка.

Пример.

```
{
    "tts": {
        "en-US-Zira": {
            "displayName": "Zira",
            "gender": "female",
            "locale": "en-US",
            "languageName": "English",
            "regionName": "United States",
            "language": "en"
        }
}
```

### <a name="localization"></a>Локализация
Эта служба возвращает все имена на языке из заголовка Accept-Language для всех языков, поддерживаемых для перевода текста.

### <a name="response-class-status-200"></a>Класс ответа (состояние 200)
Объект, описывающий набор поддерживаемых языков.

Значение ModelExample:

Langagues { speech (object, optional), text (object, optional), tts (object, optional) }
