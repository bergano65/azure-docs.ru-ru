# <a name="what-is-the-anomaly-detector-api"></a>Что такое API обнаружения аномалий?

API обнаружения аномалий позволяет отслеживать и выявлять аномалии в данных временных рядов с помощью машинного обучения. API обнаружения аномалий адаптируется, автоматическое определение и применение наиболее подгонки моделей к данным, независимо от отрасли, сценария или объем данных. Используя данные временных рядов, API определяет границы для обнаружения аномалий, ожидаемые значения, и какие точки данных расположены аномалии.

![Обнаружение изменений в тенденциях запросов на обслуживание](./media/anomaly_detection2.png)

С помощью средства обнаружения аномалий не требует любой опыта в машинного обучения и REST API позволяет легко интегрировать службу в приложениях и процессах.

## <a name="features"></a>Функции

С помощью средства обнаружения аномалий, можно автоматически обнаруживать аномалии в данных временных рядов, или по мере их появления в режиме реального времени. 

|Функция  |ОПИСАНИЕ  |
|---------|---------|
|Обнаруживать аномалии в реальном времени. | Обнаруживать аномалии в данных потоковой передачи с помощью ранее доступ к данным для определения того, является ли ваш последнюю из них аномалий. Эта операция приводит к возникновению ошибки модели с помощью точек данных, отправки и определяет, является ли целевой точки аномалии. Путем вызова API с каждой новая точка данных, которые вы создаете, можно отслеживать данные, так как оно создается. |
|Обнаруживать аномалии на протяжении всего набора данных в пакетном режиме. | Используйте ваши временных рядов для обнаружения аномалий, которые могут существовать на протяжении всего ваши данные. Эта операция приводит к возникновению ошибки модели с помощью данных всей временных рядов, с каждой точкой, проанализировать с помощью той же модели.         |
| Получите дополнительные сведения о данных. | Получите полезные сведения о данных и наблюдаемые аномалий, включая ожидаемые значения, границы аномалий и позиций. |
| Настройка границ обнаружения аномалий. | API обнаружения аномалий автоматически создаваться границы для обнаружения аномалий. Настройте эти границы, чтобы увеличить или уменьшить API чувствительности к аномалии, а наиболее подходящими для ваших данных. |

## <a name="demo"></a>Демонстрация

Чтобы быстро приступить к использованию API обнаружения аномалий, попробуйте [интерактивную демонстрацию](https://notebooks.azure.com/AzureAnomalyDetection/projects/anomalydetector) , которые выполняются в браузере. В этой демонстрации выполняется в записной книжке Jupyter веб сервере и показано, как отправить запрос API и визуализировать результат.

Чтобы выполнить пример, выполните следующие действия:

1. Получите действительный ключ подписки API обнаружения аномалий и конечную точку API. В следующем разделе содержатся инструкции по регистрации. 
2. Войдите и щелкните клон, в правом верхнем углу.
3. Нажмите кнопку **запускать на свободных вычислительных ресурсов**
4. Выберите один из записных книжек для этого примера.
5. Добавьте допустимый ключ подписки API обнаружения аномалий для `subscription_key` переменной. Изменение `endpoint` переменной к конечной точке. Например: `https://westus2.api.cognitive.microsoft.com`
1. В верхней строке меню, щелкните **ячейки**, затем **запустить все**.

## <a name="workflow"></a>Рабочий процесс

API обнаружения аномалий — это веб-службу RESTful, что позволяет легко вызывать из любого языка программирования, можно внести HTTP-запросы и анализ JSON.

[!INCLUDE [cognitive-services-anomaly-detector-data-requirements](../../../includes/cognitive-services-anomaly-detector-data-requirements.md)]

[!INCLUDE [cognitive-services-anomaly-detector-signup-requirements](../../../includes/cognitive-services-anomaly-detector-signup-requirements.md)]

После регистрации:

1. Получить данные временных рядов и преобразовать его в допустимом формате JSON. Используйте [рекомендации](concepts/anomaly-detection-best-practices.md) при подготовке данных для получения наилучших результатов.
1. Отправка запроса к API обнаружения аномалий с данными.
1. Обработайте ответ API путем анализа возвращенного сообщения JSON.
