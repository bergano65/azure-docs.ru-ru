# <a name="face-recognition-concepts"></a>Основные понятия распознавания лиц

В этой статье объясняются концепции проверки, поиска похожих, групповых и выявляет операций распознавания лиц и базовых структур данных. В общем, Распознавание описывает работу по сравнению двух разных лиц, чтобы определить, похожи ли они или принадлежат одному лицу.

## <a name="recognition-related-data-structures"></a>Структуры данных, связанные с распознаванием

Операции распознавания используют главным образом следующие структуры данных. Эти объекты хранятся в облаке, и на них могут ссылаться строки ИДЕНТИФИКАТОРов. Строки ИДЕНТИФИКАТОРов всегда уникальны в пределах подписки. Поля имени могут быть повторяющимися.

## <a name="recognition-operations"></a>Операции распознавания

В этом разделе подробно описано, как четыре операции распознавания используют описанные ранее структуры данных. Общее описание каждой операции распознавания см. в разделе [Обзор](../Overview.md).

### <a name="verify"></a>Проверка

Операция [проверки](https://westus.dev.cognitive.microsoft.com/docs/services/563879b61984550e40cbbe8d/operations/563879b61984550f3039523a) принимает идентификатор лица из Детектедфаце или персистедфаце, а также другой идентификатор лица или объект Person и определяет, принадлежат ли они одному и тому же человеку. Если передать объект Person, при необходимости можно передать Персонграуп, к которому относится этот человек, чтобы повысить производительность.

### <a name="find-similar"></a>Поиск похожих лиц

Операция [Найти похожие](https://westus.dev.cognitive.microsoft.com/docs/services/563879b61984550e40cbbe8d/operations/563879b61984550f30395237) принимает идентификатор лица из Детектедфаце или персистедфаце и либо фацелист, либо массив других идентификаторов лиц. С помощью Фацелист Возвращает меньшее Фацелист сторон, которые похожи на заданную грань. При использовании массива идентификаторов лиц он также возвращает массив меньшего размера.

### <a name="group"></a>Группа

Операция [Group](https://westus.dev.cognitive.microsoft.com/docs/services/563879b61984550e40cbbe8d/operations/563879b61984550f30395238) принимает массив идентификаторов из ассортимента из Детектедфаце или персистедфаце и возвращает те же идентификаторы, которые сгруппированы в несколько массивов меньшего размера. Каждый массив "groups" содержит идентификаторы лиц, которые выглядят аналогичными. Один массив "Мессиграуп" содержит идентификаторы лиц, для которых не найдено сходство.

### <a name="identify"></a>Выявление

Операция [идентификации](https://westus.dev.cognitive.microsoft.com/docs/services/563879b61984550e40cbbe8d/operations/563879b61984550f30395239) принимает один или несколько идентификаторов лиц из Детектедфаце или Персистедфаце и персонграуп и возвращает список объектов Person, к которым может принадлежать каждая лицо. Возвращенные объекты Person упаковываются как объекты-кандидаты, имеющие значение достоверности прогноза.

## <a name="input-data"></a>Входные данные

Используйте следующие советы, чтобы убедиться, что входные изображения дают наиболее точные результаты распознавания:

* Поддерживаются форматы входных изображений JPEG, PNG, GIF (первый кадр), BMP.
* Размер файла изображения должен быть не больше 4 МБ.
* При создании объектов Person используйте фотографии, которые имеют различные виды углов и освещения.
* Некоторые лица могут не распознаться из-за технических проблем, например:
  * Образы с экстремальным освещением, например серьезное появление.
  * Препятствия, блокирующие один или оба глаза.
  * Различия в типе волосы или волосы.
  * Изменение внешнего вида лица из-за возраста.
  * Выражения с крайними лицами.
