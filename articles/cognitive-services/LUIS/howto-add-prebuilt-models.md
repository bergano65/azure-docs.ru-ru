# <a name="add-prebuilt-models-for-common-usage-scenarios"></a>Добавление предварительно созданных моделей для распространенных сценариев использования

LUIS включает набор предварительно созданных моделей для быстрого добавления типичных, диалоговых пользовательских сценариев. Это быстрый и простой способ добавления возможностей в клиентское приложение без необходимости проектирования моделей для этих возможностей.

## <a name="add-a-prebuilt-domain"></a>Добавление готовой предметной области

1. Войдите на [портал Luis](https://www.luis.ai)и выберите свою **подписку** и ресурс для **разработки** , чтобы просмотреть приложения, назначенные этому ресурсу для разработки.
1. Откройте приложение, выбрав его имя на странице " **Мои приложения** ".

1. Выберите предварительно **созданные домены** на левой панели инструментов.

1. Найдите домен, который нужно добавить в приложение, а затем нажмите кнопку **Добавить домен** .

    > [!div class="mx-imgBorder"]
    > ![Добавление предварительно созданного домена календаря](./media/luis-prebuilt-domains/add-prebuilt-domain.png)

## <a name="add-a-prebuilt-intent"></a>Добавление предварительно созданного намерения

1. Войдите на [портал Luis](https://www.luis.ai)и выберите свою **подписку** и ресурс для **разработки** , чтобы просмотреть приложения, назначенные этому ресурсу для разработки.
1. Откройте приложение, выбрав его имя на странице " **Мои приложения** ".

1. На странице **«** цели» выберите добавить предварительно **подготовленный доменное намерение** на панели инструментов над списком «намерения».

1. Выберите цель во всплывающем диалоговом окне.

    > [!div class="mx-imgBorder"]
    > ![Добавление предварительно созданного намерения](./media/luis-prebuilt-domains/add-prebuilt-domain-intents.png)

1. Нажмите кнопку **Готово**.

## <a name="add-a-prebuilt-entity"></a>Добавить предварительно созданную сущность
1. Войдите на [портал Luis](https://www.luis.ai)и выберите свою **подписку** и ресурс для **разработки** , чтобы просмотреть приложения, назначенные этому ресурсу для разработки.
1. Откройте приложение, выбрав его имя на странице " **Мои приложения** ".
1. Выберите **сущности** в левой части.

1. На странице **сущности** выберите добавить предварительно **созданную сущность**.

1. В диалоговом окне Добавление предварительно **подготовленных сущностей** выберите предварительно созданную сущность.

    > [!div class="mx-imgBorder"]
    > ![Диалоговое окно добавления предварительно созданной сущности](./media/luis-prebuilt-domains/add-prebuilt-entity.png)

1. Нажмите кнопку **Готово**. После добавления сущности не требуется проводить обучение приложения.

## <a name="add-a-prebuilt-domain-entity"></a>Добавление предварительно созданной доменной сущности
1. Войдите на [портал Luis](https://www.luis.ai)и выберите свою **подписку** и ресурс для **разработки** , чтобы просмотреть приложения, назначенные этому ресурсу для разработки.
1. Откройте приложение, выбрав его имя на странице " **Мои приложения** ".
1. Выберите **сущности** в левой части.

1. На странице **сущности** выберите добавить предварительно **созданную сущность домена**.

1. В диалоговом окне Добавление предварительно **подготовленных моделей домена** выберите предварительно созданную сущность домена.

1. Нажмите кнопку **Готово**. После добавления сущности не требуется проводить обучение приложения.

## <a name="publish-to-view-prebuilt-model-from-prediction-endpoint"></a>Опубликовать для предварительно созданной модели из конечной точки прогнозирования

Самый простой способ просмотреть значение предварительно построенной модели — запрос из опубликованной конечной точки.

## <a name="entities-containing-a-prebuilt-entity-token"></a>Сущности, содержащие предварительно собранный токен сущности

Если у вас есть сущность машинного обучения, которая требует обязательного компонента предварительно созданной сущности, добавьте подсущность в сущность машинного обучения, а затем добавьте _необходимый_ компонент предварительно созданной сущности.
