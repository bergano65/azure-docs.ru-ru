# <a name="speaker-recognition-api---preview"></a>API Распознавания говорящего — предварительная версия

API Распознавания говорящего — это облачные API, которые обеспечивают эффективные алгоритмы ИИ для идентификации и проверки говорящего. Распознавание говорящего разделяется на две категории: проверка говорящего и идентификация говорящего.

## <a name="speaker-verification"></a>Проверка говорящего

У голоса есть уникальные характеристики, которые моно соотнести с отдельным пользователем.  Приложения могут использовать голос в качестве дополнительного фактора для проверки в таких сценариях, как центры обработки вызовов и веб-службы.

API Проверки говорящего служат интеллектуальным средством для проверки пользователей по голосовым и речевым парольным фразам.

### <a name="enrollment"></a>Регистрация

Регистрация для проверки говорящего основана на тексте, то есть говорящему необходимо выбрать определенную парольную фразу для использования во время регистрации и проверки.

На этапе регистрации говорящего записывается голос, произносящий конкретную фразу. При распознавании выбранной фразы извлекаются голосовые характеристики, чтобы сформировать уникальную голосовую подпись. Вместе эти данные о регистрации говорящего будут использоваться для проверки говорящего. Данные о регистрации говорящего хранятся в защищенной системе. Клиент определяет, как долго они должны храниться. Клиенты могут создавать, обновлять и удалять данные о регистрации для отдельных говорящих через вызовы API.  При удалении подписки все данные о регистрации говорящего, связанные с подпиской, также будут удалены.

Клиенты должны убедиться в получении соответствующих разрешений от пользователей для проверки говорящего.

### <a name="verification"></a>Проверка

На этапе проверки клиент должен вызвать API проверки говорящего с использованием идентификатора, связанного с конкретным пользователем, которого необходимо проверить.  Служба извлекает голосовые характеристики и парольную фразу из входной записи речи. Затем она сравнивает характеристики с соответствующими элементами данных о регистрации говорящего, которого хочет проверить клиент, и определяет соответствия.  Ответ возвращает значение "Принять" или "Отклонить" с различными уровнями достоверности.  Затем клиент определяет, как использовать результаты, чтобы решить, является ли этот пользователь зарегистрированным говорящим.

Пороговый уровень достоверности следует задать в зависимости от сценария и других используемых факторов проверки. Мы рекомендуем поэкспериментировать с уровнем достоверности и выбрать соответствующие параметры для каждого применения. Интерфейсы API не предназначены для определения того, откуда исходит звук: от реального пользователя, от имитации или записи зарегистрированного говорящего.

Служба не сохраняет запись речи или извлеченные голосовые характеристики, которые отправляются в службу на этапе проверки.

Для получения дополнительных сведений о проверке говорящего, пожалуйста, обратитесь к API [Говорящий — Проверка](https://westus.dev.cognitive.microsoft.com/docs/services/563309b6778daf02acc0a508/operations/563309b7778daf06340c9652).

## <a name="speaker-identification"></a>Идентификация говорящего

Приложения могут использовать голос для распознавания говорящего в группе зарегистрированных говорящих. API Идентификации говорящего можно использовать в таких сценариях, как эффективность проведения встречи, персонализация и расшифровка в центре обработки вызовов.

### <a name="enrollment"></a>Регистрация

Регистрация для идентификации говорящего не зависит от текста, то есть не важно, что именно говорит пользователь в аудиозаписи. Парольная фраза не требуется.

На этапе регистрации записывается голос говорящего и извлекаются голосовые характеристики для формирования уникальной голосовой подписи. Звук речи и извлеченные характеристики хранятся в защищенной системе. Клиент определяет, как долго они будут храниться. Клиенты могут создавать, обновлять и удалять эти данные о регистрации говорящего для отдельных говорящих через вызовы API. При удалении подписки все данные о регистрации говорящего, связанные с подпиской, также будут удалены.

Клиенты должны убедиться в получении соответствующих разрешений от пользователей для идентификации говорящего.

### <a name="identification"></a>Идентификация

На этапе идентификации служба идентификации говорящего извлекает голосовые характеристики из записи речи. Затем она сравнивает характеристики с данными о регистрации указанного списка говорящих. При обнаружении соответствия с зарегистрированным говорящим в ответе возвращается идентификатор говорящего с уровнем достоверности.  Если ни один из говорящих не соответствует зарегистрированному говорящему, в ответе возвращается "Отклонить".

Пороговый уровень достоверности следует задать в зависимости от сценария. Мы рекомендуем поэкспериментировать с уровнем достоверности и выбрать соответствующие параметры для каждого применения. Интерфейсы API не предназначены для определения того, откуда исходит звук: от реального пользователя, от имитации или записи зарегистрированного говорящего.

Служба не сохраняет запись речи или извлеченные голосовые характеристики, которые отправляются в службу на этапе проверки идентификации.

Дополнительные сведения об идентификации говорящего см. в  [справочнике по API распознавания говорящего](https://westus.dev.cognitive.microsoft.com/docs/services/563309b6778daf02acc0a508/operations/5645c068e597ed22ec38f42e).
