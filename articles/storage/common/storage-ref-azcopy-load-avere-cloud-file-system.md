# <a name="azcopy-load-clfs"></a>azcopy load clfs

Передает локальные данные в контейнер и сохраняет их в формате файловой системы Microsoft Авере Cloud FileSystem (CLFS).

## <a name="synopsis"></a>Краткий обзор

Команда Load копирует данные в контейнеры хранилища BLOB-объектов Azure, а затем сохраняет их в формате файловой системы Microsoft Авере Cloud FileSystem (CLFS). Собственный формат CLFS используется кэшем Azure HPC и Авере Вфкст для продуктов Azure.

Чтобы использовать эту команду, установите необходимое расширение с помощью команды: pip3 Install клфслоад ~ = 1.0.23. Убедитесь, что CLFSLoad.py находится в вашем пути. Дополнительные сведения об этом шаге см. в статье [https://aka.ms/azcopy/clfs](https://aka.ms/azcopy/clfs) .

Эта команда является простым вариантом перемещения существующих данных в облачное хранилище для использования с конкретными продуктами для кэширования высокопроизводительных вычислений Майкрософт. 

Так как эти продукты используют собственный формат файловой системы для управления данными, эти данные не могут быть загружены с помощью собственной команды копирования. 

Вместо этого данные должны быть загружены с помощью самого продукта кэша или с помощью этой команды Load, которая использует правильный собственный формат.
Эта команда позволяет передавать данные без использования кэша. Например, чтобы предварительно заполнить хранилище или добавить файлы в рабочий набор, не увеличивая нагрузку на кэш.

Назначение — это пустой контейнер хранилища Azure. После завершения перемещения целевой контейнер можно использовать с экземпляром кэша Azure HPC или Авере Вфкст для кластера Azure.

> [!NOTE] 
> Это предварительный выпуск команды Load. Сообщите о проблемах в репозитории GitHub AzCopy.

```
azcopy load clfs [local dir] [container URL] [flags]
```

## <a name="examples"></a>Примеры

Загрузка всего каталога в контейнер с SAS в формате CLFS:

```azcopy
azcopy load clfs "/path/to/dir" "https://[account].blob.core.windows.net/[container]?[SAS]" --state-path="/path/to/state/path"
```

## <a name="options"></a>Параметры

**--Compression-Type** строка укажите тип сжатия, используемый для передачи. Доступные значения: `DISABLED` , `LZ4` . (по умолчанию `LZ4` )

**--**    Справка по `azcopy load clfs` команде.

**--строка уровня журнала** определяет уровень детализации журнала для файла журнала, доступные уровни:,, `DEBUG` `INFO` `WARNING` , `ERROR` . (по умолчанию `INFO` )

**--Max-Errors** UInt32 укажите максимальное число сбоев при переносе. При возникновении достаточного количества ошибок немедленно прервите выполнение задания.

**--New-Session**   Запустите новое задание, а не продолжите его, в котором хранятся данные отслеживания `--state-path` . (по умолчанию true)

**--Preserve-жестких связей**    Сохранение связей с жесткими связями.

**--State — строка пути** , в которой требуется указать путь к локальному каталогу для отслеживания состояния задания. Путь должен указывать на существующий каталог, чтобы возобновить задание. Для нового задания оно должно быть пустым.
