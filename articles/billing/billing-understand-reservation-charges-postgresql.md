# <a name="how-a-reservation-discount-is-applied-to-azure-database-for-postgresql-single-server"></a>Как скидка на резервирование применяется к отдельному серверу Базы данных Azure для PostgreSQL

После того как вы купили зарезервированную емкость отдельного сервера Базы данных Azure для PostgreSQL, скидка на резервирование автоматически применяется к базам данных отдельных серверов PostgreSQL, атрибуты и количество которых совпадают с резервированием. Резервирование охватывает только затраты на вычисление для отдельного сервера Базы данных Azure для PostgreSQL. С вас взимается плата за хранение и сеть по обычным тарифам. 

## <a name="how-reservation-discount-is-applied"></a>Применение скидки на резервирование

Скидка на резервирование ***сгорает, если не используется***. Поэтому, если у вас не существует подходящих ресурсов в течение некоторого часа, вы потеряете количество ресурсов, зарезервированных на этот час. Вы не можете переносить неиспользуемые зарезервированные часы.</br>

При завершении работы ресурса скидка на резервирование автоматически переносится на другой подходящий ресурс в указанной области резервирования. Если в указанной области не найдены подходящие ресурсы, зарезервированные часы теряются.

## <a name="discount-applied-to-azure-database-for-postgresql-single-server"></a>Скидка, применяемая к Базе данных Azure для PostgreSQL (отдельный сервер)

Скидка на зарезервированную емкость отдельного сервера Базы данных Azure для PostgreSQL применяется к работающему отдельному серверу PostgreSQL на почасовой основе. Приобретенное резервирование должно соответствовать потреблению вычислительных ресурсов работающим отдельным сервером Базы данных для PostgreSQL. Если отдельные серверы PostgreSQL работают менее часа, резервирование автоматически применяется к другому отдельному серверу Базы данных Azure для PostgreSQL, соответствующему атрибутам резервирования. Скидка может применяться к отдельным серверам Базы данных Azure для PostgreSQL, работающим параллельно. Если у вас нет отдельного сервера PostgreSQL, работающего в течение целого часа и соответствующего атрибутам резервирования, вы не получите полную скидку на резервирование за этот час.

В следующих примерах показано применение скидки на зарезервированную емкость отдельного сервера Базы данных Azure для PostgreSQL в зависимости от количества приобретенных вами ядер и времени их работы.

* **Пример 1**: Вы приобретаете зарезервированную емкость отдельного сервера Базы данных Azure для PostgreSQL с 8 виртуальными ядрами. Если вы используете отдельный сервер Базы данных Azure для PostgreSQL с 16 виртуальными ядрами, который соответствует остальным атрибутам резервирования, вы оплачиваете цену по модели с оплатой по мере использования за потребленные вычислительные ресурсы отдельным сервером PostgreSQL с 8 виртуальными ядрами и получаете скидку на резервирование за один час использования вычислений отдельным сервером PostgreSQL с 8 виртуальными ядрами.</br>

В остальных примерах предполагается, что вы покупаете зарезервированную емкость для отдельного сервера Базы данных Azure для PostgreSQL с 16 виртуальными ядрами, а остальные атрибуты резервирования соответствуют работающим отдельными серверам PostgreSQL.

* **Пример 2**. Вы запускаете два отдельных сервера Базы данных Azure для PostgreSQL с 8 виртуальными ядрами на час. Скидка на резервирование емкости с 16 виртуальными ядрами применяется к использованию вычислений для обоих отдельных серверов Базы данных Azure для PostgreSQL с 8 виртуальными ядрами.

* **Пример 3**: Вы запускаете один отдельный сервер Базы данных Azure для PostgreSQL с 16 виртуальными ядрами с 13.00 до 13.30. Вы запускаете другой отдельный сервер Базы данных Azure для PostgreSQL с 16 виртуальными ядрами с 13.30 до 14.00. Для обеих Баз данных будет применяться скидка на резервирование.

* **Пример 4**. Вы запускаете один отдельный сервер Базы данных Azure для PostgreSQL с 16 виртуальными ядрами с 13.00 до 13.45. Вы запускаете другой отдельный сервер Базы данных Azure для PostgreSQL с 16 виртуальными ядрами с 13.30 до 14.00. За превышение работы на 15 минут с вас взимается оплата по мере использования. Скидка на резервирование применяется к использованию вычислительных ресурсов за остальное время.

Чтобы больше узнать о применении резервирований Azure и просмотреть отчеты о начислении платы за использование, ознакомьтесь со статьей [Общие сведения об использовании зарезервированных экземпляров Azure с Соглашением о регистрации Enterprise](https://docs.microsoft.com/azure/billing/billing-understand-reserved-instance-usage-ea).
