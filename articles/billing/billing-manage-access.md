# <a name="manage-access-to-billing-information-for-azure"></a>Управление доступом к данным для выставления счетов в Azure

Для большинства подписок доступ к сведениям о выставлении счетов можно предоставлять членам команды через раздел **Подписки** на портале Azure. Если вы являетесь клиентом Azure с соглашением Enterprise и выполняете функции корпоративного администратора, вы можете предоставлять разрешения администраторам отделов и (или) владельцам учетной записи на корпоративном портале.

## <a name="give-access-to-billing"></a>Предоставление доступа к выставлению счетов

Все клиенты, не использующие соглашение Enterprise, могут предоставлять доступ к данным для выставления счетов путем назначения членам команды одной из следующих ролей пользователя:

- Администратор учетной записи
- Администратор служб
- Соадминистратор
- Владелец.
- участник;
- Читатель
- Читатель счетов

Сведения о назначении ролей см. в статье [Управление доступом с помощью RBAC и портала Azure](../role-based-access-control/role-assignments-portal.md).

Эти роли имеют доступ к данным для выставления счетов на [портале Azure](https://portal.azure.com/). Также сотрудники с этими ролями могут использовать [API выставления счетов](billing-usage-rate-card-overview.md) для получения счетов-фактур и сведений о потреблении программным способом. Дополнительные сведения см. в статье [о ролях в Azure RBAC](../role-based-access-control/built-in-roles.md).

### <a name="opt-in"></a> Предоставление пользователям разрешений на скачивание счетов

После назначения ролей членам команды нужно, чтобы администратор учетной записи включил доступ к скачиванию счетов на портале Azure. Все счета, созданные до декабря 2016 года, доступны только администратору учетной записи.

1. Войдите на [портале Azure](https://portal.azure.com/).
1. Являясь администратором учетной записи, выберите свою подписку в [колонке подписок](https://portal.azure.com/#blade/Microsoft_Azure_Billing/SubscriptionsBlade) на портале Azure.

1. Выберите **Счета** и **Access to invoices** (Доступ к счетам).

    ![На снимке экрана показано, как делегировать доступ к счетам](./media/billing-manage-access/AA-optin.png)

1. Выберите **On** (Включено) и сохраните изменения.

    ![На снимке экрана показано, как включать и выключать делегирование доступа к счету](./media/billing-manage-access/AA-optinAllow.png)

Администратор учетной записи также может настроить отправку счетов по электронной почте. Дополнительные сведения см. в статье [Получение счета по электронной почте](billing-download-azure-invoice-daily-usage-date.md).

## <a name="give-read-only-access-to-billing"></a>Предоставление доступа только для чтения к выставлению счетов

Если пользователю нужен доступ к сведениям о счетах в подписке, но не требуется управлять службами Azure, назначьте ему роль читателя счетов. Эта роль хорошо подходит для сотрудников организации, которые отвечают за управление бюджетами и затратами для подписок Azure.

Если вы используете соглашение Enterprise, владелец учетной записи или администратор отдела могут назначать участникам команды роль читателя счетов. Но для того, чтобы читатель счетов получил доступ к сведениям для выставления счетов по отделу или всей учетной записи, корпоративный администратор предприятия должен включить политики **Возможность просмотра затрат для владельца учетной записи включена** или **Возможность просмотра затрат для администратора отдела включена** на корпоративном портале.

Функция чтения счетов находится в предварительной версии и пока не поддерживает неглобальные облака.

1. Войдите на [портале Azure](https://portal.azure.com/).
1. Выберите свою подписку в [колонке подписок](https://portal.azure.com/#blade/Microsoft_Azure_Billing/SubscriptionsBlade) на портале Azure.

1. Выберите **Управление доступом (IAM)**.
1. Выберите **Назначения ролей**, чтобы просмотреть все назначения ролей для этой подписки.
1. Выберите **Добавить назначение ролей**.
1. В раскрывающемся списке **Роль** выберите **Читатель счетов**.
1. В текстовом поле **Выбрать** введите имя или адрес электронной почты пользователя, которого вы хотите добавить.
1. Выберите пользователя.
1. Щелкните **Сохранить**.
1. Через несколько секунд выбранному пользователю будет назначена роль читателя счетов в области действия подписки.
1. Читатель счетов получит по электронной почты сообщение со ссылкой для входа.

    ![Снимок экрана, показывающий, что читатель счетов может видеть на портале Azure](./media/billing-manage-access/billing-reader-view.png)

## <a name="allow-department-administrator-or-account-owner-billing-access"></a>Разрешение доступа к выставлению счетов для администратора отдела или владельца учетной записи

Корпоративный администратор может разрешить администраторам отдела и владельцу учетной записи просматривать сведения о потреблении и расходах, связанных с отделами или учетными записями, которыми они управляют.

1. Войдите на [портал EA](https://ea.azure.com/) от имени корпоративного администратора.
1. Выберите **Управление**
1. В разделе **Регистрация** укажите для параметра **Возможность просмотра затрат для администратора отдела включена** значение **Включено**, чтобы администратор отдела смог просматривать сведения о потреблении и затратах.
1. Укажите для параметра **Возможность просмотра затрат для владельца учетной записи включена** значение **Включено**, чтобы владелец учетной записи смог просматривать сведения о потреблении и затратах.


Дополнительные сведения см. в статье [Understand Azure Enterprise Agreement administrative roles in Azure](billing-understand-ea-roles.md) (Сведения об административных ролях в Azure для соглашения Azure Enterprise).

## <a name="only-account-admins-can-access-account-center"></a>Только администраторы учетной записи могут получить доступ к центру управления учетной записью

Администратор учетной записи является полноправным владельцем подписки. По умолчанию пользователь, который зарегистрировал или приобрел подписку Azure, является администратором учетной записи, если только [право владения подпиской не было передано](billing-subscription-transfer.md) кому-то другому. Администратор учетной записи может создавать и отменять подписки, менять адрес выставления счетов и политики доступа для подписки, используя [Центр управления учетной записью](https://account.azure.com/Subscriptions).
