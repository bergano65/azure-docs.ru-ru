# <a name="policy-settings-for-web-application-firewall-with-azure-front-door"></a>Параметры политики для брандмауэра веб-приложения с помощью передовой Azure

Политика брандмауэра веб-приложения (WAF) позволяет управлять доступом к веб-приложений, набор правил, пользовательские и управляемые. Имя политики WAF должен быть уникальным. При попытке использовать имя существующей, вы получите ошибку проверки. Существует несколько параметров уровня политики, применимые для всех правил, заданных для этой политики, как описано в этой статье.

## <a name="waf-state"></a>Состояние WAF

Политику WAF для входной двери может находиться в одном из следующих двух состояний:
- **Включено**. Если политика включена, WAF активно проверяет входящие запросы и выполняет соответствующие действия в соответствии с определениями правил
- **Отключено:** — при отключении политики проверки WAF приостановлена. Входящие запросы будут обходить WAF и отправляются в серверные решения на основе маршрута двери.

## <a name="waf-mode"></a>Режим WAF

WAF политики можно настроить для запуска в двух режимов:

- **Режим обнаружения** при запуске в режиме обнаружения, WAF не принимать никаких мер, отличный от монитора и записи запроса и его из этих правил WAF журналы WAF. Включить ведение журнала диагностики для входной двери (Если вы используете портал, это можно сделать, выбрав **диагностики** на портале Azure).

- **Режим предотвращения** при настроен для работы в режиме предотвращения, предпринимаемые WAF указанное действие, если запрос соответствует правилу. Любые соответствующие запросы также записываются в журналы WAF.

## <a name="waf-response-for-blocked-requests"></a>Ответ WAF для заблокированных запросов

По умолчанию, когда WAF блокирует запрос из-за соответствующих правило, она возвращает код состояния 403- **запрос блокирован** сообщения. Строка ссылки также возвращается для ведения журнала.

Можно определить настраиваемый ответ код состояния и сообщение ответа при запрос блокируется WAF. Поддерживаются следующие коды пользовательских состояний:

- 200 ОК
- «403 запрещено»
- 405 Метод запрещен
- Страница недоступна 406 приемлемым
- 429 — слишком много запросов

Настраиваемый ответ сообщение о состоянии кода и ответ — это параметр уровня политики. После настройки, все заблокированные запросы получите такое же состояние настраиваемый ответ и ответное сообщение.

## <a name="uri-for-redirect-action"></a>URI для перенаправления действия

Вы должны определить URI для перенаправления запросов, если **ПЕРЕНАПРАВЛЕНИЯ** выбрано действие для любого из правила, содержащиеся в политике WAF. Это перенаправление URI должен быть действительным сайтом HTTP (S) и после настройки все запросы, правила сопоставления с помощью действия «REDIRECT» будут перенаправляться в указанном сайте.
