# <a name="geo-filtering-on-a-domain-for-azure-front-door"></a>Географическая фильтрация в домене для передней дверцы Azure

По умолчанию Передняя дверца Azure будет отвечать на все запросы пользователей независимо от расположения, из которого поступает запрос. В некоторых сценариях может потребоваться ограничить доступ к веб-приложению по странам или регионам. Служба брандмауэра веб-приложения (WAF) в передней дверце позволяет определить политику с помощью настраиваемых правил доступа для конкретного пути в конечной точке, чтобы разрешить или заблокировать доступ из указанных стран или регионов. 

Политика WAF содержит набор настраиваемых правил. Правило состоит из условий соответствия, действия и приоритета. В условии соответствия определяются переменная сопоставления, оператор и значение match. Для правила геофильтрации переменная соответствия REMOTE_ADDR, оператором является географическое соответствие, а значение — это 2-значный интересующий код страны или региона. Код страны "ZZ" или "Unknown" захватывает IP-адреса, которые еще не сопоставлены со страной в нашем наборе данных. Вы можете добавить ZZ в условие соответствия, чтобы избежать ложных срабатываний. Можно объединить условие геосоответствия и условие соответствия REQUEST_URI строки, чтобы создать правило географической фильтрации на основе пути. 


Вы можете настроить политику географической настройки для своей передней дверцы с помощью [Azure PowerShell](front-door-tutorial-geo-filtering.md) или [шаблона](https://github.com/Azure/azure-quickstart-templates/tree/master/101-front-door-geo-filtering)быстрого запуска.
