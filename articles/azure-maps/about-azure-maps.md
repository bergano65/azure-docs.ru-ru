# <a name="what-is-azure-maps"></a>Что такое Azure Maps?

Azure Maps — это коллекция геопространственных служб, открепленных актуальными данными сопоставления, доступными для предоставления географического контекста веб-приложениям и мобильным приложениям. Службы Azure Maps состоят из REST API для отображения **карт** в нескольких стилях и снимков со спутника, **поиска** адресов, мест и точек интереса по всему миру; **маршрутизации** подключения "точка — точка", нескольких точек, оптимизации нескольких точек, изохрона, коммерческого транспортного средства, оказания влияния на трафик и матричную маршрутизацию; просмотра лучших в своем классе потоков трафика и инцидентов, установления местоположения пользователя посредством **геолокации** и конвертации расположения в **часовых поясах**, как и выбора времени в расположении. Кроме этого, Azure Maps предоставляют службы для хранилища **геозон**, **данных** карт, в которых находится информация о расположении размещения в Azure, и **пространственных операций**, чтобы предоставить аналитику расположения с помощью геопространственной аналитики. Службы Azure Maps доступны непосредственно как REST API или с помощью надежного **веб-пакета SDK** или **пакета SDK для Android**. Эти инструменты позволяют разработчикам быстро найти и масштабировать решения, которые интегрируют сведения о расположении из облака Azure в решения Azure. Уже сегодня зарегистрируйтесь для бесплатной [учетной записи в Azure Maps](https://azure.microsoft.com/services/azure-maps/) и начинайте разработку.

В следующем видео подробно объясняется принцип действия Azure Maps.

## <a name="map-controls"></a>Элементы управления картами

### <a name="web-sdk"></a>веб-пакет SDK.

Веб-пакет SDK Azure Maps позволяет настраивать интерактивные карты, используя собственные изображения и содержимое для отображения в веб-приложениях или мобильных приложениях. Этот элемент управления использует WebGL, позволяя вам отображать большие наборы данных с высокой производительностью. Вы можете выполнять разработку с помощью пакета SDK, используя JavaScript или TypeScript.

![Веб-пакет SDK Azure Maps](media/about-azure-maps/Introduction_WebMapControl.png)

### <a name="android-sdk"></a>Android SDK

Пакет SDK для Android в Azure Maps позволяет создавать мощные мобильные приложения карт. 

![Пакет SDK для Android в Azure Maps](media/about-azure-maps/AndroidSDK.png)

## <a name="services-in-azure-maps"></a>Службы в Azure Maps

Служба Azure Maps состоит из следующих шести служб, которые могут предоставить географический контекст для приложений Azure.

### <a name="render-service"></a>Служба отрисовки

Служба отрисовки предназначена для разработчиков, создающих веб-приложения и мобильные приложения для картографирования. Служба использует либо высококачественные растровые графические изображения, доступные в 19 уровнях масштабирования, либо полностью настраиваемых изображениях в векторном формате.

![Map.png: служба "Карты Azure"](media/about-azure-maps/Introduction_Map.png)

Служба отрисовки теперь предоставляет предварительную версию API-интерфейсов, чтобы разработчики могли работать со спутниковыми изображениями. Дополнительные сведения см. в статье [Render](https://docs.microsoft.com/rest/api/maps/render) (Служба отрисовки).

### <a name="route-service"></a>Служба построения маршрутов

Служба построения маршрутов содержит надежные геометрические вычисления инфраструктуры и направлений в реальных условиях для нескольких режимов движения. Эта служба позволяет разработчикам вычислять направления для нескольких режимов движения, в том числе на автомобиле, грузовике, велосипеде или пешком. Служба также может учитывать входные данные, например условия трафика, ограничения массы или транспортировку опасных материалов.

![Служба "Карты Azure": Route.png](media/about-azure-maps/Introduction_Route.png)

Служба построения маршрутов теперь предлагает предварительную версию расширенных возможностей, в том числе пакетной обработки запросов нескольких маршрутов, матриц времени и расстояния между исходными показателями и местами назначения, а также поиск маршрутов или расстояний для путешествий, исходя из времени или запаса топлива. Дополнительные сведения о возможностях маршрутизации см. в статье [Route](https://docs.microsoft.com/rest/api/maps/route) (Маршрут).

### <a name="search-service"></a>Служба поиска

Служба поиска предназначена для разработчиков. Она позволяет искать адреса, расположения, деловые справочники по имени или категории и другую географическую информацию. Служба поиска также может применять [обратное геокодирование](https://en.wikipedia.org/wiki/Reverse_geocoding) к адресам и пересечениям улиц на основе широты и долготы.

![Служба "Карты Azure": Search.png](media/about-azure-maps/Introduction_Search.png)

Служба поиска также предоставляет дополнительные возможности поиска по маршруту, поиска на более широкой территории, запросы пакетных групп поиска, а также поиск более крупной зоны вместо точки расположения. API-интерфейсы пакетной службы и область поиска в настоящее время доступны в предварительной версии. Дополнительные сведения о возможности поиска см. на странице [Search](https://docs.microsoft.com/rest/api/maps/search) (Поиск).

### <a name="time-zone-service"></a>Служба часовых поясов

Служба часовых поясов позволяет запрашивать сведения о текущем, хронологическом и будущем часовом поясах с использованием пар "широта — долгота" или [идентификатора IANA](https://www.iana.org/). Служба часовых поясов также позволяет преобразовывать идентификаторы часовых поясов Microsoft Windows в часовые пояса IANA, получать данные о смещении часовых поясов в формате UTC и о текущем времени в соответствующем часовом поясе. Обычный ответ JSON на запрос к службе часовых поясов выглядит следующим образом.

```JSON
{
    "Version": "2017c",
    "ReferenceUtcTimestamp": "2017-11-20T23:09:48.686173Z",
    "TimeZones": [{
        "Id": "America/Los_Angeles",
        "ReferenceTime": {
            "Tag": "PST",
            "StandardOffset": "-08:00:00",
            "DaylightSavings": "00:00:00",
            "WallTime": "2017-11-20T15:09:48.686173-08:00",
            "PosixTzValidYear": 2017,
            "PosixTz": "PST+8PDT,M3.2.0,M11.1.0"
        }
    }]
}
```

Дополнительные сведения об этой службе см. на странице [Timezone](https://docs.microsoft.com/rest/api/maps/timezone) (Часовой пояс).

### <a name="traffic-service"></a>Служба отслеживания дорожного движения

Служба трафика представляет собой набор веб-служб для разработчиков, позволяющий создавать веб-приложения и мобильные приложения, требующие трафика. Эта служба предоставляет два типа данных.

* "Поток транспорта": сведения о скорости и времени проезда в реальном времени для всех ключевых дорог в сети.
* "Проблемы на дороге": точное представление о пробках и ДТП на дороге в сети.

![Трафик в службе "Карты Azure"](media/about-azure-maps/Introduction_Traffic.png)

Посетите страницу [Traffic](https://docs.microsoft.com/rest/api/maps/traffic) (Трафик), чтобы получить дополнительные сведения.

### <a name="ip-to-location"></a>Определение расположения по IP-адресу

Служба сопоставления IP-адресов и расположений позволяет просматривать полученный двухбуквенный код страны для заданного IP-адреса. Эта служба поможет вам оптимизировать и повысить удобство работы пользователей путем размещения содержимого настраиваемых приложений исходя из географического расположения.

Сведения об использовании интерфейсов REST API для службы определения расположения по IP-адресу см. [здесь](https://docs.microsoft.com/rest/api/maps/geolocation).

## <a name="programming-model"></a>Модель программирования

Azure Maps разрабатываются для мобильности и могут использоваться в приложениях на различных платформах. Служба использует модель программирования, которая не влияет на язык и поддерживает выходные данные JSON через [интерфейсы REST API](https://docs.microsoft.com/rest/api/maps/).

Кроме того, служба Azure Maps предлагает удобный [элемент управления картой JavaScript](https://docs.microsoft.com/javascript/api/azure-maps-control) с простой моделью программирования для быстрой и простой разработки как веб-приложений, так и мобильных приложений.

## <a name="usage"></a>Использование

Для доступа к службам Azure Maps нужно перейти на [портал Azure](https://portal.azure.com) и создать учетную запись службы Azure Maps.

Служба Azure Maps использует схему проверки подлинности на основе ключа. Учетная запись поставляется с двумя предварительно созданными ключами. Интегрируйте эти возможности использования географического расположения в приложение с помощью ключа или с помощью запроса в службу Azure Maps.

## <a name="supported-regions"></a>Поддерживаемые регионы

API-интерфейс Azure Maps сейчас доступен во всех странах за исключением следующих:

* Аргентина
* Китай
* Индия
* Марокко
* Пакистан
* Южная Корея

Убедитесь, что текущее расположение вашего IP-адреса не относится к одной из неподдерживаемых стран, перечисленных выше.
