# <a name="choose-the-right-pricing-tier-in-azure-maps"></a>Выбор подходящей ценовой категории в Azure Maps

Для Azure Maps предлагается две ценовых категории. Цель этой статьи — помочь вам выбрать правильную ценовую категорию, которая соответствует вашим потребностям. Задайте себе следующие два вопроса, которые помогут вам выбрать правильную ценовую категорию.

## <a name="what-geospatial-capabilities-do-i-plan-to-use"></a>Какие геопространственные возможности планируется использовать?
Если вы считаете, что основные геопространственные модули API соответствуют требованиям службы, тогда вам подходит ценовая категория S0. Если вам нужны более продвинутые возможности для приложения, рассмотрите возможность выбора ценовой категории S1. Примером возможностей являются площадь плюс гибридные образы, получение диапазона маршрута и пакетное геокодирование. Таблица **Возможности ценовой категории** помогает лучше представить потребности вашего приложения. С ее помощью вы можете выбрать ценовую категорию, наиболее подходящую для вашего приложения.

## <a name="how-many-concurrent-users-do-i-plan-to-support"></a>Сколько одновременно работающих пользователей планируется поддерживать? 
Ценовые категории S0 и S1 обрабатывают различные объемы пропускной способности данных. Прежде чем выбрать ценовую категорию Azure Maps, задайте себе несколько вопросов. Например: Сколько одновременно работающих пользователей планируется поддерживать? Ценовая категория S0 обрабатывает до **50 запросов в секунду**. Ценовая категория S1 обрабатывает **более 50 запросов в секунду**.

### <a name="pricing-tier-capabilities"></a>Возможности ценовых категорий

| Функция                              |        S0           |  S1      |
|-----------------------------------------|:-------------------:|:--------:|
| поиска                                  |        ✓           |     ✓    |
| Маршрутизация                                 |        ✓           |     ✓    |
| Отображение                                  |        ✓           |     ✓    |
| Трафик                                 |        ✓           |     ✓    |
| Часовые пояса                              |        ✓           |     ✓    |
| * Снимки плюс гибридные снимки (предварительная версия)  |        ✓           |     ✓    |
| * Диапазон маршрута (предварительная версия)                  |        ✓           |     ✓    |
| * Расположение IP-адреса 2 (предварительная версия)                |        ✓           |     ✓    |
| * Многоугольники из поиска (предварительная версия)         |        ✓           |     ✓    |
| * Пакетное геокодирование (предварительная версия)              |        ✓           |     ✓    |
| * Пакетная маршрутизация (предварительная версия)                |        ✓           |     ✓    |
| * Матричная маршрутизация (предварительная версия)               |        ✓           |     ✓    |

\* Доступ к этим возможностям с помощью ценовой категории S0 перестанет поддерживаться после 4 февраля 2019 года.

Следует обратить внимание на такие дополнительные точки данных.
* Какое у вас предприятие?
* Насколько критично строится приложение?

Ознакомьтесь с таблицей **ценовых категорий для целевых клиентов**, чтобы лучше понять смысл ценовых категорий S0 и S1. Дополнительные сведения см. на странице [Цены на Azure Maps](https://azure.microsoft.com/pricing/details/azure-maps/). 

### <a name="pricing-tier-targeted-customers"></a>Целевые клиенты ценовой категории

| Ценовой уровень  |     Целевые клиенты                                                                |
|---------------|:-----------------------------------------------------------------------------------------|
| S0            |    <p>Ценовая категория S0 предназначена для клиентов-предприятий небольшого и среднего размера. Этот ценовой уровень подходит вам, если вы не ожидаете большого количества одновременных пользователей. А также если основные геопространственные API, показанные в предыдущей таблице, соответствуют требованиям службы. Этот уровень находится в общем доступе. Он применяется для приложений на всех этапах производства: от подтверждения концепции и ранних стадий тестирования до вывода приложения в рабочую среду и развертывания.<p>|
| S1            |    <p>Ценовая категория S1 предназначена для клиентов, которым необходима поддержка крупномасштабных критически важных корпоративных приложений, большого количества одновременно работающих пользователей. Она также доступна для тех клиентов, которым необходимы дополнительные геопространственные службы.</p>|
