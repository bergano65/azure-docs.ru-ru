# <a name="firmware-security"></a>Безопасность встроенного по
В этой статье описывается, как корпорация Майкрософт обеспечивает защиту облачных экосистем оборудования и цепочек поставок.

## <a name="securing-the-cloud-hardware-ecosystem"></a>Обеспечение безопасности облачной экосистемы оборудования
Активные партнеры корпорации Майкрософт в экосистеме облачных оборудования для обеспечения непрерывной безопасности:

- Совместная работа с партнерами по оборудованию и микропрограммному обеспечению Azure (например, производителям компонентов и системным интеграторам) соответствует требованиям к безопасности оборудования и встроенного по Azure.

- Предоставление партнерам возможности проводить непрерывную оценку и улучшение безопасности продуктов с помощью определенных корпорацией Майкрософт требований в таких областях, как:

  - Безопасная загрузка встроенного по
  - Безопасное восстановление встроенного по
  - Безопасное обновление встроенного по
  - Шифрование встроенного по
  - Заблокированное оборудование
  - Детальная телеметрии отладки
  - Системная поддержка аппаратного обеспечения TPM 2,0 для включения измеряемой загрузки

- Участие в проекте безопасности [Open COMPUTE Project (OCP)](https://www.opencompute.org/wiki/Security) и вклад в разработку спецификаций. Спецификации обеспечивают согласованность и ясность для безопасного проектирования и архитектуры в экосистеме.

   > [!NOTE]
   > Примером нашего вклада в проект безопасности OCP является [аппаратная спецификация безопасной загрузки](https://docs.google.com/document/d/1Se1Dd-raIZhl_xV3MnECeuu_I0nF-keg4kqXyK4k4Wc/edit#heading=h.5z2d7x9gbhk0) .

## <a name="securing-hardware-and-firmware-supply-chains"></a>Обеспечение безопасности цепочек поставок оборудования и встроенного по
Поставщики и поставщики облачных оборудования для Azure также должны придерживаться процессов безопасности цепочки и требований, разработанных корпорацией Майкрософт. Процессы разработки и развертывания оборудования и микропрограммного обеспечения должны соответствовать процессам [жизненного цикла разработки безопасности](https://www.microsoft.com/securityengineering/sdl) Майкрософт (SDL), таким как:

- Моделирование угроз
- Безопасная проверка проектирования
- Проверка встроенного по и тестирование на проникновение
- Защита сред сборки и тестирования
- Управление уязвимостью системы безопасности и реагирование на инциденты
