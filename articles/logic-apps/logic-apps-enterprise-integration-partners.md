# <a name="add-trading-partners-to-integration-accounts-for-azure-logic-apps"></a>Добавление торговых партнеров в учетные записи интеграции для Azure Logic Apps

В [Azure Logic Apps](../logic-apps/logic-apps-overview.md)можно создавать автоматизированные рабочие процессы интеграции "бизнес-бизнес" (B2B) с помощью [учетной записи интеграции](../logic-apps/logic-apps-enterprise-integration-create-integration-account.md) с приложениями логики. Чтобы представить вашу организацию и других пользователей, вы создаете и добавляете торговые партнеры в качестве артефактов в учетную запись интеграции. Партнеры — это сущности, которые участвуют в транзакциях B2B и обмениваются сообщениями друг с другом.

Перед созданием этих партнеров необходимо обсудить и поделиться информацией с партнерами о том, как выявление и проверка сообщений, отправляемых другими участниками. После того как вы согласны с этими сведениями, вы можете создавать партнеров в учетной записи интеграции.

## <a name="partner-roles-in-integration-accounts"></a>Роли партнеров в учетных записях интеграции

Чтобы определить сведения об обмене сообщениями с партнерами, создайте и добавьте [соглашения](../logic-apps/logic-apps-enterprise-integration-agreements.md) в качестве артефактов в учетную запись интеграции. Для соглашений требуются по крайней мере два партнера в учетной записи интеграции. Ваша организация всегда является *ведущим партнером* в соглашении. Организация, которая обменивается сообщениями с вашей организацией, является *гостевым партнером*. Гостевой партнер может быть другой компанией или даже штатом в вашей организации. Добавив этих партнеров, можно приступать к созданию соглашения.

В соглашении указываются сведения для обработки входящих и исходящих сообщений с точки зрения партнера узла. Для входящих сообщений **Параметры получения** указывают, каким способом главный партнер получает сообщения от гостевого партнера в соглашении. Для исходящих сообщений **Параметры отправки** указывают, каким способом главный партнер отправляет сообщения гостевому партнеру.

## <a name="prerequisites"></a>Технические условия

* Подписка Azure. Если у вас еще нет подписки Azure, [зарегистрируйтесь для получения бесплатной учетной записи Azure](https://azure.microsoft.com/free/).

* [Учетная запись интеграции](../logic-apps/logic-apps-enterprise-integration-create-integration-account.md) для хранения партнеров, соглашений и других артефактов B2B. Эта учетная запись интеграции должна быть связана с подпиской Azure.

## <a name="create-partner"></a>Создание партнера

1. Войдите на [портале Azure](https://portal.azure.com).

1. В главном меню Azure выберите **Все службы**. В поле поиска введите "Интеграция" и выберите **учетные записи интеграции**.

   ![Выберите "учетные записи интеграции".](./media/logic-apps-enterprise-integration-partners/find-integration-accounts.png)

1. В разделе **Учетные записи интеграции** выберите учетную запись интеграции, в которую нужно добавить партнеров.

   ![Выбор учетной записи интеграции](./media/logic-apps-enterprise-integration-partners/select-integration-account.png)

1. Выберите плитку **Партнеры**.

   ![Выбор плитки "Партнеры"](./media/logic-apps-enterprise-integration-partners/choose-partners.png)

1. В разделе **Партнеры** выберите **Добавить**. В разделе **Добавление партнера**укажите сведения о партнере, как описано в таблице ниже.

   ![Выберите "Добавить" и укажите сведения о партнере](./media/logic-apps-enterprise-integration-partners/add-partners.png)

   | Свойство | Обязательно для заполнения | Описание |
   |----------|----------|-------------|
   | **Имя** | ДА | Имя партнера |
   | **Квалификатор** | ДА | Основной текст для проверки подлинности, который предоставляет организациям уникальные бизнес-удостоверения, например **D-U-N-S (Dun & Bradstreet)** . <p>Партнеры могут выбрать взаимно определенную бизнес-идентификацию. Для этих **сценариев выберите параметр** для EDIFACT или **взаимного определения (X12)** для X12. <p>Для RosettaNet выберите только **Дунс**, который является стандартом. |
   | **Значение** | ДА | Значение, идентифицирующее документы, получаемые приложениями логики. <p>Для RosettaNet это значение должно быть четырехзначным числом, которое соответствует номеру ДУНС. |
   ||||

   > [!NOTE]
   > Для партнеров, использующих RosettaNet, можно указать дополнительные сведения, сначала создав этих партнеров, а затем [отредактировав их позже](#edit-partner).

1. Когда все будет готово, нажмите кнопку **ОК**.

   Теперь ваш новый партнер появится в списке **партнеров** . Кроме того, плитка **партнеры** обновляет текущее число партнеров.

   ![Новый партнер](./media/logic-apps-enterprise-integration-partners/new-partner.png)

## <a name="edit-partner"></a>Изменение данных партнера

1. На [портале Azure](https://portal.azure.com) найдите и выберите свою учетную запись интеграции.
Выберите плитку **Партнеры**.

   ![Выбор плитки "Партнеры"](./media/logic-apps-enterprise-integration-partners/edit.png)

1. В разделе **партнеры**выберите партнера, которого нужно изменить, и нажмите кнопку **изменить**. В разделе **изменить**внесите изменения.

   ![Внесение и сохранение изменений](./media/logic-apps-enterprise-integration-partners/edit-partner.png)

   Для RosettaNet в разделе **Свойства партнера RosettaNet**можно указать следующие дополнительные сведения:

   | Свойство | Обязательно для заполнения | Описание |
   |----------|----------|-------------|
   | **Классификация партнеров** | Нет | Тип организации партнера |
   | **Код цепочки поставок** | Нет | Код цепочки поставок партнера, например "информационные технологии" или "электронные компоненты" |
   | **Имя контакта** | Нет | Имя контактного лица партнера |
   | **Электронная почта** | Нет | Адрес электронной почты партнера |
   | **Fax** | Нет | Номер факса партнера |
   | **Телефонов** | Нет | Номер телефона партнера |
   ||||

1. Когда все будет готово, нажмите кнопку **ОК** , чтобы сохранить изменения.

## <a name="delete-partner"></a>Удаление партнера

1. На [портале Azure](https://portal.azure.com) найдите и выберите свою учетную запись интеграции. Выберите плитку **Партнеры**.

   ![Выбор плитки "Партнеры"](./media/logic-apps-enterprise-integration-partners/choose-partners-to-delete.png)

1. В разделе **Партнеры** выберите партнера, которого нужно удалить. Щелкните **Удалить**.

   ![Удаление партнера](./media/logic-apps-enterprise-integration-partners/delete-partner.png)
