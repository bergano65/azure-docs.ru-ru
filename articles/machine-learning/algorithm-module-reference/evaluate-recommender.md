# <a name="evaluate-recommender"></a>Ознакомление с рекомендацией

В этой статье описывается, как использовать модуль "Оценка рекомендаций" в конструкторе Машинное обучение Azure (Предварительная версия). Целью является измерение точности прогнозов, выполненных моделью рекомендаций. С помощью этого модуля можно оценивать различные виды рекомендаций:  
  
-   Оценки, прогнозируемые для пользователя и элемента    
-   Элементы, Рекомендуемые для пользователя  
  
При создании прогнозов с использованием модели рекомендаций для каждого из поддерживаемых типов прогнозов возвращается несколько разных результатов. Модуль рекомендации по оценке выводит тип прогноза из формата столбца оцененного набора данных. Например, оцененный набор данных может содержать:

- тройные элементы оценки пользовательских элементов
- Пользователи и их Рекомендуемые элементы

Модуль также применяет соответствующие метрики производительности в зависимости от типа выполняемого прогноза. 

  
## <a name="how-to-configure-evaluate-recommender"></a>Настройка рекомендаций по оценке

Модуль рекомендации по оценке сравнивает выходные данные прогноза с помощью модели рекомендаций с соответствующими данными «Земля». Например, модуль [рекомендуемого SVDа оценки](score-svd-recommender.md) позволяет получить оцененные наборы данных, которые можно проанализировать с помощью инструкции Evaluate рекомендует.

### <a name="requirements"></a>Требования

Для вычисления рекомендаций в качестве входных данных требуются следующие наборы данных. 
  
#### <a name="test-dataset"></a>Тестовый набор данных

Тестовый набор данных содержит данные "Земля" в виде тройных оценок пользовательских элементов.  

#### <a name="scored-dataset"></a>Оцененный набор данных

Оцененный набор данных содержит прогнозы, созданные моделью рекомендаций.  
  
Столбцы в этом втором наборе данных зависят от типа прогноза, который был выполнен во время процесса оценки. Например, оцененный набор данных может содержать один из следующих элементов:

- Пользователи, элементы и оценки, которые пользователь скорее всего предоставит для элемента
- Список пользователей и элементов, рекомендуемых для них 

### <a name="metrics"></a>Метрики

Метрики производительности для модели создаются на основе типа входных данных. Сведения см. в следующих разделах.

## <a name="evaluate-predicted-ratings"></a>Оценка прогнозируемых оценок  

При оценке прогнозируемых оценок набор данных с оценками (второй вход для вычисления рекомендуется) должен содержать тройные показатели элементов, которые соответствуют этим требованиям:
  
-   Первый столбец набора данных содержит идентификаторы пользователей.    
-   Второй столбец содержит идентификаторы элементов.  
-   Третий столбец содержит соответствующие оценки пользовательских элементов.  
  
> [!IMPORTANT] 
> Чтобы оценка прошла, имена столбцов должны быть `User`, `Item`и `Rating`соответственно.  
  
Оцените рекомендацию сравнивает оценки в наборе данных "Земля" с прогнозируемыми рейтингами оцененного набора данных. Затем вычисляется средняя абсолютная ошибка (MAE) и корневое среднее значение ошибки (корень СРЕДНЕКВАДРАТИЧНОЙ ПОГРЕШНОСТИ).



## <a name="evaluate-item-recommendations"></a>Оценка рекомендаций по элементам

При оценке рекомендаций по элементам используйте оцененный набор данных, который включает Рекомендуемые элементы для каждого пользователя:
  
-   Первый столбец набора данных должен содержать идентификатор пользователя.    
-   Все последующие столбцы должны содержать соответствующие рекомендованные идентификаторы элементов, упорядоченные по релевантности элемента пользователю. 

Перед подключением к этому набору данных рекомендуется отсортировать набор данных, чтобы сначала приступить к наиболее релевантным элементам.  

> [!IMPORTANT] 
> Для работы с рекомендациями по оценке необходимо, чтобы имена столбцов были `User`, `Item 1`, `Item 2`, `Item 3` и т. д.  
  
Метод Evaluate рекомендует вычислить среднее нормализованное вычисление со скидкой (НДКГ) и возвратить его в выходной набор данных.  
  
Так как невозможно определить фактическую истинность для рекомендуемых элементов, при оценке рекомендуется использовать рейтинги пользовательских элементов в тестовом наборе данных, что дает выигрыш в вычислении НДКГ. Чтобы оценить, модуль оценки рекомендаций должен создать только рекомендации для элементов с рейтингом "Земля" (в тестовом наборе данных).
