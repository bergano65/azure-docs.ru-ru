# <a name="track-the-progress-of-data-science-projects"></a>Отслеживать ход выполнения проектов обработки и анализа данных

Руководители групп обработки и анализа данных, руководители групп и руководители проектов могут отслеживать ход выполнения своих проектов.  Руководители хотят узнать, какая работа выполнена, кто проработал и какая работа остается.   Управление ожиданиями — важный элемент успеха.

## <a name="azure-devops-dashboards"></a>Панели мониторинга Azure DevOps

Если вы используете Azure DevOps, вы можете создавать панели мониторинга для наблюдения за действиями и рабочими элементами, связанными с данным проектом Agile. Дополнительные сведения о панелях мониторинга см. в разделе [панели мониторинга, отчеты и мини](/azure/devops/report/dashboards/)-приложения.

Инструкции по созданию и настройке панелей мониторинга и мини-приложений в Azure DevOps см. в следующих кратких руководствах:

- [Добавление панелей мониторинга и управление ими](/azure/devops/report/dashboards/dashboards)
- [Добавление мини-приложений на панель мониторинга](/azure/devops/report/dashboards/add-widget-to-dashboard)

## <a name="example-dashboard"></a>Пример панели мониторинга

Ниже приведен простой пример панели мониторинга, который отслеживает действия спринта для проекта гибкой обработки и анализа данных, включая количество фиксаций в связанных репозиториях. 

- Плитка **отсчета** показывает количество дней, остающихся в текущем спринте. 

- В двух **фрагментах кода** показано количество фиксаций в двух репозиториях проекта за последние семь дней. 

- **Рабочие элементы для проекта клиента TDSP** отображает результаты запроса для всех рабочих элементов и их состояний. 

- **Общая схема потока** (CFD) показывает количество закрытых и активных рабочих элементов.

- На **диаграмме «выработка** » показано, что все еще нужно завершить работу в течение оставшегося времени спринта.

- На **диаграмме выгорание** показана завершенная работа по сравнению с общим объемом работы в спринте.
