# <a name="options"></a>Параметры #

## <a name="setting-options"></a>Значения параметра ##

Код внедрения Проигрыватель мультимедиа Azure — это просто тег видео HTML5, поэтому во многих параметрах можно использовать стандартные атрибуты тега для установки параметров.

`<video controls autoplay ...>`

Кроме того, можно использовать атрибут настройки данных для предоставления параметров в формате [JSON](http://json.org/example.html) . Кроме того, здесь можно задать параметры, которые не являются стандартными для тега Video.

`<video data-setup='{ "controls": true, "autoplay": false }'...>`

Наконец, если вы не используете атрибут настройки данных для активации программы установки проигрывателя, можно передать объект с параметрами проигрывателя в качестве второго аргумента в функции установки JavaScript.

`amp("vid1", { "controls": true, "autoplay": false });`

> [!NOTE]
> Параметры в конструкторе устанавливаются только при первой инициализации до того, как будет задан источник.  Если вы хотите изменить параметры для того же инициализированного элемента Проигрыватель мультимедиа Azure, необходимо обновить параметры перед изменением источника. Параметры в JavaScript можно обновить с помощью `myPlayer.options({/*updated options*/});` . Обратите внимание, что только измененные параметры будут затронуты, все остальные параметры, заданные ранее, будут сохранены.

## <a name="individual-options"></a>Отдельные параметры ##

> [!NOTE]
>Атрибуты видео могут иметь только значение true или false (логический). просто добавьте атрибут (не знак равенства), чтобы включить его, или исключите его, чтобы отключить. Например, чтобы включить элементы управления: НЕПРАВИЛЬНая `<video controls="true" ...>` `<video controls ...>` самая крупная проблема, с которой работают люди, пытается установить для этих значений значение false, используя false в качестве значения (например, Controls = "false"), которое фактически делает противоположным и устанавливает значение true, так как атрибут по-прежнему включен.

### <a name="controls"></a>controls ###

Параметр Controls определяет, есть ли у проигрывателя элементы управления, с которыми может взаимодействовать пользователь. Без управления единственным способом запуска воспроизведения видео является атрибут автозапуска или API.

`<video controls ...>` или `{ "controls": true }`

### <a name="autoplay"></a>autoplay ###

Если автозапуск имеет значение true, воспроизведение видео начнется сразу же после загрузки страницы (без взаимодействия с пользователем).

> [!NOTE]
> Этот параметр не поддерживается мобильными устройствами, такими как Windows Phone, Apple iOS и Android. Мобильные устройства блокируют функцию автозапуска, чтобы предотвратить использование планов месячных данных потребителя (часто ресурсоемких). Для запуска видео в этом случае требуется пользовательское касание или щелчок.

`<video autoplay ...>`ни `{ "autoplay": true }`

### <a name="poster"></a>Плакат ###
Атрибут афиши задает изображение, отображаемое до начала воспроизведения видео. Часто это кадр видеоролика или пользовательского окна приветствия. Как только пользователь нажмет кнопку Воспроизведение, вы откинете этот образ.

`<video poster="myPoster.jpg" ...>` или `{ "poster": "myPoster.jpg" }`

### <a name="width"></a>width ###

Атрибут width задает отображаемую ширину видео.

`<video width="640" ...>` или `{ "width": 640 }`

### <a name="height"></a>рост ###

Атрибут Height задает отображаемую высоту видео.

`<video height="480" ...>` или `{ "height": 480 }`

### <a name="plugins"></a>plugins ###

Модули JSON подключаемых модулей определяют, какие подключаемые модули будут загружены с помощью этого экземпляра AMP, позволяя настраивать любые параметры, которые может иметь подключаемый модуль.

   `<video... data-setup='{plugins: { "contentTitle": {"name": "Azure Medi Services Overview"}}}'...>`

Дополнительные сведения о разработке и использовании подключаемых модулей см. в разделе [Создание подключаемых модулей](azure-media-player-writing-plugins.md) .

### <a name="other-options"></a>другие параметры ###

Другие параметры для тега можно задать `<video>` с помощью `data-setup` параметра, который принимает JSON.
`<video ... data-setup='{"nativeControlsForTouch": false}'>`

#### <a name="nativecontrolsfortouch"></a>нативеконтролсфортауч ####

Для этого параметра явно задано значение false. Если установить значение false, то оболочка Проигрыватель мультимедиа Azure будет отображаться одинаково на разных платформах.  Кроме того, в противоположность имени, сенсорный ввод по-прежнему будет включен.

### <a name="fluid"></a>Гидродинамика ###

Установив для этого параметра значение true, элемент видео будет иметь полную ширину родительского контейнера, а высота будет скорректирована в соответствии с видео со стандартным пропорциями 16:9.

`<video ... data-setup='{"fluid": true}'>`

`fluid` параметр переопределяет `width` явные `height` Параметры и. Этот параметр доступен только в Проигрыватель мультимедиа Azure версии `2.0.0` и более поздних версиях.

### <a name="playbackspeed"></a>плайбаккспид ###

`playbackSpeed` параметр управляет Плайбаккспид элементом управления и набором параметров скорости воспроизведения, доступных для пользователя. `playbackSpeed` принимает объект. Чтобы включить управление скоростью воспроизведения на панели управления, свойство `enabled` объекта должно иметь значение true. Пример включения скорости воспроизведения в разметке:

`<video ... data-setup='{"playbackSpeed": {"enabled": true}}'>`


Другие свойства `playbackSpeed` параметра задаются объектом [плайбаккспидоптионс](/javascript/api/azuremediaplayer/playbackspeedoptions) .

Пример настройки параметров скорости воспроизведения в JavaScript:

```javascript
    var myPlayer = amp('vid1', {
        fluid: true,
        playbackSpeed: {
            enabled: true,
            initialSpeed: 1.0,
            speedLevels: [
                { name: "x4.0", value: 4.0 },
                { name: "x3.0", value: 3.0 },
                { name: "x2.0", value: 2.0 },
                { name: "x1.75", value: 1.75 },
                { name: "x1.5", value: 1.5 },
                { name: "x1.25", value: 1.25 },
                { name: "normal", value: 1.0 },
                { name: "x0.75", value: 0.75 },
                { name: "x0.5", value: 0.5 },
            ]
        }
    });
```

Этот параметр доступен только в Проигрыватель мультимедиа Azure версии 2.0.0 и более поздних версиях.

### <a name="staledatatimelimitinsec"></a>сталедататимелимитинсек ###

`staleDataTimeLimitInSec`Параметр — это оптимизация, позволяющая настроить количество секунд, в течение которых следует удерживать устаревшие данные, которые вы хотите разместить в буферах медиасаурце. Эта функция отключена по умолчанию.

### <a name="cea708captionssettings"></a>cea708CaptionsSettings ###

Параметр Enabled имеет значение true позволяет отображать субтитры Live CEA в динамических потоках и в динамических архивах. Атрибут Label не является обязательным. Если он не включен, игрок вернется к метке по умолчанию.

```javascript
     cea708CaptionsSettings: {
                enabled: true,
                srclang: 'en',
                label: 'Live CC'
            }
```

Этот параметр доступен только в Проигрыватель мультимедиа Azure версии не ниже.
