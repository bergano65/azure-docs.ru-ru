# <a name="security-module"></a>Модуль безопасности

> [!IMPORTANT]
> Центр безопасности Azure для Интернета вещей находится в общедоступной предварительной версии.
> Эта предварительная версия предоставляется без соглашения об уровне обслуживания и не рекомендована для использования рабочей среде. Некоторые функции могут не поддерживаться или их возможности могут быть ограничены. Дополнительные сведения см. в статье [Дополнительные условия использования предварительных выпусков Microsoft Azure](https://azure.microsoft.com/support/legal/preview-supplemental-terms/).

В этой статье объясняется, как центр безопасности Azure (ASC) для Интернета вещей использует двойники устройств и модулей. 

## <a name="device-twins"></a>Двойники устройства

Для решений Интернета вещей, созданных в Azure двойники устройств играют ключевую роль в управлении устройствами и автоматизации процессов.  

ASC для Интернета вещей обеспечивает полную интеграцию с вашей существующей Интернета вещей платформой управления устройствами, давая возможность управлять состояние безопасности устройства и сделать использовать существующие возможности управления устройствами. Интеграция достигается за счет центра Интернета вещей использует механизм двойников.  

Дополнительные сведения о концепции [двойников](https://docs.microsoft.com/azure/iot-hub/iot-hub-devguide-device-twins) в центре Интернета вещей Azure. 

## <a name="security-module-twins"></a>Двойников модулей безопасности

ASC для Интернета вещей сохраняет двойник модуля безопасности для каждого устройства в службе.
Двойник модуля безопасности содержатся все сведения, относящиеся к безопасности устройств для каждого конкретного устройства в решении.
Свойства безопасности устройства хранятся в двойник модуля выделенного сервера для безопасного обмена данными, а также для обновления и обслуживания, которые требуют меньше ресурсов.  

См. в разделе [двойник модуля безопасности создать](quickstart-create-security-twin.md) и [настройке агентов безопасности](how-to-agent-configuration.md) вы научитесь создавать, настраивать и устанавливать двойника. См. в разделе [двойников модулей понимание](https://docs.microsoft.com/azure/iot-hub/iot-hub-devguide-module-twins) Дополнительные сведения о концепции двойников модуля в центре Интернета вещей.
