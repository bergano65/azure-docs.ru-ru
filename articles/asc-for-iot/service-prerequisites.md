# <a name="azure-security-center-for-iot-prerequisites"></a>Предварительные требования для центра безопасности Azure для IoT

В этой статье представлено описание различных стандартных блоков центра безопасности Azure для службы IoT, необходимые для начала действия и объясняются основные понятия, помогающие понять службу. 

## <a name="minimum-requirements"></a>Минимальные требования

- Уровень "Стандартный" центра Интернета вещей
    - Привилегии уровня **владельца** роли RBAC 
- [Рабочая область Log Analytics](https://docs.microsoft.com/azure/azure-monitor/learn/quick-create-workspace) 
- Центр безопасности Azure (рекомендуется)
    - Использование центра безопасности Azure является рекомендацией, а не требованием. Без центра безопасности Azure вы не сможете просматривать другие ресурсы Azure в центре Интернета вещей. 
 
## <a name="working-with-azure-security-center-for-iot-service"></a>Работа с центром безопасности Azure для службы IoT

Центр безопасности Azure для IoT Insights и отчеты доступны в центре Интернета вещей Azure и центра безопасности Azure. Чтобы включить центр безопасности Azure для Интернета вещей в центре Интернета вещей Azure, требуется учетная запись с правами уровня **владельца** . После включения ASC для Интернета вещей в центре Интернета вещей центр безопасности Azure для IoT Insights отображается в качестве функции **безопасности** центра Интернета вещей Azure и центра **Интернета вещей** в центре безопасности Azure. 

## <a name="supported-service-regions"></a>Поддерживаемые регионы для службы 

В настоящее время центр безопасности Azure для Интернета вещей поддерживает центры Интернета вещей в следующих регионах Azure:
  - Центральная часть США  
  - East US 
  - Восточная часть США 2
  - Центрально-западная часть США
  - Западная часть США
  - Западная часть США 2
  - Центрально-южная часть США
  - Центрально-северная часть США
  - Центральная Канада
  - Восточная Канада 
  - Северная Европа    
  - Южная Бразилия
  - Центральная Франция  
  - Западная часть Соединенного Королевства 
  - Южная часть Соединенного Королевства
  - Западная Европа 
  - Северная Европа 
  - Западная Япония  
  - Восточная Япония  
  - Юго-восточная Австралия
  - Восточная Австралия
  - Восточная Азия   
  - Юго-Восточная Азия
  - Центральная Корея
  - Республика Корея, южный регион 
  - Центральная Индия
  - Южная Индия

Центр безопасности Azure для Интернета вещей направляет весь трафик из всех стран в региональный центр обработки данных Западной Европы и все остальные регионы в Центральный Региональный центр обработки данных США.  
  
## <a name="wheres-my-iot-hub"></a>Где находится центр Интернета вещей?

Перед началом работы проверьте расположение центра Интернета вещей, чтобы проверить доступность службы. 

1. Откройте Центр Интернета вещей. 
2. Нажмите **Обзор**. 
3. Убедитесь, что указанное расположение соответствует одному из [поддерживаемых регионов службы](#supported-service-regions). 


## <a name="supported-platforms-for-agents"></a>Поддерживаемые платформы для агентов 

Центр безопасности Azure для агентов IoT поддерживает растущей список устройств и платформ. Просмотрите [список поддерживаемых платформ](how-to-deploy-agent.md) , чтобы проверить существующую или запланированную библиотеку устройств.
