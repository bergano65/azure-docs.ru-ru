# <a name="azure-data-box-frequently-asked-questions"></a>Azure Data Box. Часто задаваемые вопросы

Гибридное решение Microsoft Azure Data Box позволяет быстро, надежно и недорого передать в Azure много терабайт данных. В этой статье собраны ответы на вопросы, которые могут возникнуть при работе с Data Box на портале Azure.

Вопросы и ответы упорядочены по следующим категориям:

- Сведения о службе
- Заказ устройства
- Настройка и подключение 
- Отслеживание состояния
- Копирование данных 
- Доставка устройства
- проверка и отправка данных. 
- Поддержка процедур обеспечения сохранности

## <a name="about-the-service"></a>Сведения о службе

### <a name="q-what-is-azure-data-box-service"></a>У. Что такое служба Azure Data Box? 
A.  Служба Azure Data Box предназначена для приема данных в автономном режиме. Эта служба управляет массивом продуктов для хранения с разными объемами, которые предназначены для физической транспортировки данных. 

### <a name="q-what-is-azure-data-box"></a>У. Что такое Azure Data Box?
A. Azure Data Box позволяет быстро, недорого и безопасно передавать терабайты данных в Azure. Устройство Data Box можно заказать на портале Azure. Корпорация Майкрософт отправит вам устройство хранения емкостью 80 ТБ через региональную транспортную службу. 

Получив устройство, вы сможете быстро настроить его через локальный пользовательский веб-интерфейс. Скопируйте данные с серверов на устройство или с устройства на серверы и отправьте устройство обратно в Azure. Для заказа импорта в центре обработки данных Azure ваши данные автоматически передаются с устройства в Azure. Весь процесс от начала и до конца контролируется службой Data Box на портале Azure.

### <a name="q-when-should-i-use-data-box"></a>У. Когда следует использовать Data Box?
A. Если у вас есть 40-500 ТБ данных, которые необходимо переместить в Azure или из него, вам будет выгодно использовать Data Box. Для данных размером менее 40 ТБ используйте Диск Data Box, а для данных размером более 500 ТБ зарегистрируйтесь в службе [Data Box Heavy](data-box-heavy-overview.md).

### <a name="q-what-is-the-price-of-data-box"></a>У. Какова стоимость Data Box?
A. Базовая стоимость Data Box включает его использование до 10 дней. Выбрав модель продукта при создании заказа на портале Azure, вы увидите стоимость конкретного устройства. Также применяется стандартная плата за доставку и плата за хранилище Azure. В заказах на экспорт используются аналогичные модели ценообразования для импорта заказов, хотя могут взиматься дополнительные платежи за исходящий трафик. 

Дополнительные сведения см. в [Azure Data Box ценах](https://azure.microsoft.com/pricing/details/storage/databox/) и исходящих [платежах](https://azure.microsoft.com/pricing/details/bandwidth/). 

### <a name="q-what-is-the-maximum-amount-of-data-i-can-transfer-with-data-box-in-one-instance"></a>У. Какой максимальный объем данных можно передать за один раз с помощью Data Box?
A. Номинальная емкость Data Box составляет 100 ТБ, доступная емкость — 80 ТБ. Вы можете передать с помощью Data Box не более 80 ТБ данных. Для данных большего объема следует заказать больше устройств.

### <a name="q-how-can-i-check-if-data-box-is-available-in-my-region"></a>У. Как проверить доступность Data Box в моем регионе? 
A.  Сведения о доступности Data Box в разных странах и регионах см. на странице [доступности по регионам](data-box-overview.md#region-availability).  

### <a name="q-which-regions-can-i-store-data-in-with-data-box"></a>У. В каких регионах можно сохранять данные с помощью Data Box?
A. Data Box поддерживается во всех регионах США, Западной и Северной Европы, во Франции, в Великобритании, Японии, Австралии и Канаде. Дополнительные сведения см. в разделе [Регионы доступности](data-box-overview.md#region-availability).

### <a name="q-how-can-i-import-source-data-at-my-location-in-a-particular-country-to-an-azure-region-in-a-different-countryregion-or-export-data-from-an-azure-region-in-one-country-to-a-different-countryregion"></a>У. Как импортировать исходные данные в моем расположении в определенной стране в регион Azure в другой стране или регионе или экспортировать данные из региона Azure в одной стране в другую страну или регион?

Data Box поддерживает прием и передачу данных только в той же стране или регионе, что и назначение, и не пересекает международные границы. Единственное исключение — заказы в Европейского союза (ЕС), где поля данных могут поставляться в любую страну или регион ЕС.

Например, в сценарии импорта, если исходные данные в Канаде были выбраны для перемещения в учетную запись хранения Западная часть США Azure, ее можно получить следующим образом:

1. Закажите Data Box в Канаде, выбрав учетную запись хранения в Канаде. Устройство поставляется из центра обработки данных Azure в Канаде в адрес доставки (в Канаде), предоставленный при создании заказа.

2. После завершения копирования локальной копии данных в Data Box верните устройство в центр обработки данных Azure в Канаде. Данные, представленные в Data Box, затем передаются в целевую учетную запись хранения в регионе Azure для Канады, выбранном во время создания заказа.

3. Затем можно использовать такой инструмент, как AzCopy, для копирования данных в учетную запись хранения в западной части США. На этом этапе взимается плата за использование [стандартного хранилища](https://azure.microsoft.com/pricing/details/storage/) и [пропускной способности](https://azure.microsoft.com/pricing/details/bandwidth/) , которая не включается в Data Box выставления счетов.

### <a name="q-who-should-i-contact-if-i-come-across-any-issues-with-data-box"></a>У. Кому следует связаться, если у меня возникают проблемы с Data Box?
A. При возникновении проблем с Data Box [обратитесь в службу Служба поддержки Майкрософт](data-box-disk-contact-microsoft-support.md).

### <a name="q-i-lost-my-data-box-is-there-a-lost-device-charge"></a>У. Я потерял Data Box. Взимается ли плата в таком случае?
A. Да. Плата взимается за потерянное или поврежденное устройство. Эта плата взимается на [странице цен](https://azure.microsoft.com/pricing/details/storage/databox/) и [условиях обслуживания продукта](https://www.microsoft.com/licensing/product-licensing/products).


## <a name="order-device"></a>Заказ устройства

### <a name="q-how-do-i-get-data-box"></a>У. Как получить Data Box? 
A.  Чтобы получить Azure Data Box, войдите на портал Azure и создайте заказ Data Box. Введите свои контактные данные и сведения об уведомлениях. Data Box будет доставлен вам в течение 10 дней с момента размещения заказа в зависимости от доступности. Дополнительные сведения см. в описании [процесса заказа Data Box](data-box-deploy-ordered.md).

### <a name="q-i-couldnt-create-a-data-box-order-in-the-azure-portal-why"></a>У. Не удалось создать порядок Data Box в портал Azure. Почему?
A. Если вы не можете создать Data Box заказ, существует проблема с типом подписки или доступом.

Сначала проверьте подписку. Data Box доступен только для подписок с Соглашением Enterprise (EA) и поставщика облачных решений (CSP). Если у вас нет ни одного из этих типов подписки, обратитесь в служба поддержки Майкрософт, чтобы обновить подписку.

Если же подписка уже относится к поддерживаемому типу, проверьте уровень доступа для подписки. Чтобы создать заказ, нужно иметь роль участника или владельца для подписки.

### <a name="q-how-long-will-my-order-take-from-order-creation-to-data-uploaded-to-azure"></a>У. Сколько времени будет отправлено из заказа для создания данных в Azure?

A. Приведенное ниже оценочное время выполнения каждого этапа обработки заказов даст вам хорошее представление о том, что следует рассчитывать.  

Это время опережения — *Оценка*. Время для каждого этапа обработки заказов зависит от загрузки в центре данных, параллельных заказов и других условий окружающей среды.

**Предполагаемое время опережения для Data Box заказа:**

1. Заказ Data Box: несколько минут с портала
2. Выделение и подготовка устройств: 1–2 рабочих дня
3. Отправка: 2–3 рабочих дня
4. Копирование данных на сайте клиента. зависит от характера данных, размера и количества файлов.
5. Обратная доставка: 2–3 рабочих дня
6. Обработка устройства в центре обработки данных: 1-2 рабочих дней
7. Отправка данных в Azure: начинает работу сразу после завершения обработки и подключения устройства. Время отправки зависит от характера данных, размера и количества файлов.

### <a name="q-i-ordered-a-couple-of-data-box-devices-i-cant-create-any-additional-orders-why"></a>У. Заказав несколько устройств Data Box, Я не могу создать дополнительные заказы. Почему?
A. Мы допуским не более пяти активных заказов на одну подписку на границу каждой из них (сочетание страны и региона). Если вам нужно больше устройств, обратитесь в службу поддержки Майкрософт и попросите увеличить лимит для вашей подписки.

### <a name="q-when-i-try-to-create-an-order-i-receive-a-notification-that-the-data-box-service-is-not-available-what-does-this-mean"></a>У. При попытке создать заказ я получаю уведомление, что служба Data Box не доступна. Что это означает?
A. Служба Data Box недоступна для выбранной комбинации страны и региона. Изменив это сочетание, скорее всего, вы сможете воспользоваться службой Data Box. Список регионов, где доступна эта служба, см. в описании [доступности Data Box по регионам](data-box-overview.md#region-availability).

### <a name="q-i-placed-my-data-box-order-few-days-back-when-will-i-receive-my-data-box"></a>У. Мой заказ на Data Box размещен несколько дней назад. Когда я получу устройство Data Box?
A. После размещения заказа мы проверяем, есть ли доступное для этого заказа устройство. Если устройство доступно, мы отправим его в течение 10 дней. Но разумеется, бывают периоды повышенного спроса. В такие периоды заказ помещается в очередь, состояние которой вы можете отслеживать на портале Azure. Если заказ не будет выполнен в течение 90 дней, он автоматически отменяется.

### <a name="q-i-have-filled-up-my-data-box-with-data-and-need-to-order-another-one-is-there-a-way-to-quickly-place-the-order"></a>У. Я заполнил Data Box данными и должен заказать еще один. Есть ли способ быстро разместить заказ?
A. Можно клонировать предыдущий заказ. В этом случае создается тот же заказ, сведения которого можно изменять, только без необходимости вводить сведения об адресе, контактные данные и информацию для уведомлений. Клонирование допускается только для заказов импорта.

## <a name="configure-and-connect"></a>Настройка и подключение

### <a name="q-how-do-i-unlock-the-data-box"></a>У. Как разблокировать Data Box? 
A.  На портале Azure перейдите к своему заказу Data Box и перейдите к разделу **Сведения об устройстве**. Скопируйте пароль разблокирования. Используя этот пароль, войдите в локальный пользовательский интерфейс Data Box. Дополнительные сведения см. в статье [Руководство. Присоединение кабелей и подключение к Azure Data Box](data-box-deploy-set-up.md).

### <a name="q-can-i-use-a-linux-host-computer-to-connect-and-copy-the-data-on-to-the-data-box"></a>У. Можно ли использовать главный компьютер Linux для подключения Data Box и копирования данных?
A.  Да. Data Box можно подключать к клиентам SMB и NFS. Дополнительные сведения см. в списке [поддерживаемых операционных систем](data-box-system-requirements.md) для главного компьютера.

### <a name="q-my-data-box-is-dispatched-but-now-i-want-to-cancel-this-order-why-is-the-cancel-button-not-available"></a>У. Data Box уже отправлен, но я хочу отменить этот заказ. Почему кнопка "Отмена" недоступна?
A.  Заказ можно отменить только в период от момента заказа и до его обработки. Когда заказ на Data Box уже обработан, его невозможно отменить. 

### <a name="q-can-i-connect-a-data-box-at-the-same-to-multiple-host-computers-to-transfer-data"></a>У. Можно ли подключить Data Box для передачи данных одновременно к нескольким компьютерам?
A. Да. К одному Data Box можно подключить несколько компьютеров для передачи данных, также можно параллельно выполнять несколько заданий копирования. Дополнительные сведения см. в статье [Руководство. Копирование данных в Azure Data Box через SMB](data-box-deploy-copy-data.md)

### <a name="q-can-i-connect-to-both-the-10-gbe-interfaces-on-the-data-box-to-transfer-data"></a>У. Можно ли подключиться к обоим интерфейсам 10-GbE на устройств Data Box для передачи данных?
A. Да. Оба интерфейса 10 GbE можно подключить на устройстве Data Box для одновременного копирования данных. См. дополнительные сведения в [руководстве по Копирование данных в Azure Data Box через SMB](data-box-deploy-copy-data.md)

## <a name="track-status"></a>Отслеживание состояния

### <a name="q-how-do-i-track-the-data-box-from-when-i-placed-the-order-to-shipping-the-device-back"></a>У. Как отслеживать Data Box с момента размещения заказа до отправки устройства обратно? 
A.  Состояние заказа на Data Box можно отслеживать на портале Azure. При создании заказа вам будет предложено указать электронное письмо с уведомлением. Если вы указали его, вы получаете уведомление по электронной почте обо всех изменениях состояния заказа. Дополнительные сведения см. в разделе об [изменении адреса электронной почты для уведомлений](data-box-portal-ui-admin.md#edit-notification-details).

### <a name="q-how-do-i-return-the-device"></a>У. Как вернуть устройство? 
A.  Корпорация Майкрософт выводит транспортную этикетку на дисплей E-ink. Если метка отгрузки не отображается на экране электронного ввода, перейдите к **обзору > загрузить метку доставки**. Скачайте и распечатайте метку, вставьте метку в очистите пластиковый тег на устройстве и отпустите устройство в расположении перевозчика. 

### <a name="q-i-received-an-email-notification-that-my-device-has-reached-the-azure-datacenter-how-do-i-find-out-if-the-data-upload-is-in-progress"></a>У. Я получил по электронной почте уведомление о том, что устройство прибыло в центр обработки данных Azure. Как мне узнать, выполняется ли передача данных?
A. Вы можете открыть заказ на Data Box на портале Azure и перейти к разделу **Обзор**. Если передача данных в Azure уже началась, в области справа вы увидите ход выполнения копирования. 

## <a name="migrate-data"></a>Перенос данных

### <a name="q-what-is-the-maximum-data-size-that-can-be-used-with-data-box"></a>У. Какой максимальный размер данных для Data Box?  
A.  Рабочий объем Data Box составляет 80 ТБ. Одно устройство Data Box можно использовать для данных размером от 40 до 80 ТБ. Для более крупных размеров, вплоть до 500 ТБ, следует заказать несколько устройств Data Box. Если же размер данных превышает 500 ТБ, зарегистрируйтесь в службе Data Box Heavy.  

### <a name="q-what-are-the-maximum-block-blob-and-page-blob-sizes-supported-by-data-box"></a>У. Какой максимальный размер блочного и страничного BLOB-объектов поддерживает Data Box? 
A.  Максимальные размеры регулируются ограничениями службы хранилища Azure. Максимальный размер блочного BLOB-объекта составляет приблизительно 4,768 ТиБ, а максимальный размер большого двоичного объекта составляет 8 ТиБ. Дополнительные сведения см. в статье [Scalability and performance targets for Blob storage](../storage/blobs/scalability-targets.md) (Целевые показатели масштабируемости и производительности для хранилища BLOB-объектов).

### <a name="q-how-do-i-know-that-my-data-is-secure-during-transit"></a>У. Как узнать, что мои данные защищены во время передачи? 
A. Мы применяем несколько механизмов безопасности, защищающих Data Box на этапе транспортировки. Сюда входят печати с контролем вскрытия, аппаратные и программные средства обнаружения вскрытия, пароль для разблокировки устройства. Дополнительные сведения см. в статье [Azure Data Box security and data protection](data-box-security.md) (Безопасность Azure Data Box и защита данных).

### <a name="q-how-do-i-copy-the-data-to-the-data-box"></a>У. Как скопировать данные на Data Box? 
A.  При использовании клиента SMB для копирования данных на устройство можно использовать средство копирования SMB, такое как `Robocopy` , `Diskboss` или даже в проводнике Windows. 

Если вы используете клиент NFS, воспользуйтесь средствами [rsync](https://rsync.samba.org/), [FreeFileSync](https://www.freefilesync.org/), [Unison](https://www.cis.upenn.edu/~bcpierce/unison/) или [Ultracopier](https://ultracopier.first-world.info/). 

Дополнительные сведения см. в статье [Руководство. Копирование данных в Azure Data Box через SMB](data-box-deploy-copy-data.md)

### <a name="q-are-there-any-tips-to-speed-up-the-data-copy"></a>У. Есть ли советы по ускорению копирования данных?
A.  Чтобы ускорить процесс копирования:

- Используйте несколько потоков для копирования данных. Например, с `Robocopy` используйте параметр многопоточность. Дополнительные сведения об используемой команде см. в статье [Руководство. Копирование данных в Azure Data Box через SMB](data-box-deploy-copy-data.md).
- Используйте несколько сеансов.
- Вместо того, чтобы копировать в сетевую общую папку (где скорость копирования может ограничиться скоростью передачи данных), храните данные локально на компьютере, к которому подключена Data Box.
- Оцените производительность компьютера, используемого для копирования данных. Загрузите и используйте [ `Bluestop` средство FIO](https://ci.appveyor.com/project/axboe/fio) , чтобы протестировать производительность серверного оборудования. Выберите последнюю сборку x86 или x64, откройте вкладку **Артефакты** и скачайте MSI.

## <a name="ship-device"></a>Доставка устройства

### <a name="q-my-device-was-delivered-but-the-device-seems-to-be-damaged-what-should-i-do"></a>У. Устройство доставлено, но устройство кажется поврежденным. Что делать?
A. Если устройство было повреждено или есть свидетельство о незаконном вмешательстве, не используйте устройство. [Обратитесь в Служба поддержки Майкрософт](data-box-disk-contact-microsoft-support.md) и верните устройство как можно скорее. Вы также можете создать новый заказ на Data Box, чтобы получить другое устройство. В этом случае вам не придется рассчитывать на замену устройства.

### <a name="q-can-i-pick-up-my-data-box-order-myself-can-i-return-the-data-box-via-a-carrier-that-i-choose"></a>У. Можно ли забрать заказ на Data Box самостоятельно? Можно ли вернуть Data Box с помощью перевозчика, которого я выбрал?
A. Да. Корпорация Майкрософт также предлагает самостоятельно управляемую доставку. При размещении заказа на Data Box можно выбрать вариант "самоуправляемая доставка". Дополнительные сведения см. в разделе [Самостоятельное управление доставкой для Data Box](data-box-portal-customer-managed-shipping.md).

### <a name="q-will-my-data-box-devices-cross-countryregion-borders-during-shipping"></a>У. Будут ли мои устройства Data Box пересекать границы страны или региона во время доставки?
A. Пересылка всех устройств Data Box осуществляется в пределах страны или региона получателя. Следовательно, устройства не будут пересекать международные границы. Единственное исключение — заказы в пределах Европейского союза (ЕС), когда устройства могут перемещаться между разными странами и регионами ЕС. Это относится к устройствам Data Box и Data Box Heavy.

### <a name="q-i-ordered-a-data-box-in-us-east-but-i-received-a-device-that-was-shipped-from-a-location-in-us-west-where-should-i-return-the-device-to"></a>У. Я заказывал Data Box в восточной части США, но получил устройство, которое было доставлено из западной части США. Куда мне возвращать устройство?
A. Мы пытаемся попытаться получить Data Box устройство как можно быстрее. Мы планируем отправку из центра обработки данных, находящегося ближе всего к региону регистрации учетной записи хранения, но фактическая отправка может выполняться из любого центра обработки данных Azure, в котором устройства есть в наличии. Data Box следует возвращать по тому же адресу, с которого оно было отправлено и который указан на этикетке отгрузки.

### <a name="q-e-ink-display-is-not-showing-the-return-shipment-label-what-should-i-do"></a>У. На дисплее E-ink не отображается транспортная этикетка для обратной доставки. Что делать?
A. Если на экране с рукописным вводом не отображается метка возвратная отгрузка, выполните следующие действия.
- Удалите старую транспортную этикетку и любые наклейки, сохранившиеся от предыдущих доставок.
- Перейдите к странице заказа на портале Azure. Выберите **Обзор** и **Скачать транспортную этикетку**. Дополнительные сведения см. в разделе о [скачивании транспортной этикетки](data-box-portal-admin.md#download-shipping-label).
- Распечатайте эту транспортную этикетку и поместите ее в прозрачный пластиковый конверт, прикрепленный к устройству. 
- Проконтролируйте, чтобы этикетка была отчетливо видна. 

### <a name="q-how-is-my-data-protected-during-transit"></a>У. Как мои данные защищаются во время передачи? 
A.  Во время транспортировки защиту данных на Data Box обеспечивают следующие механизмы.
 - Все диски Data Box шифруются по стандарту AES 256 бит. 
 - Устройство блокируется и доступ к данным возможен только после ввода пароля разблокировки.
Дополнительные сведения см. в статье [о функциях безопасности для Data Box](data-box-security.md).  

### <a name="q-i-have-finished-prepare-to-ship-for-my-import-order-and-shut-down-the-device-can-i-still-add-more-data-to-the-data-box"></a>У. Закончена подготовка к отправке заказа на импорт и завершение работы устройства. Можно ли по-прежнему добавлять дополнительные данные в Data Box?
A. Да. Вы можете включить устройство и добавить дополнительные данные. Но после этого следует снова запустить процесс **подготовки к отправке**.

### <a name="q-i-received-my-device-and-it-is-not-booting-up-how-do-i-ship-the-device-back"></a>У. Я получил устройство, но оно не запускается? Как отправить устройство обратно?
A. Если устройство не загружается, перейдите к своему заказу в портал Azure. Скачайте метку доставки и подключите ее к устройству. Дополнительные сведения см. в разделе о [скачивании транспортной этикетки](data-box-portal-admin.md#download-shipping-label).

## <a name="verify-and-upload"></a>Проверка и отправка данных

### <a name="q-how-soon-can-i-access-my-data-in-azure-once-ive-shipped-the-data-box-back"></a>У. Как скоро я получу доступ в Azure к своим данным, отправленным на Data Box? 
A.  Как только для **Копирование данных** будет отображаться состояние **Завершено**, вы сможете использовать свои данные.

### <a name="q-where-is-my-data-located-in-azure-after-the-upload"></a>У. Где располагаются мои данные в Azure после передачи?
A.  При копировании данных в Data Box в зависимости от того, являются ли данные блочными или страничными BLOB-объектами или файлами Azure, данные передаются в один из следующих путей в учетной записи хранения Azure.
- `https://<storage_account_name>.blob.core.windows.net/<containername>` 
- `https://<storage_account_name>.file.core.windows.net/<sharename>`
 
  Кроме того, можно переходить к учетной записи хранения Azure в портал Azure и переходить отсюда.

### <a name="q-i-just-noticed-that-i-did-not-follow-the-azure-naming-requirements-for-my-containers-will-my-data-fail-to-upload-to-azure"></a>У. Как оказалось, требования к именованию Azure для контейнеров не были соблюдены. Будут ли мои данные отправлены в Azure?
A.  Если имена контейнеров имеют прописные буквы, эти имена автоматически преобразуются в нижний регистр. Если имена не совместимы иным образом (специальные символы, другие языки и т. д.), передача завершается ошибкой. Дополнительные рекомендации по именованию общих ресурсов, контейнеров и файлов см. по адресу:
- [Именование общих папок и ссылка на них](/rest/api/storageservices/naming-and-referencing-shares--directories--files--and-metadata)
- [Соглашения по блочным и страничным BLOB-объектам](/rest/api/storageservices/understanding-block-blobs--append-blobs--and-page-blobs).

### <a name="q-how-do-i-verify-the-data-i-copied-onto-data-box"></a>У. Как мне проверить данные, скопированные на Data Box?
A.  Данные автоматически проверяются после копирования, когда вы запускаете процесс **Подготовка к отправке**. В процессе этой проверки Data Box создает список файлов и контрольные суммы. Вы можете скачать этот список файлов и сравнить его с исходными файлами. Дополнительные сведения см. в разделе о [подготовке к отправке](data-box-deploy-picked-up.md#prepare-to-ship).

### <a name="q-what-happens-to-my-data-after-i-return-the-data-box"></a>У. Что происходит с моими данными после возврата Data Box?
A.  Когда завершится копирование данных в Azure, все данные безопасно удаляются с Data Box с соблюдением рекомендаций NIST SP 800-88 в редакции 1. Дополнительные сведения см. в статье [Стирание данных из Data Box](data-box-deploy-picked-up.md#erasure-of-data-from-data-box).

## <a name="audit-report"></a>Отчет аудита

### <a name="how-does-azure-data-box-service-help-support-customers-chain-of-custody-procedure"></a>Как служба Azure Data Box помогает соблюдать клиентские процедуры обеспечения сохранности?
A.  Служба Azure Data Box предоставляет встроенные отчеты, которые можно использовать в документации по обеспечению сохранности. Журналы аудита и копирования доступны в учетной записи хранения в Azure, а после выполнения заказа вы можете скачать [журнал заказов](data-box-portal-admin.md#download-order-history) на портале Azure.


### <a name="what-type-of-reporting-is-available-to-support-chain-of-custody"></a>Какие доступны отчеты для поддержки обеспечения сохранности?
A.  Для поддержки обеспечения сохранности предоставляются следующие отчеты:

- логистика транспортировки от UPS;
- журналы включения и доступа пользователей;
- Файл спецификации или манифеста с 64-разрядной циклической проверкой избыточности (CRC-64) или контрольной суммой для каждого файла, который успешно получен в Data Box.
- отчеты о файлах, которые не удалось отправить в учетную запись хранения Azure;
- протокол очистки устройства Data Box (в соответствии со стандартом NIST 800 88R1) после копирования данных в учетную запись хранения Azure.

### <a name="are-the-carrier-tracking-logs-from-ups-available"></a>Доступны ли журналы отслеживания от перевозчика (UPS)? 
A.  Журналы отслеживания перевозчика включаются в журнал заказов Data Box. Этот отчет будет вам доступен после того, как устройство вернется в центр обработки данных Azure и будет выполнена очистка его дисков. Если возникнет срочная потребность в этих данных, вы можете получить сведения об отслеживании на веб-сайте перевозчика, указав номер отслеживания.

### <a name="can-i-transport-the-data-box-to-azure-datacenter"></a>Могу ли я сам привезти Data Box в центр обработки данных Azure? 
A.  Нет. Если вы выбрали Управление нагрузкой Майкрософт, вы не сможете передавать данные. В настоящее время центр обработки данных Azure не принимает доставку Data Box от клиентов или от перевозчиков, отличных от ИБП.

Если вы выбрали самостоятельно управляемую доставку, вы можете выбрать или отбросить Data Box из центра обработки данных Azure.
