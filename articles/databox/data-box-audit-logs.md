# <a name="audit-logs-for-your-azure-data-box-and-azure-data-box-heavy"></a>Журналы аудита для Azure Data Box и Azure Data Box Heavy

Журналы представляют собой неизменяемые записи с метками времени для дискретных событий, произошедших с течением времени. Журналы содержат сведения о диагностике, аудите и безопасности на устройстве.  

В ходе выполнения операции Data Box или Data Box Heavy пошаговые инструкции выполняются в следующих шагах: заказ, Настройка, копирование данных, возврат, передача в Azure и проверка и очистки данных. Для каждого из этих шагов выполняется аудит и регистрация всех событий.

Эта статья содержит сведения о журналах аудита Data Box, включая типы журналов и собираемые сведения, а также расположение журналов. 

Сведения в этой статье применимы как к, так Data Box и Data Box Heavy. В последующих разделах все ссылки на Data Box также применяются к Data Box Heavy. Журналы, собранные из службы Data Box, работающей в Azure, не рассматриваются в этой статье. 


## <a name="about-audit-logs"></a>Сведения о журналах аудита 

На Data Box собираются следующие журналы:

- **Системные журналы** — Data Box это устройство на основе Windows, в журнал заносятся все события оборудования, программного обеспечения и системных событий. Набор этих событий собираются и сообщается в журналах аудита системы. 

- **Security** -Data Box — это устройство под управлением Windows, регистрируются все события безопасности. Набор этих событий собираются и регистрируется в журналах аудита безопасности. 

- **Приложение** . Эти журналы относятся только к Data Box. Эти журналы содержат все события, созданные на устройстве в ответ на выполняемые службы Data Box.

Каждый из этих журналов обсуждается в следующем разделе.

### <a name="application-logs"></a>Журналы приложений

Следующие идентификаторы событий журнала приложений собираются как часть журналов аудита пакетов на Data Box.     

- **Microsoft-Azure-датабокс-OOBE-Audit** — содержит события, происходящие в локальном пользовательском интерфейсе. 
- **Microsoft-Azure-датабокс-reprovisionть аудит** — содержит события, связанные с повторной инициализацией устройства Data Box. Повторная инициализация Data Box происходит при сбросе устройства через локальный пользовательский интерфейс. Этот параметр выбирается при необходимости стереть скопированные данные путем удаления существующих общих ресурсов и повторного создания общих ресурсов в процессе повторной инициализации или сброса устройства.
- **Microsoft-Azure-датабокс-хксмгмт-Audit** — содержит события, связанные только с шагом **подготовить к отправке** , прежде чем устройство будет отправлено обратно в центр обработки данных Azure. 
- **Microsoft-Azure-датабокс-ифксаудит** — содержит сообщения, регистрируемые различными сущностями продукта о заданиях, журналы, указывающие дополнительные сведения о том, что происходит в некоторых потоках.

Помимо журналов аудита ИФКС, цепочка журналов аудита поставок также собирается для Data Box. Эти журналы невозможно просмотреть в режиме реального времени, но только после завершения задания и удаления данных с Data Box дисков. Эти журналы содержат подмножество сведений, содержащихся в журналах аудита ИФКС.

Дополнительные сведения о цепочке журналов аудита поставок см. в разделе [Получение цепочки журналов поставок после очистки данных](data-box-logs.md#get-chain-of-custody-logs-after-data-erasure).

## <a name="access-audit-logs"></a>Доступ к журналам аудита

Эти журналы хранятся в Azure и недоступны напрямую. Если вам нужно получить доступ к этим журналам, отправьте запрос в службу поддержки. Дополнительные сведения см. в статье [Обращение в службу поддержки Майкрософт](data-box-disk-contact-microsoft-support.md). 

После того как обращение в службу поддержки будет завершено, корпорация Майкрософт загрузит и предоставит вам доступ к этим журналам.
