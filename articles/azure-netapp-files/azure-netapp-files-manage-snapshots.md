# <a name="manage-snapshots-by-using-azure-netapp-files"></a>Управление моментальными снимками с помощью Azure NetApp Files

Вы можете использовать Azure NetApp Files для создания моментального снимка по запросу для тома или восстановления из моментального снимка в новый том.

## <a name="create-an-on-demand-snapshot-for-a-volume"></a>Создание моментального снимка по запросу для тома

Моментальные снимки можно создавать только по запросу. Политики моментальных снимков сейчас не поддерживаются.

1.  В колонке "Управление томом", щелкните **Моментальные снимки**.

    ![Переход к моментальным снимкам](../media/azure-netapp-files/azure-netapp-files-navigate-to-snapshots.png)

2.  Чтобы создать моментальный снимок по запросу для тома, щелкните **+ Добавить моментальный снимок**.

    ![Добавление моментального снимка](../media/azure-netapp-files/azure-netapp-files-add-snapshot.png)

3.  В окне "Создание моментального снимка" введите имя создаваемого моментального снимка.   

    ![Новый моментальный снимок](../media/azure-netapp-files/azure-netapp-files-new-snapshot.png)

4. Последовательно выберите **ОК**. 

## <a name="restore-a-snapshot-to-a-new-volume"></a>Восстановление моментального снимка в новый том

Сейчас моментальный снимок можно восстановить только в новый том. 
1. Перейдите в колонку **Manage Snapshots** (Управление моментальными снимками) из колонки "Том" для отображения списка моментальных снимков. 
2. Выберите восстанавливаемый моментальный снимок.  
3. Щелкните правой кнопкой мыши имя моментального снимка и выберите пункт **Restore to new volume** (Восстановить в новый том).  

    ![Восстановление моментального снимка в новый том](../media/azure-netapp-files/azure-netapp-files-snapshot-restore-to-new-volume.png)

4. В окне "Новый том" укажите сведения о новом томе:  
    * **ИМЯ**   
        Укажите имя создаваемого тома.  
        
        Имя должно быть уникальным в пределах группы ресурсов. Длина имени должна быть не менее трех знаков.  Имя может состоять из любых алфавитно-цифровых символов.

    * **Путь к файлу**     
        Укажите путь к файлу, который будет использоваться для создания пути экспорта для нового тома. Путь экспорта используется для подключения и получения доступа к тому.   
        
        Подключенный объект является конечной точкой IP-адреса службы NFS. Он создается автоматически.   
        
        Имя пути к файлу может содержать только буквы, цифры и дефисы. Длина имени находится в диапазоне от 16 до 40 знаков. 

    * **Quota**  
        Укажите объем логического хранилища, выделенный для тома.  

        В поле **Доступная квота** отображается объем неиспользуемого пространства выбранного пула емкости, которое можно использовать для создания нового тома. Размер нового тома не должен превышать доступную квоту.

    *   **Виртуальная сеть**  
        Укажите Виртуальную сеть Azure, из которой будет осуществляться доступ к тому.  
        В указанной виртуальной сети должна быть подсеть, делегированная службе Azure NetApp Files. Благодаря пиринговой связи вы можете получить доступ к службе Azure NetApp Files только из той же виртуальной сети, либо из виртуальной сети, которая находится в том же регионе, что и том. Том доступен в вашей локальной сети через Express Route. 

    * **Подсеть**  
        Укажите подсеть, которую нужно использовать для тома.  
        Указанная подсеть должна быть делегирована службе Azure NetApp Files. Вы можете создать новую подсеть, выбрав параметр **Создать** под полем подсети.  
   <!--
    ![Restored new volume](../media/azure-netapp-files/azure-netapp-files-snapshot-new-volume.png) 
   -->

5. Последовательно выберите **ОК**.   
    Новый том, на котором восстанавливается моментальный снимок, отображается в колонке "Тома".
