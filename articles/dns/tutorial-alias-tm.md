# <a name="tutorial-configure-an-alias-record-to-support-apex-domain-names-with-traffic-manager"></a>Руководство. Настройка записи псевдонима для поддержки вершинных доменных имен с помощью диспетчера трафика 

Вы можете создать запись псевдонима для вашего вершинного доменного имени для ссылки на профиль диспетчера трафика Microsoft Azure. Например, contoso.com. Вместо использования службы перенаправления можно настроить Azure DNS для ссылки на профиль диспетчера трафика непосредственно из вашей зоны. 


Из этого руководства вы узнаете, как выполнять следующие задачи:

> [!div class="checklist"]
> * Создание виртуальной машины узла и сетевой инфраструктуры.
> * Создание профиля диспетчера трафика.
> * Создание записи псевдонима.
> * Тестирование записи псевдонима.


Если у вас еще нет подписки Azure, [создайте бесплатную учетную запись Azure](https://azure.microsoft.com/free/?WT.mc_id=A261C142F), прежде чем начинать работу.

## <a name="prerequisites"></a>Предварительные требования
Необходимо иметь доступное имя домена, который можно будет разместить в Azure DNS и использовать для тестирования. У вас должен быть полный контроль над этим доменом, включая возможность определять записи сервера имен (NS) для домена.

Инструкции по размещению домена в Azure DNS см. в [этом руководстве](dns-delegate-domain-azure-dns.md).

В качестве примера в этом руководстве используется домен contoso.com — замените его на собственное доменное имя.

## <a name="create-the-network-infrastructure"></a>Создание сетевой инфраструктуры
Сначала создайте виртуальную сеть и подсеть для размещения веб-серверов.
1. Войдите на портал Azure по адресу http://portal.azure.com.
2. На портале слева вверху выберите **Создать ресурс**. В поле поиска введите *группу ресурсов* и создайте группу ресурсов с именем **RG-DNS-Alias-TM**.
3. Щелкните **Создать ресурс** > **Сети** > **Виртуальная сеть**.
4. Создайте виртуальную сеть с именем **VNet-Servers**. Поместите ее в группу ресурсов **RG-DNS-Alias-TM** и присвойте этой подсети имя **SN-Web**.

## <a name="create-two-web-server-virtual-machines"></a>Создание двух виртуальных машин для веб-серверов
1. Выберите **Создать ресурс** > **Виртуальная машина Windows Server 2016**.
2. Введите **Web-01** в качестве имени и поместите эту виртуальную машину в группу ресурсов **RG-DNS-Alias-TM**. Введите имя пользователя и пароль и нажмите кнопку **ОК**.
3. В разделе **Размер** выберите номер SKU с 8 ГБ ОЗУ.
4. В разделе **Параметры** выберите виртуальную сеть **VNet-Servers** и подсеть **SN-Web**.
5. Выберите **Общедоступный IP-адрес**. В поле **Назначение** выберите **Статический**, а затем нажмите кнопку **ОК**.
6. В качестве общедоступных входящих портов выберите **HTTP** > **HTTPS** > **RDP (3389)**, а затем нажмите кнопку **ОК**.
7. На странице **Сводка** выберите **Создать**. На завершение процедуры может потребоваться несколько минут.

Повторите этот процесс, чтобы создать еще одну виртуальную машину с именем **Web-02**.

### <a name="add-a-dns-label"></a>Добавление метки DNS
Общедоступным IP-адресам требуется метка DNS для работы с диспетчером трафика.
1. В группе ресурсов **RG-DNS-Alias-TM** выберите общедоступный IP-адрес **Web-01-ip**.
2. В разделе **Параметры** выберите пункт **Конфигурация**.
3. В текстовом поле метки DNS-имени введите **web01pip**.
4. Щелкните **Сохранить**.

Повторите эту процедуру для общедоступного IP-адреса **Web-02-ip**, используя **web02pip** в качестве метки DNS-имени.

### <a name="install-iis"></a>Установка служб IIS

Установите службы IIS на обе виртуальные машины, **Web-01** и **Web-02**.

1. Подключитесь к виртуальной машине **Web-01** и войдите в систему.
2. На панели мониторинга **Диспетчер серверов** нажмите **Добавить роли и компоненты**.
3. Нажмите кнопку **Далее** три раза. На странице **Роли сервера** выберите **Веб-сервер (IIS)**.
4. Выберите **Добавить компоненты**, а затем нажмите кнопку **Далее**.
5. Нажмите кнопку **Далее** четыре раза. Щелкните **Установить**. На завершение процедуры может потребоваться несколько минут.
6. По завершении установки выберите **Закрыть**.
7. Откройте веб-браузер. Перейдите к **localhost** и убедитесь, что открывается веб-страница IIS по умолчанию.

Повторите эту процедуру для установки служб IIS на виртуальную машину **Web-02**.


## <a name="create-a-traffic-manager-profile"></a>Создание профиля диспетчера трафика

1. Откройте группу ресурсов **RG-DNS-Alias-TM** и выберите общедоступный IP-адрес **Web-01-ip**. Запишите этот IP-адрес для последующего использования. Повторите это действие для общедоступного IP-адреса **Web-02-ip**.
1. Последовательно выберите **Создать ресурс** > **Сети** > **Профиль диспетчера трафика**.
2. Введите имя **TM-alias-test**. Поместите его в группу ресурсов **RG-DNS-Alias-TM**.
3. Нажмите кнопку **Создать**.
4. После завершения развертывания выберите **Перейти к ресурсу**.
5. На странице профиля диспетчера трафика в разделе **Параметры** выберите **Конечные точки**.
6. Выберите **Добавить**.
7. В качестве **типа** выберите **Внешняя конечная точка**, а в качестве **имени** введите **EP-Web01**.
8. В текстовом поле **Полное доменное имя (FQDN) или IP-адрес** введите IP-адрес для **Web-01-ip**, который вы записали ранее.
9. Выберите то же **расположение**, что и для других ресурсов, и нажмите кнопку **ОК**.

Повторите эту процедуру, чтобы добавить конечную точку **Web-02**, используя IP-адрес, записанный ранее для **Web-02-ip**.

## <a name="create-an-alias-record"></a>Создание записи псевдонима

Создайте запись псевдонима, которая указывает на профиль диспетчера трафика.

1. Выберите свою зону Azure DNS, чтобы открыть ее.
2. Выберите **Набор записей**.
3. Оставьте текстовое поле **Имя** пустым для представления вершинного доменного имени. Например, contoso.com.
4. Оставьте в поле **Тип** запись **A**.
5. Установите флажок **Набор записей псевдонимов**.
6. Щелкните **Выбрать службу Azure** и выберите профиль диспетчера трафика **TM-alias-test**.

## <a name="test-the-alias-record"></a>Тестирование записи псевдонима

1. В веб-браузере перейдите к вашему вершинному доменному имени. Например, contoso.com. Откроется веб-страница служб IIS по умолчанию. Закройте веб-браузер.
2. Завершите работу виртуальной машины **Web-01**. Подождите несколько минут для того, чтобы полностью завершить работу.
3. Откройте новый веб-браузер и снова перейдите к вашему вершинному доменному имени.
4. Появится веб-страница служб IIS по умолчанию, так как диспетчер трафика обработал ситуацию и направил трафик на виртуальную машину **Web-02**.

## <a name="clean-up-resources"></a>Очистка ресурсов

Если ресурсы, созданные для этого руководства, больше не нужны, удалите группу ресурсов **RG-DNS-Alias-TM**.
