# <a name="what-is-a-virtual-network-link"></a>Что такое связь виртуальной сети?

После создания частной зоны DNS в Azure она не будет доступна сразу из любой виртуальной сети. Необходимо связать его с виртуальной сетью, прежде чем виртуальная машина, размещенная в этой сети, сможет получить доступ к частной зоне DNS.
Чтобы связать частную зону DNS с виртуальной сетью, необходимо создать ссылку на виртуальную сеть в частной зоне DNS. У каждой частной зоны DNS есть коллекция дочерних ресурсов для связи с виртуальной сетью. Каждый из этих ресурсов представляет подключение к виртуальной сети.

Вы можете связать виртуальную сеть с частной зоной DNS в качестве виртуальной сети регистрации или как виртуальную сеть разрешения.

## <a name="registration-virtual-network"></a>Виртуальная сеть регистрации

При [создании связи](https://docs.microsoft.com/azure/dns/private-dns-getstarted-portal#link-the-virtual-network) между частной зоной DNS и виртуальной сетью у вас есть возможность включить [авторегистрацию](./private-dns-autoregistration.md) записей DNS для виртуальных машин. При выборе этого параметра виртуальная сеть станет виртуальной сетью регистрации для частной зоны DNS. Для виртуальных машин, развертываемых в сети, автоматически создается запись DNS. Записи DNS создаются для виртуальных машин, которые уже развернуты в виртуальной сети. С точки зрения виртуальной сети частная зона DNS станет зоной регистрации для этой виртуальной сети.
Одна частная зона DNS может иметь несколько виртуальных сетей регистрации, но каждая виртуальная сеть может иметь только одну связанную с ней зону регистрации.

## <a name="resolution-virtual-network"></a>Разрешение виртуальной сети

При создании ссылки на виртуальную сеть в частной зоне DNS и выборе варианта не включать авторегистрацию записей DNS виртуальная сеть рассматривается как виртуальная сеть только для разрешения. Записи DNS для виртуальных машин, развернутых в таких сетях, не будут автоматически создаваться в связанной частной зоне DNS. Однако виртуальные машины, развернутые в такой сети, могут успешно запрашивать записи DNS из частной зоны DNS. Эти записи могут создаваться вручную или заполняться другими виртуальными сетями, которые были связаны как сети регистрации с частной зоной DNS.
Одна частная зона DNS может иметь несколько виртуальных сетей разрешения, и виртуальная сеть может иметь несколько связанных с ней зон разрешения.

## <a name="limits"></a>Ограничения

Чтобы понять, сколько сетей регистрации и разрешения можно связать с частными зонами DNS, см. [Azure DNS ограничения](https://docs.microsoft.com/azure/azure-resource-manager/management/azure-subscription-service-limits#azure-dns-limits) .

## <a name="other-considerations"></a>Другие замечания

* Виртуальные сети, развернутые с помощью классической модели развертывания, не поддерживаются.

* Вы можете создать только одну связь между частной зоной DNS и виртуальной сетью.

* Каждый канал виртуальной сети в частной зоне DNS должен иметь уникальное имя в контексте частной зоны DNS. Ссылки с одинаковыми именами можно использовать в разных частных зонах DNS.

* После создания ссылки на виртуальную сеть проверьте поле "состояние канала" для ресурса связи виртуальной сети. В зависимости от размера виртуальной сети может пройти несколько минут, прежде чем связь будет выполнена, а состояние связи изменится на " *завершено*".

* При удалении виртуальной сети все ссылки на виртуальные сети и автоматически зарегистрированные записи DNS, связанные с ним в разных частных зонах DNS, автоматически удаляются.
