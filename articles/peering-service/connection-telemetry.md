# <a name="peering-service-connection-telemetry"></a>Телеметрии подключения службы пиринга

Данные телеметрии подключений содержат аналитические сведения, собираемые для подключения, настроенного между расположением клиента и сетью Майкрософт. Клиенты могут получать данные телеметрии для подключения службы пиринга Azure путем регистрации подключения в портал Azure. Эта функция обеспечивает безопасность и получение сведений о производительности сети.


Данные телеметрии подключений состоят из следующих областей:

### <a name="latency-measurement"></a>Измерение задержки

 Задержка измеряется от клиента до PoP Microsoft ребра для зарегистрированных префиксов в службе пиринга.

### <a name="route-prefix-monitoring-and-protection"></a>Мониторинг и защита префикса маршрута

Пути маршрутизации отслеживаются для любых подозрительных действий, которые затем фиксируются в журналах событий. Например, журналы событий создаются для некоторых из следующих факторов.

- Захваты префиксов
- Снятие префикса
- Утечки маршрутов
