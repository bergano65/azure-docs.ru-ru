# <a name="tutorial-create-a-peering-service-connection"></a>Руководство по Создание подключения Службы пиринга

В этом руководстве показано, как создать ресурс Службы пиринга и настроить подключение Службы пиринга. 

1. Чтобы зарегистрировать подключение Службы пиринга, выберите действие  **Создать ресурс** > **Служба пиринга**.

 
    ![Регистрация в Службе пиринга](./media/peering-service-portal/peering-servicecreate.png)

2. Введите следующие сведения на вкладке **Основные сведения** страницы **Create a peering service connection** (Создание подключения Службы пиринга).
 
3. Выберите подписку и группу ресурсов, связанную с этой подпиской.

    ![Вкладка общих сведений для регистрации Службы пиринга](./media/peering-service-portal/peering-servicebasics.png)

4. В поле **Имя** введите имя для регистрации экземпляра Службы пиринга.

5. Теперь нажмите кнопку **Далее: конфигурация** в нижней части страницы. Откроется страница **Конфигурация**.
## <a name="configure-the-peering-service-connection"></a>Настройка подключения Службы пиринга

1. На странице **Конфигурация** укажите расположение, в котором вы намерены включить Службу пиринга, выбрав нужное значение в раскрывающемся списке **Peering service location** (Расположение Службы пиринга).

1. Выберите поставщика услуг, у которого нужно получить Службу пиринга, выбрав его имя в раскрывающемся списке **Peering service provider** (Поставщик Службы пиринга).
 
1. Выберите действие **Create new prefix** (Создать новый префикс) в нижней части раздела **Prefixes** (Префиксы), чтобы открыть соответствующие текстовые поля. Введите в них имя ресурса префикса и список префиксов, связанных с этим поставщиком услуг.

1. Выберите **Prefix Key** (Ключ префикса) и добавьте ключ префикса, предоставленный вам поставщиком (ISP или IXP). Этот ключ позволяет корпорации Майкрософт проверять префикс и поставщика, который выделил вам этот префикс IP-адресов.

    ![Снимок экрана: вкладка "Конфигурация" на странице "Create a peering service connection" (Создание подключения Службы пиринга), на которой можно ввести ключ префикса.](./media/peering-service-portal/peering-serviceconfiguration.png)

1. Нажмите кнопку **Просмотр и создание** в левом нижнем углу страницы. Откроется страница **Просмотр и создание**, и в Azure произойдет проверка вашей конфигурации.

 1. Когдая появится сообщение **Проверка пройдена**, щелкните **Создать**.

> ![Снимок экрана: вкладка "Просмотр и создание" на странице "Create a peering service connection" (Создание подключения Службы пиринга).](./media/peering-service-portal/peering-service-prefix.png)

1. После регистрации подключения Службы пиринга выполняется дополнительная проверка для включенных в нее префиксов. Состояние проверки можно узнать в разделе **Префиксы** для имени ресурса. Если проверка завершается неудачно, отображается одно из следующих сообщений об ошибке:

   - Invalid Peering Service prefix, the prefix should be valid format, only Ipv4 prefix is supported (Недопустимый префикс Службы пиринга, префикс должен иметь допустимый формат, поддерживается только префикс IPv4)
   - Prefix was not received from Peering Service provider (Префикс не был получен от поставщика Службы пиринга)
   - Prefix announcement does not have a valid BGP community, please contact Peering Service provider (Объявление префикса не содержит допустимого сообщества BGP. Обратитесь к поставщику Службы пиринга)
   - Backup route not found, please contact Peering Service provider (Резервный маршрут не найден. Обратитесь к поставщику Службы пиринга)
   - Prefix received with longer AS path, please contact Peering Service provider (Получен префикс с более длинным путем AS. Обратитесь к поставщику Службы пиринга)
   - Prefix received with private AS in the path, please contact Peering Service provider (Получен префикс с частным AS в пути. Обратитесь к поставщику Службы пиринга)

### <a name="add-or-remove-a-prefix"></a>Добавление или удаление префикса

Чтобы добавить префиксы, выберите **Добавить префиксы** на странице **Префиксы**.

Нажмите кнопку с многоточием (...) рядом с указанным префиксом и выберите действие **Удалить**.

### <a name="delete-a-peering-service-connection"></a>Удаление подключения Службы пиринга

На странице **Все ресурсы** установите флажок для Службы пиринга и выберите действие **Удалить** в верхней части страницы.
