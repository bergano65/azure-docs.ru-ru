# <a name="understand-azure-security-center-container-recommendations"></a>Общие сведения о рекомендациях центра безопасности Azure в отношении контейнеров

По мере миграции монолитных приложений для выполнения критически важных, полностью облачных контейнерных приложений в рабочей среде вы можете воспользоваться преимуществами контейнеров, включая простое и быстрое развертывание и обновление. Количество развернутых контейнеров продолжает увеличиваться, и вам нужны решения по обеспечению безопасности, которые предоставят вам сведения о состоянии безопасности контейнеров и помогут защитить их от угроз.

Центр безопасности Azure предоставляет следующие возможности по защите контейнеров:

- **Просмотр контейнеров, размещенных на машинах IaaS Linux**<br>В Центре безопасности Azure на вкладке "Контейнеры" отображаются все виртуальные машины, развернутые с помощью Docker. При изучении проблем с безопасностью виртуальной машины Центр безопасности предоставляет дополнительную информацию о контейнерах компьютера, такую как версия Docker и число образов, выполняемых на узле.

    ![Вкладка контейнера](./media/security-center-container-recommendations/docker-recommendation.png)


- **Рекомендации по обеспечению безопасности на основе теста производительности CIS для Docker**<br>Центр безопасности сканирует конфигурации Docker и показывает сведения о неверных настройках, предоставляя список всех невыполненных оцененных правил. Центр безопасности предоставляет рекомендации, которые помогут вам быстро устранить эти проблемы и сэкономить время. Центр безопасности непрерывно оценивает конфигурации Docker и сообщает актуальные сведения об их состоянии.

    ![Вкладка контейнера](./media/security-center-container-recommendations/container-cis-benchmark.png)

- **Обнаружение угроз для контейнера в реальном времени**<br> Центр безопасности предоставляет возможность обнаружения угроз в реальном времени для ваших контейнеров на компьютерах Linux с компонентом AuditD. Оповещения идентифицируют несколько подозрительных действий Docker, таких как создание привилегированного контейнера на узле, указание о сервере Secure Shell (SSH), выполняемом в контейнере Docker, или использование майнеров криптовалют. Эти сведения можно использовать для быстрого устранения проблем безопасности и повышения безопасности контейнеров.

    ![Вкладка контейнера](./media/security-center-container-recommendations/docker-threat-detection.png)
