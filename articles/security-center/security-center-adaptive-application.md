# <a name="adaptive-application-controls-in-azure-security-center"></a>Адаптивные элементы управления приложениями в Центре безопасности Azure
В этом руководстве приведены сведения о настройке адаптивного элемента управления приложениями в центре безопасности Azure.

## <a name="what-are-adaptive-application-controls-in-security-center"></a>Общие сведения об адаптивных элементах управления приложениями в центре безопасности Azure
Адаптивные элементы управления приложениями — это интеллектуальное автоматизированное сквозное решение для формирования списка разрешенных приложений в центре безопасности Azure. Эти элементы позволяют управлять выполнением приложений на виртуальных машинах, расположенных в Azure, что, в свою очередь, помимо других преимуществ помогает повысить уровень защиты виртуальных машин от вредоносных программ. Центр безопасности анализирует приложения, выполняющиеся на ваших виртуальных машинах, с помощью машинного обучения и на основе этих аналитических данных помогает применять конкретные правила списка разрешений. Эта возможность значительно упрощает настройку политик разрешенных приложений и управление ими, а также позволяет:

- блокировать попытки запуска вредоносных приложений (в том числе тех, которые могли пропустить антивредоносные решения) и создавать оповещения об этих попытках;
- обеспечивать соответствие политике безопасности организации, которая предписывает использовать только лицензированное программное обеспечение;
- избегать использования нежелательного программного обеспечения в среде;
- избегать выполнения старых и неподдерживаемых приложений;
- избегать использования определенных программных средств, запрещенных в организации;
- управлять доступом к конфиденциальным данным через использование приложений для ИТ-отдела.

## <a name="how-to-enable-adaptive-application-controls"></a>Как включить адаптивные элементы управления приложениями
Адаптивные элементы управления приложениями позволяют определить набор приложений, которые можно запускать в настроенных группах виртуальных машин. Эта функция доступна только на виртуальных машинах Windows (все версии, классическая модель развертывания или модель развертывания с помощью Azure Resource Manager). Чтобы настроить список разрешенных приложений в центре безопасности, можно использовать следующие шаги:

1. Откройте панель мониторинга **центра безопасности**.
2. На левой панели в разделе **Расширенная защита облака** выберите **Адаптивное управление приложениями**.

    ![Защита](./media/security-center-adaptive-application/security-center-adaptive-application-fig1-new.png)

Откроется страница **Адаптивное управление приложениями**.

![controls](./media/security-center-adaptive-application/security-center-adaptive-application-fig2.png)

В разделе **Группы виртуальных машин** содержатся три вкладки:

* **Настроено.** Список групп виртуальных машин с настроенным элементом управления приложениями.
* **Рекомендуется.** Список групп, в которых рекомендуется использовать элемент управления приложениями. Используя машинное обучение, центр безопасности определяет виртуальные машины, на которых следует использовать элемент управления приложениями, в зависимости от того, согласованно ли они используют одинаковое приложение.
* **Нет рекомендации.** Список групп виртуальных машин, для которых нет рекомендаций по использованию элемента управления приложениями. К ним, например, относятся виртуальные машины, на которых приложения постоянно изменяются, не достигая устойчивого состояния.

> [!NOTE]
> Центр безопасности использует собственный алгоритм кластеризации для создания групп виртуальных машин. Так похожие виртуальные машины гарантированно получат оптимальную рекомендуемую политику управления приложениями.
>
>

### <a name="configure-a-new-application-control-policy"></a>Настройка новой политики управления приложениями
1. Чтобы просмотреть список групп, в которых рекомендуется использовать элемент управления приложениями, перейдите на вкладку **Рекомендуется**:

  ![Рекомендуется](./media/security-center-adaptive-application/security-center-adaptive-application-fig3.png)

  Этот список содержит следующие параметры:

  - **Имя.** Название подписки и группы.
  - **Виртуальные машины.** Число виртуальных машин в группе.
  - **Состояние** — состояние рекомендаций.
  - **Серьезность.** Уровень серьезности рекомендаций.

2. Выберите группу, чтобы открыть элемент **Создание правил управления приложениями**.

  ![Правила управления приложениями](./media/security-center-adaptive-application/security-center-adaptive-application-fig4.png)

3. В разделе **Выбрать виртуальные машины** просмотрите список рекомендуемых виртуальных машин и снимите флажки рядом с теми, к которым не следует применять политику разрешения приложений. Вы увидите два списка:

  - **Рекомендуемые приложения** — список приложений, которые часто используются на виртуальных машинах в пределах этой группы; их можно запускать.
  - **Дополнительные приложения** — список приложений, которые менее часто используются на виртуальных машинах в пределах этой группы. Так как они подвержены злонамеренному использованию (см. ниже), рекомендуется проверять их.

4. Просмотрите приложения в каждом из списков и снимите флажки рядом с теми, которые не нужно использовать. Каждый список содержит следующие параметры:

  - **Имя**. Сведения о сертификате или полный путь приложения.
  - **Типы файлов.** Тип файла приложения. Это может быть EXE-файл, скрипт, MSI-файл или любая вариация этих типов.
  - **Подвержено злонамеренному использованию**. Значок предупреждения отображается рядом с теми приложениями, с помощью которых злоумышленники могут обходить решение для списка разрешенных приложений. Мы рекомендуем просмотреть эти приложения перед их утверждением.
  - **Пользователи.** Пользователи, которым рекомендуется разрешить запуск приложения.

5. Выбрав все необходимые элементы, нажмите **Создать**. <br>
После этого центр безопасности Azure автоматически создает соответствующие правила на основе встроенного решения для списка разрешенных приложений, доступного на серверах Windows (AppLocker).


> [!NOTE]
> - Центр безопасности использует данные как минимум за две недели в целях создания базовой инфраструктуры и указания уникальных рекомендаций на каждую группу виртуальных машин. Новым клиентам центра безопасности уровня "Стандартный" следует ожидать, что сначала их группы виртуальных машин отобразятся на вкладке *Нет рекомендации*.
> - Адаптивные элементы управление приложениями в Центре безопасности не поддерживают виртуальные машины, для которых политика AppLocker уже включена объектом групповой политики или локальной политикой безопасности.
> -  По соображениям безопасности центр безопасности всегда пытается создать правило издателя для приложений, которые добавляются в список разрешенных. Если приложение не содержит сведений об издателе (то есть не подписано), создается правило полного пути определенного приложения.
>   

### <a name="editing-and-monitoring-a-group-configured-with-application-control"></a>Изменение и мониторинг групп, настроенных с элементом управления

1. Чтобы изменить группу с политикой разрешенных приложений и наблюдать за ней, вернитесь на страницу **Адаптивное управление приложениями** и в разделе **Группы виртуальных машин** щелкните **Настроено**:

  ![Группы](./media/security-center-adaptive-application/security-center-adaptive-application-fig5.png)

  Этот список содержит следующие параметры:

  - **Имя**. Имя подписки и группы.
  - **Виртуальные машины.** Число виртуальных машин в группе.
  - **Режим**. В режиме аудита попытки запуска приложений не из списка разрешений записываются в журнал, а в режиме принудительного выполнения их запуск блокируется.
  - **Оповещения**. Все текущие нарушения.

2. На странице **Изменение политики управления приложениями** выберите группу, которую следует изменить.

  ![Защита](./media/security-center-adaptive-application/security-center-adaptive-application-fig6.png)

3. В разделе **Режим защиты** вы можете выбрать один из следующих параметров:

  - **Аудит.** В этом режиме решение по управлению приложениями не применяет правила принудительно, а только выполняет аудит действий на защищенных виртуальных машинах. Мы советуем применять этот режим в сценариях, где сначала нужно оценить общее поведение приложения, прежде чем блокировать его выполнение на целевой виртуальной машине.
  - **Принудительное применение.** В этом режиме решение по управлению приложениями принудительно применяет правила и блокирует неразрешенные приложения.

   > [!NOTE]
   > Как уже отмечалось ранее, новая политика управления приложениями по умолчанию применяется в режиме *аудита*. 
   >

4. В разделе **Расширение политики** вы можете добавить собственные пути к приложениям, которые следует добавить в список разрешений. Когда вы добавите эти пути, центр безопасности обновляет политику разрешенных приложений на виртуальных машинах в выбранной группе виртуальных машин и создает соответствующие правила для этих приложений в дополнение к правилам, которые уже существуют.

5. Просмотрите текущие нарушения, перечисленные в разделе **Последние оповещения**. Щелкните каждую строку, чтобы перейти на страницу **Оповещения** в центре безопасности Azure и просмотреть все оповещения, обнаруженные с помощью центра безопасности Azure на связанных виртуальных машинах.
  - **Оповещения**: все нарушения, которые были зарегистрированы.
  - **Число виртуальных машин**: число виртуальных машин с этим типом проблемы.

6. В разделе **Правила списка разрешенных издателей**, **Правила для списка разрешенных путей** и **Правила для списка разрешенных хэшей** вы увидите правила для разрешенных приложений, которые сейчас настроены на виртуальных машинах в группе, в соответствии с типом коллекции правил. Для каждого правила можно просмотреть следующие сведения.

  - **Правило**: определенные параметры, согласно которым приложение проверяется в AppLocker для определения того, разрешен ли его запуск.
  - **Тип файла**: типы файлов, связанные с определенным правилом. Это может быть EXE, скрипт, MSI или любая вариация этих типов файлов.
  - **Пользователи**: имя или число пользователей, которым разрешено запускать приложение, на которое распространяются правила для разрешенных приложений.

   ![Правила списка разрешений](./media/security-center-adaptive-application/security-center-adaptive-application-fig9.png)

7. Щелкнув многоточие в конце каждой строки, можно удалить конкретное правило или изменить пользователей.

8. После внесения изменений в политику **адаптивного управления приложениями** нажмите кнопку **Сохранить**.

### <a name="not-recommended-list"></a>Список нерекомендуемых приложений

Центр безопасности рекомендует применять политики разрешенных приложений только для виртуальных машин, где используется стабильный набор приложений. Если приложения на связанных виртуальных машинах постоянно изменяются, рекомендации не создаются.

![Рекомендации](./media/security-center-adaptive-application/security-center-adaptive-application-fig11.png)

Этот список содержит следующие параметры:
- **Имя.** Название подписки и группы.
- **Виртуальные машины.** Число виртуальных машин в группе.

Центр безопасности Azure позволяет также определить политику разрешенных приложений для нерекомендуемых групп виртуальных машин. Следуйте тем же принципам, что были описаны выше, чтобы настроить политику разрешенных приложений и в этих группах.
