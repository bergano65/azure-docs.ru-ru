# <a name="ueba-for-azure-resources-and-users"></a>Аналитика поведения пользователей и сущностей для ресурсов и пользователей Azure 

Центр безопасности Azure использует Microsoft Cloud App Security, чтобы предоставлять оповещения на основе аналитики поведения пользователей и сущностей (UEBA) для ресурсов и пользователей Azure (действия в Azure). Эти оповещения сообщают об аномалиях в поведении пользователей. Они основаны на аналитике поведения пользователей и сущностей, а также машинном обучении и позволяют вам сразу запустить расширенное обнаружение угроз в действиях подписок. Автоматически активируемая функция обнаружения новых аномалий немедленно предоставляет результаты, распознавая многочисленные аномалии поведения пользователей и ресурсов, связанных с вашей подпиской. Кроме того, эти оповещения используют дополнительные данные, которые уже существуют в механизме обнаружения Microsoft Cloud App Security, ускоряя процесс исследования и обнаружения текущих угроз. 

> [!NOTE]
> Центр безопасности Azure может хранить копию данных безопасности клиента, собранных с ресурса клиента или связанных с ним (например, виртуальной машиной или клиентом Azure Active Directory): а) в регионе ресурса, за исключением тех регионов, в которых корпорация Майкрософт еще не развернула центр безопасности Azure (в этом случае копия таких данных будет храниться в США); б) в расположении, где центр безопасности Azure использует другую службу Microsoft Online Service для обработки таких данных, он может хранить такие данные в соответствии с правилами геолокации этой службы.

## <a name="prerequisites"></a>Предварительные требования

- Действительная, активированная [лицензия Microsoft Cloud App Security](https://docs.microsoft.com/cloud-app-security/getting-started-with-cloud-app-security).
- [Центр безопасности ценовой категории "Стандартный"](https://azure.microsoft.com/pricing/details/security-center/).
 
## <a name="threat-detection-alerts"></a>Оповещения об обнаружении угроз

Центр безопасности поддерживает приведенные ниже оповещения об обнаружении аномалий Cloud App Security.

**Неосуществимое перемещение**
-  Это обнаружение идентифицирует два действия пользователя (в одном или нескольких сеансах), выполняемых в географически отдаленных расположениях в течение периода времени, недостаточного для того, чтобы переместиться из первого расположения во второе. Оно указывает на то, что другой пользователь использует те же учетные данные. Это обнаружение использует алгоритм машинного обучения, который игнорирует очевидные "ложные срабатывания", такие как VPN и расположения, которые постоянно используют другие пользователи в организации, и позволяет получить результаты, соответствующие невозможным условиям перемещения. В обнаружении предусмотрен первоначальный период обучения — 7 дней, в течение которых оно изучает последовательность действий нового пользователя.

**Действие из редко упоминаемой страны**
- Это обнаружение изучает расположения, где выполнялись предыдущие действия, чтобы определить новые редко упоминаемые расположения. Механизм обнаружения аномалий хранит информацию о предыдущих расположениях, используемых пользователями в организации. Оповещение активируется при выполнении действия в расположении, которое ни один пользователь организации давно или же никогда не посещал. 

**Действия, выполняемые с анонимных IP-адресов**
- Эта функция обнаруживает, что пользователи использовали IP-адрес, который был идентифицирован как анонимный IP-адрес прокси-сервера. Эти прокси-серверы используют пользователи, желающие скрыть IP-адреса своих устройств. Иногда их используют злоумышленники. Это обнаружение использует алгоритм машинного обучения, который уменьшает количество "ложных срабатываний" (например, ошибочно помеченных IP-адресов, которые часто используются корпоративными пользователями).
 
  ![оповещение об обнаружении угроз](./media/security-center-ueba-mcas/security-center-mcas-alert.png)

## <a name="disabling-threat-detection-alerts"></a>Отключение оповещений об обнаружении угроз

Эти оповещения включены по умолчанию, но их можно отключить.

1. В колонке "Центр безопасности" выберите **Политика безопасности**. Для подписки, которую нужно изменить, выберите **Изменить параметры**.
2.  Щелкните **Обнаружение угроз**.
3. В разделе **Enable integrations** (Включить интеграцию) снимите флажок **Allow Microsoft Cloud App Security to access my data** (Разрешить Microsoft Cloud App Security доступ к моим данным) и щелкните **Сохранить**.

   ![оповещение об обнаружении угроз](./media/security-center-ueba-mcas/security-center-mcas-optout.png)

> [!NOTE]
> Начальный период обучения составляет семь дней, в течение которых активируются не все оповещения об обнаружении аномалий. После этого каждый сеанс сравнивается с действиями, выполняемыми пользователями, IP-адресами, устройствами и т. д., обнаруженными за последний месяц, а также с оценкой риска этих действий. Эти обнаружения являются частью механизма обнаружения аномалий машинного обучения, который оценивает вашу среду и активирует оповещения в соответствии с изученными базовыми тенденциями вашей организации. Кроме того, обнаружения используют алгоритмы машинного обучения, предназначенные для оценки пользователей и шаблона входа, чтобы уменьшить количество ложных срабатываний.
