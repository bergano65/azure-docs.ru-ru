# <a name="protect-app-service-with-azure-security-center"></a>Защита службы приложений с помощью центра безопасности Azure
Эта статья позволяет вам использовать центр безопасности Azure для мониторинга и защиты своих приложений, работающих на базе службы приложений.

Служба приложений позволяет создавать и размещать веб-приложения на любых языках программирования без необходимости управлять инфраструктурой. Служба приложений поддерживает Windows и Linux, а также автоматическое масштабирование, высокий уровень доступности и автоматическое развертывание из GitHub, Azure DevOps или любого репозитория Git. 

Уязвимостями веб-приложений часто пользуются злоумышленники, так как они обладают распространенным и динамическим интерфейсом почти ко всем организациям в Интернете. Запросы к приложениям, работающим на основе службы приложений, проходят через несколько шлюзов, развертываемых в центрах обработки данных Azure по всему миру, ответственных за маршрутизацию каждого запроса к соответствующему приложению. 

Центр безопасности Azure может выполнять оценки и предоставлять рекомендации в приложениях, работающих в службе приложений, в песочнице, в виртуальной машине или в зарезервированных экземплярах. Благодаря прозрачности Azure как поставщика облачных служб центр безопасности анализирует внутренние журналы службы приложений для мониторинга распространенных атак на веб-приложения, которые часто направлены на несколько целей.

Центр безопасности использует масштабирование облака для выявления атак на приложения службы приложений и сосредотачивается на новых атаках, пока злоумышленники находятся на этапе разведки и выполняют сканирование для выявления уязвимостей на нескольких веб-сайтах, размещенных в Azure. Центр безопасности использует аналитику и модели машинного обучения, чтобы охватить все интерфейсы, позволяя клиентам взаимодействовать с приложениями по протоколу HTTP или с помощью методов управления. Более того, являясь службой в Azure, получающей данные напрямую из источника, центр безопасности также находится в уникальной позиции, позволяющей предлагать размещенную на узле аналитику по вопросам безопасности, охватывая базовые вычислительные узлы для этого PaaS, позволяя центру безопасности обнаруживать атаки на веб-приложения, которыми уже воспользовались.

## <a name="prerequisites"></a>Предварительные требования

Для контроля и защиты службы приложений вам потребуется план службы приложений, который связан с выделенными виртуальными машинами. Это такие планы: "Базовый", "Стандартный", "Премиум", "Изолированный" или Linux. Центр безопасности Azure не поддерживает планы уровня "Бесплатный", "Общий" или план использования. Дополнительные сведения см. в статье [Планы службы приложений](https://azure.microsoft.com/pricing/details/app-service/plans/).

## <a name="security-center-protection"></a>Защита центра безопасности

Центр безопасности Azure защищает экземпляр виртуальной машины, в которой выполняется служба приложений и находится интерфейс управления. Он также отслеживает запросы и ответы, отправляемые и принимаемые приложением, работающим в службе приложений.

Центр безопасности изначально интегрирован со службой приложений, что устраняет необходимость развертывания и адаптации. Интеграция полностью прозрачна.



## <a name="enabling-monitoring-and-protection-of-app-service"></a>Включение мониторинга и защиты службы приложений

1. Выберите центр безопасности в Azure.
2. Выберите **Политика безопасности** и выберите подписку.
3. В конце строки подписки выберите **Изменить параметры**.
4. В разделе **Ценовая категория** строки **Служба приложений** переведите план в состояние **Включено**.

![Переключатель службы приложений](./media/security-center-app-services/app-services-toggle.png)

>[!NOTE]
> Количество экземпляров, перечисленных для вашего количества ресурсов, представляет собой количество соответствующих экземпляров службы приложений, активных в момент, когда вы открыли колонку ценовой категории. Так как это число может меняться в зависимости от выбранных параметров масштабирования, количество экземпляров, которые вы оплачиваете, изменится соответствующим образом.

Чтобы отключить мониторинг и рекомендации для службы приложений, повторите этот процесс и переключите план **службы приложений** в значение **Отключено**.
