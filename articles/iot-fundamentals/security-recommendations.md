# <a name="security-recommendations-for-azure-internet-of-things-iot-deployment"></a>Рекомендации по безопасности для развертывания Azure "Интернет вещей" (IoT)

Эта статья содержит рекомендации по безопасности для Интернета вещей. Реализация этих рекомендаций поможет вам выполнить обязательства по обеспечению безопасности, которые описаны в нашей модели общей ответственности. Дополнительные сведения о принимаемых Майкрософт мерах для выполнения обязанностей поставщика услуг см. в статье [об общей ответственности в отрасли облачных вычислений](https://gallery.technet.microsoft.com/Shared-Responsibilities-81d0ff91).

Выполнение некоторых из описанных в этой статье рекомендаций можно автоматически отслеживать с помощью Центра безопасности Azure. Центр безопасности Azure служит первой линией обороны ваших ресурсов в Azure. Он периодически анализирует состояние безопасности для ресурсов Azure, чтобы выявить потенциальные уязвимости. Затем он предоставляет рекомендации по их устранению.

- Дополнительные сведения о рекомендациях Центра безопасности Azure см. в [этой статье](../security-center/security-center-recommendations.md).
- Дополнительные сведения о самом Центре безопасности Azure см. в статье [Что такое Центр безопасности Azure?](../security-center/security-center-introduction.md)

## <a name="general"></a>Общие сведения

| Рекомендация | Комментарии | Поддерживается по УБЫВАНИЮ |
|-|----|--|
| Последние новости | Используйте последние версии поддерживаемых платформ, языков программирования, протоколов и платформ. | - |
| Обеспечение безопасности ключей проверки подлинности | Обеспечьте физическую безопасность идентификаторов устройств и их ключей проверки подлинности после развертывания. Это позволит избежать имитации вредоносного устройства в качестве зарегистрированного устройства. | - |
| Используйте пакеты SDK для устройств, когда это возможно | Пакеты SDK для устройств реализуют различные функции безопасности, такие как шифрование, проверка подлинности и т. д., которые помогут вам в разработке надежного и безопасного приложения для устройств. Дополнительные сведения см. в статье [изучение и использование пакетов SDK для центра Интернета вещей Azure](../iot-hub/iot-hub-devguide-sdks.md) . | - |

## <a name="identity-and-access-management"></a>Управление удостоверениями и доступом 

| Рекомендация | Комментарии | Поддерживается по УБЫВАНИЮ |
|-|----|--|
| Определение контроля доступа для концентратора | [Изучите и определите тип доступа](iot-security-deployment.md#securing-the-cloud) каждого компонента в решении центра Интернета вещей на основе функциональных возможностей. Допустимые разрешения: *Чтение реестра*, *RegistryReadWrite*, *ServiceConnect*и *DeviceConnect*. [Политики общего доступа по умолчанию в центре Интернета вещей](../iot-hub/iot-hub-devguide-security.md#access-control-and-permissions) также могут помочь определить разрешения для каждого компонента в зависимости от его роли. | - |
| Определение контроля доступа для серверных служб | Данные, принимаемые решением центра Интернета вещей, можно использовать в других службах Azure, таких как [Cosmos DB](../cosmos-db/index.yml), [Stream Analytics](../stream-analytics/index.yml), [служба приложений](../app-service/index.yml), [Logic Apps](../logic-apps/index.yml)и [хранилище BLOB-объектов](../storage/blobs/storage-blobs-introduction.md). Убедитесь, что вы понимаете и разрешите соответствующие разрешения на доступ, как описано для этих служб. | - |

## <a name="data-protection"></a>Защита данных

| Рекомендация | Комментарии | Поддерживается по УБЫВАНИЮ |
|-|----|--|
| Безопасная проверка подлинности устройства | Обеспечьте безопасную связь между устройствами и центром Интернета вещей с помощью [уникального ключа удостоверения или токена безопасности](iot-security-deployment.md#iot-hub-security-tokens)или [сертификата X. 509 на устройстве](iot-security-deployment.md#x509-certificate-based-device-authentication) для каждого устройства. Используйте соответствующий метод для [использования маркеров безопасности на основе выбранного протокола (MQTT, AMQP или HTTPS)](../iot-hub/iot-hub-devguide-security.md). | - |
| Безопасное взаимодействие устройств | Центр Интернета вещей обеспечивает безопасность подключения к устройствам по стандарту TLS, поддерживающим версии 1,2 и 1,0. Используйте [TLS 1,2](https://tools.ietf.org/html/rfc5246) для обеспечения максимальной безопасности. | - |
| Защита взаимодействия со службой | Центр Интернета вещей предоставляет конечные точки для подключения к серверным службам, таким как служба [хранилища Azure](../storage/index.yml) или [концентраторы событий](../event-hubs/index.yml) , используя только протокол TLS, а конечная точка не предоставляется в незашифрованном канале. Когда эти данные поступает в эти серверные службы для хранения или анализа, необходимо применить соответствующие методы обеспечения безопасности и шифрования для этой службы, а также защитить конфиденциальные данные в серверной части. | - |

## <a name="networking"></a>Сеть

| Рекомендация | Комментарии | Поддерживается по УБЫВАНИЮ |
|-|----|--|
| Защита доступа к устройствам | Чтобы избежать нежелательного доступа, обеспечьте минимальное число устройств для аппаратных портов на своих устройствах. Кроме того, механизмы сборки для предотвращения или обнаружения физического изменения устройства. Дополнительные сведения см. в статье [рекомендации по обеспечению безопасности IOT](iot-security-best-practices.md) . | - |
| Создание безопасного оборудования | Реализуйте такие функции безопасности, как зашифрованное хранилище или доверенный платформенный модуль (TPM) (доверенный платформенный модуль), чтобы обеспечить безопасность устройств и инфраструктуры. Обеспечьте обновление операционной системы и драйверов устройства до последних версий, а также, если место позволяет, установить антивирусные и Антивредоносные функции. Ознакомьтесь с [архитектурой безопасности IOT](iot-security-architecture.md) , чтобы понять, как это может помочь устранить несколько угроз безопасности. | - |

## <a name="monitoring"></a>Наблюдение

| Рекомендация | Комментарии | Поддерживается по УБЫВАНИЮ |
|-|----|--|
| Отслеживание несанкционированного доступа к устройствам |  Используйте функцию ведения журнала операционной системы устройства для отслеживания нарушений безопасности или физического изменения устройства или его портов. | - |
| Мониторинг решения Интернета вещей в облаке | Отслеживайте общую работоспособность решения центра Интернета вещей с помощью [метрик в Azure Monitor](../iot-hub/iot-hub-metrics.md). | - |
| Настройка диагностики | Внимательно наблюдайте за операциями, записывая в решение события, а затем отправляя журналы диагностики в Azure Monitor, чтобы получить представление о производительности. Дополнительные сведения см. [в статье мониторинг и диагностика проблем в центре Интернета вещей](../iot-hub/iot-hub-monitor-resource-health.md) . | - |
