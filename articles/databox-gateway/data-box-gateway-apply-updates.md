# <a name="update-your-azure-data-box-gateway"></a>Обновление Шлюз Azure Data Box

В этой статье описываются шаги, необходимые для установки обновления на Шлюз Azure Data Box через локальный веб-интерфейс и в портал Azure. Вы применяете обновления программного обеспечения или исправления, чтобы обновлять Шлюз Data Box устройства.

> [!IMPORTANT]
>
> - Обновление **1911** соответствует версии программного обеспечения **1.6.1049.786** на вашем устройстве. Дополнительные сведения об этом обновлении см. в [заметках о выпуске](data-box-gateway-1911-release-notes.md).
>
> - Помните, что при установке обновления или исправления происходит перезагрузка устройства. Поскольку Шлюз Data Box является устройством с одним узлом, все выполняющиеся на нем операции ввода-вывода прерываются, а само устройство находится в состоянии простоя до 30 минут для обновления его программного обеспечения.

Каждое из этих действий описано в следующих разделах.

## <a name="use-the-azure-portal"></a>Использование портала Azure

Рекомендуется устанавливать обновления с помощью портал Azure. Устройство автоматически сканируется на наличие обновлений раз в день. Когда обновления будут доступны, на портале отобразится уведомление. После этого обновления можно загрузить и установить.

> [!NOTE]
> Убедитесь, что устройство работоспособно, а состояние отображается как **подключено** , прежде чем продолжать установку обновлений.

1. Когда обновления будут доступны для вашего устройства, вы увидите уведомление. Выберите уведомление или на верхней панели команд **Обновите устройство**. Это позволит применить обновления программного обеспечения устройства.

    ![Снимок экрана домашней страницы Шлюз Azure Data Box с вызываемыми параметрами "Обзор" и "обновить устройство".](./media/data-box-gateway-apply-updates/portal-apply-update-01a.png)

2. В колонке **обновления устройства** проверьте условия лицензии, связанные с новыми функциями в заметках о выпуске.

    Вы можете **загрузить и установить** обновления или просто **скачать** обновления. а затем установить позже.

    ![Снимок экрана: диалоговое окно "обновления устройства" с параметрами "принять" и "понимать", а также параметром "скачать и установить".](./media/data-box-gateway-apply-updates/portal-apply-update-02.png)

    Если вы хотите загрузить и установить обновления, установите флажок автоматически обновлять установку после завершения загрузки.

    ![Снимок экрана: диалоговое окно "обновления устройства" с параметром "принять" и параметром загрузки "вызываемый".](./media/data-box-gateway-apply-updates/portal-apply-update-03.png)

3. Начнется загрузка обновлений. Появится уведомление о том, что скачивание выполняется.

    ![Снимок экрана с сообщением "скачивание и установка обновлений выполняется".](./media/data-box-gateway-apply-updates/portal-apply-update-05.png)

    Баннер уведомления также отображается в портал Azure. Это указывает на ход выполнения загрузки.

    ![Снимок экрана домашней страницы Шлюз Azure Data Box с параметром обзор и заголовком уведомления о скачивании обновлений.](./media/data-box-gateway-apply-updates/portal-apply-update-08a.png)

    Чтобы просмотреть подробное состояние обновления, можно выбрать это уведомление или **обновить устройство** .

    ![Снимок экрана: диалоговое окно "скачать и установить обновления" с сообщением о ходе выполнения и сообщением скачать обновления, вызываемыми.](./media/data-box-gateway-apply-updates/portal-apply-update-09.png)

4. По завершении загрузки баннер уведомления обновится и сообщит о завершении. Если вы решили загрузить и установить обновления, установка начнется автоматически.

    ![Снимок экрана домашней страницы Шлюз Azure Data Box с параметром обзора и заголовком уведомления о скачанных обновлениях.](./media/data-box-gateway-apply-updates/portal-apply-update-10a.png)

    Если выбрана загрузка только обновлений, выберите уведомление, чтобы открыть колонку **обновления устройства** . Выберите пункт **Установить**.
  
    ![Снимок экрана: диалоговое окно "обновления устройства" с вызываемым параметром "установить".](./media/data-box-gateway-apply-updates/portal-apply-update-11a.png)

5. Появится уведомление о том, что выполняется установка.

    ![Снимок экрана домашней страницы Шлюз Azure Data Box с параметром обзор и сообщением о ходе выполнения загрузки и установки.](./media/data-box-gateway-apply-updates/portal-apply-update-12a.png)

    На портале также отображается информационное оповещение, указывающее на то, что установка выполняется. <!-- The device goes offline and is in maintenance mode.-->

    <!-- ![Software version after update](./media/data-box-gateway-apply-updates/update-13.png)-->

6. Так как это устройство с 1 узлом, устройство будет перезагружено после установки обновлений. Критическое оповещение во время перезагрузки указывает на то, что пульс устройства будет потерян.

    ![Снимок экрана домашней страницы Шлюз Azure Data Box с параметром обзор и заголовком критического уведомления об оповещениях.](./media/data-box-gateway-apply-updates/portal-apply-update-19a.png)

    Выберите оповещение, чтобы просмотреть соответствующее событие устройства.

    ![Снимок экрана раздела событий устройства с потерянным пульсом от события устройства.](./media/data-box-gateway-apply-updates/portal-apply-update-20a.png)

7. После установки обновлений состояние устройства изменится на "в **сети** ".

    ![Снимок экрана домашней страницы Шлюз Azure Data Box с параметром обзора и состоянием «в сети», именуемым «out».](./media/data-box-gateway-apply-updates/portal-apply-update-23a.png)

    На верхней панели команд выберите **обновления устройства**. Убедитесь, что обновление успешно установлено, а версия программного обеспечения устройства отражает это.

    ![Снимок экрана: диалоговое окно "обновления устройства" с вызываемым разделом "версия установленного программного обеспечения".](./media/data-box-gateway-apply-updates/portal-apply-update-24.png)

## <a name="use-the-local-web-ui"></a>Использование локального веб-интерфейса

При использовании локального веб-интерфейса установка обновлений включает два этапа:

- Загрузка обновления или исправления
- Установка обновления или исправления

В следующих разделах каждый этап рассматривается более подробно.

### <a name="download-the-update-or-the-hotfix"></a>Загрузка обновления или исправления

Чтобы скачать обновление, выполните следующие действия. Обновление можно загрузить из расположения или каталога Центр обновления Майкрософт, предоставляемого корпорацией Майкрософт.

Чтобы скачать обновление из каталога Центр обновления Майкрософт, выполните следующие действия.

1. Запустите браузер и перейдите по адресу [https://catalog.update.microsoft.com](https://catalog.update.microsoft.com) .

   ![Снимок экрана Центр обновления Майкрософт каталога в окне браузера с Шлюз Data Box ввода в текстовом поле поиска. Выводятся как адресная строка браузера, так и текстовое поле поиска.](./media/data-box-gateway-apply-updates/download-update-1.png)

2. В поле поиска каталога Центр обновления Майкрософт введите номер исправления или условия для обновления, которое вы хотите скачать, в базе знаний Майкрософт (KB). Например, введите **шлюз Azure Data Box** и нажмите кнопку **Поиск**.

   В списке обновлений отображается **Шлюз Azure Data Box 1911**.

   ![Снимок экрана Центр обновления Майкрософт каталога в окне браузера с Шлюз Data Box отображаемых и вызываемых результатов поиска.](./media/data-box-gateway-apply-updates/download-update-2.png)

3. Щелкните **Скачать**. Существует один файл для загрузки с именем *SoftwareUpdatePackage.exe* , соответствующий обновлению программного обеспечения устройства. Скачайте файл в папку в локальной системе. Эту папку также можно скопировать в сетевую папку, доступную с вашего устройства.

   ![Снимок экрана: диалоговое окно загрузки со ссылкой на файл с пакетом обновления программного обеспечения E X E с именем out.](./media/data-box-gateway-apply-updates/download-update-3.png)

### <a name="install-the-update-or-the-hotfix"></a>Установка обновления или исправления

Перед обновлением или установкой исправлений убедитесь, что:

- Обновление или исправление скачано локально на вашем узле или доступно в сетевой папке.
- Состояние устройства — работоспособное, как показано на странице **Обзор** локального веб-интерфейса.

   ![Снимок экрана Шлюз Data Box локального веб-интерфейса U с параметром панели мониторинга и состоянием программного обеспечения: работоспособное сообщение с именем out.](./media/data-box-gateway-apply-updates/local-ui-update-1.png)

Выполнение этой процедуры займет около 20 минут. Для установки обновления или исправления выполните следующие действия.

1. В локальном веб-интерфейсе перейдите в раздел **обслуживание**  >  **Обновление программного обеспечения**. Запишите версию программного обеспечения, которое выполняется в вашей системе.

   ![Снимок экрана Шлюз Data Box локального веб-сайта U с параметром обновления программного обеспечения и сообщением о текущей версии программного обеспечения, именуемым out.](./media/data-box-gateway-apply-updates/local-ui-update-2.png)

2. Укажите путь к файлу обновления. Можно также перейти к файлу установки обновления, если он размещен в общей сетевой папке. Выберите файл обновления программного обеспечения с суффиксом *SoftwareUpdatePackage.exe* .

   ![Снимок экрана: проводник с пакетом обновления программного обеспечения с именем E X E.](./media/data-box-gateway-apply-updates/local-ui-update-3.png)

3. Нажмите кнопку **Применить**.

   ![Снимок экрана: Шлюз Data Box локальный веб-сайт U с параметром обновления программного обеспечения, текстовое поле путь к файлу обновления и параметр применить обновление вызываемым.](./media/data-box-gateway-apply-updates/local-ui-update-4.png)

4. При появлении запроса на подтверждение выберите **Да** для продолжения. Если устройство является устройством с одним узлом, после применения обновления устройство перезапускается и происходит простоя.

   ![Снимок экрана диалогового окна "обновление программного обеспечения" с выделенным параметром "Да".](./media/data-box-gateway-apply-updates/local-ui-update-5.png)

5. Начнется обновление. После успешного обновления устройство перезапускается. В течение этого времени локальный пользовательский интерфейс недоступен.

6. После перезапуска открывается страница **входа** . Чтобы убедиться, что программное обеспечение устройства Обновлено, в локальном веб-интерфейсе перейдите в раздел **обслуживание**  >  **Обновление программного обеспечения**. Отображаемая версия программного обеспечения в этом примере — **1.6.1049.786**.

   ![Снимок экрана Шлюз Data Box локального веб-сайта U с параметром обновления программного обеспечения и обновленным сообщением о текущей версии программного обеспечения.](./media/data-box-gateway-apply-updates/local-ui-update-6.png)
