# <a name="administer-your-iot-central-application"></a>Администрирование приложения IoT Central

После создания приложения IoT Central можно перейти в раздел **Администрирование** для выполнения таких задач:

- управлять параметрами приложения;
- Управление пользователями
- управлять ролями;
- Просмотр счетов
- Преобразование бесплатной пробной версии в версию с оплатой по мере использования
- Экспорт данных
- Управление подключением устройств
- Использование маркеров доступа

Чтобы получить доступ к разделу **Администрирование**, вам должна быть назначена роль **Администратор** приложения Azure IoT Central. Если вы создаете приложение Azure IoT Central, вам автоматически назначается роль **Администратор** для этого приложения. В разделе [Управление пользователями](#manage-users) в этой статье объясняется, как назначить роль **Администратор** другим пользователям.

## <a name="manage-application-settings"></a>управлять параметрами приложения;

### <a name="change-application-name-and-url"></a>Изменение имени приложения и URL-адреса
На странице **Параметры приложения** можно изменить имя и URL-адрес приложения, а затем выбрать **Сохранить**.

![Страница "Параметры приложения"](media/howto-administer-experimental/image0-a.png)

> [!Note]
> Если URL-адрес будет изменен, старый URL-адрес может быть занят другим клиентом Azure IoT Central. В этом случае он будет недоступен для использования. При изменении URL-адреса старый URL-адрес больше нельзя будет использовать, поэтому вам нужно уведомить пользователей о новом URL-адресе.

### <a name="prepare-and-upload-image"></a>Подготовка и передача изображения
Чтобы изменить изображение приложения, ознакомьтесь со статьей [Подготовка и передача изображений в приложение Azure IoT Central](howto-prepare-images-experimental.md?toc=/azure/iot-central-experimental/toc.json&bc=/azure/iot-central-experimental/breadcrumb/toc.json).

### <a name="copy-an-application"></a>Копирование приложения
Можно создать копию любого приложения без экземпляров устройства, журнала данных устройства и данных пользователя. Копия будет приложением с оплатой по мере использования, за использование которого будут выставляться счета. Таким образом, нельзя создать бесплатную пробную версию приложения.

Нажмите кнопку **Копировать**. В диалоговом окне введите сведения для нового приложения с оплатой по мере использования. Затем нажмите кнопку **Копировать**, чтобы подтвердить, что вы хотите продолжить. Дополнительные сведения о полях в этой форме см. в руководстве [Создание приложения Azure IoT Central](quick-deploy-iot-central-experimental.md?toc=/azure/iot-central-experimental/toc.json&bc=/azure/iot-central-experimental/breadcrumb/toc.json).

![Страница "Параметры приложения"](media/howto-administer-experimental/appCopy2.png)

Когда приложение будет скопировано, можно перейти к нему по появившейся ссылке.

![Страница "Параметры приложения"](media/howto-administer-experimental/appCopy3.png)

> [!Note]
> При копировании приложения также копируются определение правил или действий. Так как пользователи с правом доступа к вашему исходному приложению не копируются в скопированное приложение, вам необходимо вручную разрешить пользователям выполнять такие действия, как отправка сообщений по электронной почте, для чего назначение пользователей является необходимым. Как правило, рекомендуется проверить правила и действия, чтобы убедиться, что они обновлены в новом приложении.

### <a name="delete-an-application"></a>Удаление приложения

> [!Note]
> Чтобы удалить приложение, у вас также должны быть разрешения на удаление ресурсов в подписке Azure, которую вы выбрали при создании приложения. Дополнительные сведения см. в статье [Использование управления доступом на основе ролей для контроля доступа к ресурсам в подписке Azure](https://docs.microsoft.com/azure/active-directory/role-based-access-control-configure).

Нажмите кнопку **Удалить**, чтобы окончательно удалить приложение IoT Central. Это приведет к окончательному удалению всех данных, которые связаны с этим приложением.

## <a name="manage-users"></a>Управление пользователями

### <a name="add-users"></a>Добавление пользователей

Каждый пользователь должен иметь учетную запись, чтобы войти в систему и получить доступ к приложению Azure IoT Central. Учетные записи MSA и Azure Active Directory поддерживаются в Azure IoT Central. Группы Azure Active Directory сейчас не поддерживаются в Azure IoT Central.

Дополнительные сведения см. в статье [Общие сведения об учетной записи Майкрософт](https://support.microsoft.com/products/microsoft-account?category=manage-account) и [Практическое руководство. Добавление или удаление пользователей в Azure Active Directory](https://docs.microsoft.com/azure/active-directory/add-users-azure-active-directory).

1. Чтобы добавить пользователя в приложение IoT Central, в разделе **Администрирование** перейдите на страницу **Пользователи**.

    ![Список пользователей](media/howto-administer-experimental/image1.png)

1. На странице **Пользователи** выберите **+Добавить пользователя**.

1. Выберите роль в раскрывающемся списке **Роль**. Дополнительные сведения о ролях см. в разделе [Управление ролями](#manage-roles) этой статьи.

    ![Выбор ролей](media/howto-administer-experimental/image3.png)

    > [!NOTE]
    >  Чтобы добавить пользователей массово, введите идентификаторы соответствующих пользователей, разделив их точками с запятой. Выберите роль в раскрывающемся списке **Роль**. Затем нажмите кнопку **Save** (Сохранить).

### <a name="edit-the-roles-that-are-assigned-to-users"></a>Изменение ролей, назначенных пользователям

Роли нельзя изменить после назначения. Чтобы изменить роль, назначенную пользователю, удалите учетную запись и повторно добавьте пользователя с другой ролью.

### <a name="delete-users"></a>Удаление пользователей

Чтобы удалить пользователей, установите один или несколько флажков на странице **Пользователи**. Теперь щелкните **Удалить**.

## <a name="manage-roles"></a>управлять ролями;

Роли позволяют вам контролировать, кто в вашей организации может выполнять различные задачи в IoT Central. Вы можете назначить пользователям своего приложения три роли. 

### <a name="administrator"></a>Администратор

Пользователи с ролью **Администратор** имеют доступ ко всем возможностям в приложении.

Пользователю, создавшему приложение, автоматически назначается роль **администратора**. В роли **администратора** должен быть хотя бы один пользователь.

### <a name="application-builder"></a>Конструктор приложений

Пользователи с ролью **Конструктор приложений** могут выполнять в приложении все задачи, кроме его администрирования. Это означает, что конструкторы могут создавать, изменять и удалять шаблоны устройств и устройства, управлять наборами устройств и выполнять аналитику и задания. У конструкторов не будет доступа к разделу **Администрирование** приложения.

### <a name="application-operator"></a>Оператор приложения

Пользователи с ролью **Оператор приложений** не могут вносить изменения в шаблоны устройств и администрировать приложение. Это означает, что операторы могут добавлять и удалять устройства, управлять наборами устройств и выполнять аналитику и задания. У операторов не будет доступа к страницам **Конструктор приложений** и **Администрирование**.


## <a name="view-your-bill"></a>Просмотр счетов

Чтобы просмотреть свой счет, перейдите на страницу **Выставление счетов** в разделе **Администрирование**. В новой вкладке откроется страница выставления счетов Azure, где вы сможете просмотреть счет по каждому приложению Azure IoT Central.

### <a name="convert-your-trial-to-pay-as-you-go"></a>Преобразование бесплатной пробной версии в версию с оплатой по мере использования

Вы можете преобразовать пробную версию приложения в приложение с оплатой по мере использования. Ниже приведены различия между этими типами приложений.

- **Пробные версии** приложений предоставляются бесплатно в течение 7 дней, после чего срок их действия истекает. Их можно преобразовать для оплаты по мере использования в любое время до истечения срока действия.
- За приложения **с оплатой по мере использования** плата взимается за каждое устройство, а первые 5 устройств предоставляются бесплатно.

Узнайте больше о ценах на службу Azure IoT Central на [этой странице](https://azure.microsoft.com/pricing/details/iot-central/).
    
Для завершения этого процесса самообслуживания выполните следующие действия:

1. В разделе **Администрирование** перейдите на страницу **Выставление счетов**. 

    ![Состояние бесплатной пробной версии](media/howto-administer-experimental/freetrialbilling.png)

1. Нажмите кнопку **Convert to Pay-As-You-Go** (Преобразовать для оплаты по мере использования). 

    ![Преобразование бесплатной пробной версии](media/howto-administer-experimental/convert.png)

1. Выберите подходящий Azure Active Directory, а затем подписку Azure, которую вы хотите использовать для приложения с оплатой по мере использования.

1. После нажатия кнопки **Преобразовать** приложение преобразуется в экземпляр с оплатой по мере использования и начнется выставление счетов.

## <a name="export-data"></a>Экспорт данных

Чтобы экспортировать измерения, устройства и данные шаблонов устройства в учетную запись хранения большого двоичного объекта Azure, можно включить **непрерывный экспорт данных**. Дополнительные сведения о том, как экспортировать данные, см. в [этом разделе](#howto-export-data).

## <a name="manage-device-connection"></a>Управление подключением устройств

Подключение устройств в нужном масштабе в приложении с помощью ключей и сертификатов. Дополнительные сведения о создании устройств см. [в этом разделе](#concepts-connectivity).

## <a name="use-access-tokens"></a>Использование маркеров доступа

Создайте маркеры доступа для использования в средствах для разработчиков. В настоящее время доступен один инструмент разработки — обозреватель IoT Central, который отслеживает сообщения устройства и изменения в свойствах и параметрах. Дополнительные сведения см. в разделе [об обозревателе IoT Central](#howto-use-iotc-explorer). 

## <a name="use-the-azure-sdks-for-control-plane-operations"></a>Используйте пакеты SDK Azure для операций уровня управления

Пакеты SDK IoT Central Azure Resource Manager доступны для Node, Python, C#, Ruby, Java и Go. Эти библиотеки поддерживают операции уровня управления для Azure IoT Central, позволяя создавать, отображать, обновлять или удалять приложения Azure IoT Central. Они также включают вспомогательные методы для проверки подлинности и обработки ошибок в соответствии с выбранным языком. 

Примеры использования пакетов SDK для Azure Resource Manager см. здесь: [https://github.com/emgarten/iotcentral-arm-sdk-examples](https://github.com/emgarten/iotcentral-arm-sdk-examples).
