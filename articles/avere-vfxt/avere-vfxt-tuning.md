# <a name="cluster-tuning"></a>Настройка кластера

Работа большинства кластеров vFXT может быть улучшена за счет настраиваемых параметров производительности. Эти настройки помогают кластеру оптимально работать с конкретным рабочим процессом, набором данных и инструментами.

Эта настройка должна выполняться с помощью представителя службы поддержки, так как она может содержать настройки функций, недоступных на панели управления Авере.

В этом разделе описаны некоторые пользовательские настройки, которые можно выполнить.

## <a name="general-optimizations"></a>Общая оптимизация

Эти изменения можно рекомендовать в зависимости от свойств набора данных или стиля рабочего процесса.

* Если рабочая нагрузка связана с интенсивными операциями записи, увеличьте размер кэша от его значения по умолчанию на 20 %.
* Если набор данных состоит из множества небольших файлов, увеличьте предельное число файлов в кэше кластера.
* Если работа связана с копированием или перемещением данных между двумя репозиториями, отрегулируйте количество потоков, используемых для перемещения данных:
  * Чтобы повысить скорость, можно увеличить количество используемых параллельных потоков.
  * Если том внутреннего хранилища становится перегруженным, вам может потребоваться уменьшить количество используемых параллельных потоков.
* Если кластер кэширует данные для основного файлового хранилища, использующего списки управления доступом NFSv4, включите кэширование режима доступа, чтобы упростить авторизацию файлов для конкретных клиентов.

## <a name="cloud-nas-or-cloud-gateway-optimizations"></a>Оптимизация облачных запоминающих устройств, подключаемых к сети, или облачных шлюзов

В случае облачного NAS или шлюза кластер Вфкст предоставляет доступ в стиле NAS к облачному контейнеру. Чтобы воспользоваться преимуществами более высоких скоростей данных между кластером Вфкст и облачным хранилищем, ваш репрезентативный специалист может рекомендовать изменить параметры для более агрессивной принудительной передачи данных в том хранилища из кэша. Пример:

* Увеличьте количество TCP-подключений между кластером и контейнером хранилища.

## <a name="cloud-bursting-or-hybrid-wan-optimizations"></a>Оптимизация выхода в облако или WAN для гибридного хранилища

В сценарии для ускорения облака или гибридного решения для оптимизации глобальной сети кластер Вфкст обеспечивает интеграцию между облачным и локальным хранилищем оборудования. Эти изменения могут быть полезными:

* Увеличьте количество TCP-подключений, разрешенных между кластером и основным файловым хранилищем.
* Включите параметр оптимизации WAN для удаленного основного файлового хранилища (этот параметр можно использовать для удаленного локального хранилища или облачного основного файлового хранилища в другом регионе Azure).
* Увеличение размера буфера сокета TCP<sup>*</sup>
* Включить параметр "всегда пересылать" для уменьшения количества избыточных кэшированных файлов<sup>*</sup>

<sup>*</sup>Эти изменения могут быть применены не ко всем системам в зависимости от рабочей нагрузки и требований к производительности.
