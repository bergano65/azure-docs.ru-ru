# <a name="review-access-to-groups-or-applications-in-azure-ad-access-reviews"></a>Проверка доступа к группам или проверок доступа приложений в Azure AD

Azure Active Directory (Azure AD) упрощает предприятиям управлять доступом к группам и приложениям в Azure AD и просматривает остальным сетевым службам Майкрософт с помощью функции доступа Azure AD.

В этой статье описывается, как назначенный проверяющий выполняет проверку доступа для участников группы или пользователи, имеющие доступ к приложению.

## <a name="open-the-access-review"></a>Открытие проверки доступа

Чтобы выполнить проверку доступа, сначала — поиск и открытие проверки доступа.

1. Найдите сообщение от корпорации Майкрософт с просьбой проверить доступ. Ниже приведен пример сообщения электронной почты для проверки доступа для группы.

    ![Электронное сообщение о проверке доступа](./media/perform-access-review/access-review-email.png)

1. Нажмите кнопку **запуск проверки** ссылку, чтобы открыть проверку доступа.

Если у вас адрес электронной почты, можно найти на ожидающие проверки доступа, выполните следующие действия.

1. Войдите на портал MyApps в [ https://myapps.microsoft.com ](https://myapps.microsoft.com).

    ![Портал MyApps](./media/perform-access-review/myapps-access-panel.png)

1. В правом верхнем углу страницы щелкните значок пользователя, где отображается ваше имя и название организации по умолчанию. Если в списке содержится несколько организаций, выберите ту организацию, которая запросила проверку доступа.

1. В правой части страницы щелкните **проверок доступа** плитку, чтобы просмотреть список ожидающие проверки доступа.

    Если такой элемент не отображается, для этой организации нет ожидающих проверок доступа и в данный момент не требуется никаких действий.

    ![Список проверок доступа](./media/perform-access-review/access-reviews-list.png)

1. Нажмите кнопку **начать проверку** ссылку для проверки доступа, вы хотите выполнить.

## <a name="perform-the-access-review"></a>Выполнить проверку доступа

После открытия проверки доступа, вы увидите имена пользователей, которые необходимо изучить.

Если этот запрос поступает в проверке собственного доступа, страницы будут выглядеть иначе. Дополнительные сведения см. в разделе [проверка доступа для себя группам или приложениям](review-your-access.md).

![Выполнить проверку доступа](./media/perform-access-review/perform-access-review.png)

Можно утвердить или запретить доступ двумя способами:

- Вы можете утвердить или отклонить каждый запрос по отдельности, или
- Вы можете принять рекомендации системы, который является простым и быстрым способом.

### <a name="approve-or-deny-access-for-each-request"></a>Утверждение или отклонение доступа для каждого запроса

1. Просмотрите список пользователей, чтобы решить, следует ли утверждать или отклонять их непрерывного доступа.

1. Чтобы утверждать или отклонять каждый запрос, щелкните строку, чтобы открыть окно, чтобы указать действие, выполняемое.

1. Нажмите кнопку **утвердить** или **запретить**. Если вы не уверены, можно щелкнуть **неизвестен**. Это приведет к в обслуживании его/ее доступа пользователя, но выбор будет отражаться в журналах аудита.

    ![Выполнить проверку доступа](./media/perform-access-review/approve-deny.png)

    Администратор проверки доступа может потребоваться, чтобы вы указывали причину для утверждения необходимости получения доступа или членства в группе.

1. После указания действие, выполняемое, нажмите кнопку **Сохранить**.

    Если вы хотите изменить ответ, выберите строку и ответ. Например можно утвердить ранее отклоненного пользователя или отклонить ранее утвержденного пользователя. Ваш ответ можно изменить в любое время до завершения проверки доступа.

    Если имеется несколько рецензентов, записывается последнего отправленного ответа. Рассмотрим пример, где администратор назначает рецензенты — Алиса и Боб. Анна сначала открывает проверки доступа и утверждает доступа. Прежде чем проверка заканчивается, Боб открывает проверки доступа и запрещает доступ. Ответ — записанным запретов последнего.

    > [!NOTE]
    > Если пользователь получает отказ в доступе, они не будут немедленно удалены. Они удаляются при завершении проверки или проверки остановлена администратором.

### <a name="approve-or-deny-access-based-on-recommendations"></a>Утверждение или отклонение доступа на основе рекомендаций

Чтобы сделать проверки доступа для вас проще и быстрее, мы также предоставляем рекомендации, которые могут принимать одним щелчком. Создаются рекомендации на основе действий входа в систему пользователя.

1. Синюю полосу в нижней части страницы, нажмите кнопку **принять рекомендации**.

    ![Принять рекомендации](./media/perform-access-review/accept-recommendations.png)

    Вы увидите сводку рекомендуемые действия.

    ![Принять рекомендации сводки](./media/perform-access-review/accept-recommendations-summary.png)

1. Нажмите кнопку **ОК** чтобы принять эти рекомендации.
