# <a name="delete-a-directory-in-azure-active-directory"></a>Удалить каталог в Azure Active Directory

При удалении каталога Azure AD, также удаляются все ресурсы, которые содержатся в каталоге. Подготовьте свою компанию, сводя к минимуму связанные с ней ресурсы, прежде чем можно будет удалить. Только глобальный администратор Azure Active Directory (Azure AD) можно удалить каталог Azure AD на портале.

## <a name="prepare-the-directory"></a>Подготовка каталога

Не удается удалить каталог, в Azure AD, пока он проходит несколько проверок. Эти проверки снизить риск того, что удаление каталога Azure AD отрицательно влияет на доступ пользователей, такие как возможность входа Office 365 или доступа к ресурсам в Azure. Например если случайно удален каталог, связанный с подпиской, пользователям недоступны ресурсы Azure для этой подписки. Проверяются следующие условия.

* В каталоге, кроме один глобальный администратор, который является удаление каталога может быть никто из пользователей. Всех других пользователей следует удалить перед удалением каталога. Если пользователи синхронизируются из локальной, то синхронизации сначала должны быть отключены, и пользователей, которые необходимо удалить из облачного каталога с помощью портала Azure или командлетов Azure PowerShell.
* В каталоге не должно быть ни одного приложения. Любые приложения, необходимо удалить перед удалением каталога.
* Может существовать не поставщики многофакторной проверки подлинности с каталогом связаны.
* С каталогом не должны быть связаны подписки на любые службы Microsoft Online Services, такие как Microsoft Azure, Office 365 или Azure AD Premium. Например, если каталог по умолчанию создан в Azure, его невозможно удалить, если ваша подписка Azure по-прежнему использует его для проверки подлинности. Точно так же нельзя удалить каталог, если у другого пользователя есть связанная с ним подписка.

## <a name="delete-the-directory"></a>Удалить каталог

1. Войдите в [Центр администрирования Azure AD](https://aad.portal.azure.com) с учетной записью глобального администратора для вашей организации.

2. Выберите **Azure Active Directory**.

3. Перейдите в каталог, который требуется удалить.
  
   ![Подтвердите организации перед удалением](./media/directory-delete-howto/delete-directory-command.png)

4. Нажмите **Удалить каталог**.
  
   ![выберите команду, чтобы удалить организацию](./media/directory-delete-howto/delete-directory-list.png)

5. Если каталог не проходит одну или несколько проверок, вы дана ссылка на дополнительные сведения о том, как для передачи. Когда клиент пройдет все проверки, нажмите **Удалить** для завершения процесса.

## <a name="if-you-cant-delete-the-directory"></a>Если не удается удалить каталог

При настройке вашего каталога Azure AD может подписки на основе лицензии, также активирована для вашей организации, таких как Azure AD Premium P2, Office 365 Business Premium или Enterprise Mobility + Security E5. Чтобы избежать случайной потери данных, нельзя удалить каталог, пока не будут полностью удалены подписки. Подписки должны быть в **отозвана** состояния, чтобы разрешить удаление каталога. **Истек** или **Canceled** подписка перемещается **отключено** состояние, а последний этап — **отозвана** состояния.

Дополнительные сведения о жизненном цикле хранения данных и подписок в Office 365 см. в разделе [Что происходит с моими данными и доступом при завершении подписки Office 365 для бизнеса?](https://support.office.com/article/what-happens-to-my-data-and-access-when-my-office-365-for-business-subscription-ends-4436582f-211a-45ec-b72e-33647f97d8a3)

## <a name="delete-a-subscription"></a>Удаление подписки

Подписки можно поместить в состоянии отозвана будет удален в течение трех дней, в центре администрирования Microsoft 365.

1. Войдите в [Центр администрирования Microsoft 365](https://admin.microsoft.com) с помощью учетной записи глобального администратора в вашей организации. Если вы пытаетесь удалить каталог «Contoso», с contoso.onmicrosoft.com домена первоначальное значение по умолчанию, войдите с помощью имени участника-пользователя например admin@contoso.onmicrosoft.com.

2. Выберите **выставления счетов** и выберите **подписок**, затем выберите подписку, которую нужно отменить. Нажмите **Отменить** и обновите страницу.
  
   ![Удаление связи для удаления подписки](./media/directory-delete-howto/delete-command.png)
  
3. Нажмите **Удалить**, чтобы удалить подписку, и примите условия. Все данные будут безвозвратно удалены в течение трех дней. Если вы передумаете, вы можете повторно активировать подписку в течение трех дней.
  
   ![Внимательно прочитайте условия](./media/directory-delete-howto/delete-terms.png)

4. Теперь состояние подписки изменилось, подписка помечена для удаления. Подписка входит в состояние **Отозвано** через 72 часа.

5. После удаления подписки в каталоге и истечении 72 часов, вы можете войти обратно в центр администрирования Azure AD еще раз и должны быть требуется вмешательство и подписки, не блокирует удаление вашего каталога. Можно успешно удалить каталог Azure AD.
  
   ![пройденная проверка подписки на экране удаления](./media/directory-delete-howto/delete-checks-passed.png)

## <a name="i-have-a-trial-subscription-that-blocks-deletion"></a>У меня есть пробную подписку, которая блокирует удаление

Существуют [самообслуживания регистрации продуктов](https://docs.microsoft.com/office365/admin/misc/self-service-sign-up?view=o365-worldwide) , как Microsoft Power BI, службы управления правами, Microsoft Power Apps или Dynamics 365, отдельные пользователи могут зарегистрироваться с помощью Office 365, при этом также создается гостевой пользователь для проверки подлинности в каталог Azure AD. Эти продукты самообслуживания блокировать удаления каталога, пока будут полностью удалены из каталога, чтобы избежать потери данных. Их можно удалить только администратором Azure AD пользователь зарегистрировался по отдельности или был назначен продукт.

Существует два типа самообслуживания регистрации продуктов, в том, как они назначены: 

* Назначение уровня организации: Администратора Azure AD назначает продукта для всей организации, и пользователь может активно использование службы с это назначение уровня организации даже если они не лицензируются отдельно.
* Назначение уровня пользователя: Отдельного пользователя во время самостоятельную регистрацию по сути назначает продукта на самих себя без прав администратора Когда организация управляется администратором (см. в разделе [смены администратором неуправляемого каталога](domains-admin-takeover.md), то администратор сможет назначить продукт непосредственно для пользователей без самостоятельной регистрации.  

После начала удаления регистрации продукта самообслуживания, действие окончательно удаляет данные и блокирует доступ пользователя к службе. Любой пользователь, который был назначен предложение по отдельности или на уровне организации затем блокируется входе или доступе к все существующие данные. Если вы хотите избежать потери данных за счет самообслуживания регистрации продукта, например [панелей мониторинга Microsoft Power BI](https://docs.microsoft.com/power-bi/service-export-to-pbix) или [конфигурация политики службы управления правами](https://docs.microsoft.com/azure/information-protection/configure-policy#how-to-configure-the-azure-information-protection-policy), убедитесь, что резервное копирование данных и сохранены в других местах.

Дополнительные сведения о доступных самообслуживания регистрации продуктов и служб, см. в разделе [доступных программ самообслуживания](https://docs.microsoft.com/office365/admin/misc/self-service-sign-up?view=o365-worldwide#available-self-service-programs).

В следующей таблице описано, что происходит, когда истекает срок действия пробной подписки Office 365 (не включая платную партнерскую подписку/CSP, соглашение Enterprise или корпоративные лицензии). Дополнительные сведения о жизненном цикле хранения и подписки данных Office 365, см. в разделе [что происходит с данными и доступе при завершении Office 365 для бизнеса подписки?](https://docs.microsoft.com/office365/admin/subscriptions-and-billing/what-if-my-subscription-expires?view=o365-worldwide).

------------- | ---- | --------------
Активно (пробная версия — 30 дней) | Данные доступны для всех | Пользователи имеют обычный доступ к самообслуживания регистрации продукта, файлах или приложениях<br>Администраторы имеют обычный доступ к центру администрирования Microsoft 365 и ресурсы

## <a name="how-can-i-delete-a-self-service-sign-up-product-in-the-azure-portal"></a>Как удалить самообслуживания регистрации продукта на портале Azure?

Вы можете поместить самообслуживания регистрации продукта Microsoft Power BI или Azure Rights Management Services в **удалить** состояние будет немедленно удален в на портале Azure AD.

1. Войдите в [Центр администрирования Azure AD](https://aad.portal.azure.com/#blade/Microsoft_AAD_IAM/ActiveDirectoryMenuBlade/Overview) с учетной записью, которая является глобальным администратором в организации. Если вы пытаетесь удалить каталог «Contoso», с contoso.onmicrosoft.com домена первоначальное значение по умолчанию, для входа с именем участника-например admin@contoso.onmicrosoft.com.

2. Выберите **лицензии**, а затем выберите **самообслуживания регистрации продуктов**. Вы увидите все самообслуживания регистрации продукты отдельно от подписки на основе рабочих мест. Выберите продукт, который вы хотите удалить без возможности восстановления. Ниже приведен пример, в Microsoft Power BI:

    ![имя пользователя неправильно ввел или не найден](./media/directory-delete-howto/licenses-page.png)

3. Выберите **удалить** для удаления продукта и примите условия данные будут удалены немедленно и отменить его нельзя. Это действие удаления будет удалите всех пользователей и организации доступ к этому продукту. Нажмите "Да" для перемещения вперед с удалением.  

    ![имя пользователя неправильно ввел или не найден](./media/directory-delete-howto/delete-product.png)

4. При выборе **Да**, инициируется удаление продукта самообслуживания. Отсутствует уведомление, которое сообщит об удалении в процессе выполнения.  

    ![имя пользователя неправильно ввел или не найден](./media/directory-delete-howto/progress-message.png)

5. Теперь изменения состояния самообслуживания регистрации продукта на **Deleted**. При обновлении страницы продукта должны быть удалены из **самообслуживания регистрации продуктов** страницы.  

    ![имя пользователя неправильно ввел или не найден](./media/directory-delete-howto/product-deleted.png)

6. После удаления всех продуктов, можно снова войти обратно в центр администрирования Azure AD, и должен быть не требуемое действие и продукты, не блокирует удаление вашего каталога. Можно успешно удалить каталог Azure AD.

    ![имя пользователя неправильно ввел или не найден](./media/directory-delete-howto/delete-organization.png)
