# <a name="sign-in-problems-with-an-azure-ad-gallery-app-configured-for-sso"></a>Проблемы при входе в приложение из коллекции Azure AD, для которого настроен единый вход

Панель доступа — это веб-портал. Она позволяет пользователям, у которых есть учетные записи Azure Active Directory (Azure AD), использовать облачные приложения, для которых у них есть разрешения. Пользователи, имеющие выпуски Azure AD, также могут использовать возможности самостоятельного управления группами и приложениями с помощью панели доступа.

Панель доступа отличается от портал Azure. Для использования панели доступа пользователям не требуется подписка Azure.

Чтобы использовать единый вход на основе пароля на панели доступа, необходимо установить расширение панели доступа в браузере. Расширение загружается автоматически при выборе приложения, настроенного для единого входа на основе пароля.

## <a name="browser-requirements-for-access-panel"></a>Требования к браузеру для панели доступа

Для панели доступа требуется браузер, поддерживающий JavaScript и поддерживающий CSS.

Следующие браузеры поддерживают единый вход на основе пароля:

- Internet Explorer 8, 9, 10 и 11 в Windows 7 или более поздней версии

- Chrome в Windows 7 или более поздней версии или MacOS X или более поздней версии

- Firefox 26,0 или более поздней версии в Windows XP SP2 или более поздней версии или Mac OS X 10,6 или более поздней версии

>[!NOTE]
>Расширение единого входа на основе пароля будет доступно для Microsoft ребро в Windows 10, когда в Microsoft погранично добавлена поддержка расширений браузера.

## <a name="install-the-access-panel-browser-extension"></a>Установить расширение "панель доступа" для браузера

Выполните следующие действия.

1. Откройте [панель доступа](https://myapps.microsoft.com) в поддерживаемом браузере и войдите в систему как пользователь в Azure AD.

2. Выберите приложение с поддержкой единого входа с помощью пароля на панели доступа.

3. При появлении запроса выберите **Install Now (установить сейчас**).

4. Вы будете направлены на ссылку для скачивания на основе браузера. Нажмите кнопку **Добавить** , чтобы установить расширение браузера.

5. При появлении запроса выберите **включить** или **Разрешить**.

6. После установки перезапустите браузер.

7.  Войдите на панель доступа и проверьте, можете ли вы запустить приложения с поддержкой единого входа с помощью пароля.

Вы также можете напрямую скачать расширения для Chrome и Firefox с помощью следующих ссылок:

-   [Расширение панели доступа Chrome](https://chrome.google.com/webstore/detail/access-panel-extension/ggjhpefgjjfobnfoldnjipclpcfbgbhl)

-   [Расширение панели доступа Firefox](https://addons.mozilla.org/firefox/addon/access-panel-extension/)

## <a name="set-up-a-group-policy-for-internet-explorer"></a>Настройка групповой политики для Internet Explorer

Можно настроить групповую политику, которая позволяет удаленно устанавливать расширение панели доступа для Internet Explorer на компьютерах пользователей.

Ниже перечислены необходимые условия.

-   [Домен Active Directory службы](https://msdn.microsoft.com/library/aa362244%28v=vs.85%29.aspx) должны быть настроены, а компьютеры пользователей должны быть присоединены к домену.

-   У вас есть разрешение на изменение параметров для изменения объекта групповая политика (GPO). По умолчанию это разрешение имеют члены следующих групп безопасности: "Администраторы домена", "Администраторы предприятия" и "Владельцы-создатели групповой политики". [Узнайте больше](https://technet.microsoft.com/library/cc781991%28v=ws.10%29.aspx).

Сведения о настройке групповой политики и ее развертывании для пользователей см. в статье [Развертывание расширения панели доступа для Internet Explorer с помощью групповой политики](https://docs.microsoft.com/azure/active-directory/active-directory-saas-ie-group-policy).

## <a name="troubleshoot-access-panel-in-internet-explorer"></a>Устранение неполадок с панелью доступа в Internet Explorer

## <a name="configure-password-sso-for-an-azure-ad-gallery-app"></a>Настройка единого входа с помощью пароля для приложения из коллекции Azure AD

Чтобы настроить приложение из коллекции Azure AD, необходимо выполнить следующие действия.

-   Добавление приложения из коллекции Azure AD
-   [Настроить приложение для единого входа по паролю](#configure-the-app-for-password-sso)
-   [Назначение пользователей для приложения](#assign-users-to-the-app)

### <a name="add-the-app-from-the-azure-ad-gallery"></a>Добавление приложения из коллекции Azure AD

Выполните следующие действия.

1. Откройте [портал Azure](https://portal.azure.com) и войдите в систему как глобальный администратор или Соадминистратор.

2. Выберите **все службы** в верхней части области навигации в левой части окна, чтобы открыть расширение Azure AD.

3. В поле фильтра поиска введите **Azure Active Directory** , а затем выберите **Azure Active Directory**.

4. В области навигации Azure AD выберите **корпоративные приложения** .

5. Щелкните **Добавить** в правом верхнем углу области **корпоративные приложения** .

6. В разделе **Добавить из коллекции** введите имя приложения в поле **введите имя** .

7. Выберите приложение, которое требуется настроить для единого входа.

8. *Необязательно*. Перед добавлением приложения его имя можно изменить в поле **имя** .

9. Нажмите кнопку **Добавить** , чтобы добавить приложение.

   После небольшой задержки вы сможете увидеть область конфигурации приложения.

### <a name="configure-the-app-for-password-sso"></a>Настройка приложения для единого входа с помощью пароля

Выполните следующие действия.

1. Откройте [портал Azure](https://portal.azure.com/) и войдите в систему как глобальный администратор или Соадминистратор.

2. Выберите **все службы** в верхней части области навигации в левой части окна, чтобы открыть расширение Azure AD.

3. В поле фильтра поиска введите **Azure Active Directory** , а затем выберите **Azure Active Directory**.

4. Выберите **корпоративные приложения** в области навигации Azure AD.

5. Выберите **все приложения** , чтобы просмотреть список приложений.

   > [!NOTE]
   > Если нужное приложение не отображается, используйте элемент управления **Фильтр** в верхней части **списка все приложения**. Задайте для параметра " **Показывать** " значение "все приложения".

6. Выберите приложение, которое требуется настроить для единого входа.

7. После загрузки приложения выберите **единый вход** в области в левой части приложения.

8. Выберите режим **входа на основе пароля** .

9. Назначьте пользователей приложению.

10. Также можно указать учетные данные для пользователей. (В противном случае пользователям будет предложено ввести учетные данные при запуске приложения.) Для этого выберите строки пользователей. Затем выберите **обновить учетные данные** и введите свои имена пользователей и пароли.

### <a name="assign-users-to-the-app"></a>Назначение пользователей приложению

Чтобы напрямую назначить пользователей приложению, выполните следующие действия.

1. Откройте [портал Azure](https://portal.azure.com/) и войдите в систему как глобальный администратор.

2. Выберите **все службы** в области навигации в левой части экрана, чтобы открыть расширение Azure AD.

3. В поле фильтра поиска введите **Azure Active Directory** , а затем выберите **Azure Active Directory**.

4. Выберите **корпоративные приложения** в области навигации Azure AD.

5. Выберите **все приложения** , чтобы просмотреть список приложений.

   > [!NOTE]
   > Если нужное приложение не отображается, используйте элемент управления **Фильтр** в верхней части **списка все приложения**. Задайте для параметра " **Показывать** " значение "все приложения".

6. В списке выберите приложение, которому нужно назначить пользователя.

7. После загрузки приложения выберите **Пользователи и группы** в области навигации приложения в левой части окна.

8. Выберите **Добавить** в верхней части списка **Пользователи и группы** , чтобы открыть область **Добавление назначения** .

9. На панели **Добавление назначения** выберите **Пользователи и группы** .

10. В поле **Искать по имени или адресу электронной почты** введите полное имя или адрес электронной почты пользователя, которого нужно назначить.

11. Наведите указатель мыши на пользователя в списке. Установите флажок рядом с фотографией или логотипом профиля пользователя, чтобы добавить этого пользователя в **выбранный** список.

12. *Необязательно*. Чтобы добавить другого пользователя, введите другое имя или адрес электронной почты в поле **Поиск по имени или адресу электронной почты** , а затем установите флажок, чтобы добавить этого пользователя в **выбранный** список.

13. Завершив выбор пользователей, нажмите кнопку **выбрать** , чтобы добавить их в список пользователей и групп, назначенных приложению.

14. *Необязательно*. Щелкните **выбрать роль** на панели **Добавление назначения** , чтобы выбрать роль для назначения выбранным пользователям.

15. Выберите **назначить** , чтобы назначить приложение выбранным пользователям.

    После небольшой задержки пользователи смогут получить доступ к этим приложениям из панели доступа.

## <a name="request-support"></a>Создание запроса 
Если при настройке единого входа и назначении пользователей появляется сообщение об ошибке, откройте запрос в службу поддержки. Включите как можно больше следующих сведений:

-   идентификатор ошибки корреляции;
-   имя участника-пользователя (адрес электронной почты пользователя);
-   идентификатор клиента;
-   тип браузера;
-   Часовой пояс и интервал времени и времени, когда произошла ошибка
-   трассировки Fiddler.
