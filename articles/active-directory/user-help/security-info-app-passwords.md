# <a name="manage-app-passwords-from-your-security-info-preview-page"></a>Управление паролями приложений на странице сведений для защиты (предварительная версия)
Некоторые приложения, например Outlook 2010, не поддерживают двухфакторную проверку подлинности. Это означает, что такое приложение не будет работать, если вы включите в организации двухфакторную проверку подлинности. Чтобы избежать этой проблемы, помимо обычного пароля, создайте автоматически сгенерированный пароль для использования с каждым внебраузерным приложением.

[!INCLUDE [preview-notice](../../../includes/active-directory-end-user-preview-notice-security-info.md)]

>[!Important]
>Администратор может запретить использование паролей приложений. Если вы не видите параметр **Пароли приложений**, значит он недоступен для вашей организации.

При использовании паролей приложений необходимо помнить следующее:

- Пароли приложений автоматически создаются и вводятся только один раз для каждого приложения.

- Максимальное количество паролей на пользователя — 40. Если вы попытаетесь создать пароль после достижения этого предела, вам будет предложено удалить один из существующих паролей, чтобы создать новый.

- Мы рекомендуем использовать один пароль приложения для каждого устройства, а не для отдельных приложений. Например, создайте один пароль для всех приложений на ноутбуке и еще один — для всех приложений на настольном компьютере.

    >[!Note]
    >Все клиенты Office 2013 (включая Outlook) поддерживают новые протоколы идентификации и двухфакторную проверку подлинности. Это означает, что после включения двухфакторной проверки подлинности отпадает потребность в использовании паролей приложений для клиентов Office 2013. Дополнительные сведения см. в статье [Современная проверка подлинности в клиентских приложениях Office 2013 и Office 2016](https://support.office.com/article/how-modern-authentication-works-for-office-2013-and-office-2016-client-apps-e4c45989-4b1a-462e-a81b-2a13191cf517).

## <a name="create-new-app-passwords"></a>Создание паролей приложений
Если вы используете двухфакторную проверку подлинности для рабочей или учебной учетной записи, а администратор разрешил применить функцию сведений для защиты, вы можете создавать и удалять пароли приложений на странице **Сведения для защиты**.

>[!Note]
>Если администратор не включил функцию использования сведений для защиты, необходимо выполнить инструкции в статье [Что такое пароли приложений в службе Многофакторной идентификации Azure](multi-factor-authentication-end-user-app-passwords.md).

### <a name="to-create-a-new-app-password"></a>Создание пароля приложения
1. Войдите в рабочую или учебную учетную запись, а затем перейдите на страницу https://myprofile.microsoft.com/.

    ![Страница "Мой профиль" с выделенными ссылками на сведения для защиты](media/security-info/securityinfo-myprofile.png)

2. На панели навигации слева выберите **Сведения для защиты** или щелкните ссылку в блоке **Сведения для защиты**, а затем выберите **Добавить метод** на странице **Сведения для защиты**.

    ![Страница сведений для защиты с выделенным параметром "Добавить метод"](media/security-info/securityinfo-myprofile-addmethod.png)

3. На странице **Добавить метод** выберите **Пароль приложения** из раскрывающегося списка, а затем щелкните **Добавить**.

    ![Окно добавления метода с выбранным методом "Пароль приложения"](media/security-info/securityinfo-myprofile-addpassword.png)

4. Введите имя приложения, которому требуется пароль приложения, а затем выберите **Далее**.

    ![Страница пароля приложения, где указано имя приложения](media/security-info/securityinfo-myprofile-password-appname.png)

5. Скопируйте текст из поля **Пароль**, вставьте этот пароль в область для ввода пароля приложения (в нашем примере это Outlook 2010) и щелкните **Готово**.

    ![Страница пароля приложения, где указано имя приложения](media/security-info/securityinfo-myprofile-password-copytext.png)
    
    Это действие добавляет пароль, который теперь позволит вам успешно входить в приложение.

## <a name="delete-your-app-passwords"></a>Удаление паролей приложений
Когда вы прекратите использовать приложение, для которого требовался пароль приложения, этот пароль можно удалить. Удаление пароля приложения освобождает доступное место, позволяя в будущем создать новый пароль приложения.

>[!Important]
>Если вы удалите пароль приложения случайно, эту операцию невозможно отменить. Вам придется создать новый пароль приложения и повторно ввести его в приложение по инструкциям из раздела [Создание паролей приложений](#create-new-app-passwords) в этой статье.

### <a name="to-delete-an-app-password"></a>Удаление пароля приложения

1. На странице **Сведения для защиты** щелкните ссылку **Удалить** рядом с параметром **Пароль приложения** для нужного приложения.

    ![Ссылка на метод удаления пароля со страницы сведений для защиты](media/security-info/securityinfo-myprofile-password-appdelete.png)

2. Выберите **Да** в окне подтверждения, чтобы удалить **пароль приложения**. После удаления пароль приложения удаляется из сведений для защиты и не отображается на странице **Сведения для защиты**.
