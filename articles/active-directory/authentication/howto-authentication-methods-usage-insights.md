# <a name="authentication-methods-usage--insights-preview"></a>Использование методов проверки подлинности & Insights (Предварительная версия)

Использование & Insights позволяет понять, как методы проверки подлинности для таких функций, как многофакторная идентификация Azure и самостоятельный сброс пароля, работают в вашей организации. Эта возможность создания отчетов предоставляет вашей организации средства для понимания того, какие методы регистрируются и как они используются.

## <a name="permissions-and-licenses"></a>Разрешения и лицензии

Следующие роли могут получать доступ к использованию и аналитике:

- глобального администратора;
- Читатель сведений о безопасности
- Администратор безопасности
- Читатель отчетов

Для доступа к использованию и аналитике не требуется никаких дополнительных лицензий. Сведения о лицензировании Azure для многофакторной идентификации и самостоятельного сброса пароля (SSPR) можно найти на [сайте цен на Azure Active Directory](https://azure.microsoft.com/pricing/details/active-directory/).

## <a name="how-it-works"></a>Принцип работы

Для доступа к использованию и анализу методов проверки подлинности:

1. Перейдите на [портал Azure](https://portal.azure.com).
1. Перейдите к **Azure Active Directory** > **Сброс пароля** > **Использование & Insights**.
1. Из просмотра **регистрации** или **использования** можно открыть предварительно отфильтрованные отчеты для фильтрации в соответствии с вашими потребностями.

![Общие сведения об использовании & Insights](./media/howto-authentication-methods-usage-insights/usage-insights-overview.png)

Чтобы получить доступ к & подробным сведениям об использовании, перейдите на страницу [https://portal.azure.com/#blade/Microsoft_AAD_IAM/AuthMethodsOverviewBlade](https://portal.azure.com/#blade/Microsoft_AAD_IAM/AuthMethodsOverviewBlade). По этой ссылке вы сможете открыть обзор регистрации.

Плитки зарегистрированные пользователи, пользователи с поддержкой и пользователи могут отображать следующие данные регистрации для пользователей:

- Буферизаци Пользователь считается зарегистрированным, если он (или администратор) зарегистрировал достаточно методов проверки подлинности в соответствии с политикой SSPR или многофакторной проверки подлинности вашей организации.
- Включено: Пользователь считается включенным, если он находится в области действия политики SSPR. Если SSPR включен для группы, то он считается включенным, если он находится в этой группе. Если SSPR включен для всех пользователей, все пользователи в клиенте (за исключением гостей) считаются включенными.
- Возможности Пользователь считается поддерживающим, если он зарегистрирован и включен. Это состояние означает, что при необходимости они могут выполнять SSPR в любое время.

Если щелкнуть любую из этих плиток или аналитические сведения, показанные в них, откроется предварительно отфильтрованный список сведений о регистрации.

На диаграмме **регистрации** на вкладке **Регистрация** отображается количество успешных и неудачных регистраций методов проверки подлинности по методу проверки подлинности. Диаграмма **сбросов** на вкладке **Использование** показывает количество успешных и неудачных проверок подлинности в ходе потока сброса пароля методом проверки подлинности.

Если щелкнуть любую из диаграмм, откроется предварительно отфильтрованный список событий регистрации или сброса.

С помощью элемента управления в правом верхнем углу можно изменить диапазон дат для данных аудита, отображаемых при регистрации, и сбросить диаграммы до 24 часов, 7 дней или 30 дней.

### <a name="registration-details"></a>Сведения о регистрации

Щелкнув элемент **зарегистрированные**пользователи, **Пользователи с поддержкой**или **плитки или** аналитика, вы перейдете к сведениям о регистрации.

В отчете сведения о регистрации отображаются следующие сведения для каждого пользователя:

- ИМЯ
- Имя пользователя
- Состояние регистрации (все, зарегистрировано, не зарегистрировано)
- Включенное состояние (все, включено, не включено)
- Состояние поддержки (все, поддерживается, не поддерживается)
- Методы (уведомление приложения, код приложения, телефонный звонок, SMS, электронная почта, контрольные вопросы)

С помощью элементов управления в верхней части списка можно найти пользователя и отфильтровать список пользователей на основе отображаемых столбцов.

### <a name="reset-details"></a>Сведения о сбросе

Если щелкнуть диаграмму регистрации или сброса, вы перейдете к сведениям о сбросе.

В отчете о сбросе отображаются события регистрации и сброса за последние 30 дней, включая:

- ИМЯ
- Имя пользователя
- Функция (все, регистрация, сброс)
- Метод проверки подлинности (уведомление приложения, код приложения, телефонный звонок, вызов Office, SMS, электронная почта, контрольные вопросы)
- Состояние (все, успешно, сбой)

С помощью элементов управления в верхней части списка можно найти пользователя и отфильтровать список пользователей на основе отображаемых столбцов.

## <a name="limitations"></a>Ограничения

Данные, показанные в этих отчетах, будут задержаны до 60 минут. Поле Last обновилed (Последнее обновление) существует в портал Azure, чтобы узнать, насколько актуальны данные.

Данные об использовании и аналитике не заменяют отчеты о действиях многофакторной идентификации Azure или сведения, содержащиеся в отчете о входе в Azure AD.
