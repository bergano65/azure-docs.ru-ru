# <a name="tutorial-azure-active-directory-single-sign-on-sso-integration-with-shopify-plus"></a>Руководство по интеграции единого входа Azure Active Directory с Shopify Plus

В этом учебнике описано, как интегрировать Shopify Plus с Azure Active Directory (Azure AD). Интеграция Shopify Plus с Azure AD обеспечивает следующие возможности.

* С помощью Azure AD вы можете контролировать доступ к приложению Shopify Plus.
* Вы можете включить автоматический вход пользователей в Shopify Plus с помощью учетных записей Azure AD.
* Централизованное управление учетными записями через портал Azure.

Чтобы узнать больше об интеграции приложений SaaS с Azure AD, прочитайте статью [Что такое доступ к приложениям и единый вход с помощью Azure Active Directory?](../manage-apps/what-is-single-sign-on.md)

## <a name="prerequisites"></a>Предварительные требования

Чтобы приступить к работе, потребуется следующее.

* Подписка Azure AD. Если у вас нет подписки, вы можете получить [бесплатную учетную запись](https://azure.microsoft.com/free/).
* Подписка Shopify Plus с поддержкой единого входа.

## <a name="scenario-description"></a>Описание сценария

В рамках этого руководства вы настроите и проверите единый вход Azure AD в тестовой среде.

* Shopify Plus поддерживает единый вход, инициированный **поставщиком услуг и поставщиком удостоверений** .

* После настройки Shopify Plus можно применять элемент управления сеансами, которые защищают от хищения и несанкционированного доступа к конфиденциальным данным вашей организации в режиме реального времени. Управление сеансом является расширением функции условного доступа. [Узнайте, как применять управление сеансами с помощью Microsoft Cloud App Security](/cloud-app-security/proxy-deployment-any-app).

## <a name="adding-shopify-plus-from-the-gallery"></a>Добавление Shopify Plus из коллекции

Чтобы настроить интеграцию Shopify Plus с Azure AD, необходимо добавить Shopify Plus из коллекции в список управляемых приложений SaaS.

1. Войдите на [портал Azure](https://portal.azure.com) с помощью личной учетной записи Майкрософт либо рабочей или учебной учетной записи.
1. В области навигации слева выберите службу **Azure Active Directory** .
1. Перейдите в колонку **Корпоративные приложения** и выберите **Все приложения** .
1. Чтобы добавить новое приложение, выберите **Новое приложение** .
1. В разделе **Добавление из коллекции** в поле поиска введите **Shopify Plus** .
1. Выберите **Shopify Plus** в области результатов и добавьте это приложение. Подождите несколько секунд, пока приложение не будет добавлено в ваш клиент.


## <a name="configure-and-test-azure-ad-single-sign-on-for-shopify-plus"></a>Настройка и проверка единого входа Azure AD для Shopify Plus

Настройте и проверьте единый вход AAD в Shopify Plus с использованием данных тестового пользователя **B. Simon** . Для обеспечения работы единого входа необходимо установить связь между пользователем AAD и соответствующим пользователем в Shopify Plus.

Чтобы настроить и проверить единый вход AAD в Shopify Plus, выполните действия в следующих стандартных блоках.

1. **[Настройка единого входа Azure AD](#configure-azure-ad-sso)** необходима, чтобы пользователи могли использовать эту функцию.
    1. **[Создание тестового пользователя Azure AD](#create-an-azure-ad-test-user)** требуется для проверки работы единого входа Azure AD с помощью пользователя B.Simon.
    1. **[Назначение тестового пользователя Azure AD](#assign-the-azure-ad-test-user)** необходимо, чтобы позволить пользователю B.Simon использовать единый вход Azure AD.
1. **[Настройка единого входа в Shopify Plus](#configure-shopify-plus-sso)** необходима, чтобы настроить параметры единого входа на стороне приложения.
    1. **[Создание тестового пользователя Shopify Plus](#create-shopify-plus-test-user)** требуется для того, чтобы в Shopify Plus существовал пользователь B. Simon, связанный с одноименным пользователем в Azure AD.
1. **[Проверка единого входа](#test-sso)** позволяет убедиться в правильности конфигурации.

## <a name="configure-azure-ad-sso"></a>Настройка единого входа Azure AD

Выполните следующие действия, чтобы включить единый вход Azure AD на портале Azure.

1. На [портале Azure](https://portal.azure.com/) на странице интеграции с приложением **Shopify Plus** найдите раздел **Управление** и выберите **Единый вход** .
1. На странице **Выбрать метод единого входа** выберите **SAML** .
1. На странице **Настройка единого входа с помощью SAML** щелкните значок "Изменить" (значок пера), чтобы открыть диалоговое окно **Базовая конфигурация SAML** и изменить параметры.

   ![Изменение базовой конфигурации SAML](common/edit-urls.png)

1. Если вы хотите настроить приложение в режиме, инициируемом **поставщиком удостоверений** , в разделе **Базовая конфигурация SAML** введите значения следующих полей.

    В текстовом поле **URL-адрес ответа** введите URL-адрес в следующем формате: `https://accounts.shopify.com/saml/consume/organization/<ORGANIZATION_ID>`.

1. Чтобы настроить приложение для работы в режиме, инициируемом **поставщиком услуг** , щелкните **Задать дополнительные URL-адреса** и выполните следующие действия.

    В текстовом поле **URL-адрес входа** введите URL-адрес: `https://shopify.plus/login`.

    > [!NOTE]
    > Значение URL-адреса ответа приведено для примера. Вместо него нужно указать фактический URL-адрес ответа. Чтобы получить это значение, обратитесь в [группу поддержки клиентов Shopify Plus](mailto:plus-user-management@shopify.com). Можно также посмотреть шаблоны в разделе **Базовая конфигурация SAML** на портале Azure.

1. Приложение Shopify Plus ожидает проверочные утверждения SAML в определенном формате, который требует добавить настраиваемые сопоставления атрибутов в конфигурацию атрибутов токена SAML. На следующем снимке экрана показан список атрибутов по умолчанию.

    ![Изображение](common/default-attributes.png)

1. В дополнение к описанному выше приложение Shopify Plus ожидает несколько дополнительных атрибутов в ответе SAML, как показано ниже. Эти атрибуты также заранее заполнены, но вы можете изменить их в соответствии со своими требованиями.

    | Имя | Исходный атрибут|
    | ---- | --------------- |
    | email | user.mail |

1. Для параметра **Идентификатор имени** укажите формат **Постоянный** . Выберите вариант **Уникальный идентификатор пользователя (идентификатор имени)** , а затем выберите формат **Идентификатор имени** . Для этого параметра укажите значение **Постоянный** . Сохраните изменения.
1. На странице **Настройка единого входа с помощью SAML** в разделе **Сертификат подписи SAML** щелкните кнопку копирования, чтобы скопировать **URL-адрес метаданных федерации приложений** и сохранить его на компьютере.

    ![Ссылка для скачивания сертификата](common/copy-metadataurl.png)

### <a name="create-an-azure-ad-test-user"></a>Создание тестового пользователя Azure AD

В этом разделе описано, как на портале Azure создать тестового пользователя с именем B.Simon.

1. На портале Azure в области слева выберите **Azure Active Directory** , **Пользователи** , а затем — **Все пользователи** .
1. В верхней части экрана выберите **Новый пользователь** .
1. В разделе **Свойства пользователя** выполните следующие действия.
   1. В поле **Имя** введите `B.Simon`.  
   1. В поле **Имя пользователя** введите username@companydomain.extension. Например, `B.Simon@contoso.com`.
   1. Установите флажок **Показать пароль** и запишите значение, которое отображается в поле **Пароль** .
   1. Нажмите кнопку **Создать** .

### <a name="assign-the-azure-ad-test-user"></a>Назначение тестового пользователя Azure AD

В этом разделе описано, как включить единый вход в Azure для пользователя B. Simon, предоставив этому пользователю доступ к Shopify Plus.

1. На портале Azure выберите **Корпоративные приложения** , а затем — **Все приложения** .
1. Из списка приложений выберите **Shopify Plus** .
1. На странице "Обзор" приложения найдите раздел **Управление** и выберите **Пользователи и группы** .

   ![Ссылка "Пользователи и группы"](common/users-groups-blade.png)

1. Выберите **Добавить пользователя** , а в диалоговом окне **Добавление назначения**  выберите **Пользователи и группы** .

    ![Ссылка "Добавить пользователя"](common/add-assign-user.png)

1. В диалоговом окне **Пользователи и группы** выберите **B.Simon** в списке пользователей, а затем в нижней части экрана нажмите кнопку **Выбрать** .
1. Если ожидается, что в утверждении SAML будет получено какое-либо значение роли, то в диалоговом окне **Выбор роли** нужно выбрать соответствующую роль для пользователя из списка и затем нажать кнопку **Выбрать** , расположенную в нижней части экрана.
1. В диалоговом окне **Добавление назначения** нажмите кнопку **Назначить** .

## <a name="configure-shopify-plus-sso"></a>Настройка единого входа Shopify Plus

Подробные инструкции см. в [документации Shopify по настройке интеграции SAML](https://help.shopify.com/en/manual/shopify-plus/saml).

Чтобы настроить единый вход на стороне **Shopify Plus** , скопируйте **URL-адрес метаданных федерации приложения** из Azure Active Directory. Затем войдите с учетной записью [администратора организации](https://shopify.plus) и перейдите в раздел **Users** > **Security** (Пользователи > Безопасность). Щелкните **Set up configuration** (Настроить конфигурацию), а затем вставьте URL-адрес метаданных федерации приложений в раздел **Identity provider metadata URL** (URL-адрес метаданных поставщика удостоверений). Чтобы завершить этот шаг, щелкните **Add** (Добавить).

### <a name="create-shopify-plus-test-user"></a>Создание тестового пользователя Shopify Plus

На этом шаге вы создадите пользователя с именем B. Simon в приложении Shopify Plus. Вернитесь к разделу **Users** (Пользователи) и добавьте пользователя, указав адрес электронной почты и нужные разрешения. Перед использованием единого входа необходимо создать и активировать пользователей.

### <a name="enforce-saml-authentication"></a>Применение проверки подлинности SAML

> [!NOTE]
> Мы рекомендуем проверить интеграцию для отдельных пользователей, прежде чем применять ее ко всем.

Отдельные пользователи:
1. Откройте страницу отдельного пользователя в Shopify Plus, для которого указан домен электронной почты, управляемый Azure AD и утвержденный в Shopify Plus.
1. В разделе SAML authentication (Проверка подлинности SAML) выберите **Edit** (Изменить), **Required** (Требуется) и **Save** (Сохранить).
1. Убедитесь, что этот пользователь может успешно выполнять вход через потоки, инициированные поставщиком удостоверений и поставщиком услуг.

Для всех пользователей в домене электронной почты сделайте следующее:
1. Вернитесь на страницу **Security** (Безопасность).
1. Выберите **Required** (Обязательный) для параметра аутентификации SAML. В результате SAML будет принудительно применяться для всех пользователей Shopify Plus с таким доменом электронной почты.
1. Щелкните **Сохранить** .

> [!IMPORTANT]
> Включение SAML для всех пользователей в домене электронной почты влияет на всех пользователей, использующих это приложение. Теперь пользователи не смогут использовать для входа обычную страницу входа. Доступ к приложению будет возможен только с помощью Azure Active Directory. Shopify не предоставляет резервных URL-адресов для входа пользователей с использованием обычных имен пользователя и паролей. Если потребуется отключить SAML, свяжитесь со службой поддержки Shopify.
