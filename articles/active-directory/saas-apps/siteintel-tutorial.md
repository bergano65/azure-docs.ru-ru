# <a name="tutorial-azure-active-directory-single-sign-on-sso-integration-with-siteintel"></a>Руководство по интеграции единого входа Azure Active Directory с SiteIntel

В этом руководстве описано, как интегрировать SiteIntel с Azure Active Directory (Azure AD). Интеграция Azure AD с SiteIntel обеспечивает перечисленные ниже возможности.

* С помощью AAD вы можете контролировать доступ к SiteIntel.
* Поддержка автоматического входа пользователей в SiteIntel с учетными записями Azure AD.
* Централизованное управление учетными записями через портал Azure.

Дополнительные сведения об интеграции приложений SaaS с Azure AD см. в статье [Что такое доступ к приложениям и единый вход с помощью Azure Active Directory?](../manage-apps/what-is-single-sign-on.md).

## <a name="prerequisites"></a>Предварительные требования

Чтобы приступить к работе, потребуется следующее.

* Подписка Azure AD. Если у вас нет подписки, вы можете получить [бесплатную учетную запись](https://azure.microsoft.com/free/).
* Подписка SiteIntel с поддержкой единого входа.

## <a name="scenario-description"></a>Описание сценария

В рамках этого руководства вы настроите и проверите единый вход Azure AD в тестовой среде.

* SiteIntel поддерживает единый вход, инициируемый поставщиком службы и поставщиком удостоверений.
* После настройки SiteIntel вы можете применить функцию управления сеансом, которая в реальном времени защищает конфиденциальные данные вашей организации от хищения и несанкционированного доступа. Управление сеансом является расширением функции условного доступа. [Узнайте, как применять управление сеансами с помощью Microsoft Cloud App Security](/cloud-app-security/proxy-deployment-any-app).

## <a name="add-siteintel-from-the-gallery"></a>Добавление SiteIntel из коллекции

Чтобы настроить интеграцию SiteIntel с Azure AD, необходимо добавить SiteIntel из коллекции в список управляемых приложений SaaS.

1. Войдите на [портал Azure](https://portal.azure.com) с помощью личной учетной записи Майкрософт либо рабочей или учебной учетной записи.
1. В области слева выберите **Azure Active Directory**.
1. Перейдите в колонку **Корпоративные приложения** и выберите **Все приложения**.
1. Чтобы добавить новое приложение, выберите **Новое приложение**.
1. В поле **Добавить из коллекции** введите **SiteIntel**.
1. Выберите **SiteIntel** в списке результатов и добавьте это приложение. Подождите несколько секунд, пока приложение не будет добавлено в ваш клиент.

## <a name="configure-and-test-azure-ad-single-sign-on-for-siteintel"></a>Настройка и проверка единого входа Azure AD в SiteIntel

Настройте и проверьте единый вход Azure AD в SiteIntel с помощью тестового пользователя *B. Simon*. Для обеспечения работы единого входа необходимо установить связь между пользователем AAD и соответствующим пользователем в SiteIntel.

Чтобы настроить и проверить единый вход Azure AD в SiteIntel, выполните действия в следующих стандартных блоках:

1. **[Настройка единого входа Azure AD](#configure-azure-ad-sso)** необходима, чтобы пользователи могли использовать эту функцию.  

    а. **[Создание тестового пользователя Azure AD](#create-an-azure-ad-test-user)** требуется для проверки работы единого входа Azure AD от имени пользователя B. Simon.  

    b. **[Назначение тестового пользователя Azure AD](#assign-the-azure-ad-test-user)** необходимо, чтобы разрешить пользователю B. Simon использовать единый вход Azure AD.

1. **[Настройка единого входа в SiteIntel](#configure-siteintel-sso)** необходима, чтобы настроить параметры единого входа на стороне приложения.

    * **[Создание тестового пользователя SiteIntel](#create-a-siteintel-test-user)** требуется для того, чтобы в SiteIntel существовал пользователь B. Simon, связанный с одноименным пользователем в Azure AD.

1. **[Проверка единого входа](#test-sso)** необходима, чтобы убедиться в корректной работе конфигурации.

## <a name="configure-azure-ad-sso"></a>Настройка единого входа Azure AD

Чтобы включить единый вход Azure AD на портале Azure, выполните следующие действия:

1. На [портале Azure](https://portal.azure.com/) на странице интеграции с приложением **SiteIntel** перейдите к разделу **Управление** , а затем выберите **Единый вход**.
1. На странице **Выбрать метод единого входа** выберите **SAML**.
1. На странице **Настройка единого входа с помощью SAML** выберите **Изменить** (значок карандаша) рядом с областью **Базовая конфигурация SAML**.

   ![Снимок экрана "Страница настройки единого входа с помощью SAML"](common/edit-urls.png)

1. Если вы хотите настроить приложение в режиме, инициируемом IdP, в разделе **Базовая конфигурация SAML** выполните следующие действия.

    а. В поле **Идентификатор** введите URL-адрес в следующем формате: `urn:amazon:cognito:sp:<REGION>_<USERPOOLID>`

    b. В поле **URL-адрес ответа** введите URL-адрес в следующем формате: `https://<CLIENT>.auth.siteintel.com/saml2/idpresponse`

    c. В поле **Состояние ретранслятора** введите URL-адрес в следующем формате: `https://<CLIENT>.siteintel.com`

1. Если вы хотите настроить приложение для работы в режиме, инициируемом поставщиком услуг, выберите **Задать дополнительные URL-адреса** и выполните следующие действия.

   * В поле **URL-адрес входа** введите URL-адрес в следующем формате: `https://<CLIENT>.siteintel.com`

    > [!NOTE]
    > Эти значения приведены для примера. Замените их фактическим идентификатором, URL-адресом ответа, URL-адресом для входа и состоянием ретранслятора. Чтобы получить эти значения, обратитесь к [группе поддержки клиентов SiteIntel](mailto:support@intalytics.com). Можно также посмотреть шаблоны в разделе **Базовая конфигурация SAML** на портале Azure.

1. На странице **Настройка единого входа с помощью SAML** в разделе **Сертификат подписи SAML** щелкните кнопку **Копировать** , чтобы скопировать URL-адрес в поле **URL-адрес метаданных федерации приложений**.

    ![Снимок экрана: кнопка копирования "URL-адрес метаданных федерации приложения"](common/copy-metadataurl.png)

### <a name="create-an-azure-ad-test-user"></a>Создание тестового пользователя Azure AD

В этом разделе описано, как на портале Azure создать тестового пользователя с именем B.Simon.

1. Слева на портале Azure выберите **Azure Active Directory** > **Пользователи** > **Все пользователи**.
1. В верхней части панели выберите **Новый пользователь**.
1. В свойствах **Пользователя** выполните следующие действия.

   а. В поле **Имя** введите **B.Simon**.  

   b. В поле **Имя пользователя** введите имя пользователя в следующем формате: `username@companydomain.extension` (например, `B.Simon@contoso.com`).

   c. Установите флажок **Показать пароль** и запишите значение, которое отображается в поле **Пароль**.

   d. Нажмите кнопку **создания**.

### <a name="assign-the-azure-ad-test-user"></a>Назначение тестового пользователя Azure AD

В этом разделе описано, как разрешить пользователю B. Simon использовать единый вход Azure, предоставив этому пользователю доступ к SiteIntel.

1. На портале Azure выберите **Корпоративные приложения** > **Все приложения**.
1. В списке **Приложения** выберите **SiteIntel**.
1. В разделе **Управление** на странице сводных данных о приложении выберите **Пользователи и группы** :

   ![Снимок экраны ссылки "Пользователи и группы"](common/users-groups-blade.png)

1. Выберите **Добавить пользователя** , а затем на панели **Добавление назначения** щелкните **Пользователи и группы**.

    ![Снимок экрана: кнопка Add user (Добавить пользователя)](common/add-assign-user.png)

1. На панели **Пользователи и группы** выберите **B. Simon** , а затем нажмите кнопку **Выбрать** в нижней части экрана.
1. Если вы хотите получить значение роли в утверждении SAML, выберите в области **Выбор роли** подходящую роль для пользователя в предложенном списке, а затем щелкните **Выбрать**.
1. В области **Добавление назначения** нажмите кнопку **Назначить**.

## <a name="configure-siteintel-sso"></a>Настройка единого входа SiteIntel

Чтобы настроить единый вход на стороне SiteIntel, отправьте URL-адрес, скопированный из поля **URL-адрес метаданных федерации приложений** , в [группу поддержки SiteIntel](mailto:support@intalytics.com). Специалисты службы поддержки правильно настроят это значение для подключения единого входа SAML для приложения на обеих сторонах.

В этом разделе описано, как создать пользователя *Britta Simon* в приложении SiteIntel. Чтобы добавить пользователей на платформу SiteIntel, обратитесь в [службу поддержки SiteIntel](mailto:support@intalytics.com). Перед использованием единого входа необходимо создать и активировать пользователей.
