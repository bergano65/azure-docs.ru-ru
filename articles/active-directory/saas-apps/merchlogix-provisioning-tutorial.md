# <a name="tutorial-configure-merchlogix-for-automatic-user-provisioning"></a>Руководство по настройке MerchLogix для автоматической подготовки пользователей

В этом учебнике описаны шаги, которые нужно выполнить в MerchLogix и Azure Active Directory (Azure AD), чтобы настроить Azure AD для автоматической подготовки и отзыва пользователей и групп в MerchLogix.

> [!NOTE]
> В этом руководстве рассматривается соединитель, созданный на базе службы подготовки пользователей Azure AD. Подробные сведения о том, что делает эта служба, как она работает, и часто задаваемые вопросы см. в статье [Автоматическая подготовка пользователей и ее отзыв для приложений SaaS в Azure Active Directory](../app-provisioning/user-provisioning.md).

## <a name="prerequisites"></a>Предварительные требования

В сценарии, описанном в этом руководстве, предполагается, что у вас уже имеется:

* клиент Azure AD;
* клиент MerchLogix;
* контактное лицо по техническим вопросам в MerchLogix, которое может предоставить URL-адрес конечной точки SCIM и секретный токен, необходимые для подготовки пользователей.

## <a name="adding-merchlogix-from-the-gallery"></a>Добавление MerchLogix из коллекции

Перед настройкой MerchLogix для автоматической подготовки пользователей в Azure AD необходимо добавить MerchLogix из коллекции приложений Azure AD в список управляемых приложений SaaS.

**Чтобы добавить MerchLogix из коллекции приложений Azure AD, сделайте следующее.**

1. На **[портале Azure](https://portal.azure.com)** в области навигации слева щелкните значок **Azure Active Directory**. 

    ![Кнопка Azure Active Directory][1]

2. Перейдите к элементу **Корпоративные приложения** > **Все приложения**.

    ![Раздел "Корпоративные приложения"][2]

3. Чтобы добавить MerchLogix, в верхней части диалогового окна нажмите кнопку **Новое приложение**.

    ![Кнопка "Создать приложение"][3]

4. В поле поиска введите **MerchLogix**.

5. В области результатов выберите **MerchLogix** и нажмите кнопку **Добавить**, чтобы добавить это приложение в список приложений SaaS.

    ![Снимок экрана, на котором показан раздел добавления из коллекции с выделенным текстовым полем "Введите имя".][4]

## <a name="assigning-users-to-merchlogix"></a>Назначение пользователей MerchLogix

В Azure Active Directory для определения того, какие пользователи должны получать доступ к выбранным приложениям, используется концепция, называемая "назначение". В контексте автоматической подготовки синхронизированы будут только те пользователи и группы, которые были "назначены" приложению в Azure AD. 

Перед настройкой и включением автоматической подготовки пользователей нужно решить, какие пользователи или группы в Azure AD должны иметь доступ к MerchLogix. Когда этот вопрос будет решен, этих пользователей и (или) группы можно будет назначить приложению MerchLogix, следуя приведенным ниже указаниям.

* [Назначение корпоративному приложению пользователя или группы](../manage-apps/assign-user-or-group-access-portal.md)

### <a name="important-tips-for-assigning-users-to-merchlogix"></a>Важные рекомендации по назначению пользователей в MerchLogix

* Рекомендуется назначить одного пользователя Azure AD в MerchLogix для тестирования исходной конфигурации автоматической подготовки пользователей. Дополнительные пользователи и (или) группы можно будет назначить позже, после успешного прохождения тестов.

* При назначении пользователя в MerchLogix в диалоговом окне назначения необходимо выбрать действительную роль для конкретного приложения (если она доступна). Пользователи с ролью **Доступ по умолчанию** исключаются из подготовки.

## <a name="configuring-automatic-user-provisioning-to-merchlogix"></a>Настройка автоматической подготовки пользователей в MerchLogix 

В этом разделе описывается, как настроить службу подготовки Azure AD для создания, обновления и отключения пользователей и (или) групп в MerchLogix на основе их назначений в Azure AD.

> [!TIP]
> Для MerchLogix также можно включить единый вход на основе SAML. Для этого следуйте инструкциям, приведенным в [учебнике по единому входу в MerchLogix](merchlogix-tutorial.md). Единый вход можно настроить независимо от автоматической подготовки пользователей, хотя эти две возможности дополняют друг друга.

### <a name="to-configure-automatic-user-provisioning-for-merchlogix-in-azure-ad"></a>Чтобы настроить автоматическую подготовку пользователей MerchLogix в Azure AD, сделайте следующее.

1. Войдите на [портал Azure](https://portal.azure.com) и перейдите к **Azure Active Directory > Корпоративные приложения > Все приложения**.

2. В списке приложений SaaS выберите MerchLogix.

3. Выберите вкладку **Подготовка**.

4. Для параметра **Режим подготовки к работе** выберите значение **Automatic** (Автоматически).

    ![Снимок экрана, на котором показан раздел "MerchLogix — Подготовка", где выбран подраздел "Подготовка", для параметра "Режим подготовки" задано значение "Авто" и выделен параметр "Проверить подключение".](./media/merchlogix-provisioning-tutorial/Merchlogix1.png)

5. В разделе **Учетные данные администратора** выполните следующие действия:

    * В поле **URL-адрес клиента** введите URL-адрес конечной точки SCIM, предоставленный контактным лицом по техническим вопросам в MerchLogix.

    * В поле **Секретный токен** введите секретный токен, предоставленный контактным лицом по техническим вопросам в MerchLogix.

6. После заполнения полей, указанных на шаге 5, щелкните **Проверить подключение**, чтобы убедиться, что Azure AD может подключиться к MerchLogix. Если установить подключение не удалось, убедитесь, что у учетной записи MerchLogix есть разрешения администратора, и повторите попытку.

7. В поле **Почтовое уведомление** введите адрес электронной почты пользователя или группы, которые должны получать уведомления об ошибках подготовки, а также установите флажок **Send an email notification when a failure occurs** (Отправить уведомление по электронной почте при сбое).

8. Выберите команду **Сохранить**.

9. В разделе **Сопоставления** выберите **Синхронизировать пользователей Azure Active Directory с MerchLogix**.

10. В разделе **Сопоставление атрибутов** просмотрите пользовательские атрибуты, которые синхронизируются из Azure AD в MerchLogix. Атрибуты, выбранные как свойства в разделе **Сопоставление**, используются для сопоставления учетных записей пользователей в MerchLogix в операциях обновления. Нажмите кнопку **Сохранить**, чтобы зафиксировать все изменения.

11. В разделе **Сопоставления** выберите **Синхронизировать группы Azure Active Directory с MerchLogix**.

12. В разделе **Сопоставление атрибутов** просмотрите атрибуты группы, которые синхронизируются из Azure AD в MerchLogix. Атрибуты, выбранные как свойства в разделе **Сопоставление**, используются для сопоставления групп в MerchLogix в операциях обновления. Нажмите кнопку **Сохранить**, чтобы зафиксировать все изменения.

13. Чтобы включить службу подготовки Azure AD для MerchLogix, измените значение параметра **Состояние подготовки** на **Включено** в разделе **Параметры**.

14. Когда будете готовы выполнить подготовку, нажмите кнопку **Сохранить**.

После этого начнется начальная синхронизация пользователей и (или) групп, определенных в поле **Область** раздела **Параметры**. Начальная синхронизация занимает больше времени, чем последующие операции синхронизации. Если служба запущена, они выполняются примерно каждые 40 минут. В разделе **Сведения о синхронизации** можно отслеживать ход выполнения и переходить по ссылкам для просмотра отчетов о подготовке, в которых зафиксированы все действия, выполняемые службой подготовки Azure AD с приложением MerchLogix.

Дополнительные сведения о чтении журналов подготовки Azure AD см. в руководстве по [отчетам об автоматической подготовке учетных записей](../app-provisioning/check-status-user-account-provisioning.md).
