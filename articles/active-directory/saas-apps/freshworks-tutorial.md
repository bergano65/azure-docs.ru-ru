# <a name="tutorial-azure-active-directory-single-sign-on-sso-integration-with-freshworks"></a>Руководство по интеграции единого входа Azure Active Directory с Freshworks

В этом учебнике описано, как интегрировать Freshworks с Azure Active Directory (Azure AD). Интеграция Freshworks с Azure AD обеспечивает следующие возможности:

* С помощью Azure AD вы можете контролировать доступ к Freshworks.
* Вы можете включить автоматический вход пользователей во Freshworks с помощью учетных записей Azure AD.
* Централизованное управление учетными записями через портал Azure.

## <a name="prerequisites"></a>Предварительные требования

Чтобы приступить к работе, потребуется следующее.

* Подписка Azure AD. Если у вас нет подписки, вы можете получить [бесплатную учетную запись](https://azure.microsoft.com/free/).
* Подписка Freshworks с поддержкой единого входа.

> [!NOTE]
> Эту интеграцию также можно использовать в облачной среде Azure AD для государственных организаций США. Это приложение можно найти в коллекции облачных приложений с поддержкой Azure AD для государственных организаций США и настроить таким же образом, как и в общедоступном облаке.

## <a name="scenario-description"></a>Описание сценария

В рамках этого руководства вы настроите и проверите единый вход Azure AD в тестовой среде.

* Freshworks поддерживает единый вход, инициированный **поставщиком услуг**.

## <a name="add-freshworks-from-the-gallery"></a>Добавление Freshworks из коллекции

Чтобы настроить интеграцию Freshworks с Azure AD, необходимо добавить Freshworks из коллекции в список управляемых приложений SaaS.

1. Войдите на портал Azure с помощью личной учетной записи Майкрософт либо рабочей или учебной учетной записи.
1. В области навигации слева выберите службу **Azure Active Directory**.
1. Перейдите в колонку **Корпоративные приложения** и выберите **Все приложения**.
1. Чтобы добавить новое приложение, выберите **Новое приложение**.
1. В разделе **Добавление из коллекции** в поле поиска введите **Freshworks**.
1. Выберите **Freshworks** в области результатов и добавьте это приложение. Подождите несколько секунд, пока приложение не будет добавлено в ваш клиент.

## <a name="configure-and-test-azure-ad-sso-for-freshworks"></a>Настройка и проверка единого входа Azure AD для Freshworks

Настройте и проверьте единый вход Azure AD в Freshworks с помощью тестового пользователя **B. Simon**. Для обеспечения работы единого входа необходимо установить связь между пользователем Azure AD и соответствующим пользователем во Freshworks.

Чтобы настроить и проверить единый вход Azure AD в Freshworks, выполните следующие действия:

1. **[Настройка единого входа Azure AD](#configure-azure-ad-sso)** необходима, чтобы пользователи могли использовать эту функцию.
    1. **[Создание тестового пользователя Azure AD](#create-an-azure-ad-test-user)** требуется для проверки работы единого входа Azure AD с помощью пользователя B.Simon.
    1. **[Назначение тестового пользователя Azure AD](#assign-the-azure-ad-test-user)** необходимо, чтобы позволить пользователю B.Simon использовать единый вход Azure AD.
1. **[Настройка единого входа во Freshworks](#configure-freshworks-sso)** необходима, чтобы настроить параметры единого входа на стороне приложения.
    1. **[Создание тестового пользователя Freshworks](#create-freshworks-test-user)** требуется для того, чтобы в Freshworks существовал пользователь B. Simon, связанный с одноименным пользователем в Azure AD.
1. **[Проверка единого входа](#test-sso)** позволяет убедиться в правильности конфигурации.

## <a name="configure-azure-ad-sso"></a>Настройка единого входа Azure AD

Выполните следующие действия, чтобы включить единый вход Azure AD на портале Azure.

1. На портале Azure на странице интеграции с приложением **Freshworks** найдите раздел **Управление** и выберите **Единый вход**.
1. На странице **Выбрать метод единого входа** выберите **SAML**.
1. На странице **Настройка единого входа с помощью SAML** щелкните значок карандаша, чтобы открыть диалоговое окно **Базовая конфигурация SAML** для изменения параметров.

   ![Изменение базовой конфигурации SAML](common/edit-urls.png)

1. На странице **Базовая конфигурация SAML** введите значения следующих полей.

    а. В текстовом поле **URL-адрес для входа** введите URL-адрес в следующем формате: `https://<SUBDOMAIN>.freshworks.com/login`.

    b. В текстовом поле **Идентификатор (сущности)** введите URL-адрес в следующем формате: `https://<SUBDOMAIN>.freshworks.com/sp/SAML/<MODULE_ID>/metadata`.

    > [!NOTE]
    > Эти значения приведены для примера. Необходимо обновить эти значения действующим URL-адресом для входа и идентификатором. Чтобы получить эти значения, обратитесь в [службу поддержки клиентов Freshworks](mailto:support@freshworks.com). Можно также посмотреть шаблоны в разделе **Базовая конфигурация SAML** на портале Azure.

1. На странице **Настройка единого входа с помощью SAML** в разделе **Сертификат подписи SAML** найдите пункт **Сертификат (Base64)** и щелкните **Скачать**, чтобы скачать сертификат. Сохраните этот сертификат на компьютере.

    ![Ссылка для скачивания сертификата](common/certificatebase64.png)

1. Чтобы изменить параметры в разделе **Подпись** в соответствии со своими требованиями, нажмите кнопку **Изменить**, чтобы открыть диалоговое окно **Сертификат подписи SAML**.

     ![Изображение](common/edit-certificate.png)

     ![Снимок экрана, на котором показано диалоговое окно "Сертификат подписи SAML" с выбранной кнопкой "Изменить"](./media/freshworks-tutorial/response.png)

    а. Для параметра **Вариант подписывания** выберите значение **Ответ знака SAML**.

    b. Выберите команду **Сохранить**.

1. Требуемые URL-адреса можно скопировать из раздела **Настройка Freshworks**.

    ![Копирование URL-адресов настройки](common/copy-configuration-urls.png)

### <a name="create-an-azure-ad-test-user"></a>Создание тестового пользователя Azure AD

В этом разделе описано, как на портале Azure создать тестового пользователя с именем B.Simon.

1. На портале Azure в области слева выберите **Azure Active Directory**, **Пользователи**, а затем — **Все пользователи**.
1. В верхней части экрана выберите **Новый пользователь**.
1. В разделе **Свойства пользователя** выполните следующие действия.
   1. В поле **Имя** введите `B.Simon`.  
   1. В поле **Имя пользователя** введите username@companydomain.extension. Например, `B.Simon@contoso.com`.
   1. Установите флажок **Показать пароль** и запишите значение, которое отображается в поле **Пароль**.
   1. Нажмите кнопку **Создать**.

### <a name="assign-the-azure-ad-test-user"></a>Назначение тестового пользователя Azure AD

В этом разделе описано, как включить единый вход Azure для пользователя B. Simon, предоставив этому пользователю доступ к Freshworks.

1. На портале Azure выберите **Корпоративные приложения**, а затем — **Все приложения**.
1. В списке приложений выберите **Freshworks**.
1. На странице "Обзор" приложения найдите раздел **Управление** и выберите **Пользователи и группы**.
1. Выберите **Добавить пользователя**, а в диалоговом окне **Добавление назначения** выберите **Пользователи и группы**.
1. В диалоговом окне **Пользователи и группы** выберите **B.Simon** в списке пользователей, а затем в нижней части экрана нажмите кнопку **Выбрать**.
1. Если пользователям необходимо назначить роль, вы можете выбрать ее из раскрывающегося списка **Выберите роль**. Если для этого приложения не настроена ни одна роль, будет выбрана роль "Доступ по умолчанию".
1. В диалоговом окне **Добавление назначения** нажмите кнопку **Назначить**.

## <a name="configure-freshworks-sso"></a>Настройка единого входа для Freshworks

1. Откройте новое окно веб-браузера, зайдите на сайт Freshworks с правами администратора и сделайте следующее:

2. В левой части меню щелкните значок **Security** (Безопасность), затем установите флажок **Single sign-on** (Единый вход) и выберите **SAML SSO** (Единый вход SAML) в разделе **Authentication Methods** (Методы аутентификации).

    ![Снимок экрана, на котором показан раздел "Security > Authentication Methods" (Безопасность > Способы проверки подлинности) с включенным параметром "SAML SSO" (Единый вход SAML)](./media/freshworks-tutorial/configure01.png)

3. В разделе **Single sign-on** (Единый вход) выполните следующие действия:

    ![Настройка приложения Freshworks](./media/freshworks-tutorial/configure02.png)

    а. Щелкните **Copy** (Копировать), чтобы скопировать **идентификатор сущности поставщика служб** для вашего экземпляра и вставить его в текстовое поле **идентификатора сущности** в разделе **Базовая конфигурация SAML** на портале Azure.

    b. В текстовое поле **Entity ID provided by the IdP** (Идентификатор сущности, предоставленный поставщиком удостоверений) вставьте значение **Идентификатор Azure AD**, скопированное на портале Azure.

    c. В текстовое поле **SAML SSO URL** (URL-адрес для единого входа SAML) вставьте значение **URL-адрес входа**, скопированное на портале Azure.

    d. Откройте сертификат в кодировке Base64 в Блокноте, скопируйте его содержимое и вставьте его в текстовое поле **Security certificate** (Сертификат безопасности).

    д) Выберите команду **Сохранить**.

### <a name="create-freshworks-test-user"></a>Создание тестового пользователя во Freshworks

В этом разделе вы узнаете, как создать пользователя B. Simon в приложении Freshworks. Чтобы добавить пользователей на платформу Freshworks, обратитесь в [службу поддержки клиентов Freshworks](mailto:support@freshworks.com). Перед использованием единого входа необходимо создать и активировать пользователей. 

## <a name="test-sso"></a>Проверка единого входа 

В этом разделе описано, как проверить конфигурацию единого входа Azure AD с помощью указанных ниже способов. 

* Выберите **Тестировать приложение** на портале Azure. Вы будете перенаправлены по URL-адресу для входа в Freshworks, где можно инициировать поток входа. 

* Перейдите по URL-адресу для входа в Freshworks и инициируйте поток входа.

* Вы можете использовать портал "Мои приложения" корпорации Майкрософт. Щелкнув плитку Freshworks на портале "Мои приложения", вы автоматически войдете в приложение Freshworks, для которого настроили единый вход. Дополнительные сведения о портале "Мои приложения" см. в [этой статье](https://docs.microsoft.com/azure/active-directory/active-directory-saas-access-panel-introduction).
