# <a name="tutorial-integrate-civic-platform-with-azure-active-directory"></a>Руководство по Интеграция платформы Civic Platform с Azure Active Directory

В этом руководстве описано, как интегрировать платформу Civic Platform с Azure Active Directory (Azure AD). Интеграция платформы Civic Platform с Azure AD обеспечивает возможности, перечисленные ниже.

* С помощью Azure AD вы можете контролировать доступ к платформе Civic Platform.
* Вы можете включить автоматический вход для пользователей платформы Civic Platform с помощью учетной записи Azure AD.
* Централизованное управление учетными записями через портал Azure.

Чтобы узнать больше об интеграции приложений SaaS с Azure AD, прочитайте статью [Что такое доступ к приложениям и единый вход с помощью Azure Active Directory?](https://docs.microsoft.com/azure/active-directory/active-directory-appssoaccess-whatis)

## <a name="prerequisites"></a>Предварительные требования

Чтобы приступить к работе, потребуется следующее:

* подписка Azure AD Если у вас нет подписки, [здесь](https://azure.microsoft.com/pricing/free-trial/) вы можете получить бесплатную пробную версию сроком на один месяц.
* подписка с поддержкой единого входа на платформе Civic Platform.

## <a name="scenario-description"></a>Описание сценария

В рамках этого руководства вы настроите и проверите единый вход Azure AD в тестовой среде.

* Платформа Civic Platform поддерживает единый вход, инициированный **поставщиком услуг**.





## <a name="adding-civic-platform-from-the-gallery"></a>Добавление платформы Civic Platform из коллекции

Чтобы настроить интеграцию платформы Civic Platform с Azure AD, добавьте платформу Civic Platform из коллекции в список управляемых приложений SaaS.

1. Войдите на [портал Azure](https://portal.azure.com) с помощью личной учетной записи Майкрософт либо рабочей или учебной учетной записи.
1. В области навигации слева выберите службу **Azure Active Directory**.
1. Перейдите в колонку **Корпоративные приложения** и выберите **Все приложения**.
1. Чтобы добавить новое приложение, выберите **Новое приложение**.
1. В разделе **Добавление из коллекции** в поле поиска введите **платформа Civic Platform**.
1. Выберите **платформу Civic Platform** на панели результатов, а затем добавьте приложение. Подождите несколько секунд, пока приложение не будет добавлено в ваш клиент.


## <a name="configure-and-test-azure-ad-single-sign-on"></a>Настройка и проверка единого входа в Azure AD

Настройте и проверьте единый вход Azure AD в платформу Civic Platform с помощью тестового пользователя **B.Simon**. Для обеспечения работы единого входа необходимо установить связь между пользователем Azure AD и соответствующим пользователем на платформе Civic Platform.

Чтобы настроить и проверить единый вход Azure AD на платформу Civic Platform, выполните действия в следующих стандартных блоках:

1. **[Настройка единого входа Azure AD](#configure-azure-ad-sso)** необходима, чтобы пользователи могли использовать эту функцию.
2. **[Настройка единого входа на платформу Civic Platform](#configure-civic-platform-sso)** необходима, чтобы настроить параметры единого входа на стороне приложения.
3. **[Создание тестового пользователя Azure AD](#create-an-azure-ad-test-user)** требуется для проверки работы единого входа Azure AD с помощью пользователя B.Simon.
4. **[Назначение тестового пользователя Azure AD](#assign-the-azure-ad-test-user)** необходимо, чтобы позволить пользователю B.Simon использовать единый вход Azure AD.
5. **[Создание тестового пользователя платформы Civic Platform](#create-civic-platform-test-user)** требуется для того, чтобы на платформе Civic Platform существовал пользователь B.Simon, связанный с одноименным пользователем в Azure AD.
6. **[Проверка единого входа](#test-sso)** необходима, чтобы убедиться в корректной работе конфигурации.

### <a name="configure-azure-ad-sso"></a>Настройка единого входа Azure AD

Выполните следующие действия, чтобы включить единый вход Azure AD на портале Azure.

1. На [портале Azure](https://portal.azure.com/) на странице интеграции с приложением **платформы Civic Platform** найдите раздел **Управление** и выберите **Единый вход**.
1. На странице **Выбрать метод единого входа** выберите **SAML**.
1. На странице **Настройка единого входа с помощью SAML** щелкните значок "Изменить" (значок пера), чтобы открыть диалоговое окно **Базовая конфигурация SAML** и изменить параметры.

   ![Правка базовой конфигурации SAML](common/edit-urls.png)

1. На странице **Базовая конфигурация SAML** введите значения следующих полей.

    a. В текстовом поле **URL-адрес для входа** введите URL-адрес в следующем формате: `https://<SUBDOMAIN>.accela.com`.

    b. В текстовом поле **Идентификатор (сущности)** введите URL-адрес следующим образом: `civicplatform.accela.com`

    > [!NOTE]
    > Значение URL-адреса входа приведено только для примера. Вместо него необходимо указать фактический URL-адрес для входа. Чтобы получить эти значения, обратитесь в [Службу технической поддержки платформы Civic Platform](mailto:skale@accela.com). Можно также посмотреть шаблоны в разделе **Базовая конфигурация SAML** на портале Azure.

1. На странице **Настройка единого входа с помощью SAML** в разделе **Сертификат подписи SAML** нажмите кнопку "Копировать", чтобы копировать **URL-адрес метаданных федерации приложений** и сохранить его на компьютере.

    ![Ссылка для скачивания сертификата](common/copy-metadataurl.png)

1. Перейдите в раздел **Azure Active Directory** > **Регистрация приложений** в Azure AD и выберите нужное приложение.

1. Скопируйте **идентификатор каталога (клиента)** и сохраните его в блокноте.

    ![Скопируйте идентификатор каталога (клиента) и сохраните его в коде приложения](media/civic-platform-tutorial/directoryid.png)

1. Скопируйте **идентификатор приложения** и сохраните его в блокноте.

   ![Скопируйте идентификатор приложения (клиента)](media/civic-platform-tutorial/applicationid.png)

1. Перейдите в раздел **Azure Active Directory** > **Регистрация приложений** в Azure AD и выберите нужное приложение. Выберите **Сертификаты и секреты**.

1. Выберите **Секреты клиента -> Новый секрет клиента**.

1. Введите описание и срок действия секрета. Когда все будет готово, нажмите **Добавить**.

   > [!NOTE]
   > После сохранения секрета клиента отображается значение секрета клиента. Это значение невозможно будет получить позже, поэтому скопируйте его сразу.

   ![Скопируйте значение секрета, так как вы не сможете получить его позже](media/civic-platform-tutorial/secretkey.png)

### <a name="configure-civic-platform-sso"></a>Настройка единого входа для платформы Civic Platform

1. Откройте новое окно веб-браузера и войдите на сайт компании Atlassian Cloud в качестве администратора.

1. Нажмите кнопку **Стандартный выбор**.

    ![Ссылка для скачивания сертификата](media/civic-platform-tutorial/standard-choices.png)

1. Создайте стандартный вариант **ssoconfig**.

1. Найдите **ssoconfig** и отправьте его.

    ![Ссылка для скачивания сертификата](media/civic-platform-tutorial/sso-config.png)

1. Разверните SSOCONFIG, щелкнув красную точку.

    ![Ссылка для скачивания сертификата](media/civic-platform-tutorial/sso-config01.png)

1. Укажите сведения о конфигурации, связанные с единым входом, на следующем шаге:

    ![Ссылка для скачивания сертификата](media/civic-platform-tutorial/sso-config02.png)

    1. В поле **applicationid** введите значение **Идентификатора приложения**, скопированное с портала Azure.

    1. В поле **clientSecret** введите значение **Секрет**, скопированное с портала Azure.

    1. В поле **directoryId** введите **идентификатор каталога (клиента)** , скопированный с портала Azure.

    1. Введите idpName. Пример: `Azure`.

### <a name="create-an-azure-ad-test-user"></a>Создание тестового пользователя Azure AD

В этом разделе описано, как на портале Azure создать тестового пользователя с именем B.Simon.

1. На портале Azure в области слева выберите **Azure Active Directory**, **Пользователи**, а затем — **Все пользователи**.
1. В верхней части экрана выберите **Новый пользователь**.
1. В разделе **Свойства пользователя** выполните следующие действия.
   1. В поле **Имя** введите `B.Simon`.  
   1. В поле **Имя пользователя** введите username@companydomain.extension. Например, `B.Simon@contoso.com`.
   1. Установите флажок **Показать пароль** и запишите значение, которое отображается в поле **Пароль**.
   1. Нажмите кнопку **Создать**.

### <a name="assign-the-azure-ad-test-user"></a>Назначение тестового пользователя Azure AD

В этом разделе описано, как включить единый вход Azure для пользователя B.Simon, предоставив этому пользователю доступ к платформе Civic Platform.

1. На портале Azure выберите **Корпоративные приложения**, а затем —**Все приложения**.
1. Из списка приложений выберите **Civic Platform**.
1. На странице "Обзор" приложения найдите раздел **Управление** и выберите **Пользователи и группы**.

   ![Ссылка "Пользователи и группы"](common/users-groups-blade.png)

1. Выберите **Добавить пользователя**, а в диалоговом окне **Добавление назначения** выберите **Пользователи и группы**.

    ![Ссылка "Добавить пользователя"](common/add-assign-user.png)

1. В диалоговом окне **Пользователи и группы** выберите **B.Simon** в списке пользователей, а затем в нижней части экрана нажмите кнопку **Выбрать**.
1. Если ожидается, что в утверждении SAML будет получено какое-либо значение роли, то в диалоговом окне **Выбор роли** нужно выбрать соответствующую роль для пользователя из списка и затем нажать кнопку **Выбрать**, расположенную в нижней части экрана.
1. В диалоговом окне **Добавление назначения** нажмите кнопку **Назначить**.

### <a name="create-civic-platform-test-user"></a>Создание тестового пользователя платформы Civic Platform

Обратитесь в [Службу поддержки платформы Civic Platform](mailto:skale@accela.com), чтобы добавить пользователей в группу поддержки платформы Civic Platform. Перед использованием единого входа необходимо создать и активировать пользователей.
