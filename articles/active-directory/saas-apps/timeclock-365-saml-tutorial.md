# <a name="tutorial-azure-active-directory-single-sign-on-sso-integration-with-timeclock-365-saml"></a>Руководство по интеграции единого входа Azure Active Directory с Timeclock 365 SAML

В этом учебнике описано, как интегрировать Timeclock 365 SAML с Azure Active Directory (Azure AD). Интеграция Timeclock 365 SAML с Azure AD обеспечивает следующие возможности.

* Управление доступом к Timeclock 365 SAML в Azure AD.
* Автоматический вход пользователей в Timeclock 365 SAML с учетными записями Azure AD.
* Централизованное управление учетными записями через портал Azure.

## <a name="prerequisites"></a>Предварительные требования

Чтобы приступить к работе, потребуется следующее.

* Подписка Azure AD. Если у вас нет подписки, вы можете получить [бесплатную учетную запись](https://azure.microsoft.com/free/).
* Подписка на Timeclock 365 SAML с поддержкой единого входа.

## <a name="scenario-description"></a>Описание сценария

В рамках этого руководства вы настроите и проверите единый вход Azure AD в тестовой среде.

* Timeclock 365 SAML поддерживает единый вход, инициируемый **поставщиком услуг**.

## <a name="adding-timeclock-365-saml-from-the-gallery"></a>Добавление Timeclock 365 SAML из коллекции

Чтобы настроить интеграцию Timeclock 365 SAML с Azure AD, необходимо добавить Timeclock 365 SAML из коллекции в список управляемых SaaS-приложений.

1. Войдите на портал Azure с помощью личной учетной записи Майкрософт либо рабочей или учебной учетной записи.
1. В области навигации слева выберите службу **Azure Active Directory**.
1. Перейдите в колонку **Корпоративные приложения** и выберите **Все приложения**.
1. Чтобы добавить новое приложение, выберите **Новое приложение**.
1. В разделе **Добавление из коллекции** в поле поиска введите **Timeclock 365 SAML**.
1. В область результатов выберите **Timeclock 365 SAML** и добавьте это приложение. Подождите несколько секунд, пока приложение не будет добавлено в ваш клиент.


## <a name="configure-and-test-azure-ad-sso-for-timeclock-365-saml"></a>Настройка и проверка единого входа Azure AD для Timeclock 365 SAML

Настройте и проверьте единый вход Azure AD в Timeclock 365 SAML с помощью тестового пользователя **B.Simon**. Для обеспечения работы единого входа необходимо установить связь между пользователем Azure AD и соответствующим пользователем в Timeclock 365 SAML.

Чтобы настроить и проверить единый вход Azure AD в Timeclock 365 SAML, выполните следующие действия.

1. **[Настройка единого входа Azure AD](#configure-azure-ad-sso)** необходима, чтобы пользователи могли использовать эту функцию.
    1. **[Создание тестового пользователя Azure AD](#create-an-azure-ad-test-user)** требуется для проверки работы единого входа Azure AD с помощью пользователя B.Simon.
    1. **[Назначение тестового пользователя Azure AD](#assign-the-azure-ad-test-user)** необходимо, чтобы позволить пользователю B.Simon использовать единый вход Azure AD.
1. **[Настройка единого входа в Timeclock 365 SAML](#configure-timeclock-365-saml-sso)** необходима, чтобы настроить параметры единого входа на стороне приложения.
    1. **[Создание тестового пользователя Timeclock 365 SAML](#create-timeclock-365-saml-test-user)** требуется для создания в Timeclock 365 SAML пользователя B.Simon, связанного с одноименным пользователем в Azure Active Directory.
1. **[Проверка единого входа](#test-sso)** позволяет убедиться в правильности конфигурации.

## <a name="configure-azure-ad-sso"></a>Настройка единого входа Azure AD

Выполните следующие действия, чтобы включить единый вход Azure AD на портале Azure.

1. На портале Azure на странице интеграции с приложением **Timeclock 365 SAML** найдите раздел **Управление** и выберите **Единый вход**.
1. На странице **Выбрать метод единого входа** выберите **SAML**.
1. На странице **Настройка единого входа с помощью SAML** щелкните значок "Изменить" (значок пера), чтобы открыть диалоговое окно **Базовая конфигурация SAML** и изменить параметры.

   ![Изменение базовой конфигурации SAML](common/edit-urls.png)

1. На странице **Базовая конфигурация SAML** введите значения следующих полей.

    В текстовом поле **URL-адрес входа** введите URL-адрес: `https://live.timeclock365.com/login`.


1. На странице **Настройка единого входа с помощью SAML** в разделе **Сертификат подписи SAML** нажмите кнопку "Копировать", чтобы скопировать **URL-адрес метаданных федерации приложений** и сохранить его на компьютере.

    ![Ссылка для скачивания сертификата](common/copy-metadataurl.png)

### <a name="create-an-azure-ad-test-user"></a>Создание тестового пользователя Azure AD

В этом разделе описано, как на портале Azure создать тестового пользователя с именем B.Simon.

1. На портале Azure в области слева выберите **Azure Active Directory**, **Пользователи**, а затем — **Все пользователи**.
1. В верхней части экрана выберите **Новый пользователь**.
1. В разделе **Свойства пользователя** выполните следующие действия.
   1. В поле **Имя** введите `B.Simon`.  
   1. В поле **Имя пользователя** введите username@companydomain.extension. Например, `B.Simon@contoso.com`.
   1. Установите флажок **Показать пароль** и запишите значение, которое отображается в поле **Пароль**.
   1. Нажмите кнопку **Создать**.

### <a name="assign-the-azure-ad-test-user"></a>Назначение тестового пользователя Azure AD

В этом разделе описано, как включить единый вход в Azure для пользователя B.Simon, предоставив этому пользователю доступ к Timeclock 365 SAML.

1. На портале Azure выберите **Корпоративные приложения**, а затем — **Все приложения**.
1. Из списка приложений выберите **Timeclock 365 SAML**.
1. На странице "Обзор" приложения найдите раздел **Управление** и выберите **Пользователи и группы**.
1. Выберите **Добавить пользователя**, а в диалоговом окне **Добавление назначения** выберите **Пользователи и группы**.
1. В диалоговом окне **Пользователи и группы** выберите **B.Simon** в списке пользователей, а затем в нижней части экрана нажмите кнопку **Выбрать**.
1. Если пользователям необходимо назначить роль, вы можете выбрать ее из раскрывающегося списка **Выберите роль**. Если для этого приложения не настроена ни одна роль, будет выбрана роль "Доступ по умолчанию".
1. В диалоговом окне **Добавление назначения** нажмите кнопку **Назначить**.

## <a name="configure-timeclock-365-saml-sso"></a>Настройка единого входа в Timeclock 365 SAML

1. Откройте новую вкладку в браузере и войдите на корпоративный сайт Timeclock 365 SAML с правами администратора.

1. Выполните приведенные ниже действия.

    ![Конфигурация Timeclock](./media/timeclock-365-saml-tutorial/saml-configuration.png)

    а. Перейдите на вкладку **Параметры > Профиль компании > Параметры**.

    b. В текстовое поле **IDP metadata path** (Путь метаданных поставщика удостоверений) вставьте **URL-адрес метаданных федерации приложений**, скопированный на портале Azure.

    c. Затем щелкните **Обновить**.

### <a name="create-timeclock-365-saml-test-user"></a>Создание тестового пользователя Timeclock 365 SAML

1. Откройте новую вкладку в браузере и войдите на корпоративный сайт Timeclock 365 SAML с правами администратора.

1. Перейдите на вкладку **Пользователи > Добавить нового пользователя**.

    ![Создание тестового пользователя 1 ](./media/timeclock-365-saml-tutorial/add-user-1.png)

1. Введите все необходимые сведения на странице **Сведения о пользователе** и нажмите кнопку **Создать**.

    ![Создание тестового пользователя 2 ](./media/timeclock-365-saml-tutorial/add-user-2.png)

1. Нажмите кнопку **Создать**, чтобы создать тестового пользовател.

## <a name="test-sso"></a>Проверка единого входа 

В этом разделе описано, как проверить конфигурацию единого входа Azure AD с помощью указанных ниже способов. 

* Выберите **Тестировать приложение** на портале Azure. Вы будете перенаправлены по URL-адресу для входа в Timeclock 365 SAML, где можно инициировать поток входа. 

* Перейдите по URL-адресу для входа в Timeclock 365 SAML и инициируйте поток входа.

* Вы можете использовать портал "Мои приложения" корпорации Майкрософт. Щелкнув элемент Timeclock 365 SAML на портале "Мои приложения", вы перейдете по URL-адресу для входа в Timeclock 365 SAML. Дополнительные сведения о портале "Мои приложения" см. в [этой статье](../user-help/my-apps-portal-end-user-access.md).
