# <a name="tutorial-azure-active-directory-single-sign-on-sso-integration-with-wootric"></a>Руководство по интеграции единого входа Azure Active Directory с Wootric

В этом руководстве описано, как интегрировать Wootric с Azure Active Directory (Azure AD). Интеграция Wootric с Azure AD обеспечивает следующие возможности^

* Управление доступом к Wootric в Azure AD.
* Поддержка автоматического входа пользователей в Wootric с учетными записями Azure AD.
* Централизованное управление учетными записями через портал Azure.

Чтобы узнать больше об интеграции приложений SaaS с Azure AD, прочитайте статью [Что такое доступ к приложениям и единый вход с помощью Azure Active Directory?](../manage-apps/what-is-single-sign-on.md)

## <a name="prerequisites"></a>Предварительные требования

Чтобы приступить к работе, потребуется следующее.

* Подписка Azure AD. Если у вас нет подписки, вы можете получить [бесплатную учетную запись](https://azure.microsoft.com/free/).
* Подписка Wootric с поддержкой единого входа.

## <a name="scenario-description"></a>Описание сценария

В рамках этого руководства вы настроите и проверите единый вход Azure AD в тестовой среде.

* Wootric поддерживает единый вход, инициируемый **поставщиком удостоверений**.
* Wootric поддерживает **JIT** -подготовку пользователей.
* После настройки Wootric вы можете применить управление сеансами, которое в реальном времени защищает от хищения конфиденциальных данных вашей организации и несанкционированного доступа к ним. Управление сеансом является расширением функции условного доступа. [Узнайте, как применять управление сеансами с помощью Microsoft Cloud App Security](/cloud-app-security/proxy-deployment-any-app).

## <a name="adding-wootric-from-the-gallery"></a>Добавление Wootric из коллекции

Чтобы настроить интеграцию Wootric с Azure AD, нужно добавить Wootric из коллекции в список управляемых приложений SaaS.

1. Войдите на [портал Azure](https://portal.azure.com) с помощью личной учетной записи Майкрософт либо рабочей или учебной учетной записи.
1. В области навигации слева выберите службу **Azure Active Directory**.
1. Перейдите в колонку **Корпоративные приложения** и выберите **Все приложения**.
1. Чтобы добавить новое приложение, выберите **Новое приложение**.
1. В разделе **Добавление из коллекции** в поле поиска введите **Wootric**.
1. Выберите **Wootric** в области результатов и добавьте это приложение. Подождите несколько секунд, пока приложение не будет добавлено в ваш клиент.


## <a name="configure-and-test-azure-ad-sso-for-wootric"></a>Настройка и проверка единого входа Azure AD для Wootric

Настройте и проверьте единый вход Azure AD в Wootric с помощью тестового пользователя **B. Simon**. Для обеспечения работы единого входа необходимо установить связь между пользователем AAD и соответствующим пользователем в Wootric.

Чтобы настроить и проверить единый вход Azure AD в Wootric, выполните действия в следующих стандартных блоках:

1. **[Настройка единого входа Azure AD](#configure-azure-ad-sso)** необходима, чтобы пользователи могли использовать эту функцию.
    1. **[Создание тестового пользователя Azure AD](#create-an-azure-ad-test-user)** требуется для проверки работы единого входа Azure AD с помощью пользователя B.Simon.
    1. **[Назначение тестового пользователя Azure AD](#assign-the-azure-ad-test-user)** необходимо, чтобы позволить пользователю B.Simon использовать единый вход Azure AD.
1. **[Настройка единого входа в Wootric](#configure-wootric-sso)** необходима, чтобы настроить параметры единого входа на стороне приложения.
    1. **[Создание тестового пользователя в Wootric](#create-wootric-test-user)** нужно для того, чтобы в Wootric существовал пользователь B. Simon, связанный с одноименным пользователем в Azure AD.
1. **[Проверка единого входа](#test-sso)** позволяет убедиться в правильности конфигурации.

## <a name="configure-azure-ad-sso"></a>Настройка единого входа Azure AD

Выполните следующие действия, чтобы включить единый вход Azure AD на портале Azure.

1. На [портале Azure](https://portal.azure.com/) на странице интеграции с приложением **Wootric** найдите раздел **Управление** и выберите **Единый вход**.
1. На странице **Выбрать метод единого входа** выберите **SAML**.
1. На странице **Настройка единого входа с помощью SAML** щелкните значок "Изменить" (значок пера), чтобы открыть диалоговое окно **Базовая конфигурация SAML** и изменить параметры.

   ![Изменение базовой конфигурации SAML](common/edit-urls.png)

1. В разделе **Базовая конфигурация SAML** приложение предварительно настроено и ему заданы требуемые URL-адреса. Пользователь должен сохранить конфигурацию, нажав кнопку **Сохранить**.


1. Приложение Wootric ожидает утверждения SAML в определенном формате, который требует добавить настраиваемые сопоставления атрибутов в конфигурацию атрибутов токена SAML. На следующем снимке экрана показан список атрибутов по умолчанию.

    ![Изображение](common/default-attributes.png)

1. В дополнение к описанному выше приложение Wootric ожидает несколько дополнительных атрибутов в ответе SAML, как показано ниже. Эти атрибуты также заранее заполнены, но вы можете изменить их в соответствии со своими требованиями.
    
    | Имя |  Исходный атрибут |
    | -------------- | --------- |
    | идентификатор | user.objectid |

1. На странице **Настройка единого входа с помощью SAML** в разделе **Сертификат подписи SAML** найдите пункт **Сертификат (Base64)** и щелкните **Скачать** , чтобы скачать сертификат. Сохраните этот сертификат на компьютере.

    ![Ссылка для скачивания сертификата](common/certificatebase64.png)

1. Требуемые URL-адреса можно скопировать в разделе **Настройка Wootric**.

    ![Копирование URL-адресов настройки](common/copy-configuration-urls.png)

### <a name="create-an-azure-ad-test-user"></a>Создание тестового пользователя Azure AD

В этом разделе описано, как на портале Azure создать тестового пользователя с именем B.Simon.

1. На портале Azure в области слева выберите **Azure Active Directory** , **Пользователи** , а затем — **Все пользователи**.
1. В верхней части экрана выберите **Новый пользователь**.
1. В разделе **Свойства пользователя** выполните следующие действия.
   1. В поле **Имя** введите `B.Simon`.  
   1. В поле **Имя пользователя** введите username@companydomain.extension. Например, `B.Simon@contoso.com`.
   1. Установите флажок **Показать пароль** и запишите значение, которое отображается в поле **Пароль**.
   1. Нажмите кнопку **Создать**.

### <a name="assign-the-azure-ad-test-user"></a>Назначение тестового пользователя Azure AD

В этом разделе описано, как включить единый вход в Azure для пользователя B. Simon, предоставив этому пользователю доступ к Wootric.

1. На портале Azure выберите **Корпоративные приложения** , а затем — **Все приложения**.
1. В списке приложений выберите **Wootric**.
1. На странице "Обзор" приложения найдите раздел **Управление** и выберите **Пользователи и группы**.

   ![Ссылка "Пользователи и группы"](common/users-groups-blade.png)

1. Выберите **Добавить пользователя** , а в диалоговом окне **Добавление назначения**  выберите **Пользователи и группы**.

    ![Ссылка "Добавить пользователя"](common/add-assign-user.png)

1. В диалоговом окне **Пользователи и группы** выберите **B.Simon** в списке пользователей, а затем в нижней части экрана нажмите кнопку **Выбрать**.
1. Если ожидается, что в утверждении SAML будет получено какое-либо значение роли, то в диалоговом окне **Выбор роли** нужно выбрать соответствующую роль для пользователя из списка и затем нажать кнопку **Выбрать** , расположенную в нижней части экрана.
1. В диалоговом окне **Добавление назначения** нажмите кнопку **Назначить**.

## <a name="configure-wootric-sso"></a>Настройка единого входа в Wootric

1. В другом окне браузера войдите в приложение Wootric с правами администратора.

1. Щелкните **значок параметров** в верхнем меню.

    ![Снимок экрана: на сайте Wootric выбран значок параметров.](./media/wootric-tutorial/configure-1.PNG)

1. В разделе **INTEGRATIONS** (Интеграции) выберите **Authentication** (Проверка подлинности) в меню слева, затем щелкните **Enable Single Sign On with Azure Active Directory** (Включить единый вход через Azure Active Directory).

    ![Снимок экрана: в разделе проверки подлинности выделен элемент для включения единого входа через Azure Active Directory с состоянием Connected (Подключено).](./media/wootric-tutorial/configure-2.PNG)

1. На следующей странице выполните приведенные ниже шаги.

    ![Снимок экрана для страницы параметров, где можно ввести описанные здесь значения.](./media/wootric-tutorial/configure-3.PNG)

    а. В текстовое поле **Identity Provider Single Sign-On URL** (URL-адрес единого входа для поставщика удостоверений) вставьте значение **URL-адреса входа** , скопированное на портале Azure.

    b. В текстовое поле **Identity Provider Issuer** (Издатель поставщика удостоверений) вставьте значение **Entity ID** (Идентификатор сущности), скопированное на портале Azure.

    c. Откройте в блокноте скачанный с портала Azure файл **сертификата (Base64)** и вставьте его содержимое в текстовое поле **X.509 Certificate** (Сертификат X.509).

    d. Установите флажок **Automatically grant access to new users** (Автоматически предоставлять доступ новым пользователям).
    
    д) Щелкните **Save** (Сохранить).

### <a name="create-wootric-test-user"></a>Создание тестового пользователя Wootric

В этом разделе описано, как создать в приложении Wootric пользователя с именем B. Simon. Приложение Wootric поддерживает JIT-подготовку пользователей, которая включена по умолчанию. В этом разделе никакие действия с вашей стороны не требуются. Если пользователь еще не существует в Wootric, он создается после проверки подлинности.
