# <a name="tutorial-integrate-abstract-with-azure-active-directory"></a>Руководство по Интеграция Abstract с Azure Active Directory

В этом руководстве описано, как интегрировать Abstract с Azure Active Directory (Azure AD). Интеграции Abstract с Azure AD обеспечивает следующие возможности.

* Контроль доступа к Abstract с помощью Azure AD.
* Автоматический вход пользователей в Abstract с помощью учетных записей Azure AD.
* Централизованное управление учетными записями через портал Azure.

Чтобы узнать больше об интеграции приложений SaaS с Azure AD, прочитайте статью [Что такое доступ к приложениям и единый вход с помощью Azure Active Directory?](https://docs.microsoft.com/azure/active-directory/active-directory-appssoaccess-whatis)

## <a name="prerequisites"></a>Предварительные требования

Чтобы приступить к работе, потребуется следующее:

* подписка Azure AD Если у вас нет подписки, вы можете получить [бесплатную учетную запись](https://azure.microsoft.com/free/).
* подписка Abstract с поддержкой единого входа (SSO).

## <a name="scenario-description"></a>Описание сценария

В рамках этого руководства вы настроите и проверите единый вход Azure AD в тестовой среде.

* Abstract поддерживает единый вход, инициированный **поставщиком услуг и поставщиком удостоверений**.

## <a name="adding-abstract-from-the-gallery"></a>Добавление Abstract из коллекции

Чтобы настроить интеграцию Abstract с Azure AD, необходимо добавить Abstract из коллекции в список управляемых приложений SaaS.

1. Войдите на [портал Azure](https://portal.azure.com) с помощью личной учетной записи Майкрософт либо рабочей или учебной учетной записи.
1. В области навигации слева выберите службу **Azure Active Directory**.
1. Перейдите в колонку **Корпоративные приложения** и выберите **Все приложения**.
1. Чтобы добавить новое приложение, выберите **Новое приложение**.
1. В разделе **Добавление из коллекции** в поле поиска введите **Abstract**.
1. Выберите **Abstract** в области результатов и добавьте это приложение. Подождите несколько секунд, пока приложение не будет добавлено в ваш клиент.


## <a name="configure-and-test-azure-ad-single-sign-on"></a>Настройка и проверка единого входа в Azure AD

Настройте и проверьте единый вход Azure AD в Abstract с помощью тестового пользователя **B.Simon**. Чтобы обеспечить работу единого входа, необходимо установить связь между пользователем Azure AD и соответствующим пользователем в Abstract.

Чтобы настроить и проверить единый вход Azure AD в Abstract, выполните действия в следующих стандартных блоках:

1. **[Настройка единого входа Azure AD](#configure-azure-ad-sso)** необходима, чтобы пользователи могли использовать эту функцию.
2. **[Настройка единого входа в Abstract](#configure-abstract-sso)** необходима, чтобы настроить параметры единого входа на стороне приложения.
3. **[Создание тестового пользователя Azure AD](#create-an-azure-ad-test-user)** требуется для проверки работы единого входа Azure AD от имени пользователя Britta Simon.
4. **[Назначение тестового пользователя Azure AD](#assign-the-azure-ad-test-user)** необходимо, чтобы разрешить пользователю Britta Simon использовать единый вход Azure AD.
5. **[Создание тестового пользователя Abstract](#create-abstract-test-user)** требуется для того, чтобы во Abstract существовал пользователь Britta Simon, связанный с одноименным пользователем в Azure AD.
6. **[Проверка единого входа](#test-sso)** необходима, чтобы убедиться в корректной работе конфигурации.

### <a name="configure-azure-ad-sso"></a>Настройка единого входа Azure AD

Выполните следующие действия, чтобы включить единый вход Azure AD на портале Azure.

1. На [портале Azure](https://portal.azure.com/) на странице интеграции с приложением **Abstract** найдите раздел **Управление** и выберите **Единый вход**.
1. На странице **Выбрать метод единого входа** выберите **SAML**.
1. На странице **Настройка единого входа с помощью SAML** щелкните значок "Изменить" (значок пера), чтобы открыть диалоговое окно **Базовая конфигурация SAML** и изменить параметры.

   ![Правка базовой конфигурации SAML](common/edit-urls.png)

1. В разделе **Базовая конфигурация SAML** приложение предварительно настроено в режиме инициированном **IDP** и ему заданы требуемые URL-адреса в Azure. Пользователь должен сохранить конфигурацию, нажав кнопку **Сохранить**.

1. Чтобы настроить приложение для работы в режиме, инициируемом **поставщиком услуг**, щелкните **Задать дополнительные URL-адреса** и выполните следующие действия.

    В текстовом поле **URL-адрес входа** введите URL-адрес: `https://app.abstract.com/signin`.

4. На странице **Настройка единого входа с помощью SAML** в разделе **Сертификат подписи SAML** нажмите кнопку "Копировать", чтобы копировать **URL-адрес метаданных федерации приложений** и сохранить его на компьютере.

    ![Ссылка для скачивания сертификата](common/copy-metadataurl.png)

### <a name="configure-abstract-sso"></a>Настройка единого входа Abstract

Не забудьте получить ваш `App Federation Metadata Url` и `Azure AD Identifier` из портала Azure, так как они понадобятся для настройки единого входа для Abstract.

Эти сведения можно найти на странице **Настройка единого входа с помощью SAML**.

* `App Federation Metadata Url` находится в разделе **Сертификат подписи SAML**.
* `Azure AD Identifier` находится в разделе **Настройка Abstract**.


Теперь вы готовы к настройке единого входа для Abstract.

>[!Note]
>Чтобы получить доступ к параметрам единого входа для Abstract, необходимо пройти проверку подлинности с учетной записью администратора организации.

1. Откройте [Веб-приложение Abstract](https://app.abstract.com/).
2. Перейдите на страницу**Разрешения** на левой панели.
3. В разделе **Настройка единого входа** введите ваш **URL-адрес метаданных** и **Идентификатор сущности**.
4. Введите любые исключения, которые могут возникнуть вручную. Сообщения электронной почты, перечисленные вручную в разделе исключений, будут обходить единый вход и смогут выполнять вход с помощью электронной почты и пароля. 
5. Нажмите кнопку **Сохранить изменения**.

>[!Note] 
>В списке исключений введенных вручную необходимо использовать основные адреса электронной почты. Активация единого входа завершится ошибкой, если адрес электронной почты, указанный в вашем списке, является дополнительным адресом электронной почты пользователя. Если это произойдет, вы увидите сообщение об ошибке с основным адресом электронной почты для неработающей учетной записи. Добавьте этот основной адрес электронной почты в список исключений вручную после того, как убедитесь, что знаете пользователя.

### <a name="create-an-azure-ad-test-user"></a>Создание тестового пользователя Azure AD

В этом разделе описано, как на портале Azure создать тестового пользователя с именем B.Simon.

1. На портале Azure в области слева выберите **Azure Active Directory**, **Пользователи**, а затем — **Все пользователи**.
1. В верхней части экрана выберите **Новый пользователь**.
1. В разделе **Свойства пользователя** выполните следующие действия.
   1. В поле **Имя** введите `B.Simon`.  
   1. В поле **Имя пользователя** введите username@companydomain.extension. Например, `B.Simon@contoso.com`.
   1. Установите флажок **Показать пароль** и запишите значение, которое отображается в поле **Пароль**.
   1. Нажмите кнопку **Создать**.

### <a name="assign-the-azure-ad-test-user"></a>Назначение тестового пользователя Azure AD

В этом разделе описано, как включить единый вход в Azure для пользователя B.Simon, предоставив этому пользователю доступ к Abstract.

1. На портале Azure выберите **Корпоративные приложения**, а затем —**Все приложения**.
1. В списке приложений выберите **Abstract**.
1. На странице "Обзор" приложения найдите раздел **Управление** и выберите **Пользователи и группы**.

   ![Ссылка "Пользователи и группы"](common/users-groups-blade.png)

1. Выберите **Добавить пользователя**, а в диалоговом окне **Добавление назначения** выберите **Пользователи и группы**.

    ![Ссылка "Добавить пользователя"](common/add-assign-user.png)

1. В диалоговом окне **Пользователи и группы** выберите **B.Simon** в списке пользователей, а затем в нижней части экрана нажмите кнопку **Выбрать**.
1. Если ожидается, что в утверждении SAML будет получено какое-либо значение роли, то в диалоговом окне **Выбор роли** нужно выбрать соответствующую роль для пользователя из списка и затем нажать кнопку **Выбрать**, расположенную в нижней части экрана.
1. В диалоговом окне **Добавление назначения** нажмите кнопку **Назначить**.

### <a name="create-abstract-test-user"></a>Создание тестового пользователя в Abstract

Чтобы проверить единый вход для Abstract:

1. Откройте [Веб-приложение Abstract](https://app.abstract.com/).
2. Перейдите на страницу**Разрешения** на левой панели.
3. Щелкните **Test with my Account** (Тест с моей учетной записью). Если тест выполнен с ошибкой, [Свяжитесь со службой поддержки](https://www.abstract.com/help/contact/).

>[!Note]
>Чтобы получить доступ к параметрам единого входа для Abstract, необходимо пройти проверку подлинности с учетной записью администратора организации.
Эта учетная запись администратора организации должна быть назначена для Abstract на портал Azure.

### <a name="test-sso"></a>Проверка единого входа 

В этом разделе описано, как проверить конфигурацию единого входа Azure AD с помощью панели доступа.

Щелкнув плитку Abstract на панели доступа, вы автоматически войдете в приложение Abstract, для которого настроили единый вход. См. дополнительные сведения о [панели доступа](https://docs.microsoft.com/azure/active-directory/active-directory-saas-access-panel-introduction)
