# <a name="tutorial-integrate-azure-active-directory-single-sign-on-sso-with-splan-visitor"></a>Руководство по интеграции единого входа Azure Active Directory со Splan Visitor

В этом учебнике описано, как интегрировать Splan Visitor с Azure Active Directory (Azure AD). При интеграции Splan Visitor с Azure AD вы можете выполнять операции перечисленные далее.

* Использовать Azure AD, чтобы контролировать доступ к Splan Visitor.
* Включить автоматический вход пользователей в Splan Visitor с учетными записями Azure AD.
* Централизованное управление учетными записями через портал Azure.

## <a name="prerequisites"></a>Предварительные требования

Для начала работы необходимы перечисленные ниже компоненты и данные.

* Подписка Azure AD. Если у вас нет подписки, вы можете получить [бесплатную учетную запись](https://azure.microsoft.com/free/).
* Подписка Splan Visitor с поддержкой единого входа.

## <a name="scenario-description"></a>Описание сценария

В рамках этого учебника вы настроите и проверите единый вход Azure AD в тестовой среде.

Splan Visitor поддерживает единый вход, инициированный поставщиком удостоверений.

## <a name="add-splan-visitor-from-the-gallery"></a>Добавление Splan Visitor из коллекции

Чтобы настроить интеграцию Splan Visitor с Azure AD, необходимо добавить Splan Visitor из коллекции в список управляемых приложений SaaS.

1. Войдите на портал Azure с помощью личной учетной записи Майкрософт либо рабочей или учебной учетной записи.
1. В области слева выберите службу **Azure Active Directory**.
1. Перейдите в колонку **Корпоративные приложения** и выберите **Все приложения**.
1. Чтобы добавить новое приложение, выберите **Новое приложение**.
1. В разделе **Добавление из коллекции** в поле поиска введите **Splan Visitor**.
1. Выберите **Splan Visitor** в области результатов и добавьте это приложение. Подождите несколько секунд, пока приложение не будет добавлено в ваш клиент.

## <a name="configure-and-test-azure-ad-sso-for-splan-visitor"></a>Настройка и проверка единого входа Azure AD для Splan Visitor

Настройте и проверьте единый вход Azure AD в Splan Visitor с использованием данных тестового пользователя **B.Simon**. Для обеспечения единого входа необходимо установить связь между пользователем Azure AD и соответствующим пользователем в Splan Visitor.

Чтобы настроить и проверить единый вход Azure AD в Splan Visitor, выполните действия из следующих стандартных блоков:

1. **[Настройка единого входа Azure AD](#configure-azure-ad-sso)** необходима, чтобы пользователи могли использовать эту функцию.
    1. **[Создание тестового пользователя Azure AD](#create-an-azure-ad-test-user)** требуется для проверки работы единого входа Azure AD от имени тестового пользователя B.Simon.
    1. **[Назначение тестового пользователя Azure AD](#assign-the-azure-ad-test-user)** необходимо, чтобы позволить B.Simon использовать единый вход Azure AD.
1. **[Настройка единого входа в Splan Visitor](#configure-splan-visitor-sso)** необходима, чтобы настроить параметры единого входа на стороне Splan Visitor.
    1. **[Создание тестового пользователя Splan Visitor](#create-a-splan-visitor-test-user)** требуется для того, чтобы в Splan Visitor существовал пользователь B.Simon, связанный с одноименным пользователем в Azure AD.
1. **[Проверка единого входа](#test-sso)** необходима, чтобы убедиться в корректной работе конфигурации.

## <a name="configure-azure-ad-sso"></a>Настройка единого входа Azure AD

Выполните следующие действия, чтобы включить единый вход Azure AD на портале Azure:

1. На портале Azure на странице интеграции с приложением **Splan Visitor** найдите раздел **Управление** и выберите **Единый вход**.
1. На странице **Выбрать метод единого входа** выберите **SAML**.
1. На странице **Настройка единого входа с помощью SAML** щелкните значок **правки (перо)** , чтобы открыть диалоговое окно **Базовая конфигурация SAML** и изменить эти параметры.

   ![Снимок экрана: выделенный значок редактирования (перо) для базовой конфигурации SAML.](common/edit-urls.png)

1. В разделе **Базовая конфигурация SAML** приложение предварительно настроено и заданы требуемые URL-адреса в Azure. Нажмите кнопку **Сохранить**, чтобы сохранить конфигурацию.

1. На странице **Настройка единого входа с помощью SAML** в разделе **Сертификат подписи SAML** найдите поле **XML метаданных федерации**. Щелкните **Скачать**, чтобы получить сертификат и сохранить его на локальном компьютере.

    ![Снимок экрана: выделенная ссылка для скачивания XML-файла метаданных федерации.](common/metadataxml.png)

1. Требуемые URL-адреса можно скопировать в разделе **настройки Splan Visitor**.

    ![Снимок экрана: раздел с URL-адресами конфигурации.](common/copy-configuration-urls.png)

### <a name="create-an-azure-ad-test-user"></a>Создание тестового пользователя Azure AD

В этом разделе описано, как создать на портале Azure тестового пользователя с именем B.Simon.

1. На портале Azure в области слева последовательно выберите элементы **Azure Active Directory**, **Пользователи** и **Все пользователи**.
1. В верхней части экрана выберите **Новый пользователь**.
1. В разделе **Свойства пользователя** выполните следующие действия.
   1. В поле **Имя** введите **B.Simon**.  
   1. В поле **Имя пользователя** введите свое имя пользователя в формате _username@companydomain.extension_ . Например, введите **B.Simon@contoso.com** .
   1. Установите флажок **Показать пароль** и запишите значение, которое отображается в поле **Пароль**.
   1. Нажмите кнопку **Создать**.

### <a name="assign-the-azure-ad-test-user"></a>Назначение тестового пользователя Azure AD

В этом разделе описано, как включить единый вход в Azure для пользователя B.Simon, предоставив этому пользователю доступ к Splan Visitor.

1. На портале Azure выберите **Корпоративные приложения**, а затем — **Все приложения**.
1. В списке приложений выберите **Splan Visitor**, чтобы открыть общие сведения о приложении.
1. Найдите раздел **Управление** и выберите вкладку **Пользователи и группы**.
1. Выберите **Добавить пользователя**, а в диалоговом окне **Добавление назначения** — **Пользователи и группы**.
1. В диалоговом окне **Пользователи и группы** выберите **B.Simon** в списке **Пользователи**, а затем в нижней части экрана нажмите кнопку **Выбрать**.
1. Если пользователю будет назначена роль, выберите ее в раскрывающемся меню **Выбрать роль**. Если для этого приложения не настроена ни одна роль, оставьте выбранной роль **Доступ по умолчанию**.
1. В диалоговом окне **Добавление назначения** выберите **Назначить**.

## <a name="configure-splan-visitor-sso"></a>Настройка единого входа для Splan Visitor

Чтобы настроить единый вход на стороне Splan Visitor, отправьте скачанный **XML-файл метаданных федерации** и соответствующие URL-адреса, скопированные на портале Azure, в [службу технической поддержки Splan Visitor](mailto:support@splan.com). Ее специалисты обеспечат правильную настройку единого входа SAML с обеих сторон.

### <a name="create-a-splan-visitor-test-user"></a>Создание тестового пользователя Splan Visitor

Создайте в Splan Visitor тестового пользователя с именем **Britta Simon**. Обратитесь в [службу технической поддержки Splan Visitor](mailto:support@splan.com), чтобы добавить этого пользователя в приложение Splan Visitor. Перед использованием единого входа необходимо создать и активировать этого пользователя.

## <a name="test-sso"></a>Проверка единого входа

Проверьте конфигурацию единого входа Azure AD с помощью одного из следующих способов:

* **Портал Azure**: Выберите элемент **Тестировать приложение**, чтобы автоматически войти в приложение Splan Visitor, для которого настроен единый вход.
* **Портал Майкрософт "Мои приложения".** Выберите плитку **Splan Visitor**, чтобы автоматически войти в приложение Splan Visitor, для которого настроен единый вход. Дополнительные сведения о портале "Мои приложения" см. в статье о [входе на портал "Мои приложения" и запуске приложений на нем](https://docs.microsoft.com/azure/active-directory/active-directory-saas-access-panel-introduction).
