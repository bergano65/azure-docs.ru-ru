# <a name="tutorial-azure-active-directory-single-sign-on-sso-integration-with-mail-luck"></a>Руководство по интеграции единого входа Azure Active Directory с Mail Luck!

В этом руководстве вы узнаете, как интегрировать Mail Luck! с Azure Active Directory (Azure AD). Интеграция Mail Luck! с Azure AD обеспечивает приведенные ниже возможности:

* Контроль доступа к Mail Luck! с помощью Azure AD.
* Разрешить автоматический вход пользователей в Mail Luck! с помощью учетных записей Azure AD.
* Централизованное управление учетными записями через портал Azure.

Чтобы узнать больше об интеграции приложений SaaS с Azure AD, прочитайте статью [Что такое доступ к приложениям и единый вход с помощью Azure Active Directory?](https://docs.microsoft.com/azure/active-directory/active-directory-appssoaccess-whatis)

## <a name="prerequisites"></a>Предварительные требования

Чтобы приступить к работе, потребуется следующее:

* подписка Azure AD Если у вас нет подписки, вы можете получить [бесплатную учетную запись](https://azure.microsoft.com/free/).
* Mail Luck! Подписка Aha! с поддержкой единого входа.

## <a name="scenario-description"></a>Описание сценария

В рамках этого руководства вы настроите и проверите единый вход Azure AD в тестовой среде.

* Mail Luck! поддерживает единый вход, инициированный **поставщиком услуг**.

## <a name="adding-mail-luck-from-the-gallery"></a>Добавление Mail Luck! из коллекции

Чтобы настроить интеграцию Mail Luck! с Azure AD, необходимо добавить Mail Luck! из коллекции в список управляемых приложений SaaS.

1. Войдите на [портал Azure](https://portal.azure.com) с помощью личной учетной записи Майкрософт либо рабочей или учебной учетной записи.
1. В области навигации слева выберите службу **Azure Active Directory**.
1. Перейдите в колонку **Корпоративные приложения** и выберите **Все приложения**.
1. Чтобы добавить новое приложение, выберите **Новое приложение**.
1. В разделе **Добавление из коллекции** введите **Mail Luck!** в поле поиска.
1. Выберите **Mail Luck!** в области результатов и добавьте это приложение. Подождите несколько секунд, пока приложение не будет добавлено в ваш клиент.

## <a name="configure-and-test-azure-ad-single-sign-on-for-mail-luck"></a>Настройка и проверка единого входа Azure AD для Mail Luck!

Настройка и проверка единого входа Azure AD в Mail Luck! с помощью тестового пользователя с именем **B.Simon**. Для обеспечения работы единого входа необходимо установить связь между пользователем Azure AD и соответствующим пользователем в Mail Luck!.

Чтобы настроить и проверить единый вход Azure AD в Mail Luck!, выполните действия в следующих стандартных блоках:

1. **[Настройка единого входа Azure AD](#configure-azure-ad-sso)** необходима, чтобы пользователи могли использовать эту функцию.
    * **[Создание тестового пользователя Azure AD](#create-an-azure-ad-test-user)** требуется для проверки работы единого входа Azure AD с помощью пользователя B.Simon.
    * **[Назначение тестового пользователя Azure AD](#assign-the-azure-ad-test-user)** необходимо, чтобы позволить пользователю B.Simon использовать единый вход Azure AD.
1. **[Настройка Mail Luck! единого входа Mail Luck!](#configure-mail-luck-sso)** . Для настройки параметров единого входа на стороне приложения.
    * **[Создание тестового пользователя Mail Luck!](#create-mail-luck-test-user)** требуется для создания в Mail Luck! пользователя B.Simon, связанного с соответствующим представлением в Azure AD.
1. **[Проверка единого входа](#test-sso)** необходима, чтобы убедиться в корректной работе конфигурации.

## <a name="configure-azure-ad-sso"></a>Настройка единого входа Azure AD

Выполните следующие действия, чтобы включить единый вход Azure AD на портале Azure.

1. На [портале Azure](https://portal.azure.com/), на странице интеграции с приложением **Mail Luck!** найдите раздел **Управление** и выберите **Единый вход**.
1. На странице **Выбрать метод единого входа** выберите **SAML**.
1. На странице **Настройка единого входа с помощью SAML** щелкните значок "Изменить" (значок пера), чтобы открыть диалоговое окно **Базовая конфигурация SAML** и изменить параметры.

   ![Правка базовой конфигурации SAML](common/edit-urls.png)

1. На странице **Базовая конфигурация SAML** введите значения следующих полей.

    a. В текстовом поле **URL-адрес для входа** введите URL-адрес в следующем формате: `https://manage<UNITID>.ml-sgw.jp/<TENANT_NAME>/saml/sign_in`.

    b. В текстовом поле **Идентификатор (сущности)** введите URL-адрес в следующем формате: `https://manage<UNITID>.ml-sgw.jp/<TENANT_NAME>/saml/`.

    > [!NOTE]
    > Эти значения приведены для примера. Необходимо обновить эти значения действующим URL-адресом для входа и идентификатором. Свяжитесь с [Mail Luck! поддержки Aha!](https://customer.nttpc.co.jp/cgi-bin/form/inquiry_index.cgi), чтобы получить эти значения. Можно также посмотреть шаблоны в разделе **Базовая конфигурация SAML** на портале Azure.

1. На странице **Настройка единого входа с помощью SAML** в разделе **Сертификат подписи SAML** нажмите кнопку "Копировать", чтобы скопировать **URL-адрес метаданных федерации приложений** и сохранить его на компьютере.

    ![Ссылка для скачивания сертификата](common/copy-metadataurl.png)

### <a name="create-an-azure-ad-test-user"></a>Создание тестового пользователя Azure AD

В этом разделе описано, как на портале Azure создать тестового пользователя с именем B.Simon.

1. На портале Azure в области слева выберите **Azure Active Directory**, **Пользователи**, а затем — **Все пользователи**.
1. В верхней части экрана выберите **Новый пользователь**.
1. В разделе **Свойства пользователя** выполните следующие действия.
   1. В поле **Имя** введите `B.Simon`.  
   1. В поле **Имя пользователя** введите username@companydomain.extension. Например, `B.Simon@contoso.com`.
   1. Установите флажок **Показать пароль** и запишите значение, которое отображается в поле **Пароль**.
   1. Нажмите кнопку **Создать**.

### <a name="assign-the-azure-ad-test-user"></a>Назначение тестового пользователя Azure AD

В этом разделе описано, как включить единый вход Azure для пользователя B.Simon, предоставив этому пользователю доступ к Mail Luck!.

1. На портале Azure выберите **Корпоративные приложения**, а затем —**Все приложения**.
1. В списке приложений выберите **Mail Luck!** .
1. На странице "Обзор" приложения найдите раздел **Управление** и выберите **Пользователи и группы**.

   ![Ссылка "Пользователи и группы"](common/users-groups-blade.png)

1. Выберите **Добавить пользователя**, а в диалоговом окне **Добавление назначения** выберите **Пользователи и группы**.

    ![Ссылка "Добавить пользователя"](common/add-assign-user.png)

1. В диалоговом окне **Пользователи и группы** выберите **B.Simon** в списке пользователей, а затем в нижней части экрана нажмите кнопку **Выбрать**.
1. Если ожидается, что в утверждении SAML будет получено какое-либо значение роли, то в диалоговом окне **Выбор роли** нужно выбрать соответствующую роль для пользователя из списка и затем нажать кнопку **Выбрать**, расположенную в нижней части экрана.
1. В диалоговом окне **Добавление назначения** нажмите кнопку **Назначить**.

## <a name="configure-mail-luck-sso"></a>Настройка Mail Luck! Единый вход

Чтобы настроить единый вход на стороне приложения **Mail Luck!** необходимо отправить **URL-адрес метаданных федерации приложений** для [группы поддержки Mail Luck!](https://customer.nttpc.co.jp/cgi-bin/form/inquiry_index.cgi). Специалисты службы поддержки настроят подключение единого входа SAML на обеих сторонах.

### <a name="create-mail-luck-test-user"></a>Создание Mail Luck! пользователя Aha!

В этом разделе вы узнаете, как создать пользователя B.Simon в приложении Mail Luck!. Поработайте с [командой поддержки Mail Luck!](https://customer.nttpc.co.jp/cgi-bin/form/inquiry_index.cgi), чтобы добавить пользователей на платформу Mail Luck! . Перед использованием единого входа необходимо создать и активировать пользователей.
