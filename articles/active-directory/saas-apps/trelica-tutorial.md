# <a name="tutorial-azure-active-directory-single-sign-on-sso-integration-with-trelica"></a>Руководство по интеграции единого входа Azure Active Directory с Trelica

В этом руководстве описано, как интегрировать Trelica с Azure Active Directory (Azure AD).

Такая интеграция предоставит следующие возможности:

* Контроль доступа к Trelica с помощью Azure AD.
* Поддержка автоматического входа пользователей в Trelica с учетными записями Azure AD.
* Централизованное управление учетными записями через портал Azure.

Дополнительные сведения об интеграции приложений SaaS с Azure AD см. в статье о [едином входе в приложения с помощью Azure Active Directory](https://docs.microsoft.com/azure/active-directory/manage-apps/what-is-single-sign-on).

## <a name="prerequisites"></a>Предварительные требования

Чтобы приступить к работе, потребуется следующее.

* Подписка Azure AD. Если у вас нет подписки, вы можете получить [бесплатную учетную запись](https://azure.microsoft.com/free/).
* Подписка Trelica с поддержкой единого входа.

## <a name="scenario-description"></a>Описание сценария

В рамках этого руководства вы настроите и проверите единый вход Azure AD в тестовой среде.

* Trelica поддерживает единый вход, инициируемый поставщиком удостоверений.
* Trelica поддерживает JIT-подготовку пользователей.
* После настройки Trelica вы сможете включить управление сеансом. Этот механизм защищает от утечки данных и несанкционированного доступа к конфиденциальным данным вашей организации в режиме реального времени. Управление сеансом является расширением функции условного доступа. [Узнайте, как применять управление сеансами с помощью Microsoft Cloud App Security](https://docs.microsoft.com/cloud-app-security/proxy-deployment-any-app).

## <a name="adding-trelica-from-the-gallery"></a>Добавление Trelica из коллекции

Чтобы настроить интеграцию Trelica с Azure AD, необходимо добавить Trelica из коллекции в список управляемых приложений SaaS.

1. Войдите на [портал Azure](https://portal.azure.com) с помощью личной учетной записи Майкрософт либо рабочей или учебной учетной записи.
1. В области навигации слева выберите службу **Azure Active Directory**.
1. Перейдите в колонку **Корпоративные приложения** и выберите **Все приложения**.
1. Чтобы добавить новое приложение, выберите **Новое приложение**.
1. В разделе **Добавление из коллекции** в поле поиска введите **Trelica**.
1. Выберите **Trelica** в результатах поиска и добавьте это приложение. Подождите несколько секунд, пока приложение не будет добавлено в ваш клиент.

## <a name="configure-and-test-azure-ad-single-sign-on-for-trelica"></a>Настройка и проверка единого входа Azure AD для Trelica

Настройте и проверьте единый вход Azure AD в Trelica с помощью тестового пользователя **B.Simon**. Чтобы обеспечить работу единого входа, необходимо установить связь между пользователем Azure AD и соответствующим пользователем в Trelica.

Чтобы настроить и проверить единый вход Azure AD в Trelica, выполните действия в следующих стандартных блоках:

1. **[Настройка единого входа Azure AD](#configure-azure-ad-sso)** необходима, чтобы пользователи могли использовать эту функцию.
    1. **[Создание тестового пользователя Azure AD](#create-an-azure-ad-test-user)** требуется для проверки работы единого входа Azure AD от имени пользователя B.Simon.
    1. **[Назначение тестового пользователя Azure AD](#assign-the-azure-ad-test-user)** необходимо, чтобы позволить B.Simon использовать единый вход Azure AD.
1. **[Настройка единого входа в Trelica](#configure-trelica-sso)** необходима, чтобы настроить параметры единого входа на стороне приложения.
    1. **[Создание тестового пользователя Trelica](#create-a-trelica-test-user)** требуется для создания в Trelica пользователя B.Simon. связанный с одноименным пользователем в Azure AD.
1. **[Проверка единого входа](#test-sso)** необходима, чтобы убедиться в корректной работе конфигурации.

## <a name="configure-azure-ad-sso"></a>Настройка единого входа Azure AD

Выполните следующие действия, чтобы включить единый вход Azure AD на портале Azure:

1. На [портале Azure](https://portal.azure.com/) на странице интеграции с приложением **Trelica** перейдите к разделу **Управление**. Щелкните **Единый вход**.
1. На странице **Выбрать метод единого входа** выберите **SAML**.
1. На странице **Настройка единого входа с помощью SAML** щелкните значок карандаша, чтобы открыть диалоговое окно **Базовая конфигурация SAML** для изменения параметров.

   ![Страница "Настройка единого входа с помощью SAML", где выделен значок карандаша для перехода к базовой конфигурации SAML](common/edit-urls.png)

1. На странице **Настройка единого входа с помощью SAML** введите следующие значения:

    1. В поле **Идентификатор** введите URL-адрес **https://app.trelica.com** .

    1. В поле **URL-адрес ответа** введите URL-адрес в формате `https://app.trelica.com/Id/Saml2/<CUSTOM_IDENTIFIER>/Acs`.

    > [!NOTE]
    > Значение URL-адреса ответа приведено для примера. Укажите вместо него фактический URL-адрес ответа (известный как ACS).
    > Его можно найти, выполнив вход в Trelica и открыв [страницу конфигурации поставщиков конфигурации SAML](https://app.trelica.com/Admin/Profile/SAML) (Admin [Администратор] > Account [Учетная запись] > SAML). Нажмите кнопку копирования рядом с **URL-адресом службы обработчика утверждений (ACS)** , чтобы поместить его в буфер обмена и вставить в текстовое поле **URL-адрес ответа** в Azure AD.
    > Если у вас есть вопросы, изучите [справочную документацию Trelica](https://docs.trelica.com/admin/saml/azure-ad) или обратитесь в [группу поддержки клиентов Trelica](mailto:support@trelica.com).

1. На странице **Настройка единого входа с помощью SAML** в разделе **Сертификат подписи SAML** нажмите кнопку "Копировать", чтобы скопировать **URL-адрес метаданных федерации приложений** и сохранить его на компьютере.

    ![Раздел "Сертификат подписи SAML", где выделена кнопка копирования рядом с полем "URL-адрес метаданных федерации приложений"](common/copy-metadataurl.png)

### <a name="create-an-azure-ad-test-user"></a>Создание тестового пользователя Azure AD

В этом разделе описано, как создать на портале Azure тестового пользователя с именем B.Simon.

1. На панели слева на портале Azure выберите **Azure Active Directory** > **Пользователи** > **Все пользователи**.
1. В верхней части экрана щелкните **Новый пользователь**.
1. В разделе **Свойства пользователя** выполните следующие действия.
   1. В поле **Имя** введите **B.Simon**.
   1. В поле **Имя пользователя** введите **B.Simon@** _домен_компании_ **.** _расширение_. Например, B.Simon@contoso.com.
   1. Установите флажок **Показать пароль** и запишите значение, которое отображается в поле **Пароль**.
   1. Нажмите кнопку **создания**.

### <a name="assign-the-azure-ad-test-user"></a>Назначение тестового пользователя Azure AD

В этом разделе описано, как включить единый вход в Azure для пользователя B.Simon, предоставив этому пользователю доступ к Trelica.

1. На портале Azure выберите **Корпоративные приложения** > **Все приложения**.
1. В списке приложений выберите **Trelica**.
1. На странице "Обзор" приложения перейдите в раздел **Управление** и выберите **Пользователи и группы**.

   ![Раздел "Управление" с выделенным пунктом "Пользователи и группы"](common/users-groups-blade.png)

1. Выберите **Добавить пользователя**. В диалоговом окне **Добавление назначения** выберите **Пользователи и группы**.

   ![Раздел "Пользователи и группы" с выделенным элементом "Добавить пользователя"](common/add-assign-user.png)

1. В диалоговом окне **Пользователи и группы** выберите **B. Simon** из списка пользователей. Затем в нижней части экрана нажмите кнопку **Выбрать**.
1. Если ожидается, что в утверждении SAML будет получено какое-либо значение роли, то в диалоговом окне **Выбор ролей** нужно выбрать соответствующую роль для пользователя из списка. Затем в нижней части экрана нажмите кнопку **Выбрать**.
1. В диалоговом окне **Добавление назначения** выберите **Назначить**.

## <a name="configure-trelica-sso"></a>Настройка единого входа в Trelica

Чтобы настроить единый вход на стороне **Trelica**, перейдите на [страницу конфигурации поставщиков удостоверений SAML](https://app.trelica.com/Admin/Profile/SAML) (Admin [Администратор] > Account [Учетная запись] > SAML). Нажмите кнопку **New** (Создать). Введите **Azure AD** в качестве имени и выберите **Metadata from url** (Метаданные из URL-адреса) в качестве типа метаданных. Вставьте **URL-адрес метаданных федерации приложений** из Azure AD в поле **Metadata url** (URL-адрес метаданных) в Trelica.

Если у вас есть вопросы, изучите [справочную документацию Trelica](https://docs.trelica.com/admin/saml/azure-ad) или обратитесь в [группу поддержки клиентов Trelica](mailto:support@trelica.com).

### <a name="create-a-trelica-test-user"></a>Создание тестового пользователя Trelica

Приложение Trelica поддерживает JIT-подготовку пользователей, которая включена по умолчанию. В этом разделе никакие действия с вашей стороны не требуются. Если пользователь еще не существует в Trelica, он создается после проверки подлинности.

## <a name="test-sso"></a>Проверка единого входа

В этом разделе объясняется, как проверить конфигурацию единого входа Azure AD с помощью портала "Мои приложения".

Щелкнув элемент Trelica на портале "Мои приложения", вы автоматически войдете в приложение Trelica, для которого настроили единый вход. Дополнительные сведения о портале "Мои приложения" см. в статье [Вход и запуск приложений с портала "Мои приложения"](https://docs.microsoft.com/azure/active-directory/active-directory-saas-access-panel-introduction).
