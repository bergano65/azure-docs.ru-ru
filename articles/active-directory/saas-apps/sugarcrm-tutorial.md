# <a name="tutorial-azure-active-directory-single-sign-on-sso-integration-with-sugar-crm"></a>Руководство по интеграции единого входа Azure Active Directory с Sugar CRM

В этом руководстве описано, как интегрировать Sugar CRM с Azure Active Directory (Azure AD). Интеграция Azure AD с Sugar CRM обеспечивает перечисленные ниже возможности.

* Контроль доступа к Sugar CRM с помощью Azure AD.
* Вы можете включить автоматический вход пользователей в Sugar CRM с помощью учетных записей Azure AD.
* Централизованное управление учетными записями через портал Azure.

## <a name="prerequisites"></a>Предварительные требования

Чтобы приступить к работе, потребуется следующее.

* Подписка Azure AD. Если у вас нет подписки, вы можете получить [бесплатную учетную запись](https://azure.microsoft.com/free/).
* Подписка Sugar CRM с поддержкой единого входа.

## <a name="scenario-description"></a>Описание сценария

В рамках этого руководства вы настроите и проверите единый вход Azure AD в тестовой среде.

* Sugar CRM поддерживает единый вход, инициируемый **поставщиком услуг**.

> [!NOTE]
> Идентификатор этого приложения — фиксированное строковое значение, поэтому в одном клиенте можно настроить только один экземпляр.

## <a name="add-sugar-crm-from-the-gallery"></a>Добавление Sugar CRM из коллекции

Чтобы настроить интеграцию SugarCRM с Azure AD, необходимо добавить SugarCRM из коллекции в список управляемых приложений SaaS.

1. Войдите на портал Azure с помощью личной учетной записи Майкрософт либо рабочей или учебной учетной записи.
1. В области навигации слева выберите службу **Azure Active Directory**.
1. Перейдите в колонку **Корпоративные приложения** и выберите **Все приложения**.
1. Чтобы добавить новое приложение, выберите **Новое приложение**.
1. В разделе **Добавление из коллекции** в поле поиска введите **Sugar CRM**.
1. Выберите **Sugar CRM** в области результатов и добавьте это приложение. Подождите несколько секунд, пока приложение не будет добавлено в ваш клиент.

## <a name="configure-and-test-azure-ad-sso-for-sugar-crm"></a>Настройка и проверка единого входа Azure AD для Sugar CRM

Настройте и проверьте единый вход Azure AD в Sugar CRM с помощью тестового пользователя **B.Simon**. Для обеспечения работы единого входа необходимо установить связь между пользователем Azure AD и соответствующим пользователем в Sugar CRM.

Чтобы настроить и проверить единый вход Azure AD в Sugar CRM, выполните следующие действия:

1. **[Настройка единого входа Azure AD](#configure-azure-ad-sso)** необходима, чтобы пользователи могли использовать эту функцию.
    1. **[Создание тестового пользователя Azure AD](#create-an-azure-ad-test-user)** требуется для проверки работы единого входа Azure AD с помощью пользователя B.Simon.
    1. **[Назначение тестового пользователя Azure AD](#assign-the-azure-ad-test-user)** необходимо, чтобы позволить пользователю B.Simon использовать единый вход Azure AD.
1. **[Настройка единого входа в Sugar CRM](#configure-sugar-crm-sso)** необходима, чтобы настроить параметры единого входа на стороне приложения.
    1. **[Создание тестового пользователя Sugar CRM](#create-sugar-crm-test-user)** требуется для того, чтобы в Sugar CRM существовал пользователь B.Simon, связанный с одноименным пользователем в Azure AD.
1. **[Проверка единого входа](#test-sso)** позволяет убедиться в правильности конфигурации.

## <a name="configure-azure-ad-sso"></a>Настройка единого входа Azure AD

Выполните следующие действия, чтобы включить единый вход Azure AD на портале Azure.

1. На портале Azure на странице интеграции с приложением **Sugar CRM** найдите раздел **Управление** и выберите **Единый вход**.
1. На странице **Выбрать метод единого входа** выберите **SAML**.
1. На странице **Настройка единого входа с помощью SAML** щелкните значок карандаша, чтобы открыть диалоговое окно **Базовая конфигурация SAML** для изменения параметров.

   ![Изменение базовой конфигурации SAML](common/edit-urls.png)

1. На странице **Базовая конфигурация SAML** введите значения следующих полей.

    а. В текстовом поле **URL-адрес для входа** введите URL-адрес в следующем формате:

    - `https://<companyname>.sugarondemand.com`
    - `https://<companyname>.trial.sugarcrm`

    b. В текстовом поле **URL-адрес ответа** введите URL-адрес в таком формате:

    - `https://<companyname>.sugarondemand.com/<companyname>`
    - `https://<companyname>.trial.sugarcrm.com/<companyname>`
    - `https://<companyname>.trial.sugarcrm.eu/<companyname>`

    > [!NOTE]
    > Эти значения приведены для примера. Измените их на фактические значения URL-адреса входа и URL-адреса ответа. Чтобы получить это значение, обратитесь в [службу поддержки клиентов Sugar CRM](https://support.sugarcrm.com/). Можно также посмотреть шаблоны в разделе **Базовая конфигурация SAML** на портале Azure.

1. На странице **Настройка единого входа с помощью SAML** в разделе **Сертификат подписи SAML** найдите пункт **Сертификат (Base64)** и щелкните **Скачать**, чтобы скачать сертификат. Сохраните этот сертификат на компьютере.

    ![Ссылка для скачивания сертификата](common/certificatebase64.png)

1. Требуемые URL-адреса можно скопировать из раздела **Настройка Sugar CRM**.

    ![Копирование URL-адресов настройки](common/copy-configuration-urls.png)

### <a name="create-an-azure-ad-test-user"></a>Создание тестового пользователя Azure AD

В этом разделе описано, как на портале Azure создать тестового пользователя с именем B.Simon.

1. На портале Azure в области слева выберите **Azure Active Directory**, **Пользователи**, а затем — **Все пользователи**.
1. В верхней части экрана выберите **Новый пользователь**.
1. В разделе **Свойства пользователя** выполните следующие действия.
   1. В поле **Имя** введите `B.Simon`.  
   1. В поле **Имя пользователя** введите username@companydomain.extension. Например, `B.Simon@contoso.com`.
   1. Установите флажок **Показать пароль** и запишите значение, которое отображается в поле **Пароль**.
   1. Нажмите кнопку **Создать**.

### <a name="assign-the-azure-ad-test-user"></a>Назначение тестового пользователя Azure AD

В этом разделе описано, как включить единый вход в Azure для пользователя B.Simon, предоставив этому пользователю доступ к Sugar CRM.

1. На портале Azure выберите **Корпоративные приложения**, а затем — **Все приложения**.
1. В списке приложений выберите **SugarCRM**.
1. На странице "Обзор" приложения найдите раздел **Управление** и выберите **Пользователи и группы**.
1. Выберите **Добавить пользователя**, а в диалоговом окне **Добавление назначения** выберите **Пользователи и группы**.
1. В диалоговом окне **Пользователи и группы** выберите **B.Simon** в списке пользователей, а затем в нижней части экрана нажмите кнопку **Выбрать**.
1. Если пользователям необходимо назначить роль, вы можете выбрать ее из раскрывающегося списка **Выберите роль**. Если для этого приложения не настроена ни одна роль, будет выбрана роль "Доступ по умолчанию".
1. В диалоговом окне **Добавление назначения** нажмите кнопку **Назначить**.

## <a name="configure-sugar-crm-sso"></a>Настройка единого входа Sugar CRM

1. В другом окне веб-браузера войдите на свой корпоративный сайт Sugar CRM с учетными данными администратора.

1. Откройте страницу **Администратор**.

    ![Администратор](./media/sugarcrm-tutorial/ic795888.png "Административный")

1. В разделе **Administration** (Администрирование) щелкните **Password Management** (Управление паролями).

    ![Снимок экрана: раздел администрирования, в котором можно выбрать элемент для управления паролями.](./media/sugarcrm-tutorial/ic795889.png "Администрирование")

1. Установите флажок **Включить проверку подлинности SAML**.

    ![Снимок экрана, на котором показан параметр для выбора проверки подлинности SAML.](./media/sugarcrm-tutorial/ic795890.png "Администрирование")

1. В разделе **Проверка подлинности SAML** сделайте следующее:

    ![Проверка подлинности SAML](./media/sugarcrm-tutorial/save.png "Проверка подлинности SAML")  

    а. В текстовое поле **Login URL** (URL-адрес входа) вставьте **URL-адрес входа**, скопированный на портале Azure.
  
    b. В текстовое поле **SLO URL** (URL-адрес единого выхода) вставьте значение **URL-адрес выхода**, скопированное на портале Azure.
  
    c. Откройте сертификат в кодировке Base-64 в Блокноте, скопируйте его содержимое в буфер обмена и вставьте весь сертификат в текстовое поле **Сертификат X.509** .
  
    d. Выберите команду **Сохранить**.

### <a name="create-sugar-crm-test-user"></a>Создание тестового пользователя Sugar CRM

Чтобы пользователи Azure AD могли входить в Sugar CRM, их необходимо подготовить в Sugar CRM. В случае SugarCRM подготовка выполняется вручную.

**Чтобы подготовить учетную запись пользователя, сделайте следующее:**

1. Войдите на свой корпоративный сайт **Sugar CRM** с учетными данными администратора.

1. Откройте страницу **Администратор**.

    ![Администратор](./media/sugarcrm-tutorial/ic795888.png "Административный")

1. В разделе **Administration** (Администрирование) щелкните **User Management** (Управление пользователями).

    ![Снимок экрана: раздел администрирования, в котором можно выбрать элемент для управления пользователями.](./media/sugarcrm-tutorial/ic795893.png "Администрирование")

1. Выберите **Users (Пользователи) \> Create New User (Создать нового пользователя)** .

    ![Создание пользователя](./media/sugarcrm-tutorial/ic795894.png "Создать нового пользователя")

1. На вкладке **Профиль пользователя** сделайте следующее:

    ![Снимок экрана: вкладка User Profile (Профиль пользователя) для ввода описанных здесь значений.](./media/sugarcrm-tutorial/ic795895.png "Новый пользователь")

    * В соответствующие текстовые поля введите **имя**, **фамилию** и **электронный адрес** действующей учетной записи Azure Active Directory.
  
1. Для параметра **Status** (Состояние) выберите значение **Active** (Активно).

1. На вкладке «Пароль» выполните следующие действия:

    ![Снимок экрана: вкладка Password (Пароль) для ввода описанных здесь значений.](./media/sugarcrm-tutorial/ic795896.png "Новый пользователь")

    а. Введите пароль в соответствующее текстовое поле.

    b. Выберите команду **Сохранить**.

> [!NOTE]
> Вы можете использовать любые другие инструменты создания учетных записей пользователей SugarCRM или API, предоставляемые SugarCRM, для подготовки учетных записей пользователей AAD.

## <a name="test-sso"></a>Проверка единого входа 

В этом разделе описано, как проверить конфигурацию единого входа Azure AD с помощью указанных ниже способов. 

* Выберите **Тестировать приложение** на портале Azure. Вы будете перенаправлены по URL-адресу для входа в Sugar CRM, где можно инициировать поток входа. 

* Перейдите по URL-адресу для входа в Sugar CRM и инициируйте поток входа.

* Вы можете использовать портал "Мои приложения" корпорации Майкрософт. Щелкнув плитку Sugar CRM на портале "Мои приложения", вы перейдете по URL-адресу для входа в Sugar CRM. Дополнительные сведения о портале "Мои приложения" см. в [этой статье](https://docs.microsoft.com/azure/active-directory/active-directory-saas-access-panel-introduction).
