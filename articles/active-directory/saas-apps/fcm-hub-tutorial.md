# <a name="tutorial-azure-active-directory-single-sign-on-sso-integration-with-fcm-hub"></a>Руководство по интеграции единого входа Azure Active Directory с FCM HUB

В этом руководстве описано, как интегрировать FCM HUB с Azure Active Directory (Azure AD). Интеграция FCM HUB с Azure AD обеспечивает следующие возможности.

* Контроль доступа к FCM HUB с помощью Azure AD.
* Автоматический вход пользователей в FCM HUB с помощью учетных записей Azure AD.
* Централизованное управление учетными записями через портал Azure.

Чтобы узнать больше об интеграции приложений SaaS с Azure AD, прочитайте статью [Что такое доступ к приложениям и единый вход с помощью Azure Active Directory?](https://docs.microsoft.com/azure/active-directory/manage-apps/what-is-single-sign-on)

## <a name="prerequisites"></a>Предварительные требования

Чтобы приступить к работе, потребуется следующее.

* Подписка Azure AD. Если у вас нет подписки, вы можете получить [бесплатную учетную запись](https://azure.microsoft.com/free/).
* Подписка FCM HUB с поддержкой единого входа.

> [!NOTE]
> Идентификатор этого приложения — фиксированное строковое значение, поэтому в одном клиенте можно настроить только один экземпляр.

## <a name="scenario-description"></a>Описание сценария

В рамках этого руководства вы настроите и проверите единый вход Azure AD в тестовой среде.

* FCM HUB поддерживает единый вход, инициированный **поставщиком услуг и поставщиком удостоверений**.
* После настройки FCM HUB можете применить функцию управления сеансом, которая защищает от хищения конфиденциальных данных вашей организации и несанкционированного доступа к ним в реальном времени. Управление сеансом является расширением функции условного доступа. [Узнайте, как применять управление сеансами с помощью Microsoft Cloud App Security](https://docs.microsoft.com/cloud-app-security/proxy-deployment-any-app).

## <a name="adding-fcm-hub-from-the-gallery"></a>Добавление FCM HUB из коллекции

Чтобы настроить интеграцию FCM HUB с Azure AD, необходимо добавить FCM HUB из коллекции в список управляемых приложений SaaS.

1. Войдите на [портал Azure](https://portal.azure.com) с помощью личной учетной записи Майкрософт либо рабочей или учебной учетной записи.
1. В области навигации слева выберите службу **Azure Active Directory**.
1. Перейдите в колонку **Корпоративные приложения** и выберите **Все приложения**.
1. Чтобы добавить новое приложение, выберите **Новое приложение**.
1. В разделе **Добавление из коллекции** в поле поиска введите **FCM HUB**.
1. Выберите **FCM HUB** на панели результатов и добавьте это приложение. Подождите несколько секунд, пока приложение не будет добавлено в ваш клиент.

## <a name="configure-and-test-azure-ad-single-sign-on-for-fcm-hub"></a>Настройка и проверка единого входа Azure AD для FCM HUB

Настройте и проверьте единый вход с Azure AD в FCM HUB, используя тестового пользователя **B.Simon**. Для обеспечения работы единого входа необходимо установить связь между пользователем Azure AD и соответствующим пользователем в FCM HUB.

Чтобы настроить и проверить единый вход Azure AD в FCM HUB, выполните действия в следующих стандартных блоках:

1. **[Настройка единого входа Azure AD](#configure-azure-ad-sso)** необходима, чтобы пользователи могли использовать эту функцию.
    * **[Создание тестового пользователя Azure AD](#create-an-azure-ad-test-user)** требуется для проверки работы единого входа Azure AD с помощью пользователя B.Simon.
    * **[Назначение тестового пользователя Azure AD](#assign-the-azure-ad-test-user)** необходимо, чтобы позволить пользователю B.Simon использовать единый вход Azure AD.
1. **[Настройка единого входа в FCM HUB](#configure-fcm-hub-sso)** необходима, чтобы настроить параметры единого входа на стороне приложения.
    * **[Создание тестового пользователя FCM HUB](#create-fcm-hub-test-user)** требуется для того, чтобы в FCM HUB существовал пользователь B.Simon, связанный с одноименным пользователем в Azure AD.
1. **[Проверка единого входа](#test-sso)** позволяет убедиться в правильности конфигурации.

## <a name="configure-azure-ad-sso"></a>Настройка единого входа Azure AD

Выполните следующие действия, чтобы включить единый вход Azure AD на портале Azure.

1. На [портале Azure](https://portal.azure.com/) на странице интеграции с приложением **FCM HUB** найдите раздел **Управление** и выберите **Единый вход**.
1. На странице **Выбрать метод единого входа** выберите **SAML**.
1. На странице **Настройка единого входа с помощью SAML** щелкните значок "Изменить" (значок пера), чтобы открыть диалоговое окно **Базовая конфигурация SAML** и изменить параметры.

   ![Изменение базовой конфигурации SAML](common/edit-urls.png)

1. В разделе **Базовая конфигурация SAML** не нужно выполнять никаких действий, так как приложение уже предварительно интегрировано с Azure.

1. Чтобы настроить приложение для работы в режиме, инициируемом **поставщиком услуг**, щелкните **Задать дополнительные URL-адреса** и выполните следующие действия.

    В текстовом поле **URL-адрес входа** введите URL-адрес в формате `https://hub.fcm.travel/SsoSp/SpInit?clientid=<CUSTOMID>`.

    > [!NOTE]
    > Это значение приведено для примера. Вместо него необходимо указать фактический URL-адрес входа. Обратитесь к своему менеджеру по работе с партнерами или обратитесь в [службу технической поддержки клиентов FCM HUB](mailto:fcmssoadmin@us.fcm.travel), чтобы получить значение. Можно также посмотреть шаблоны в разделе **Базовая конфигурация SAML** на портале Azure.

1. Щелкните **Сохранить**.

1. На странице **Управление утверждениями** в разделе **Атрибуты пользователей и утверждения** добавьте следующие настраиваемые атрибуты:
   - **Name** (Имя). PortalID
   - **Источник**. attribute
   - **Атрибут источника**: PortalID, предоставляемое FCM значение.

1. В разделе **Сертификат для подписи SAML** с помощью функции редактирования выберите или введите приведенные ниже параметры, а затем нажмите **Сохранить**.
   - **Вариант подписывания**: Ответ знака SAML
   - **Алгоритм подписывания**: SHA-256

1. На странице **Настройка единого входа с помощью SAML** в разделе **Сертификат подписи SAML** найдите пункт **Сертификат (Base64)** и щелкните **Скачать**, чтобы скачать сертификат. Сохраните этот сертификат на компьютере.

    ![Ссылка для скачивания сертификата](common/certificatebase64.png)

1. Требуемые URL-адреса вы можете скопировать из раздела **Настройка FCM HUB**.

    ![Копирование URL-адресов настройки](common/copy-configuration-urls.png)

### <a name="create-an-azure-ad-test-user"></a>Создание тестового пользователя Azure AD

В этом разделе описано, как на портале Azure создать тестового пользователя с именем B.Simon.

1. На портале Azure в области слева выберите **Azure Active Directory**, **Пользователи**, а затем — **Все пользователи**.
1. В верхней части экрана выберите **Новый пользователь**.
1. В разделе **Свойства пользователя** выполните следующие действия.
   1. В поле **Имя** введите `B.Simon`.  
   1. В поле **Имя пользователя** введите username@companydomain.extension. Например, `B.Simon@contoso.com`.
   1. Установите флажок **Показать пароль** и запишите значение, которое отображается в поле **Пароль**.
   1. Нажмите кнопку **Создать**.

### <a name="assign-the-azure-ad-test-user"></a>Назначение тестового пользователя Azure AD

В этом разделе описано, как включить единый вход в Azure для пользователя B.Simon, предоставив этому пользователю доступ к FCM HUB.

1. На портале Azure выберите **Корпоративные приложения**, а затем — **Все приложения**.
1. В списке приложений выберите **FCM HUB**.
1. На странице "Обзор" приложения найдите раздел **Управление** и выберите **Пользователи и группы**.

   ![Ссылка "Пользователи и группы"](common/users-groups-blade.png)

1. Выберите **Добавить пользователя**, а в диалоговом окне **Добавление назначения** выберите **Пользователи и группы**.

    ![Ссылка "Добавить пользователя"](common/add-assign-user.png)

1. В диалоговом окне **Пользователи и группы** выберите **B.Simon** в списке пользователей, а затем в нижней части экрана нажмите кнопку **Выбрать**.
1. Если ожидается, что в утверждении SAML будет получено какое-либо значение роли, то в диалоговом окне **Выбор роли** нужно выбрать соответствующую роль для пользователя из списка и затем нажать кнопку **Выбрать**, расположенную в нижней части экрана.
1. В диалоговом окне **Добавление назначения** нажмите кнопку **Назначить**.

## <a name="configure-fcm-hub-sso"></a>Настройка единого входа в FCM HUB

Чтобы настроить единый вход на стороне **FCM HUB**, необходимо отправить загруженный **сертификат (Base64)** и соответствующие скопированные URL-адреса с портала Azure своему менеджеру по работе с партнерами или обратиться в [службу технической поддержки клиентов FCM HUB](mailto:fcmssoadmin@us.fcm.travel). Специалисты службы поддержки настроят подключение единого входа SAML на обеих сторонах.

В этом разделе вы узнаете, как создать пользователя B.Simon в приложении FCM HUB. Обратитесь к менеджеру по работе с партнерами или в [службу технической поддержки клиентов FCM HUB](mailto:fcmssoadmin@us.fcm.travel), чтобы добавить пользователей на платформу FCM HUB. Перед использованием единого входа необходимо создать и активировать пользователей.
