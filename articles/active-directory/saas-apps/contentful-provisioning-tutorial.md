# <a name="tutorial-configure-contentful-for-automatic-user-provisioning"></a>Руководство. Настройка автоматической подготовки пользователей с содержимым

В этой статье описываются действия, которые необходимо выполнить в ходе работы с содержимым и в Azure Active Directory (Azure AD) для настройки автоматической подготовки пользователей. При настройке Azure AD автоматически подготавливает и отменяет подготовку пользователей и групп к [содержимому](https://www.contentful.com/) с помощью службы подготовки Azure AD. Подробные сведения о том, какие функции выполняет эта служба и каков принцип ее работы, а также ответы на часто задаваемые вопросы см. в статье [Что представляет собой автоматическая подготовка пользователей для приложений SaaS в Azure AD?](../app-provisioning/user-provisioning.md). 

## <a name="capabilities-supported"></a>Поддерживаемые возможности

> [!div class="checklist"]
> * Создание пользователей с содержимым
> * Удаление пользователей в случае, когда им больше не нужен доступ
> * Синхронизация атрибутов пользователей между Azure AD и содержимым
> * Подготавливайте группы и членство в группах в отношении содержимого
> * [Единый вход в систему](contentful-tutorial.md) с содержимым (рекомендуется)

## <a name="prerequisites"></a>Предварительные требования

В сценарии, описанном в этом руководстве, предполагается, что у вас уже имеется:

* [Клиент Azure AD.](../develop/quickstart-create-new-tenant.md) 
* Учетная запись пользователя в Azure AD, имеющая [разрешение](../roles/permissions-reference.md) на настройку подготовки (например, администратор приложения, администратор облачных приложений, владелец приложения или глобальный администратор). 
* Учетная запись организации с поддержкой содержимого, которая поддерживает подписку, поддерживающую систему для подготовки управления удостоверениями между доменами (SCIM). Если у вас есть вопросы о подписке Организации, обратитесь в [службу поддержки по содержимому](mailto:support@contentful.com).
 
## <a name="plan-your-provisioning-deployment"></a>Планирование развертывания для подготовки

1. Узнайте, [как работает служба подготовки](../app-provisioning/user-provisioning.md).
2. Определите, кто будет находиться в [области подготовки](../app-provisioning/define-conditional-rules-for-provisioning-user-accounts.md).
3. Определите, какие данные должны [сопоставляться между Azure AD и содержимым](../app-provisioning/customize-application-attributes.md). 

## <a name="configure-contentful-to-support-provisioning-with-azure-ad"></a>Настройка содержимого для поддержки подготовки с помощью Azure AD

1. В случае с содержимым создайте учетную запись **пользователя службы** . Все разрешения на подготовку для Azure предоставляются с помощью этой учетной записи. Рекомендуется выбрать в качестве роли Организации " **владелец** " для этой учетной записи.

2. Войдите в систему, чтобы получить сведения о **пользователе службы**.

3. В меню слева выберите **Параметры организации**  >  **средства доступа**  >  **Пользователи подготовка**.

   ![Снимок экрана: меню "Параметры организации" в области "содержимое" с подготовкой пользователей в разделе "средства Access Tools".](media/contentful-provisioning-tutorial/access.png)

4. Скопируйте и сохраните **URL-адрес scim**. Это значение вводится в портал Azure на вкладке **Подготовка** приложения для работы с содержимым.

5. Выберите **создать личный маркер доступа**.

    ![url](media/contentful-provisioning-tutorial/generate.png)

6. В модальном окне введите имя личного маркера доступа, а затем выберите **создать**.

7. Создается URL-адрес SCIM и секретный токен. Скопируйте и сохраните эти значения. Эти значения необходимо ввести на вкладке **Подготовка** приложения с содержимым в портал Azure.

    ![Снимок экрана: панель личного маркера доступа с выделенным именем в C F P A T и заполнителем маркера.](media/contentful-provisioning-tutorial/token.png)


Если при настройке подготовки в консоли администрирования с содержимым возникли вопросы, обратитесь в [службу поддержки по содержимому](mailto:support@contentful.com).

## <a name="add-contentful-from-the-azure-ad-application-gallery"></a>Добавление содержимого из коллекции приложений Azure AD

Чтобы управлять подготовкой к содержимому, добавьте содержимое из коллекции приложений Azure AD. Если вы ранее настроили единый вход, можно использовать то же приложение. Однако рекомендуется создать отдельное приложение для первоначального тестирования интеграции. Узнайте, как [Добавить приложение в коллекцию](../manage-apps/add-application-portal.md). 

## <a name="define-who-will-be-in-scope-for-provisioning"></a>Определение пользователей для включения в область подготовки 

Службу подготовки Azure AD можно использовать для определения того, кто будет подготовлен в соответствии с назначением приложения или на основе атрибутов пользователя или группы. 

Если вы решили указать, кто будет подготовлен для приложения на основе назначения, выполните действия по [назначению пользователей и групп приложению](../manage-apps/assign-user-or-group-access-portal.md).

Если вы решили указать, кто будет подготовлен на основе только атрибутов пользователя или группы, используйте фильтр области, чтобы [определить условные правила для подготовки учетных записей пользователей](../app-provisioning/define-conditional-rules-for-provisioning-user-accounts.md). 

* При назначении пользователей и групп для содержимого необходимо выбрать роль, отличную от **доступа по умолчанию**. Пользователи, которым назначена роль доступа по умолчанию, исключаются из подготовки и указываются в журналах подготовки, как неэффективно. Если единственной ролью, доступной в приложении, является роль доступа по умолчанию, можно [обновить манифест приложения](../develop/howto-add-app-roles-in-azure-ad-apps.md) , чтобы добавить дополнительные роли. 
* Начните с малого. Протестируйте небольшой набор пользователей и групп, прежде чем выполнять развертывание для всех. Если для области подготовки задано значение назначенные пользователи и группы, можно управлять областью, назначив приложению одного или двух пользователей или групп. Если для области задано значение все пользователи и группы, можно указать [Фильтр области на основе атрибутов](../app-provisioning/define-conditional-rules-for-provisioning-user-accounts.md). 

## <a name="configure-automatic-user-provisioning-to-contentful"></a>Настройка автоматической подготовки пользователей для содержимого 

В этом разделе описано, как настроить службу подготовки Azure AD для создания, обновления и отключения пользователей и групп в тестовом приложении на основе назначений пользователей или групп в Azure AD.

### <a name="configure-automatic-user-provisioning-for-contentful-in-azure-ad"></a>Настройка автоматической подготовки пользователей для содержимого в Azure AD

1. Войдите на [портал Azure](https://portal.azure.com). Выберите **корпоративные приложения**, а затем выберите **все приложения**.

   ![Снимок экрана, на котором показано меню "корпоративные приложения" в портал Azure, где все приложения выделены.](common/enterprise-applications.png)

2. В списке приложений выберите **Contentful**.

   ![Снимок экрана, на котором показаны первые 20 результатов, возвращенных в списке приложений.](common/all-applications.png)

3. Выберите вкладку **Подготовка**.

   ![Снимок экрана: вкладка "подготовка", выделенная в разделе "Управление" в левом меню.](common/provisioning.png)

4. Для параметра **Режим подготовки к работе** выберите значение **Автоматически**.

   ![Снимок экрана, на котором показаны параметры режима подготовки с автоматически выделенным цветом.](common/provisioning-automatic.png)

5. В разделе **учетные данные администратора** введите URL-адрес клиента для содержимого и секретный токен. Чтобы обеспечить возможность подключения Azure AD к содержимому, выберите **проверить подключение**. В случае сбоя подключения убедитесь, что учетная запись с содержимым имеет разрешения администратора, и повторите попытку.

   ![Снимок экрана, на котором показаны текстовые поля U и Secret маркера клиента с выделенной кнопкой "проверить подключение".](common/provisioning-testconnection-tenanturltoken.png)

6. В поле **уведомление по электронной почте** введите адрес электронной почты пользователя или группы, которые должны получать уведомления об ошибках подготовки, а затем установите флажок **Отправить уведомление по электронной почте при возникновении сбоя** .

   ![Снимок экрана, на котором отображается текстовое поле уведомления по электронной почте.](common/provisioning-notification-email.png)

7. Щелкните **Сохранить**.

8. В разделе **сопоставления** выберите **синхронизировать Azure Active Directory пользователей с содержимым**.

9. В разделе **сопоставление атрибутов** проверьте пользовательские атрибуты, которые синхронизируются из Azure AD с содержимым. Атрибуты, выбранные как свойства **Matching** , используются для сопоставления учетных записей пользователей в операциях обновления. Если вы решили изменить [соответствующий целевой атрибут](../app-provisioning/customize-application-attributes.md), необходимо убедиться, что API с поддержкой содержимого поддерживает фильтрацию пользователей на основе этого атрибута. Нажмите кнопку **Сохранить**, чтобы зафиксировать все изменения.

   |attribute|Тип|Поддерживается для фильтрации|
   |---|---|---|
   |userName|Строка|&check;|
   |name.givenName|Строка|
   |name.familyName|Строка|

10. В разделе **сопоставления** выберите **синхронизировать Azure Active Directory группы с содержимым**.

11. В разделе **сопоставление атрибутов** Проверьте атрибуты группы, которые синхронизируются из Azure AD с содержимым. Атрибуты, выбранные как свойства **Matching** , используются для сопоставления групп в операциях обновления с учетом содержимого. Нажмите кнопку **Сохранить**, чтобы зафиксировать все изменения.

    |attribute|Тип|Поддерживается для фильтрации|
    |---|---|---|
    |displayName|Строка|&check;|
    |members|Справочник|

12. Чтобы настроить фильтры области, выполните действия, описанные в [руководстве по фильтрам области](../app-provisioning/define-conditional-rules-for-provisioning-user-accounts.md).

13. Чтобы включить службу подготовки Azure AD для содержимого, в разделе **Параметры** для параметра **состояние подготовки** выберите значение **вкл**.

    ![Снимок экрана, на котором показано состояние подготовки и отключение переключателя.](common/provisioning-toggle-on.png)

14. Чтобы определить пользователей или группы, которые требуется подготавливать к содержимому, в разделе **Параметры** **области** выберите соответствующий параметр.

    ![Снимок экрана, на котором показаны параметры, которые можно выбрать на панели область.](common/provisioning-scope.png)

15. Когда будете готовы выполнить подготовку, нажмите кнопку **Сохранить**.

    ![Снимок экрана, на котором показаны кнопка "Сохранить" и кнопка "Отмена".](common/provisioning-configuration-save.png)

Эта операция запускает начальный цикл синхронизации всех пользователей и групп, определенных в **области** в разделе " **Параметры**". Начальный цикл занимает больше времени, чем последующие циклы. Пока служба подготовки Azure AD запущена, они выполняются примерно каждые 40 минут. 

## <a name="monitor-your-deployment"></a>Мониторинг развертывания

После настройки подготовки используйте следующие ресурсы для мониторинга развертывания.

* Чтобы определить, какие пользователи были подготовлены успешно или неудачно, просмотрите [журналы подготовки](../reports-monitoring/concept-provisioning-logs.md).
* Чтобы просмотреть состояние цикла подготовки и его завершения, проверьте [индикатор выполнения](../app-provisioning/application-provisioning-when-will-provisioning-finish-specific-user.md).
* Если конфигурация подготовки, вероятно, находится в неработоспособном состоянии, приложение переходит в карантин. Дополнительные сведения о [состояниях карантина](../app-provisioning/application-provisioning-quarantine-status.md).
