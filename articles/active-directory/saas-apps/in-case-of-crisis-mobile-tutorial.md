# <a name="tutorial-azure-active-directory-single-sign-on-sso-integration-with-in-case-of-crisis---mobile"></a>Руководство по интеграции единого входа Azure Active Directory с In Case of Crisis (мобильное приложение)

В этом руководстве описано, как интегрировать In Case of Crisis (мобильное приложение) с Azure Active Directory (Azure AD). Интеграция In Case of Crisis (мобильное приложение) с Azure AD обеспечивает следующие преимущества:

* С помощью Azure AD вы можете контролировать доступ к In Case of Crisis (мобильное приложение).
* Вы можете включить автоматический вход пользователей в In Case of Crisis (мобильное приложение) с помощью учетных записей Azure AD.
* Централизованное управление учетными записями через портал Azure.

Чтобы узнать больше об интеграции приложений SaaS с Azure AD, прочитайте статью [Что такое доступ к приложениям и единый вход с помощью Azure Active Directory?](https://docs.microsoft.com/azure/active-directory/active-directory-appssoaccess-whatis)

## <a name="prerequisites"></a>Предварительные требования

Чтобы приступить к работе, потребуется следующее:

* подписка Azure AD Если у вас нет подписки, вы можете получить [бесплатную учетную запись](https://azure.microsoft.com/free/).
* Подписка In Case of Crisis (мобильное приложение) с поддержкой единого входа.

## <a name="scenario-description"></a>Описание сценария

В рамках этого руководства вы настроите и проверите единый вход Azure AD в тестовой среде.

* В In Case of Crisis (мобильное приложение) поддерживается единый вход, инициированный **поставщиком приложений**.

> [!NOTE]
> Идентификатор этого приложения — фиксированное строковое значение, поэтому в одном клиенте можно настроить только один экземпляр.

## <a name="adding-in-case-of-crisis---mobile-from-the-gallery"></a>Добавление In Case of Crisis (мобильное приложение) из коллекции

Чтобы настроить интеграцию In Case of Crisis (мобильное приложение) с Azure AD, необходимо добавить In Case of Crisis (мобильное приложение) из коллекции в список управляемых приложений SaaS.

1. Войдите на [портал Azure](https://portal.azure.com) с помощью личной учетной записи Майкрософт либо рабочей или учебной учетной записи.
1. В области навигации слева выберите службу **Azure Active Directory**.
1. Перейдите в колонку **Корпоративные приложения** и выберите **Все приложения**.
1. Чтобы добавить новое приложение, выберите **Новое приложение**.
1. В разделе **Добавление из коллекции** в поле поиска введите **In Case of Crisis (мобильное приложение)** .
1. Выберите **In Case of Crisis (мобильное приложение)** в области результатов и добавьте это приложение. Подождите несколько секунд, пока приложение не будет добавлено в ваш клиент.

## <a name="configure-and-test-azure-ad-single-sign-on-for-in-case-of-crisis---mobile"></a>Настройка и проверка единого входа Azure AD в In Case of Crisis (мобильное приложение)

Настройте и проверьте единый вход Azure AD в In Case of Crisis (мобильное приложение) с помощью тестового пользователя **B. Simon**. Для обеспечения работы единого входа необходимо установить связь между пользователем Azure AD и соответствующим пользователем в In Case of Crisis (мобильное приложение).

Чтобы настроить и проверить единый вход Azure AD в In Case of Crisis (мобильное приложение), вам потребуется выполнить действия в следующих стандартных блоках.

1. **[Настройка единого входа Azure AD](#configure-azure-ad-sso)** необходима, чтобы пользователи могли использовать эту функцию.
    1. **[Создание тестового пользователя Azure AD](#create-an-azure-ad-test-user)** требуется для проверки работы единого входа Azure AD с помощью пользователя B.Simon.
    1. **[Назначение тестового пользователя Azure AD](#assign-the-azure-ad-test-user)** необходимо, чтобы позволить пользователю B.Simon использовать единый вход Azure AD.
1. **[Настройка единого входа в In Case of Crisis (мобильное приложение)](#configure-in-case-of-crisis---mobile-sso)** необходима, чтобы настроить параметры единого входа на стороне приложения.
    1. **[Создание тестового пользователя In Case of Crisis (мобильное приложение)](#create-in-case-of-crisis---mobile-test-user)** требуется для того, чтобы в In Case of Crisis (мобильное приложение) существовал пользователь Britta Simon, связанный с одноименным пользователем в Azure AD.
1. **[Проверка единого входа](#test-sso)** необходима, чтобы убедиться в корректной работе конфигурации.

## <a name="configure-azure-ad-sso"></a>Настройка единого входа Azure AD

Выполните следующие действия, чтобы включить единый вход Azure AD на портале Azure.

1. На [портале Azure](https://portal.azure.com/) на странице интеграции с приложением **In Case of Crisis (мобильное приложение)** найдите раздел **Управление** и выберите **Единый вход**.
1. На странице **Выбрать метод единого входа** выберите **SAML**.
1. На странице **Настройка единого входа с помощью SAML** щелкните значок "Изменить" (значок пера), чтобы открыть диалоговое окно **Базовая конфигурация SAML** и изменить параметры.

   ![Правка базовой конфигурации SAML](common/edit-urls.png)

1. В разделе  **Базовая конфигурация SAML** приложение предварительно настроено в режиме, инициированном **поставщиком удостоверений** , и для приложения заданы требуемые URL-адреса в Azure. Пользователь должен сохранить конфигурацию, нажав кнопку **Сохранить** .

1. На странице **Настройка единого входа с помощью SAML** в разделе **Сертификат подписи SAML** найдите элемент **Сертификат (необработанный)** и щелкните **Скачать**, чтобы скачать сертификат. Сохраните этот сертификат на компьютере.

    ![Ссылка для скачивания сертификата](common/certificateraw.png)

1. Перейдите в раздел **Управление** в левой части страницы, щелкните вкладку **Свойства**, а затем скопируйте **URL-адрес для доступа пользователя** и сохраните его на компьютере.

    ![Свойства единого входа](./media/in-case-of-crisis-mobile-tutorial/properties.png)

### <a name="create-an-azure-ad-test-user"></a>Создание тестового пользователя Azure AD

В этом разделе описано, как на портале Azure создать тестового пользователя с именем B.Simon.

1. На портале Azure в области слева выберите **Azure Active Directory**, **Пользователи**, а затем — **Все пользователи**.
1. В верхней части экрана выберите **Новый пользователь**.
1. В разделе **Свойства пользователя** выполните следующие действия.
   1. В поле **Имя** введите `B.Simon`.  
   1. В поле **Имя пользователя** введите username@companydomain.extension. Например, `B.Simon@contoso.com`.
   1. Установите флажок **Показать пароль** и запишите значение, которое отображается в поле **Пароль**.
   1. Нажмите кнопку **Создать**.

### <a name="assign-the-azure-ad-test-user"></a>Назначение тестового пользователя Azure AD

В этом разделе описано, как включить единый вход в Azure для пользователя B. Simon, предоставив этому пользователю доступ к In Case of Crisis (мобильное приложение).

1. На портале Azure выберите **Корпоративные приложения**, а затем —**Все приложения**.
1. В списке приложений выберите **In Case of Crisis (мобильное приложение)** .
1. На странице "Обзор" приложения найдите раздел **Управление** и выберите **Пользователи и группы**.

   ![Ссылка "Пользователи и группы"](common/users-groups-blade.png)

1. Выберите **Добавить пользователя**, а в диалоговом окне **Добавление назначения** выберите **Пользователи и группы**.

    ![Ссылка "Добавить пользователя"](common/add-assign-user.png)

1. В диалоговом окне **Пользователи и группы** выберите **B.Simon** в списке пользователей, а затем в нижней части экрана нажмите кнопку **Выбрать**.
1. Если ожидается, что в утверждении SAML будет получено какое-либо значение роли, то в диалоговом окне **Выбор роли** нужно выбрать соответствующую роль для пользователя из списка и затем нажать кнопку **Выбрать**, расположенную в нижней части экрана.
1. В диалоговом окне **Добавление назначения** нажмите кнопку **Назначить**.

## <a name="configure-in-case-of-crisis---mobile-sso"></a>Настройка единого входа в In Case of Crisis (мобильное приложение)

Чтобы настроить единый вход на стороне **In Case of Crisis (мобильное приложение)** , нужно отправить скачанный **сертификат (необработанный)** и **URL-адрес для доступа пользователя**, скопированный на портале Azure, в [группу поддержки In Case of Crisis (мобильное приложение)](https://www.rockdovesolutions.com/features/enterprise-ready). Специалисты службы поддержки настроят подключение единого входа SAML на обеих сторонах.

### <a name="create-in-case-of-crisis---mobile-test-user"></a>Создание тестового пользователя в In Case of Crisis (мобильное приложение)

В этом разделе описано, как создать пользователя Britta Simon в приложении In Case of Crisis (мобильное приложение). Обратитесь в группу поддержки  [In Case of Crisis (мобильное приложение)](https://www.rockdovesolutions.com/features/enterprise-ready), чтобы добавить пользователей на платформу In Case of Crisis (мобильное приложение). Перед использованием единого входа необходимо создать и активировать пользователей.

## <a name="test-sso"></a>Проверка единого входа 

В этом разделе описано, как проверить конфигурацию единого входа Azure AD с помощью панели доступа.

Щелкнув плитку "In Case of Crisis (мобильное приложение)" на Панели доступа, вы автоматически войдете в приложение In Case of Crisis (мобильное приложение), для которого настроили единый вход. См. дополнительные сведения о [панели доступа](https://docs.microsoft.com/azure/active-directory/active-directory-saas-access-panel-introduction)
