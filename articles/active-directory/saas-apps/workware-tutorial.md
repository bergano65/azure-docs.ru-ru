# <a name="tutorial-azure-active-directory-single-sign-on-sso-integration-with-workware"></a>Руководство по интеграции единого входа Azure Active Directory с Workware

Из этого учебника вы узнаете, как интегрировать Workware с Azure Active Directory (Azure AD). Интеграция Workware с Azure AD обеспечивает следующие возможности.

* Управление доступом к Workware в Azure AD.
* Включение автоматического входа пользователей в Workware с использованием учетных записей Azure AD.
* Централизованное управление учетными записями через портал Azure.

## <a name="prerequisites"></a>Предварительные требования

Чтобы приступить к работе, потребуется следующее.

* Подписка Azure AD. Если у вас нет подписки, вы можете получить [бесплатную учетную запись](https://azure.microsoft.com/free/).
* Подписка Workware с поддержкой единого входа.

## <a name="scenario-description"></a>Описание сценария

В рамках этого руководства вы настроите и проверите единый вход Azure AD в тестовой среде.

* Workware поддерживает единый вход, инициированный **поставщиком удостоверений**.

## <a name="adding-workware-from-the-gallery"></a>Добавление Workware из коллекции

Чтобы настроить интеграцию Workware с Azure AD, необходимо добавить Workware из коллекции в список управляемых приложений SaaS.

1. Войдите на портал Azure с помощью личной учетной записи Майкрософт либо рабочей или учебной учетной записи.
1. В области навигации слева выберите службу **Azure Active Directory**.
1. Перейдите в колонку **Корпоративные приложения** и выберите **Все приложения**.
1. Чтобы добавить новое приложение, выберите **Новое приложение**.
1. В разделе **Добавление из коллекции** в поле поиска введите **Workware**.
1. Выберите **Workware** в области результатов, а затем добавьте это приложение. Подождите несколько секунд, пока приложение не будет добавлено в ваш клиент.


## <a name="configure-and-test-azure-ad-sso-for-workware"></a>Настройка и проверка единого входа Azure AD для Workware

Настройте и проверьте единый вход Azure AD в Workware с помощью тестового пользователя с именем **B. Simon**. Чтобы единый вход работал, необходимо установить связь между пользователем Azure AD и соответствующим пользователем в Workware.

Чтобы настроить и проверить единый вход Azure AD в Workware, выполните действия из следующих стандартных блоков:

1. **[Настройка единого входа Azure AD](#configure-azure-ad-sso)** необходима, чтобы пользователи могли использовать эту функцию.
    * **[Создание тестового пользователя Azure AD](#create-an-azure-ad-test-user)** требуется для проверки работы единого входа Azure AD с помощью пользователя B.Simon.
    * **[Назначение тестового пользователя Azure AD](#assign-the-azure-ad-test-user)** необходимо, чтобы позволить пользователю B.Simon использовать единый вход Azure AD.
1. **[Настройка единого входа в Workware](#configure-workware-sso)** необходима, чтобы настроить параметры единого входа на стороне приложения.
    * **[Создание тестового пользователя в Workware](#create-workware-test-user)** требуется для того, чтобы в Workware существовал пользователь B. Simon, связанный с одноименным пользователем в Azure AD.
1. **[Проверка единого входа](#test-sso)** позволяет убедиться в правильности конфигурации.

## <a name="configure-azure-ad-sso"></a>Настройка единого входа Azure AD

Выполните следующие действия, чтобы включить единый вход Azure AD на портале Azure.

1. На портале Azure на странице интеграции с приложением **Workware** найдите раздел **Управление** и выберите **Единый вход**.
1. На странице **Выбрать метод единого входа** выберите **SAML**.
1. На странице **Настройка единого входа с помощью SAML** щелкните значок "Изменить" (значок пера), чтобы открыть диалоговое окно **Базовая конфигурация SAML** и изменить параметры.

   ![Изменение базовой конфигурации SAML](common/edit-urls.png)

1. На странице **Настройка единого входа с помощью SAML** введите значения для следующих полей:

    а. В текстовом поле **Идентификатор** введите URL-адрес в формате `<WORKWARE_URL>/WW/AuthServices`.

    b. В текстовом поле **URL-адрес ответа** введите URL-адрес в формате `<WORKWARE_URL>/WW/AuthServices/Acs`.

    > [!NOTE]
    > Эти значения приведены для примера. Измените их на фактические значения идентификатора и URL-адреса ответа. Чтобы получить эти значения, обратитесь в [группу поддержки клиентов Workware](mailto:support@activeops.com). Можно также посмотреть шаблоны в разделе **Базовая конфигурация SAML** на портале Azure.

1. На странице **Настройка единого входа с помощью SAML** в разделе **Сертификат подписи SAML** найдите элемент **XML метаданных федерации** и выберите **Скачать**, чтобы скачать сертификат и сохранить его на компьютере.

    ![Ссылка для скачивания сертификата](common/metadataxml.png)

1. Скопируйте требуемые URL-адреса в разделе **Настройка Workware**.

    ![Копирование URL-адресов настройки](common/copy-configuration-urls.png)
### <a name="create-an-azure-ad-test-user"></a>Создание тестового пользователя Azure AD

В этом разделе описано, как на портале Azure создать тестового пользователя с именем B.Simon.

1. На портале Azure в области слева выберите **Azure Active Directory**, **Пользователи**, а затем — **Все пользователи**.
1. В верхней части экрана выберите **Новый пользователь**.
1. В разделе **Свойства пользователя** выполните следующие действия.
   1. В поле **Имя** введите `B.Simon`.  
   1. В поле **Имя пользователя** введите username@companydomain.extension. Например, `B.Simon@contoso.com`.
   1. Установите флажок **Показать пароль** и запишите значение, которое отображается в поле **Пароль**.
   1. Нажмите кнопку **Создать**.

### <a name="assign-the-azure-ad-test-user"></a>Назначение тестового пользователя Azure AD

В этом разделе описано, как включить единый вход в Azure для пользователя B. Simon, предоставив этому пользователю доступ к Workware.

1. На портале Azure выберите **Корпоративные приложения**, а затем — **Все приложения**.
1. В списке приложений выберите **Workware**.
1. На странице "Обзор" приложения найдите раздел **Управление** и выберите **Пользователи и группы**.
1. Выберите **Добавить пользователя**, а в диалоговом окне **Добавление назначения** выберите **Пользователи и группы**.
1. В диалоговом окне **Пользователи и группы** выберите **B.Simon** в списке пользователей, а затем в нижней части экрана нажмите кнопку **Выбрать**.
1. Если пользователям необходимо назначить роль, вы можете выбрать ее из раскрывающегося списка **Выберите роль**. Если для этого приложения не настроена ни одна роль, будет выбрана роль "Доступ по умолчанию".
1. В диалоговом окне **Добавление назначения** нажмите кнопку **Назначить**.

## <a name="configure-workware-sso"></a>Настройка единого входа в Workware

Чтобы использовать функцию единого входа в Workware, необходимо выполнить следующую настройку:

#### <a name="enable-sso-permissions-for-workware-system-administrators"></a>Включение разрешений единого входа для системных администраторов Workware

* Чтобы разрешить системным администраторам Workware настроить проверку подлинности единого входа, для них необходимо включить разрешение на проверку подлинности единого входа (в категории разрешений **Administration > System configuration (Администрирование > Конфигурация системы) в окне System Settings > Permissions to Role** (Параметры системы > Разрешения для роли)).

    ![Разрешение на проверку подлинности единого входа](./media/workware-tutorial/permission.png)

#### <a name="set-up-sso-authentication-in-workware"></a>Настройка проверки подлинности единого входа в Workware

1. Перейдите на страницу **System Settings** (Параметры системы) и щелкните **SSO Authentication** (Проверка подлинности единого входа).

1. В разделе **SSO Authentication** (Проверка подлинности единого входа) нажмите кнопку **Add SSO Authentication** (Добавить проверку подлинности единого входа) и выполните приведенные ниже действия. 

    ![Проверка подлинности единого входа](./media/workware-tutorial/authentication.png)

    1. В поле **External Identity Provider** (Внешний поставщик удостоверений) укажите имя поставщика удостоверений.
    1. Выберите **SAML2.0** в качестве **типа проверки подлинности**.
    1. В текстовое поле **Identity Provider SignIn URL** (URL-адрес входа поставщика удостоверений) вставьте значение **URL-адреса входа**, скопированное на портале Azure.
    1. В текстовое поле **Identity Provider Issuer URL** (URL-адрес издателя поставщика удостоверений) вставьте значение **идентификатора Azure AD**, скопированное на портале Azure.
    1. В текстовое поле **Identity Provider Logout URL** (URL-адрес выхода поставщика удостоверений) вставьте значение **URL-адреса выхода**, скопированное на портале Azure.
    1. Установите флажок **Enable** (Включить).
    1. Передайте скачанный с портала Azure **сертификат** в качестве **сертификата поставщика удостоверений**.
    1. Нажмите кнопку **Сохранить**


### <a name="create-workware-test-user"></a>Создание тестового пользователя в Workware

1. Войдите на веб-сайт Workware в качестве администратора.

1. Последовательно выберите **Admin > Create / View > User Accounts > Add New** (Администратор > Создать/Просмотреть > Учетные записи пользователей > Добавить).

1. На следующей странице выполните приведенные ниже шаги.

    ![Тестовый пользователь](./media/workware-tutorial/create-user.png)

    a. Укажите допустимое имя в поле **Name** (Имя).

    b. Для параметра **Authentication Type** (Тип проверки подлинности) выберите значение **SSO** (Единый вход).

    c. Введите данные в обязательные поля и нажмите кнопку **Save** (Сохранить).

## <a name="test-sso"></a>Проверка единого входа 

В этом разделе описано, как проверить конфигурацию единого входа Azure AD с помощью указанных ниже способов.

* На портале Azure выберите "Тестировать приложение", и вы автоматически войдете в приложение Workware, для которого настроен единый вход.

* Вы можете использовать Панель доступа (Майкрософт). Щелкнув плитку Workware на Панели доступа, вы автоматически войдете в приложение Workware, для которого настроили единый вход. См. дополнительные сведения о [панели доступа](../user-help/my-apps-portal-end-user-access.md)
