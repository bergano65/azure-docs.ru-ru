# <a name="tutorial-integrate-wandera-with-azure-active-directory"></a>Руководство по Интеграция Wandera с Azure Active Directory

В этом руководстве описано, как интегрировать Wandera с Azure Active Directory (Azure AD). Интеграция Wandera с Azure AD обеспечивает следующие возможности.

* С помощью Azure AD вы можете контролировать доступ к Wandera.
* Включение автоматического входа пользователей в Wandera с помощью учетных записей Azure AD.
* Централизованное управление учетными записями через портал Azure.

Чтобы узнать больше об интеграции приложений SaaS с Azure AD, прочитайте статью [Что такое доступ к приложениям и единый вход с помощью Azure Active Directory?](https://docs.microsoft.com/azure/active-directory/active-directory-appssoaccess-whatis)

## <a name="prerequisites"></a>Предварительные требования

Чтобы приступить к работе, потребуется следующее:

* подписка Azure AD Если у вас нет подписки, вы можете получить [бесплатную учетную запись](https://azure.microsoft.com/free/).
* Подписка Wandera с поддержкой единого входа.

## <a name="scenario-description"></a>Описание сценария

В рамках этого руководства вы настроите и проверите единый вход Azure AD в тестовой среде.

* Wandera поддерживает единый вход, инициируемый **поставщиком удостоверений**

## <a name="adding-wandera-from-the-gallery"></a>Добавление Wandera из коллекции

Чтобы настроить интеграцию Wandera с Azure AD, необходимо добавить Wandera из коллекции в список управляемых приложений SaaS.

1. Войдите на [портал Azure](https://portal.azure.com) с помощью личной учетной записи Майкрософт либо рабочей или учебной учетной записи.
1. В области навигации слева выберите службу **Azure Active Directory**.
1. Перейдите в колонку **Корпоративные приложения** и выберите **Все приложения**.
1. Чтобы добавить новое приложение, выберите **Новое приложение**.
1. В разделе **Добавление из коллекции** в поле поиска введите **Wandera**.
1. Выберите **Wandera** в области результатов и добавьте это приложение. Подождите несколько секунд, пока приложение не будет добавлено в ваш клиент.


## <a name="configure-and-test-azure-ad-single-sign-on"></a>Настройка и проверка единого входа в Azure AD

Настройте и проверьте единый вход Azure AD в Wandera с помощью тестового пользователя **B. Simon**. Для обеспечения работы единого входа необходимо установить связь между пользователем Azure AD и соответствующим пользователем в Wandera.

Чтобы настроить и проверить единый вход Azure AD в Wandera, выполните действия их следующих стандартных блоков.

1. **[Настройка единого входа Azure AD](#configure-azure-ad-sso)** необходима, чтобы пользователи могли использовать эту функцию.
2. **[Настройка единого входа в Wandera](#configure-wandera-sso)** необходима чтобы настроить параметры единого входа на стороне приложения.
3. **[Создание тестового пользователя Azure AD](#create-an-azure-ad-test-user)** требуется для проверки работы единого входа Azure AD с помощью пользователя B.Simon.
4. **[Назначение тестового пользователя Azure AD](#assign-the-azure-ad-test-user)** необходимо, чтобы позволить пользователю B.Simon использовать единый вход Azure AD.
5. **[Создание тестового пользователя Wandera](#create-wandera-test-user)** требуется для того, чтобы в приложении Wandera существовал пользователь B. Simon, связанный с одноименным пользователем в Azure AD.
6. **[Проверка единого входа](#test-sso)** необходима, чтобы убедиться в корректной работе конфигурации.

### <a name="configure-azure-ad-sso"></a>Настройка единого входа Azure AD

Выполните следующие действия, чтобы включить единый вход Azure AD на портале Azure.

1. На [портале Azure](https://portal.azure.com/) на странице интеграции с приложением **Wandera** найдите раздел **Управление** и выберите **Единый вход**.
1. На странице **Выбрать метод единого входа** выберите **SAML**.
1. На странице **Настройка единого входа с помощью SAML** щелкните значок "Изменить" (значок пера), чтобы открыть диалоговое окно **Базовая конфигурация SAML** и изменить параметры.

   ![Правка базовой конфигурации SAML](common/edit-urls.png)

1. На странице **Базовая конфигурация SAML** введите значения следующих полей.

    В текстовом поле **URL-адрес ответа** введите URL-адрес в следующем формате: `https://radar.wandera.com/saml/acs/<tenant id>`.

    > [!NOTE]
    > Это значение приведено для примера. Вместо него нужно указать фактический URL-адрес ответа. Чтобы получить это значение, обратитесь к [группе поддержки клиентов Wandera](https://www.wandera.com/about-wandera/contact/#supportsection). Можно также посмотреть шаблоны в разделе **Базовая конфигурация SAML** на портале Azure.

1. На странице **Настройка единого входа с помощью SAML** в разделе **Сертификат подписи SAML** найдите элемент **XML метаданных федерации** и нажмите кнопку **Скачать**, чтобы скачать сертификат и сохранить его на компьютере.

    ![Ссылка для скачивания сертификата](common/metadataxml.png)

1. На странице **Настройка единого входа с помощью SAML** щелкните значок "Изменить" (значок пера), чтобы открыть диалоговое окно **Сертификат подписи SAML** и изменить параметры.

    ![Вариант подписывания](common/signing-option.png)

    1. Для параметра **Вариант подписывания** выберите значение **Ответ знака SAML и утверждение**.

    1. Выберите **Алгоритм подписывания** как **SHA-256**.

1. Скопируйте требуемые URL-адреса из раздела **Настройка Wandera**.

    ![Копирование URL-адресов настройки](common/copy-configuration-urls.png)

### <a name="configure-wandera-sso"></a>Настройте единый вход Wandera

1. Чтобы автоматизировать настройку в Wandera, необходимо установить **расширение браузера My Apps Secure Sign-in browser extension**, нажав кнопку **Установить расширение**.

    ![Расширение "Мои приложения"](common/install-myappssecure-extension.png)

2. После добавления расширения в браузер щелкните **Настройка Wandera**, и вы перейдете в приложение Wandera. После этого укажите учетные данные администратора для входа в Wandera. Расширение браузера автоматически настроит приложение и автоматизирует шаги 3–4.

    ![Настройка конфигурации](common/setup-sso.png)

3. Если вы хотите настроить Wandera вручную, откройте новое окно веб-браузера, войдите на свой корпоративный сайт Wandera в качестве администратора и выполните следующие действия:

4. В правом верхнем углу страницы щелкните **Settings** (Параметры) > **Administration** (Администрирование) > **Single Sign-On** (Единый вход), установите флажок **Enable SAML 2.0** (Включить SAML 2.0) и выполните следующие действия.

    ![Настройка Wandera](./media/wandera-tutorial/config01.png)

    a. Щелкните **Or manually enter the required fields** (Или ввести обязательные поля вручную).

    b. В текстовое поле **IdP EntityId** (Идентификатор сущности для поставщика удостоверений) вставьте значение **Идентификатор Azure AD**, скопированное на портале Azure.

    c. Откройте в блокноте XML-файл метаданных федерации, скопируйте его содержимое и вставьте в текстовое поле **IdP Public X.509 Certificate** (Общедоступный сертификат X.509 поставщика удостоверений).

    d. Выберите команду **Сохранить**.

### <a name="create-an-azure-ad-test-user"></a>Создание тестового пользователя Azure AD

В этом разделе описано, как на портале Azure создать тестового пользователя с именем B.Simon.

1. На портале Azure в области слева выберите **Azure Active Directory**, **Пользователи**, а затем — **Все пользователи**.
1. В верхней части экрана выберите **Новый пользователь**.
1. В разделе **Свойства пользователя** выполните следующие действия.
   1. В поле **Имя** введите `B.Simon`.  
   1. В поле **Имя пользователя** введите username@companydomain.extension. Например, `B.Simon@contoso.com`.
   1. Установите флажок **Показать пароль** и запишите значение, которое отображается в поле **Пароль**.
   1. Нажмите кнопку **Создать**.

### <a name="assign-the-azure-ad-test-user"></a>Назначение тестового пользователя Azure AD

В этом разделе описано, как включить единый вход в Azure для пользователя B. Simon, предоставив этому пользователю доступ к Wandera.

1. На портале Azure выберите **Корпоративные приложения**, а затем —**Все приложения**.
1. В списке приложений выберите **Wandera**.
1. На странице "Обзор" приложения найдите раздел **Управление** и выберите **Пользователи и группы**.

   ![Ссылка "Пользователи и группы"](common/users-groups-blade.png)

1. Выберите **Добавить пользователя**, а в диалоговом окне **Добавление назначения** выберите **Пользователи и группы**.

    ![Ссылка "Добавить пользователя"](common/add-assign-user.png)

1. В диалоговом окне **Пользователи и группы** выберите **B.Simon** в списке пользователей, а затем в нижней части экрана нажмите кнопку **Выбрать**.
1. Если ожидается, что в утверждении SAML будет получено какое-либо значение роли, то в диалоговом окне **Выбор роли** нужно выбрать соответствующую роль для пользователя из списка и затем нажать кнопку **Выбрать**, расположенную в нижней части экрана.
1. В диалоговом окне **Добавление назначения** нажмите кнопку **Назначить**.

### <a name="create-wandera-test-user"></a>Создайте тестового пользователя Wandera

В этом разделе вы создадите пользователя с именем B. Simon в приложении Wandera. Обратитесь в  [службу поддержки Wandera](https://www.wandera.com/about-wandera/contact/#supportsection), чтобы добавить пользователей на платформу Wandera. Перед использованием единого входа необходимо создать и активировать пользователей.

### <a name="test-sso"></a>Проверка единого входа

В этом разделе описано, как проверить конфигурацию единого входа Azure AD с помощью панели доступа.

Щелкнув элемент "Wandera" на Панели доступа, вы автоматически войдете в приложение Wandera, для которого настроили единый вход. См. дополнительные сведения о [панели доступа](https://docs.microsoft.com/azure/active-directory/active-directory-saas-access-panel-introduction)
