# <a name="tutorial-azure-active-directory-single-sign-on-sso-integration-with-cch-tagetik"></a>Руководство по интеграции единого входа Azure Active Directory с CCH Tagetik

В этом руководстве описано, как интегрировать CCH Tagetik с Azure Active Directory (Azure AD). Интеграция CCH Tagetik с Azure AD обеспечивает следующие возможности:

* Контроль доступа к CCH Tagetik с помощью Azure AD.
* Настройка автоматического входа в CCH Tagetik с помощью пользовательских учетных записей Azure AD.
* Централизованное управление учетными записями через портал Azure.

Чтобы узнать больше об интеграции приложений SaaS с Azure AD, прочитайте статью [Что такое доступ к приложениям и единый вход с помощью Azure Active Directory?](../manage-apps/what-is-single-sign-on.md)

## <a name="prerequisites"></a>Предварительные требования

Чтобы приступить к работе, потребуется следующее.

* Подписка Azure AD. Если у вас нет подписки, вы можете получить [бесплатную учетную запись](https://azure.microsoft.com/free/).
* Подписка CCH Tagetik с поддержкой единого входа.

## <a name="scenario-description"></a>Описание сценария

В рамках этого руководства вы настроите и проверите единый вход Azure AD в тестовой среде.

* CCH Tagetik поддерживает единый вход, который инициируют **поставщики услуг и удостоверений** .
* CCH Tagetik поддерживает **JIT** -подготовку пользователей.
* После настройки CCH Tagetik вы сможете применить функцию управления сеансом, которая защищает от кражи конфиденциальных данных вашей организации и несанкционированного доступа к ним в реальном времени. Управление сеансом является расширением функции условного доступа. [Узнайте, как применять управление сеансами с помощью Microsoft Cloud App Security](/cloud-app-security/proxy-deployment-any-app).

## <a name="adding-cch-tagetik-from-the-gallery"></a>Добавление CCH Tagetik из коллекции

Чтобы настроить интеграцию CCH Tagetik с Azure AD, необходимо добавить CCH Tagetik из коллекции в список управляемых приложений SaaS.

1. Войдите на [портал Azure](https://portal.azure.com) с помощью личной учетной записи Майкрософт либо рабочей или учебной учетной записи.
1. В области навигации слева выберите службу **Azure Active Directory** .
1. Перейдите в колонку **Корпоративные приложения** и выберите **Все приложения** .
1. Чтобы добавить новое приложение, выберите **Новое приложение** .
1. В разделе **Добавление из коллекции** в поле поиска введите **CCH Tagetik** .
1. В области результатов выберите **CCH Tagetik** и затем добавьте это приложение. Подождите несколько секунд, пока приложение не будет добавлено в ваш клиент.

## <a name="configure-and-test-azure-ad-single-sign-on-for-cch-tagetik"></a>Настройка и проверка единого входа Azure AD для CCH Tagetik

Настройте и проверьте единый вход Azure AD в CCH Tagetik с помощью тестового пользователя **B.Simon** . Для обеспечения работы единого входа необходимо установить связь между пользователем Azure AD и соответствующим пользователем в CCH Tagetik.

Чтобы настроить и проверить единый вход Azure AD в CCH Tagetik, выполните следующие действия:

1. **[Настройка единого входа Azure AD.](#configure-azure-ad-sso)** Требуется, чтобы пользователи могли использовать эту функцию.
    1. **[Создание тестового пользователя Azure AD.](#create-an-azure-ad-test-user)** Требуется для проверки работы единого входа Azure AD с помощью пользователя B.Simon.
    1. **[Назначение тестового пользователя Azure AD.](#assign-the-azure-ad-test-user)** Требуется, чтобы позволить пользователю B.Simon использовать единый вход Azure AD.
1. **[Настройка единого входа в CCH Tagetik.](#configure-cch-tagetik-sso)** Требуется, чтобы настроить параметры единого входа на стороне приложения.
    1. **[Создание тестового пользователя приложения CCH Tagetik.](#create-cch-tagetik-test-user)** Требуется для того, чтобы в CCH Tagetik существовал пользователь B.Simon, связанный с одноименным пользователем в Azure AD.
1. **[Проверка единого входа.](#test-sso)** Позволяет убедиться в правильности конфигурации.

## <a name="configure-azure-ad-sso"></a>Настройка единого входа Azure AD

Выполните следующие действия, чтобы включить единый вход Azure AD на портале Azure.

1. На [портале Azure](https://portal.azure.com/) на странице интеграции приложения **CCH Tagetik** найдите раздел **Управление** и выберите **Единый вход** .
1. На странице **Выбрать метод единого входа** выберите **SAML** .
1. На странице **Настройка единого входа с помощью SAML** щелкните значок "Изменить" (значок пера), чтобы открыть диалоговое окно **Базовая конфигурация SAML** и изменить параметры.

   ![Изменение базовой конфигурации SAML](common/edit-urls.png)

1. Если вы хотите настроить приложение в режиме, инициируемом **поставщиком удостоверений** , в разделе **Базовая конфигурация SAML** введите значения следующих полей.

    а. В текстовом поле **Идентификатор** введите URL-адрес в формате `https://<CUSTOMER_NAME>.saastagetik.com/prod/5/`.

    b. В текстовом поле **URL-адрес ответа** введите URL-адрес в формате `https://<CUSTOMER_NAME>.saastagetik.com/prod/5/`.

1. Чтобы настроить приложение для работы в режиме, инициируемом **поставщиком услуг** , щелкните **Задать дополнительные URL-адреса** и выполните следующие действия.

    В текстовом поле **URL-адрес входа** введите URL-адрес в формате `https://<CUSTOMER_NAME>.saastagetik.com/prod/5/`.

    > [!NOTE]
    > Эти значения приведены для примера. Замените их фактическими значениями идентификатора, URL-адреса ответа и URL-адреса входа. Чтобы получить эти значения, обратитесь в [группу поддержки клиентов CCH Tagetik](mailto:tgk-dl-supportmembers@wolterskluwer.com). Можно также посмотреть шаблоны в разделе **Базовая конфигурация SAML** на портале Azure.

1. На странице **Настройка единого входа с помощью SAML** в разделе **Сертификат подписи SAML** найдите элемент **XML метаданных федерации** и выберите **Скачать** , чтобы скачать сертификат и сохранить его на компьютере.

    ![Ссылка для скачивания сертификата](common/metadataxml.png)

1. Требуемые URL-адреса можно скопировать из раздела **Настройка CCH Tagetik** .

    ![Копирование URL-адресов настройки](common/copy-configuration-urls.png)

### <a name="create-an-azure-ad-test-user"></a>Создание тестового пользователя Azure AD

В этом разделе описано, как на портале Azure создать тестового пользователя с именем B.Simon.

1. На портале Azure в области слева выберите **Azure Active Directory** , **Пользователи** , а затем — **Все пользователи** .
1. В верхней части экрана выберите **Новый пользователь** .
1. В разделе **Свойства пользователя** выполните следующие действия.
   1. В поле **Имя** введите `B.Simon`.  
   1. В поле **Имя пользователя** введите username@companydomain.extension. Например, `B.Simon@contoso.com`.
   1. Установите флажок **Показать пароль** и запишите значение, которое отображается в поле **Пароль** .
   1. Нажмите кнопку **Создать** .

### <a name="assign-the-azure-ad-test-user"></a>Назначение тестового пользователя Azure AD

В этом разделе описано, как включить единый вход в Azure для пользователя B.Simon, предоставив этому пользователю доступ к CCH Tagetik.

1. На портале Azure выберите **Корпоративные приложения** , а затем — **Все приложения** .
1. В списке приложений выберите **CCH Tagetik** .
1. На странице "Обзор" приложения найдите раздел **Управление** и выберите **Пользователи и группы** .

   ![Ссылка "Пользователи и группы"](common/users-groups-blade.png)

1. Выберите **Добавить пользователя** , а в диалоговом окне **Добавление назначения**  выберите **Пользователи и группы** .

    ![Ссылка "Добавить пользователя"](common/add-assign-user.png)

1. В диалоговом окне **Пользователи и группы** выберите **B.Simon** в списке пользователей, а затем в нижней части экрана нажмите кнопку **Выбрать** .
1. Если ожидается, что в утверждении SAML будет получено какое-либо значение роли, то в диалоговом окне **Выбор роли** нужно выбрать соответствующую роль для пользователя из списка и затем нажать кнопку **Выбрать** , расположенную в нижней части экрана.
1. В диалоговом окне **Добавление назначения** нажмите кнопку **Назначить** .

## <a name="configure-cch-tagetik-sso"></a>Настройка единого входа CCH Tagetik

Чтобы настроить единый вход на стороне **CCH Tagetik** , нужно отправить загруженный **XML-файл метаданных федерации** и соответствующие URL-адреса, скопированные на портале Azure, в [группу поддержки CCH Tagetik](mailto:tgk-dl-supportmembers@wolterskluwer.com). Специалисты службы поддержки настроят подключение единого входа SAML на обеих сторонах.

### <a name="create-cch-tagetik-test-user"></a>Создание тестового пользователя CCH Tagetik

В этом разделе мы создадим в CCH Tagetik пользователя с именем Britta Simon. CCH Tagetik поддерживает JIT-подготовку пользователей. Эта возможность включена по умолчанию. В этом разделе никакие действия с вашей стороны не требуются. Если пользователь еще не существует в CCH Tagetik, он будет создан после проверки подлинности.

## <a name="test-sso"></a>Проверка единого входа 

В этом разделе описано, как проверить конфигурацию единого входа Azure AD с помощью панели доступа.

Щелкнув плитку CCH Tagetik на Панели доступа, вы автоматически войдете в приложение CCH Tagetik, для которого настроили единый вход. См. дополнительные сведения о [панели доступа](../user-help/my-apps-portal-end-user-access.md)
