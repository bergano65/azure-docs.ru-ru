# <a name="tutorial-integrate-chargebee-with-azure-active-directory"></a>Руководство по Интеграция Chargebee с Azure Active Directory

В этом руководстве описано, как интегрировать Chargebee с Azure Active Directory (AAD). Интеграция Chargebee с Azure AD обеспечивает следующие возможности.

* С помощью Azure AD вы можете контролировать доступ к Chargebee.
* Автоматический вход пользователей в Chargebee с использованием учетных записей AAD.
* Централизованное управление учетными записями через портал Azure.

Чтобы узнать больше об интеграции приложений SaaS с Azure AD, прочитайте статью [Что такое доступ к приложениям и единый вход с помощью Azure Active Directory?](https://docs.microsoft.com/azure/active-directory/active-directory-appssoaccess-whatis)

## <a name="prerequisites"></a>Предварительные требования

Чтобы приступить к работе, потребуется следующее:

* подписка Azure AD Если у вас нет подписки, вы можете получить [бесплатную учетную запись](https://azure.microsoft.com/free/).
* подписка Chargebee с поддержкой единого входа.

## <a name="scenario-description"></a>Описание сценария

В рамках этого руководства вы настроите и проверите единый вход Azure AD в тестовой среде.

* Chargebee поддерживает единый вход, инициированный **поставщиком услуг и поставщиком удостоверений**.

## <a name="adding-chargebee-from-the-gallery"></a>Добавление Chargebee из коллекции

Чтобы настроить интеграцию Chargebee с Azure AD, необходимо добавить Chargebee из коллекции в список управляемых приложений SaaS.

1. Войдите на [портал Azure](https://portal.azure.com) с помощью личной учетной записи Майкрософт либо рабочей или учебной учетной записи.
1. В области навигации слева выберите службу **Azure Active Directory**.
1. Перейдите в колонку **Корпоративные приложения** и выберите **Все приложения**.
1. Чтобы добавить новое приложение, выберите **Новое приложение**.
1. В разделе **Добавление из коллекции** в поле поиска введите **Chargebee**.
1. Выберите **Chargebee** в области результатов и добавьте это приложение. Подождите несколько секунд, пока приложение не будет добавлено в ваш клиент.

## <a name="configure-and-test-azure-ad-single-sign-on-for-chargebee"></a>Настройка и проверка единого входа AAD для Chargebee

Настройте и проверьте единый вход Azure AD в Chargebee с помощью тестового пользователя **B.Simon**. Для работы единого входа необходимо установить связь между пользователем AAD и соответствующим пользователем в Chargebee.

Чтобы настроить и проверить единый вход Azure AD в Chargebee, выполните действия в следующих стандартных блоках.

1. **[Настройка единого входа Azure AD](#configure-azure-ad-sso)** необходима, чтобы пользователи могли использовать эту функцию.
    1. **[Создание тестового пользователя Azure AD](#create-an-azure-ad-test-user)** требуется для проверки работы единого входа Azure AD с помощью пользователя B.Simon.
    1. **[Назначение тестового пользователя Azure AD](#assign-the-azure-ad-test-user)** необходимо, чтобы позволить пользователю B.Simon использовать единый вход Azure AD.
2. **[Настройка единого входа в Chargebee](#configure-chargebee-sso)** необходима, чтобы настроить параметры единого входа на стороне приложения.
    1. **[Создание тестового пользователя приложения Chargebee](#create-chargebee-test-user)** требуется для того, чтобы в Chargebee существовал пользователь B.Simon, связанный с одноименным пользователем в AAD.
3. **[Проверка единого входа](#test-sso)** необходима, чтобы убедиться в корректной работе конфигурации.

## <a name="configure-azure-ad-sso"></a>Настройка единого входа Azure AD

Выполните следующие действия, чтобы включить единый вход Azure AD на портале Azure.

1. На [портале Azure](https://portal.azure.com/) на странице интеграции с приложением **Chargebee** найдите раздел **Управление** и выберите **Единый вход**.
1. На странице **Выбрать метод единого входа** выберите **SAML**.
1. На странице **Настройка единого входа с помощью SAML** щелкните значок "Изменить" (значок пера), чтобы открыть диалоговое окно **Базовая конфигурация SAML** и изменить параметры.

   ![Правка базовой конфигурации SAML](common/edit-urls.png)

1. Если вы хотите настроить приложение в режиме, инициируемом **поставщиком удостоверений**, в разделе **Базовая конфигурация SAML** введите значения следующих полей.

    a. В текстовом поле **Идентификатор** введите URL-адрес в формате `https://<domainname>.chargebee.com`.

    б) В текстовом поле **URL-адрес ответа** введите URL-адрес в формате `https://app.chargebee.com/saml/<domainname>/acs`.

1. Чтобы настроить приложение для работы в режиме, инициируемом **поставщиком услуг**, щелкните **Задать дополнительные URL-адреса** и выполните следующие действия.

    В текстовом поле **URL-адрес входа** введите URL-адрес в формате `https://<domainname>.chargebee.com`.

    > [!NOTE]
    > `<domainname>` — имя домена, который создает пользователь после присвоения учетной записи. Для получения другой информации обратитесь в [службу поддержки клиентов Chargebee](mailto:support@chargebee.com). Можно также посмотреть шаблоны в разделе **Базовая конфигурация SAML** на портале Azure.

4. На странице **Настройка единого входа с помощью SAML** в разделе **Сертификат подписи SAML** найдите пункт **Сертификат (Base64)** и щелкните **Скачать**, чтобы скачать сертификат. Сохраните этот сертификат на компьютере.

    ![Ссылка для скачивания сертификата](common/certificatebase64.png)

6. Требуемые URL-адреса вы можете скопировать из раздела **Настройка Chargebee**.

    ![Копирование URL-адресов настройки](common/copy-configuration-urls.png)

### <a name="create-an-azure-ad-test-user"></a>Создание тестового пользователя Azure AD

В этом разделе описано, как на портале Azure создать тестового пользователя с именем B.Simon.

1. На портале Azure в области слева выберите **Azure Active Directory**, **Пользователи**, а затем — **Все пользователи**.
1. В верхней части экрана выберите **Новый пользователь**.
1. В разделе **Свойства пользователя** выполните следующие действия.
   1. В поле **Имя** введите `B.Simon`.  
   1. В поле **Имя пользователя** введите username@companydomain.extension. Например, `B.Simon@contoso.com`.
   1. Установите флажок **Показать пароль** и запишите значение, которое отображается в поле **Пароль**.
   1. Нажмите кнопку **Создать**.

### <a name="assign-the-azure-ad-test-user"></a>Назначение тестового пользователя Azure AD

В этом разделе описано, как включить единый вход в Azure для пользователя B.Simon, предоставив этому пользователю доступ к Chargebee.

1. На портале Azure выберите **Корпоративные приложения**, а затем —**Все приложения**.
1. В списке приложений выберите **Chargebee**.
1. На странице "Обзор" приложения найдите раздел **Управление** и выберите **Пользователи и группы**.

   ![Ссылка "Пользователи и группы"](common/users-groups-blade.png)

1. Выберите **Добавить пользователя**, а в диалоговом окне **Добавление назначения** выберите **Пользователи и группы**.

    ![Ссылка "Добавить пользователя"](common/add-assign-user.png)

1. В диалоговом окне **Пользователи и группы** выберите **B.Simon** в списке пользователей, а затем в нижней части экрана нажмите кнопку **Выбрать**.
1. Если ожидается, что в утверждении SAML будет получено какое-либо значение роли, то в диалоговом окне **Выбор роли** нужно выбрать соответствующую роль для пользователя из списка и затем нажать кнопку **Выбрать**, расположенную в нижней части экрана.
1. В диалоговом окне **Добавление назначения** нажмите кнопку **Назначить**.

## <a name="configure-chargebee-sso"></a>Настройка единого входа в Chargebee

1. Откройте новое окно веб-браузера и войдите на корпоративный сайт Chargebee с правами администратора.

4. В левой части меню последовательно щелкните **Параметры** > **Безопасность** > **Управление**.

    ![Конфигурация Chargebee](./media/chargebee-tutorial/config01.png)

5. На странице **Single Sign-On** (Единый вход) выполните следующие шаги.

    ![Конфигурация Chargebee](./media/chargebee-tutorial/config02.png)

    a. Выберите **SAML**.

    b. В текстовое поле **Login URL** (URL-адрес для входа) вставьте значение **URL-адрес входа**, скопированное на портале Azure.

    c. Откройте сертификат в кодировке Base64 в Блокноте, скопируйте его содержимое и вставьте его в текстовое поле **Certificate SAML** (Сертификат SAML).

    d. Щелкните **Confirm** (Подтвердить).

### <a name="create-chargebee-test-user"></a>Создание тестового пользователя Chargebee

Чтобы пользователи AAD могли выполнять вход в Chargebee, они должны быть подготовлены для Chargebee. В Chargebee подготовка выполняется вручную.

**Чтобы подготовить учетную запись пользователя, сделайте следующее:**

1. В другом окне браузера войдите в приложение Chargebee с правами администратора безопасности.

2. В левой части меню щелкните **Customers** (Клиенты) и перейдите к пункту **Create a New Customer** (Создание нового пользователя).

    ![Конфигурация Freedcamp](./media/chargebee-tutorial/config03.png)

3. На странице **New Customer** (Новый клиент) заполните соответствующие поля, как показано ниже, и щелкните **Create Customer** (Создать клиента).

    ![Конфигурация Freedcamp](./media/chargebee-tutorial/config04.png)
