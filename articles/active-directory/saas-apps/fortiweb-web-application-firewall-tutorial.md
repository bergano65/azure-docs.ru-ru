# <a name="tutorial-azure-active-directory-single-sign-on-sso-integration-with-fortiweb-web-application-firewall"></a>Руководство. Интеграция единого входа Azure Active Directory с FortiWeb Web Application Firewall

Узнайте, как интегрировать FortiWeb Web Application Firewall с Azure Active Directory (Azure AD). Интеграция FortiWeb Web Application Firewall с AAD предоставит вам следующие возможности:

* Управление доступом пользователей к FortiWeb Web Application Firewall через AAD.
* Автоматический вход пользователей в FortiWeb Web Application Firewall с учетными записями AAD.
* Централизованное управление учетными записями через портал Azure.

## <a name="prerequisites"></a>Предварительные требования

Чтобы приступить к работе, потребуется следующее.

* Подписка Azure AD. Если у вас нет подписки, вы можете получить [бесплатную учетную запись](https://azure.microsoft.com/free/).
* Подписка FortiWeb Web Application Firewall с поддержкой единого входа.

## <a name="scenario-description"></a>Описание сценария

В рамках этого руководства вы настроите и проверите единый вход Azure AD в тестовой среде.

* FortiWeb Web Application Firewall поддерживает единый вход, инициируемый **поставщиком службы**.

## <a name="adding-fortiweb-web-application-firewall-from-the-gallery"></a>Добавление FortiWeb Web Application Firewall из коллекции

Чтобы настроить интеграцию FortiWeb Web Application Firewall с Azure AD, вам нужно добавить FortiWeb Web Application Firewall из коллекции в список управляемых приложений SaaS.

1. Войдите на портал Azure с помощью личной учетной записи Майкрософт либо рабочей или учебной учетной записи.
1. В области навигации слева выберите службу **Azure Active Directory**.
1. Перейдите в колонку **Корпоративные приложения** и выберите **Все приложения**.
1. Чтобы добавить новое приложение, выберите **Новое приложение**.
1. В разделе **Добавление из коллекции** в поле поиска введите **FortiWeb Web Application Firewall**.
1. Выберите **FortiWeb Web Application Firewall** на панели результатов и добавьте это приложение. Подождите несколько секунд, пока приложение не будет добавлено в ваш клиент.


## <a name="configure-and-test-azure-ad-sso-for-fortiweb-web-application-firewall"></a>Настройка и проверка единого входа AAD для FortiWeb Web Application Firewall

Настройте и проверьте единый вход AAD в FortiWeb Web Application Firewall с помощью тестового пользователя **B. Simon**. Для обеспечения работы единого входа необходимо установить связь между пользователем AAD и соответствующим пользователем в FortiWeb Web Application Firewall.

Чтобы настроить и проверить единый вход AAD в FortiWeb Web Application Firewall, выполните действия из следующих стандартных блоков:

1. **[Настройка единого входа Azure AD](#configure-azure-ad-sso)** необходима, чтобы пользователи могли использовать эту функцию.
    1. **[Создание тестового пользователя Azure AD](#create-an-azure-ad-test-user)** требуется для проверки работы единого входа Azure AD с помощью пользователя B.Simon.
    1. **[Назначение тестового пользователя Azure AD](#assign-the-azure-ad-test-user)** необходимо, чтобы позволить пользователю B.Simon использовать единый вход Azure AD.
1. **[Настройка единого входа в FortiWeb Web Application Firewall](#configure-fortiweb-web-application-firewall-sso)** необходима, чтобы настроить параметры единого входа на стороне приложения.
    1. **[Создание тестового пользователя FortiWeb Web Application Firewall](#create-fortiweb-web-application-firewall-test-user)** требуется для того, чтобы в FortiWeb Web Application Firewall существовал пользователь B. Simon, связанный с представлением этого пользователя в AAD.
1. **[Проверка единого входа](#test-sso)** позволяет убедиться в правильности конфигурации.

## <a name="configure-azure-ad-sso"></a>Настройка единого входа Azure AD

Выполните следующие действия, чтобы включить единый вход Azure AD на портале Azure.

1. На портале Azure на странице интеграции с приложением **FortiWeb Web Application Firewall** найдите раздел **Управление** и выберите **Единый вход**.
1. На странице **Выбрать метод единого входа** выберите **SAML**.
1. На странице **Настройка единого входа с помощью SAML** щелкните значок "Изменить" (значок пера), чтобы открыть диалоговое окно **Базовая конфигурация SAML** и изменить параметры.

   ![Изменение базовой конфигурации SAML](common/edit-urls.png)

1. На странице **Базовая конфигурация SAML** введите значения следующих полей.

   а. В текстовом поле **Идентификатор (сущности)** введите URL-адрес в следующем формате: `https://www.<CUSTOMER_DOMAIN>.com`.

    b. В текстовом поле **URL-адрес ответа** введите URL-адрес в формате `https://www.<CUSTOMER_DOMAIN>.com/<FORTIWEB_NAME>/saml.sso/SAML2/POST`.

    c. В текстовом поле **URL-адрес для входа** введите URL-адрес в следующем формате: `https://www.<CUSTOMER_DOMAIN>.com`.

    d. В текстовом поле **URL-адрес выхода** введите URL-адрес в следующем формате: `https://www.<CUSTOMER_DOMAIN>.info/<FORTIWEB_NAME>/saml.sso/SLO/POST`
 
    > [!NOTE]
    > Идентификатор имени `<FORTIWEB_NAME>` будет использоваться позже при передаче конфигурации в FortiWeb.
    > Для получения фактических URL-адресов обратитесь в [службу поддержки пользователей FortiWeb Web Application Firewall](mailto:support@fortinet.com). Можно также посмотреть шаблоны в разделе **Базовая конфигурация SAML** на портале Azure.

1. На странице **Настройка единого входа с помощью SAML** в разделе **Сертификат подписи SAML** найдите элемент **XML метаданных федерации** и выберите **Скачать**, чтобы скачать сертификат и сохранить его на компьютере.

    ![Ссылка для скачивания сертификата](common/metadataxml.png)


### <a name="create-an-azure-ad-test-user"></a>Создание тестового пользователя Azure AD

В этом разделе описано, как на портале Azure создать тестового пользователя с именем B.Simon.

1. На портале Azure в области слева выберите **Azure Active Directory**, **Пользователи**, а затем — **Все пользователи**.
1. В верхней части экрана выберите **Новый пользователь**.
1. В разделе **Свойства пользователя** выполните следующие действия.
   1. В поле **Имя** введите `B.Simon`.  
   1. В поле **Имя пользователя** введите username@companydomain.extension. Например, `B.Simon@contoso.com`.
   1. Установите флажок **Показать пароль** и запишите значение, которое отображается в поле **Пароль**.
   1. Нажмите кнопку **Создать**.

### <a name="assign-the-azure-ad-test-user"></a>Назначение тестового пользователя Azure AD

В этом разделе описано, как включить единый вход Azure для пользователя B. Simon, предоставив этому пользователю доступ к FortiWeb Web Application Firewall.

1. На портале Azure выберите **Корпоративные приложения**, а затем — **Все приложения**.
1. В списке приложений выберите **FortiWeb Web Application Firewall**.
1. На странице "Обзор" приложения найдите раздел **Управление** и выберите **Пользователи и группы**.
1. Выберите **Добавить пользователя**, а в диалоговом окне **Добавление назначения** выберите **Пользователи и группы**.
1. В диалоговом окне **Пользователи и группы** выберите **B.Simon** в списке пользователей, а затем в нижней части экрана нажмите кнопку **Выбрать**.
1. Если пользователям необходимо назначить роль, вы можете выбрать ее из раскрывающегося списка **Выберите роль**. Если для этого приложения не настроена ни одна роль, будет выбрана роль "Доступ по умолчанию".
1. В диалоговом окне **Добавление назначения** нажмите кнопку **Назначить**.

## <a name="configure-fortiweb-web-application-firewall-sso"></a>Настройка единого входа в FortiWeb Web Application Firewall

1.  Перейдите по адресу `https://<address>:8443`, где `<address>` — это полное доменное имя или общедоступный IP-адрес, назначенный виртуальной машине FortiWeb.

2.  Выполните вход с учетными данными администратора, предоставленными при развертывании виртуальной машины FortiWeb.

1. На следующей странице выполните приведенные ниже шаги.

    ![Страница сервера SAML](./media/fortiweb-web-application-firewall-tutorial/configure-sso.png)

    а.  В меню слева щелкните **User** (Пользователь).

    b.  В разделе User (Пользователь) щелкните **Remote Server** (Удаленный сервер).

    c.  Щелкните **SAML Server** (Сервер SAML).

    d.  Нажмите кнопку **Создать**.

    д)  В поле **Name** (Имя) укажите значение `<fwName>`, введенное в разделе Configure Azure AD (Настройка AAD).

    е)  В текстовое поле **Идентификатор сущности** вставьте значение **Идентификатор Azure AD**, скопированное на портале Azure.

    ж. Рядом с полем **Metadata** (Метаданные) щелкните элемент **Choose File** (Выбрать файл) и выберите **XML-файл метаданных федерации**, который вы ранее скачали с портала Azure.

    h.  Щелкните **ОК**.

### <a name="create-a-site-publishing-rule"></a>Создание правила публикации сайта

1.  Перейдите по адресу `https://<address>:8443`, где `<address>` — это полное доменное имя или общедоступный IP-адрес, назначенный виртуальной машине FortiWeb.

1.  Выполните вход с учетными данными администратора, предоставленными при развертывании виртуальной машины FortiWeb.
1. На следующей странице выполните приведенные ниже шаги.

    ![Правило публикации сайта](./media/fortiweb-web-application-firewall-tutorial/site-publish-rule.png)

    а.  В меню слева щелкните **Application Delivery** (Доставка приложений).
    
    b.  В разделе **Application Delivery** (Доставка приложений) щелкните **Site Publish** (Публикация сайта).
    
    c.  В разделе **Site Publish** (Публикация сайта) щелкните **Site Publish** (Опубликовать сайт).
    
    d.  Щелкните **Site Publish Rule** (Правило публикации сайта).
    
    д)  Нажмите кнопку **Создать**.
    
    е)  Предоставьте имя для правила публикации сайта.
    
    ж.  Рядом с полем **Published Site Type** (Тип сайта публикации) щелкните **Regular Expression** (Регулярное выражение).
    
    i.  Рядом с элементом **Published Site** (Опубликованный сайт) укажите строку, которая будет сопоставляться с заголовком узла публикуемого веб-сайта.
    
    j.  Рядом с элементом **Path** (Путь) укажите /.
    
    k.  Рядом с элементом **Client Authentication Method** (Способ проверки подлинности клиента) выберите **SAML Authentication** (Проверка подлинности SAML).
    
    l.  В раскрывающемся списке **SAML Server** (Сервер SAML) выберите созданный ранее сервер SAML.
    
    m.  Щелкните **ОК**.


### <a name="create-a-site-publishing-policy"></a>Создание политики публикации сайта

1.  Перейдите по адресу `https://<address>:8443`, где `<address>` — это полное доменное имя или общедоступный IP-адрес, назначенный виртуальной машине FortiWeb.

2.  Выполните вход с учетными данными администратора, предоставленными при развертывании виртуальной машины FortiWeb.

1. На следующей странице выполните приведенные ниже шаги.

    ![Политика публикации сайта](./media/fortiweb-web-application-firewall-tutorial/site-publish-policy.png)

    а.  В меню слева щелкните **Application Delivery** (Доставка приложений).

    b.  В разделе **Application Delivery** (Доставка приложений) щелкните **Site Publish** (Публикация сайта).

    c.  В разделе **Site Publish** (Публикация сайта) щелкните **Site Publish** (Опубликовать сайт).

    d.  Щелкните **Site Publish Policy** (Политика публикации сайта).

    д)  Нажмите кнопку **Создать**.

    е)  Предоставьте имя для политики публикации сайта.

    ж.  Нажмите кнопку **ОК**.

    h.  Нажмите кнопку **Создать**.

    i.  В раскрывающемся списке **Rule** (Правило) выберите созданное ранее правило публикации сайта.

    j.  Щелкните **ОК**.

### <a name="create-and-assign-a-web-protection-profile"></a>Создание и назначение профиля защиты веб-сайта

1.  Перейдите по адресу `https://<address>:8443`, где `<address>` — это полное доменное имя или общедоступный IP-адрес, назначенный виртуальной машине FortiWeb.

2.  Выполните вход с учетными данными администратора, предоставленными при развертывании виртуальной машины FortiWeb.
3.  В меню слева выберите **Policy** (Политика).
4.  В разделе **Policy** (Политика) щелкните **Web Protection Profile** (Профиль защиты веб-сайта).
5.  Щелкните **Inline Standard Protection** (Стандартная встроенная защита) и действие **Clone** (Клонировать).
6.  Укажите имя для нового профиля защиты веб-сайта и нажмите **ОК**.
7.  Выберите новый профиль защиты веб-сайта и щелкните **Edit** (Изменить).
8.  Рядом с элементом **Site Publish** (Публикация сайта) выберите ранее созданную политику публикации сайта.
9.  Щелкните **ОК**.
 
    ![Публикация сайта](./media/fortiweb-web-application-firewall-tutorial/web-protection.png)

10. В меню слева выберите **Policy** (Политика).
11. В разделе **Policy** (Политика) щелкните **Server Policy** (Политика сервера).
12. Выберите политику сервера, используемую для публикации веб-сайта, для которого вы хотите использовать проверку подлинности Azure Active Directory.
13. Нажмите кнопку **Изменить**.
14. В раскрывающемся списке **Web Protection Profile** (Профиль защиты веб-сайта) выберите только что созданный профиль защиты веб-сайта.
15. Щелкните **ОК**.
16. Попробуйте перейти по внешнему URL-адресу с опубликованным веб-сайтом от FortiWeb. Вам будет предложено выполнить проверку подлинности Azure Active Directory.
