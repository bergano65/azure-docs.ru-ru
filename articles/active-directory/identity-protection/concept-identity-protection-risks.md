# <a name="what-is-risk"></a>Что такое риск?

Обнаружения рисков в защита идентификации Azure AD включают все обнаруженные подозрительные действия, связанные с учетными записями пользователей в каталоге.

Защита идентификации предоставляет организациям доступ к мощным ресурсам для просмотра и быстрого реагирования на эти подозрительные действия. 

![Общие сведения о безопасности, в которых показаны рискованные пользователи и входы](./media/concept-identity-protection-risks/identity-protection-security-overview.png)

## <a name="risk-types-and-detection"></a>Типы и обнаружение рисков

Существует два типа риска: **пользователь** и **Вход** , а также два типа обнаружения или вычисления в **режиме реального времени** и в **режиме «вне сети**».

### <a name="user-risk"></a>Риск пользователя

Риск для пользователя представляет вероятность компрометации данного удостоверения или учетной записи. 

Эти риски вычисляются в автономном режиме с помощью внутренних и внешних источников анализа угроз Майкрософт, включая исследователи безопасности, специалистов по принудительной защите, групп безопасности в корпорации Майкрософт и других надежных источников.

| Обнаружение рисков | Описание |
| --- | --- |
| Утерянные учетные данные | Этот тип обнаружения рисков указывает, что допустимые учетные данные пользователя были утечки. Киберпреступники зачастую предоставляют общий доступ к украденным ими действительным учетным данным законных пользователей. Этот общий доступ обычно осуществляется путем публикации на темном веб-сайте, при вставке сайтов или при торговле и продаже учетных данных на черном рынке. Когда служба утечки учетных данных Майкрософт получает учетные данные пользователя из темного веб-сайта, вставки сайтов или других источников, они проверяются на предмет текущих действительных учетных данных пользователей Azure AD для поиска допустимых совпадений. |
| Аналитика угроз Azure AD | Этот тип обнаружения рисков указывает на активность пользователей, необычных для данного пользователя, или согласуется с известными шаблонами атак, основанными на внутренних и внешних источниках аналитики угроз Майкрософт. |

### <a name="sign-in-risk"></a>Риск при входе

Риск при входе представляет вероятность того, что данный запрос аутентификации отправлен не владельцем удостоверения. 

Эти риски можно вычислить в режиме реального времени или вычислить в автономном режиме с помощью внутренних и внешних источников интеллектуального анализа угроз Майкрософт, включая исследователи безопасности, специалистов по обеспечению безопасности, а также для групп разработчиков в корпорации Майкрософт и других надежных источников.

| Обнаружение рисков | Тип обнаружения | Описание |
| --- | --- | --- |
| Анонимный IP-адрес | В режиме реального времени | Этот тип обнаружения рисков указывает на входы с анонимного IP-адреса (например, из браузера Tor или анонимного VPN). Эти IP-адреса обычно используются субъектами, которые хотят скрыть свои данные телеметрии входа (IP-адрес, местоположение, устройство и другое) для потенциально злого намерения. |
| Необычный переход | Автономно | Этот тип обнаружения рисков определяет два входа, исходящие из географически нераспределенных расположений, где по крайней мере одно из расположений может быть нетипичным для пользователя с учетом предыдущего поведения. Помимо других факторов, этот алгоритм машинного обучения учитывает промежуток времени между двумя входами, и время перехода пользователя из одного расположения в другое. Это значит, что другой пользователь использует те же учетные данные. <br><br> Этот алгоритм игнорирует очевидные ложные срабатывания, такие как VPN и расположения, которые постоянно используют другие пользователи в организации, и позволяет получить результаты, соответствующие невозможным условиям перемещения. В системе предусмотрен первоначальный период обучения — 14 дней или 10 входов в систему, в течение которых она изучает поведение нового пользователя при входе. |
| IP-адрес, который помечен как вредоносный | Автономно | Этот тип обнаружения рисков указывает на попытки входа с IP-адресов, зараженных вредоносными программами, которые активно взаимодействуют с сервером-Bot. Это обнаружение определяется путем сопоставления IP-адресов устройства пользователя с IP-адресами, которые были в связи с сервером Bot во время активации сервера Bot. |
| Неизвестные свойства входа | В режиме реального времени | Этот тип обнаружения рисков учитывает историю входа в систему (IP, Широта, Долгота и ASN) для поиска аномальных входов в систему. Система хранит сведения о предыдущих расположениях, используемые пользователем, и рассматривает эти «знакомые» расположения. Обнаружение рисков активируется, когда вход выполняется из расположения, которое еще не находится в списке привычных расположений. Недавно созданные пользователи будут находиться в режиме обучения в течение определенного периода времени, в течение которого незнакомые свойства входа в систему будут отключены, а наши алгоритмы узнают о поведении пользователя. Длительность режима обучения является динамической и зависит от того, сколько времени использует алгоритм для сбора достаточной информации о пользовательских шаблонах входа. Минимальная длительность составляет пять дней. Пользователь может вернуться в режим обучения после длительного периода бездействия. Система также игнорирует вход с помощью знакомых устройств и из расположений, географически близких к знакомому. <br><br> Мы также запускаем это обнаружение для базовой аутентификации (или устаревших протоколов). Поскольку эти протоколы не имеют современных свойств, например идентификатора клиента, существует ограниченная телеметрия для сокращения количества ложноположительных результатов. Мы рекомендуем нашим клиентам перейти на современные способы проверки подлинности. |
| Администратор подтвердил безопасность пользователя | Автономно | Это обнаружение означает, что администратор выбрал параметр "подтвердить безопасность пользователя" в пользовательском интерфейсе рискованных пользователей или с помощью API Рискюсерс. Чтобы узнать, какое из администраторов подтвердило безопасность этого пользователя, проверьте журнал рисков пользователя (через пользовательский интерфейс или API). |
| Вредоносный IP-адрес | Автономно | Это обнаружение означает вход с вредоносного IP-адреса. IP-адрес считается вредоносным, основанным на высокой частоте сбоев из-за недопустимых учетных данных, полученных от IP или других источников репутации IP-адресов. |

### <a name="other-risk-detections"></a>Другие обнаружения рисков

| Обнаружение рисков | Тип обнаружения | Описание |
| --- | --- | --- |
| Обнаружен дополнительный риск | В режиме реального времени или вне сети | Это обнаружение означает, что обнаружено одно из приведенных выше обновлений уровня "Премиум". Поскольку обнаружения уровня "Премиум" видны только Azure AD Premiumным клиентам P2, они называются "обнаружены дополнительные риски" для клиентов, не имеющих Azure AD Premiumных лицензий P2. |
