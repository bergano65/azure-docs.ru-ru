# <a name="users-flagged-for-risk-report-in-the-azure-portal"></a>Описание отчета о пользователях под угрозой на портале Azure

Azure Active Directory (Azure AD) обнаруживает подозрительные действия, связанные с учетными записями пользователей. Для каждого обнаруженного действия создается запись, которая называется [событием риска](concept-risk-events.md).

На [портале Azure](https://portal.azure.com) отчеты о безопасности можно найти в колонке **Azure Active Directory** в разделе **Безопасность**. 

Обнаруженные события риска используются для вычисления следующих параметров.

- **Входы, представляющие риск**. Вход, представляющий риск, означает, что в систему пытался войти пользователь, который не является законным владельцем учетной записи. 

- **Пользователи, находящиеся в группе риска**. Такая пометка означает, что конфиденциальность учетной записи пользователя, возможно, нарушена. 

Сведения о том, как настроить политики, которые активируют эти события риска, см. в руководстве по [настройке политика риска пользователя](../identity-protection/howto-user-risk-policy.md). 

![события входа, представляющие риск.](./media/concept-user-at-risk/10.png)


## <a name="what-azure-ad-license-do-you-need-to-access-the-users-at-risk-report"></a>Какая требуется лицензия Azure AD, чтобы получить доступ к отчету о пользователях под угрозой?  

Все выпуски Azure Active Directory предоставляют отчеты о пользователях, находящихся в группе риска. Однако уровень детализации отчета может для выпусков отличаться. 

- В **выпусках Azure Active Directory Free и Basic** отображается список пользователей, находящихся в группе риска. 

- Кроме того, выпуск **Azure Active Directory Premium 1** позволяет изучать некоторые базовые события риска, обнаруженные для каждого отчета. 

- Выпуск **Azure Active Directory Premium 2** предоставляет наиболее полные сведения о базовых событиях риска, а также позволяет настроить политики безопасности, автоматически реагирующие на настроенные уровни риска.


## <a name="users-at-risk-report-for-azure-ad-free-and-basic-editions"></a>Отчет о пользователей под угрозой для выпусков Azure AD Free и Basic

В выпусках Azure AD Free и Basic доступен список учетных записей пользователей, конфиденциальность которых могла быть нарушена. 

![события входа, представляющие риск.](./media/concept-user-at-risk/03.png)

Если выбрать пользователя, откроется колонка его данных. Вы можете просмотреть журнал входов пользователя под угрозой и сбросить пароль, если это необходимо.

![события входа, представляющие риск.](./media/concept-user-at-risk/46.png)


Это диалоговое окно предоставляет следующие возможности:

- Загрузка отчета

- Поиск пользователей

![события входа, представляющие риск.](./media/concept-user-at-risk/16.png)


## <a name="users-at-risk-report-for-azure-ad-premium-editions"></a>Отчет о пользователей под угрозой для выпуска Azure AD Premium

Отчет о пользователях под угрозой в выпуске Azure AD Premium включает следующие сведения:

- Список учетных записей пользователей, которые, возможно, были скомпрометированы. 

- Сводная информация о [типах обнаруженных событий риска](concept-risk-events.md).

- Возможность скачать отчет.

- Настройка [политики устранения рисков пользователей](../identity-protection/howto-user-risk-policy.md).  

![события входа, представляющие риск.](./media/concept-user-at-risk/71.png)

При выборе пользователя отображается подробное представление отчета об этом пользователе, предоставляющее перечисленные ниже возможности.

- Отображение представления всех событий входа.

- Сброс пароля пользователя

- Отклонение всех событий.

- Изучение обнаруженных событий риска для пользователя. 

![события входа, представляющие риск.](./media/concept-user-at-risk/324.png)

Чтобы изучить событие риска, выберите его из списка. Откроется колонка **Сведения** для этого события риска. В колонке **Сведения** вы можете либо вручную закрыть событие риска, либо повторно активировать событие риска, закрытое вручную. 

![события входа, представляющие риск.](./media/concept-user-at-risk/325.png)
