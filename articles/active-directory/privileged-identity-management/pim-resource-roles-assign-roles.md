# <a name="assign-azure-resource-roles-in-privileged-identity-management"></a>Назначение ролей ресурсов Azure в управление привилегированными пользователями

Azure Active Directory (Azure AD) управление привилегированными пользователями (PIM) может управлять встроенными ролями ресурсов Azure, а также пользовательскими ролями, включая (но не ограничиваясь ими):

- Владелец
- Администратор доступа пользователей
- Участник
- администратор безопасности;
- Диспетчер безопасности

> [!NOTE]
> Пользователи или члены группы, которым назначены роли подписки владельца или администратора доступа пользователей, и глобальные администраторы Azure AD, которые обеспечивают управление подписками в Azure AD, по умолчанию имеют разрешения администратора ресурсов. Эти администраторы могут назначать роли, настраивать параметры ролей и просматривать доступ с помощью управление привилегированными пользователями для ресурсов Azure. Пользователь не может управлять управление привилегированными пользователями для ресурсов без разрешений администратора ресурсов. Просмотрите список [встроенных ролей Azure](../../role-based-access-control/built-in-roles.md).

## <a name="assign-a-role"></a>Назначение роли

Выполните следующие действия, чтобы предоставить пользователю разрешение для роли ресурса Azure AD.

1. Войдите в [портал Azure](https://portal.azure.com/) с помощью пользователя, который является членом роли [администратора привилегированных ролей](../users-groups-roles/directory-assign-admin-roles.md#privileged-role-administrator) .

    Сведения о том, как предоставить другим администраторам доступ к управлению управление привилегированными пользователями, см. [в разделе Предоставление доступа другим администраторам для управления Управление привилегированными пользователями](pim-how-to-give-access-to-pim.md).

1. Откройте **Azure AD privileged Identity Management**.

1. Выберите **ресурсы Azure**.

1. Используйте фильтр ресурсов для поиска управляемых ресурсов, которые вы ищете.

    ![Список ресурсов Azure для управления](./media/pim-resource-roles-assign-roles/resources-list.png)

1. Выберите ресурс, которым требуется управлять, чтобы открыть страницу обзора ресурсов.

1. В разделе **Управление**выберите **роли** , чтобы просмотреть список ролей для ресурсов Azure.

    ![Роли ресурсов Azure](./media/pim-resource-roles-assign-roles/resources-roles.png)

1. Выберите **добавить назначения** , чтобы открыть панель **Добавление назначений** .

1. Щелкните **выбрать роль** , чтобы открыть страницу **Выбор роли** .

    ![Панель нового назначения](./media/pim-resource-roles-assign-roles/resources-select-role.png)

1. Выберите роль, которую вы намерены назначить, и щелкните **Выбрать**.

    Откроется область **Выбор элемента или группы** .

1. Выберите участника или группу, которые нужно назначить роли, а затем нажмите кнопку **выбрать**.

    ![Панель выбора участника или группы](./media/pim-resource-roles-assign-roles/resources-select-member-or-group.png)

1. На вкладке **Параметры** в списке **тип назначения** выберите значение разрешено или **Eligible** **активно**.

    ![Панель параметров членства](./media/pim-resource-roles-assign-roles/resources-membership-settings-type.png)

    Управление привилегированными пользователями для ресурсов Azure предоставляет два различных типа назначений:

    - Для **подходящих** назначений требуется, чтобы член роли выполнил действие для использования роли. Поддерживаются такие действия, как прохождение многофакторной проверки подлинности (MFA), предоставление коммерческого обоснования или запрос утверждения от назначенных утверждающих.

    - Для **активных** назначений член не должен выполнять никаких действий для использования роли. Члены, назначенные в качестве активных, постоянно имеют все полномочия, присвоенные роли.

1. Чтобы задать определенную длительность назначения, измените дату и время начала и окончания.

1. По завершении нажмите кнопку **назначить**.

1. После создания нового назначения роли отображается уведомление о состоянии.

    ![Новое назначение — уведомление](./media/pim-resource-roles-assign-roles/resources-new-assignment-notification.png)

## <a name="update-or-remove-an-existing-role-assignment"></a>Обновление или удаление существующего назначения роли

Выполните следующие действия, чтобы обновить или удалить существующее назначение роли.

1. Откройте **Azure AD privileged Identity Management**.

1. Выберите **ресурсы Azure**.

1. Выберите ресурс, которым требуется управлять, чтобы открыть его страницу обзора.

1. В разделе **Управление**выберите **роли** , чтобы просмотреть список ролей для ресурсов Azure.

    ![Роли ресурсов Azure — выбор роли](./media/pim-resource-roles-assign-roles/resources-update-select-role.png)

1. Щелкните роль, которую нужно обновить или удалить.

1. Найдите назначение роли на вкладках **Доступные роли** или **Активные роли**.

    ![Обновление или удаление назначения роли](./media/pim-resource-roles-assign-roles/resources-update-remove.png)

1. Нажмите кнопку **Обновить** или **Удалить**, чтобы обновить или удалить назначение роли.

    Дополнительные сведения о расширении назначения ролей см. [в разделе расширение или обновление ролей ресурсов Azure в Управление привилегированными пользователями](pim-resource-roles-renew-extend.md).
