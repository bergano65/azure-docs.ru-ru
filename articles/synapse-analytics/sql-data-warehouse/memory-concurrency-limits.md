# <a name="memory-and-concurrency-limits-for-dedicated-sql-pool-in-azure-synapse-analytics"></a>Ограничения памяти и параллелизма для выделенного пула SQL в Azure синапсе Analytics

Из этой статьи вы узнаете об ограничениях памяти и параллелизма, действующих для различных уровней производительности и классов ресурсов в Azure Synapse Analytics.

Масштабирование вычислительных ресурсов в хранилище данных SQL Azure с помощью портала Azure](quickstart-scale-compute-portal.md).

## <a name="concurrency-maximums-for-workload-groups"></a>Максимальные возможности параллелизма для групп рабочей нагрузки

Количество ресурсов на запрос выделяется в процентах и задается в определении группы рабочей нагрузки.  Однако даже при отказе от слотов выдачи для запросов требуется минимальный объем ресурсов в зависимости от уровня обслуживания. ## <a name="concurrency-maximums-for-resource-classes"></a>Максимальные возможности параллелизма для классов ресурсов

Чтобы каждый запрос имел достаточно ресурсов для эффективной работы, Synapse SQL отслеживает использование ресурсов, назначая каждому запросу определенное количества слотов выдачи. Система помещает запросы в очередь, исходя из важности и доступных слотов выдачи. Запросы ожидают в очереди, пока не будет доступно достаточно слотов выдачи. [Важность](sql-data-warehouse-workload-importance.md) и слоты выдачи также определяют приоритезацию ресурсов ЦП. Дополнительные сведения см. в статье [Анализ рабочей нагрузки](analyze-your-workload.md).
