# <a name="manage-a-private-endpoint-connection"></a>Управление подключением к частной конечной точке
Частная ссылка Azure работает на модели потока вызовов утверждения, в которой потребитель службы частной связи может запросить подключение к поставщику услуг для использования службы. Затем поставщик услуг может решить, следует ли разрешить потребителю подключаться или нет. Частная ссылка Azure позволяет поставщикам услуг управлять подключением к частной конечной точке в своих ресурсах. В этой статье приводятся инструкции по управлению подключениями к частным конечным точкам.

![Управление частными конечными точками](media/manage-private-endpoint/manage-private-endpoint.png)

Существует два метода утверждения соединения, которые может выбрать потребитель службы частной связи:
- **Автоматически**: Если потребитель службы имеет разрешения RBAC для ресурса поставщика услуг, потребитель может выбрать метод автоматического утверждения. В этом случае, когда запрос достигает ресурса поставщика услуг, не требуется никаких действий от поставщика услуг, и подключение автоматически утверждается. 
- **Вручную**: Напротив, если у потребителя службы нет разрешений RBAC на ресурс поставщика услуг, потребитель может выбрать метод утверждения вручную. В этом случае запрос на подключение отображается в ресурсах службы как **ожидающие**. Поставщик услуг должен вручную утвердить запрос, прежде чем можно будет установить соединения. В ручных случаях потребитель службы может также указать сообщение с запросом, чтобы предоставить поставщику услуг больше контекста. У поставщика услуг есть следующие варианты выбора для всех подключений к частным конечным точкам: **Утверждено**, **отклонено**, **удалено**.

В таблице ниже показаны различные действия поставщика услуг и результирующие состояния соединения для частных конечных точек.  Поставщик услуг также может изменить состояние подключения для подключения к частной конечной точке в дальнейшем без вмешательства потребителя. Действие обновит состояние конечной точки на стороне потребителя. 


|Действие поставщика услуг   |Состояние закрытой конечной точки потребителя службы   |Описание   |
|---------|---------|---------|
|Отсутствуют    |    Ожидается     |    Подключение создается вручную и ожидает утверждения владельцем ресурса закрытой ссылки.       |
|Одобрить    |  Одобрен       |  Подключение было автоматически или утверждено вручную и готово к использованию.     |
|Отклонить     | Отклонен        | Соединение было отклонено владельцем ресурса частной ссылки.        |
|Удаление    |  Отключено       | Подключение удалено владельцем ресурса частной ссылки. Частная конечная точка будет информативной и должна быть удалена для очистки.        |

## <a name="manage-private-endpoint-connections-on-azure-paas-resources"></a>Управление подключениями к частным конечным точкам в ресурсах Azure PaaS
Портал является предпочтительным методом управления подключениями к частным конечным точкам в ресурсах Azure PaaS. Сейчас у нас нет поддержки PowerShell/CLI для управления подключениями в ресурсах Azure PaaS.
1. Войдите на портал Azure по адресу https://portal.azure.com.
2. Перейдите в центр управления частными ссылками.
3. В разделе **ресурсы**выберите тип ресурса, который требуется использовать для управления подключениями к частным конечным точкам.
4. Для каждого типа ресурса можно просмотреть число связанных с ней частных подключений конечной точки. При необходимости можно отфильтровать ресурсы.
5. Выберите подключения к частным конечным точкам.  В списке подключений выберите подключение, которое требуется управлять. 
6. Состояние подключения можно изменить, выбрав один из параметров в верхней части.

## <a name="manage-private-endpoint-connections-on-a-customerpartner-owned-private-link-service"></a>Управление подключениями частной конечной точки в службе частной связи клиента или партнера

Azure PowerShell и Azure CLI являются предпочтительными методами управления подключениями к частным конечным точкам в службах Microsoft Partner Services или в службах, принадлежащих клиентам. Сейчас у нас нет поддержки портала для управления подключениями в службе частной связи.  
 
### <a name="powershell"></a>PowerShell 
  
Используйте следующие команды PowerShell для управления подключениями к частным конечным точкам.  
#### <a name="get-private-link-connection-states"></a>Получение состояний подключения к частной ссылке 
`Get-AzPrivateLinkService` Используйте командлет для получения подключений к частной конечной точке и их состояний.  
```azurepowershell
Get-AzPrivateLinkService -Name myPrivateLinkService -ResourceGroupName myResourceGroup 
 ```
 
#### <a name="approve-a-private-endpoint-connection"></a>Утверждение подключения к частной конечной точке 
 
Используйте командлет `Approve-AzPrivateEndpointConnection` , чтобы утвердить подключение к частной конечной точке. 
 
```azurepowershell
Approve-AzPrivateEndpointConnection -Name myPrivateEndpointConnection -ResourceGroupName myResourceGroup -ServiceName myPrivateLinkService
```
 
#### <a name="deny-private-endpoint-connection"></a>Запретить подключение к частной конечной точке 
 
Используйте командлет `Deny-AzPrivateEndpointConnection` , чтобы отклонить подключение к частной конечной точке. 
```azurepowershell
Deny-AzPrivateEndpointConnection -Name myPrivateEndpointConnection -ResourceGroupName myResourceGroup -ServiceName myPrivateLinkService 
```
#### <a name="remove-private-endpoint-connection"></a>Удаление подключения к частной конечной точке 
 
Используйте командлет `Remove-AzPrivateEndpointConnection` , чтобы удалить подключение к частной конечной точке. 
```azurepowershell
Remove-AzPrivateEndpointConnection -Name myPrivateEndpointConnection1 -ResourceGroupName myResourceGroup -ServiceName myPrivateLinkServiceName 
```
 
### <a name="azure-cli"></a>Azure CLI 
 
Используется `az network private-link-service update` для управления подключениями к частным конечным точкам. Состояние соединения указывается в ```azurecli connection-status``` параметре. 
```azurecli
az network private-link-service connection update -g myResourceGroup -n myPrivateEndpointConnection1 --service-name myPLS --connection-status Approved 
```
