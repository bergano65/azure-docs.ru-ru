# <a name="azure-dedicated-hsm-high-availability"></a>Высокая доступность выделенного устройства HSM Azure

Выделенное устройство HSM Azure поддерживается высокодоступными центрами обработки данных Майкрософт. Тем не менее любой высокодоступный центр обработки данных уязвим к локальным сбоям, а в экстренных ситуациях — к сбоям на уровне региона. Майкрософт развертывает устройства HSM в разных центрах обработки данных в пределах региона, чтобы гарантировать, что подготовка нескольких устройств не приведет к использованию одной стойки для этих устройств. Более высокий уровень доступности достигается путем связывания этих HSM по центрам обработки данных в регионе с помощью функции группы Thales HA. Кроме того, возможно связать устройства между регионами для выполнения отработки отказа в регионе в случае аварийного восстановления. С помощью такой многоуровневой конфигурации с высоким уровнем доступности любой сбой в работе устройства будет автоматически разрешен для обеспечения работоспособности приложения. Все центры обработки данных также включают запасные устройства и компоненты на месте, поэтому можно своевременно заменить любое неисправное устройство.

## <a name="high-availability-example"></a>Пример высокого уровня доступности

Сведения о настройке устройств HSM для высокой доступности на уровне программного обеспечения см. в разделе Руководство по администрированию HSM Thales Luna 7. Этот документ доступен на  [странице HSM Thales](https://thalesdocs.com/gphsm/Content/luna/network/luna_network_releases.htm).

На следующей схеме показана архитектура с высоким уровнем доступности. В ней используются несколько устройств в регионе и несколько устройств, связанных в отдельном регионе. В этой архитектуре используются как минимум четыре устройства HSM и компоненты виртуальной сети.
